Netlify 刚刚获得了 3000 万美元来改变开发者建立网站的方式

Netlify 希望彻底改变开发者构建网站的方式，将 web 服务器抽象化，将网站分成微服务，使这个过程更像是构建一个移动应用程序，而不是传统的网站。今天，该公司宣布了 3000 万美元的 B 轮投资，以帮助继续建立这一愿景。

凯鹏华盈领先一轮。安德森·霍洛维茨以及 Slack(斯图尔特·巴特菲尔德)、Yelp(杰瑞米·斯托佩尔曼)和 Figma(迪伦·菲尔德)的创始人都参加了。根据 Crunchbase 的数据，今天的投资使融资总额超过 4400 万美元。

联合创始人兼总裁克里斯巴赫(Chris Bach)和联合创始人兼首席执行官马特比尔曼(Matt Biilmann)认为，他们试图做出的改变是向 API 经济转变的一部分。他们想把 API 给移动环境中的程序员带来的同样的开发便利带到 web 开发中。

正如我今年早些时候所写的，当宣布支持 AWS Lambda 时，他们希望降低 web 开发的复杂性:

“Netlify 已经抽象出了网络服务器的概念，它说网络服务器部署缓慢，难以保护和扩展。通过从单一网站转向具有后端微服务的静态前端，它相信它可以解决安全和扩展问题，并更快地交付网站。”

创始人有一个宏伟的愿景，“我们基本上是用一个全球应用交付网络取代所有的网络服务器，”巴赫解释说。

投资者凯鹏华盈(Kleiner Perkins)的普通合伙人马蒙·哈米德(Mamoon Hamid)表示，虽然网站后端近年来有所发展，但前端一直保持静止，这就是 Netlify 通过其基于微服务的网站开发方法解决的问题。哈米德告诉 TechCrunch:“Netlify smack dab 击中了我们的观点，即我们需要去哪里让网络繁荣起来。”

他认为表示层 web 开发的最后一次重大转变是 15 年前 CMS 的出现，我们开始看到开发人员被 Netlify 方法深深吸引。哈米德说:“我们真的相信，Netlify 背后已经有 30 万强大的开发人员，他们正显示出进入这个平台的早期迹象，这个平台可能是(未来)很大一部分网络内容的来源。”。

Netlify 正致力于在未来几年增加使用他们方法的网站数量，并将此视为改变网络的使命。“对我们来说，保持一个开发者想去的地方非常重要，而且可以很容易地启动和运行一些东西。然后你可以从那里开始扩大规模，”巴赫说。

该公司希望建立一个更有组织的销售和营销团队，向更大的组织销售 Netlify 方法，同时继续扩大产品和开发人员的范围。所有这些都需要钱，这就是为什么他们今天进行了这么大的一轮。
