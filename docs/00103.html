Cloud Foundry扩大了对Kubernetes TechCrunch的支持

原文：https://web.archive.org/web/https://techcrunch.com/2018/10/10/cloud-foundry-expands-its-support-for-kubernetes/

不久前， Cloud Foundry Foundation 完全是关于Cloud Foundry的，这是一个开源平台即服务(PaaS)项目，现在大多数财富500强企业都在使用它。这个项目是云铸造应用运行时。一年前，该基金会还宣布了Cloud Foundry Container Runtime，帮助企业并行运行应用平台及其基于容器的应用。此外，Cloud Foundry也长期以来一直是BOSH背后的力量，BOSH是一种用于构建、部署和管理云应用程序的工具。

Container Runtime a year go的加入似乎有点混淆了组织的使命，但是现在尘埃落定，这里的意图开始变得更加清晰。正如Cloud Foundry CTO Chip Childers告诉我的，企业使用容器运行时的主要目的是运行他们从供应商那里获得的预打包应用程序。“容器运行时——或者实际上是Kubernetes的任何部署——当与应用运行时一起使用或结合使用时，人们主要是在那里登陆由独立软件供应商提供的打包软件，”他告诉我。“容器是新的光盘。你只是想把它放在一个好的编排平台上。”

因为应用程序运行时在Kubernetes出现之前就已经发布了，所以Cloud Foundry项目构建了自己的容器服务，称为Diego。

今天，Cloud Foundry foundation推出了两个与Kubernetes相关的新项目，将两者之间的集成提升到了一个新的水平。第一个是由IBM发起的Eirini 项目，Suse和SAP目前也在从事这项工作。这个项目已经酝酿了很长时间，也是社区期待已久的事情。在部署为应用程序运行时编写的应用程序时，它基本上允许开发人员在使用现有的Diego orchestrator和Kubernetes之间进行选择。这对Cloud Foundry来说是一件大事。

“Eirini所做的是利用Cloud Foundry应用运行时——与[Cloud Foundry]品牌紧密相关的核心PaaS体验，并允许用Kubernetes替换底层Diego调度程序，作为它所覆盖的用例的一个选项，”Childers解释道。他补充说，仍然有一些用例迭戈集装箱管理系统比Kubernetes更适合。其中之一是更好的Windows支持——这对使用Cloud Foundry的企业来说相当重要。Childers还指出，Kubernetes的多租户担保没有Diego的严格。

第二个新项目是 CF集装箱化，最初由Suse开发。顾名思义，CF容器化基本上允许您打包核心Cloud Foundry应用程序运行时，并借助BOSH部署工具将其部署在Kubernetes集群中。这几乎就是Suse已经用来发布其Cloud Foundry发行版的东西。

很明显，Kubernetes正在成为Cloud Foundry PaaS服务的一部分，在不久的将来，开发者将使用它来部署他们为之编写的应用程序。乍一看，这种对Kubernetes的关注可能会使Cloud Foundry变得多余，但值得记住的是，在其核心，Cloud Foundry应用程序运行时不是关于基础设施，而是关于旨在管理整个应用程序开发生命周期的开发人员体验和方法。如果Kubernetes可以用来帮助管理基础设施，那么Cloud Foundry项目也可以专注于它最擅长的领域。
