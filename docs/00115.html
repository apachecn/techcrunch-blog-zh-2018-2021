Nvidia 推出 Rapids，帮助将 GPU 加速引入数据分析

原文：https://web.archive.org/web/https://techcrunch.com/2018/10/10/nvidia-launches-rapids-to-help-bring-gpu-acceleration-to-data-analytics/

Nvidia 与 IBM、HPE、甲骨文、Databricks 等合作伙伴一起，今天推出了一个新的数据科学和机器学习开源平台。该公司称之为 Rapids，旨在让大型企业更容易利用 GPU 的能力快速分析大量数据，然后利用这些数据建立机器学习模型。

“企业越来越受数据驱动，”Nvidia 加速计算副总裁 Ian Buck 告诉我。“他们通过收集的数据来感知市场、环境、行为和业务运营。我们刚刚经历了十年的大数据，这些数据的输出正在使用分析和人工智能。但大多数 it 仍然使用传统的机器学习来识别复杂的模式，检测变化并做出直接影响其底线的预测。”

Rapids 背后的想法是与数据科学家今天使用的现有流行开源库和平台合作，并使用 GPU 加速它们。Rapids 与这些库集成在一起，以提供加速的分析、机器学习和未来的可视化。

巴克指出，Rapids 是基于 Python 的；它的接口类似于 Pandas 和 Scikit，这两个非常受欢迎的机器学习和数据分析库，它基于 Apache Arrow 进行内存数据库处理。它可以从单个 GPU 扩展到多个节点，IBM 指出，与在 CPU 上运行相同的算法相比，该平台在某些特定用例中可以实现高达 50 倍的改善(尽管这并不令人惊讶，因为我们过去已经从其他 GPU 加速的工作负载中看到了这一点)。

Buck 指出，Rapids 是多年努力的结果，旨在开发一套足够丰富的库和算法，让它们在 GPU 上良好运行，并与相关的开源项目建立关系。

“它旨在加速端到端的数据科学，”巴克解释道。“从数据准备到机器学习，对于那些想采取下一步行动的人来说，就是深度学习。通过 Arrow，Spark 用户可以轻松地将数据移入 Rapids 平台进行加速。”

事实上，Spark 肯定会成为这里的主要用例之一，所以难怪 Spark 背后的团队创建的公司 Databricks 是早期合作伙伴之一。

“我们有多个正在进行的项目，以更好地将 Spark 与原生加速器集成，包括 Apache Arrow 支持和 Project Hydrogen 的 GPU 调度，”Spark 创始人 Matei Zaharia 在今天的公告中表示。“我们相信，RAPIDS 是一个令人兴奋的新机会，可以扩展我们客户的数据科学和人工智能工作负载。"

Nvidia 还与 Anaconda、BlazingDB、PyData、Quansight 和 scikit-learn 以及 Wes McKinney,Ursa Labs的负责人以及 Apache Arrow 和 Pandas 的创作者合作。

另一个合作伙伴是 IBM，该公司计划为其许多服务和平台提供 Rapids 支持，包括其用于在 GPU 加速的 Power9 服务器上运行数据科学和人工智能工作负载的 PowerAI 工具，IBM Watson Studio 和 Watson Machine Learning 以及 IBM Cloud 及其支持 GPU 的机器。“在 IBM，我们对任何能够为数据科学和机器学习带来更高性能、更好业务成果的东西都非常感兴趣——我们认为英伟达在这方面有非常独特的东西，”IBM Analytics 总经理罗布·托马斯告诉我。

Buck 说:“对社区的主要好处是，通过一套完全免费和开源的库，这些库与他们习惯的现有算法和子例程直接兼容——他们现在可以访问 GPU 加速版本的算法和子例程。”他还强调，Rapids 并没有试图与现有的机器学习解决方案竞争。“Rapids 开源的部分原因是，你可以轻松地将那些机器学习子程序整合到他们的软件中，并从中受益。”
