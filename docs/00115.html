Nvidia推出Rapids，帮助将GPU加速引入数据分析

原文：https://web.archive.org/web/https://techcrunch.com/2018/10/10/nvidia-launches-rapids-to-help-bring-gpu-acceleration-to-data-analytics/

Nvidia与IBM、HPE、甲骨文、Databricks等合作伙伴一起，今天推出了一个新的数据科学和机器学习开源平台。该公司称之为Rapids，旨在让大型企业更容易利用GPU的能力快速分析大量数据，然后利用这些数据建立机器学习模型。

“企业越来越受数据驱动，”Nvidia加速计算副总裁Ian Buck告诉我。“他们通过收集的数据来感知市场、环境、行为和业务运营。我们刚刚经历了十年的大数据，这些数据的输出正在使用分析和人工智能。但大多数it仍然使用传统的机器学习来识别复杂的模式，检测变化并做出直接影响其底线的预测。”

Rapids背后的想法是与数据科学家今天使用的现有流行开源库和平台合作，并使用GPU加速它们。Rapids与这些库集成在一起，以提供加速的分析、机器学习和未来的可视化。

巴克指出，Rapids是基于Python的；它的接口类似于Pandas和Scikit，这两个非常受欢迎的机器学习和数据分析库，它基于 Apache Arrow 进行内存数据库处理。它可以从单个GPU扩展到多个节点，IBM指出，与在CPU上运行相同的算法相比，该平台在某些特定用例中可以实现高达50倍的改善(尽管这并不令人惊讶，因为我们过去已经从其他GPU加速的工作负载中看到了这一点)。

Buck指出，Rapids是多年努力的结果，旨在开发一套足够丰富的库和算法，让它们在GPU上良好运行，并与相关的开源项目建立关系。

“它旨在加速端到端的数据科学，”巴克解释道。“从数据准备到机器学习，对于那些想采取下一步行动的人来说，就是深度学习。通过Arrow，Spark用户可以轻松地将数据移入Rapids平台进行加速。”

事实上，Spark肯定会成为这里的主要用例之一，所以难怪Spark背后的团队创建的公司Databricks是早期合作伙伴之一。

“我们有多个正在进行的项目，以更好地将Spark与原生加速器集成，包括Apache Arrow支持和Project Hydrogen的GPU调度，”Spark创始人Matei Zaharia在今天的公告中表示。“我们相信，RAPIDS是一个令人兴奋的新机会，可以扩展我们客户的数据科学和人工智能工作负载。"

Nvidia还与Anaconda、BlazingDB、PyData、Quansight和scikit-learn以及Wes McKinney,Ursa Labs的负责人以及Apache Arrow和Pandas的创作者合作。

另一个合作伙伴是IBM，该公司计划为其许多服务和平台提供Rapids支持，包括其用于在GPU加速的Power9服务器上运行数据科学和人工智能工作负载的PowerAI工具，IBM Watson Studio和Watson Machine Learning以及IBM Cloud及其支持GPU的机器。“在IBM，我们对任何能够为数据科学和机器学习带来更高性能、更好业务成果的东西都非常感兴趣——我们认为英伟达在这方面有非常独特的东西，”IBM Analytics总经理罗布·托马斯告诉我。

Buck说:“对社区的主要好处是，通过一套完全免费和开源的库，这些库与他们习惯的现有算法和子例程直接兼容——他们现在可以访问GPU加速版本的算法和子例程。”他还强调，Rapids并没有试图与现有的机器学习解决方案竞争。“Rapids开源的部分原因是，你可以轻松地将那些机器学习子程序整合到他们的软件中，并从中受益。”
