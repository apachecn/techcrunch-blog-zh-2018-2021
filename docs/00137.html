New Relic收购比利时CoScale，扩大其对Kubernetes容器和微服务的监控范围TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2018/10/11/new-relic-acquires-belgiums-coscale-to-expand-its-monitoring-of-kubernetes-containers-and-microservices/

New Relic是一家分析和监控公司内部和外部应用和服务的提供商，旨在帮助优化其性能，该公司今天将进行收购，继续扩展其业务的新领域，即容器和微服务。该公司宣布已经收购了CoScale ，这是一家集装箱和微服务监控提供商，特别关注Kubernetes。

交易条款——包括团队和技术——尚未披露，因为它不会对New Relic的收益产生实质性影响。这家较大的公司在纽约证券交易所上市(股票代码:NEWR)，在过去两年中一直强劲上涨，目前的市值约为46亿美元。

根据的PitchBook ，最初成立于比利时的CoScale已经筹集了640万美元，最后估值为790万美元。投资者包括微软(通过其向上扩展加速器)、PMV和两家比利时投资者Qbic基金。

New Relic的产品管理高级总监Ramon Guiu表示:我们很高兴将CoScale的知识和深厚的技术团队带入新的Relic系列。“加入 New Relic 的CoScale团队成员将致力于将CoScale的能力和经验融入到 New Relic 平台的持续创新中。”

这笔交易突显出New Relic在过去几年不得不做出的转变:当该公司多年前成立时，应用程序监控是一项相对容易的任务，网络和特定数量的服务是需要关注的极限。但是，服务、应用和功能已经变得越来越复杂，现在需要跨一系列位置和设备存储数据，处理一切会产生大量计算需求。

New Relic在2016年首次将容器和微服务监控添加到其堆栈中。这对于该领域来说有点晚了，New Relic首席执行官Lew Cirne认为这正是时候，使New Relic的变化与市场的更广泛变化相吻合。

他上个月告诉我的同事罗恩·米勒，我们认为这些变化实际上是一个机会，让我们进一步区分并进一步加强我们的论点，即新遗迹方式确实是解决这个问题的最合理方式。正如Ron所写的，Cirne认为New Relic一直以代码为中心，而不是交付它的基础设施，这有助于它随着交付机制的变化而做出调整。

New Relic已经为Kubernetes、Google Kubernetes Engine(GKE)、Amazon Elastic Container Service for Kubernetes(EKS)、Microsoft Azure Kubernetes Service(AKS)和 RedHat Openshift 提供了监控，其想法是CoScale将帮助它在该范围内提升，同时还将Docker和Openshift添加到组合中，并提供新的服务来服务DevOps社区。

“新遗迹和CoScale的愿景非常一致，因此我们的团队很高兴能够加入新 遗迹并继续我们的旅程，通过提供他们对其现代架构性能的可见性来帮助公司更快地创新，”CoScale首席执行官Stijn Polfliet在一份声明中说。“[联合创始人] Fred [Ryckbosch]和我都觉得这是一个令人兴奋的市场空间和时间，我们很高兴能与现代应用和基础设施监控领域的领导者 New Relic 的出色团队合作。”
