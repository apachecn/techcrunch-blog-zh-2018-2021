Stripe 从谷歌聘请尼尔斯·普罗沃斯担任新的安全主管 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2018/10/17/stripe-hires-niels-provos-away-from-google-to-be-its-new-head-of-security/

在以 200 亿美元的估值筹集新资金之后，支付和金融服务初创公司 Stripe 正在进行一项关键的招聘，以向世界强调它正在认真对待其云服务的安全性。今天，该公司宣布，它已聘请尼尔斯普罗沃斯作为其新的安全主管。Provos 最近是谷歌的一名杰出工程师，在那里他领导了搜索巨头为确保其服务安全所做的一些最大的努力，可能最著名的是安全浏览，但最近也领导了谷歌云平台和谷歌的生产基础设施。

他还是一名著名的密码学家和研究员，对其他旨在提高互联网安全性的项目产生了重大影响，如 bcrypt 、 honeyd 和 OpenSSH 。他也是制造剑和刀的铁匠(我猜是另一种安全工具)。

据我所知，在离开谷歌之前，Provos 已经和 Stripe 谈过一段时间了。

他在一份声明中表示:“在过去的 20 年里，我致力于改善和提高互联网的安全性，不仅是为了数十亿用户，也是为了急需安全基础的企业。”。“我相信谷歌要搜索的内容，将会是商业:重塑在线可能性的全球基础设施。能够加入 Stripe 的优秀安全团队并与他们合作帮助在 Stripe 上运行的企业提高安全性，我感到非常兴奋。"

Provos 是 Stripe 一系列增强安全人才举措中的最新一个，而此时安全漏洞正处于历史最高水平，金融服务行业是这一增长的最大目标。我们都知道 Equifax 是一个漏洞(不幸的是，我们中的一些人比其他人更痛苦)，但是还有更多。埃森哲的一项研究估计，违规数量每年增长近 10%，修复违规的成本每年增长 40%左右，目前每个组织的成本约为 1800 万美元。

在这种情况下，Stripe 引入顶尖人才来领导其工作并增强行业对其服务的信心也就不足为奇了(因为，尽管 Stripe 拥有数百万客户，但其核心业务支付领域仍存在大量竞争)。但这是一个长期的过程，不仅仅需要招聘人才，还需要推出类似雷达的产品来防止欺诈。

有趣的是，据我所知，普罗沃斯被另一位安全明星穆奇雇佣，他在 2017 年 5 月加入 Stripe，也是该公司的安全主管。这是怎么回事？穆奇显然还在，但不是那个角色。

Stripe 的工程总监大卫·辛格尔顿(David Singleton)说:“我们很幸运能够与世界上一些顶尖的安全专家合作，特别是穆奇，他对的贡献是巨大的。“Mudge 加入我们的具体目标是建立一个世界一流的安全团队，这正是他所做的——雇用数十名安全领导者并组建团队，专注于数据隐私、安全工程、威胁操作、应用程序安全等。他还帮助我们雇佣了尼尔斯，他认识尼尔斯已经超过 20 年了。虽然他的大部分初步工作已经完成，但我们很感谢 Mudge 在 Stripe 的继续支持。”
