脸书的 GraphQL 拥有自己的开源基础

原文：https://web.archive.org/web/https://techcrunch.com/2018/11/06/facebooks-graphql-gets-its-own-open-source-foundation/

脸书孵化的数据查询语言 GraphQL 正在进入自己的开源基础。像许多其他类似的开源基金会一样，名副其实的 GraphQL 基金会将由 T2 Linux 基金会托管。

脸书早在 2012 年就宣布了 GraphQL，并在 2015 年将其开源。如今，从 Airbnb 到奥迪、GitHub、网飞、Shopify、Twitter 和《纽约时报》都在使用它。在脸书，GraphQL API 每天支持数十亿次 API 调用。在其核心，GraphQL 基本上是一种用于从客户端应用程序查询数据库的语言，以及后端 API 应该如何将这些数据呈现给客户端的一组规范。它为基于 REST 的 API 提供了一种替代方案，并承诺为开发人员提供更大的灵活性以及编写更快、更安全的应用程序的能力。几乎每一种主要的编程语言现在都通过各种各样的库支持它。

“GraphQL 重新定义了开发人员使用 API 和客户端-服务器交互的方式。我们期待着与 GraphQL 社区合作，成为一个独立的基金会，起草他们的治理，并继续促进 GraphQL 的发展和采用，”Linux 基金会开发人员关系副总裁 Chris Aniszczyk 说。

正如 Aniszczyk 指出的，新的基金会将有一个开放的治理模型，类似于其他 Linux 基金会项目。不过，具体细节仍在研究中。创始成员的名单也仍在变化，但目前包括 Airbnb、Apollo、Coursera、Elementl、脸书、GitHub、Hasura、Prisma、Shopify 和 Twitter。

“我们很高兴欢迎 GraphQL 基金会加入 Linux 基金会，”Linux 基金会的执行董事吉姆·泽姆林说。“这一进步非常重要，因为它允许长期支持和加速这一重要的突破性技术的发展，这一技术正在改变任何语言的云连接应用程序的 API 设计方法。”

目前，创始成员预计, GraphQL 规范、 GraphQL.js 、参考实现、数据加载器、库和graph QL开发者工具将成为基金会的核心技术项目，但这也仍然可能发生变化。

在这一点上，Linux 基金会本质上是基金会的基金会。它现在支持几十个项目，Linux 本身就是其中之一。这些其他基金会包括云本地计算基金会(Kubernetes 的家)、云铸造基金会、汽车级 Linux、JS 基金会(即将与 Node.js 基金会合并)等等。

随着越来越多的大公司发布开源项目，那些受欢迎的项目通常会发展到这样的地步:让一家公司来管理项目的生命周期既不可行，也不符合社区的最佳利益。 Spinnaker ，网飞和 Google 支持的持续交付平台，最近达到了这一点，比如。当然，GraphQL 现在也在这一点上，它是稳定的，并被广泛采用，但可以受益于从母公司分离出来，并获得自己的供应商中立的基础。
