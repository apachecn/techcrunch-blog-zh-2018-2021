Android 开发者现在可以强迫用户更新他们的应用

原文：https://web.archive.org/web/https://techcrunch.com/2018/11/07/android-developers-can-now-force-users-to-update-their-apps/

在 Android 开发峰会上，谷歌今天宣布了一系列新的工具和功能，供为其移动操作系统编写应用程序的开发者使用。其中一些不足为奇，包括对最新发布的 Kotlin 语言的支持，该语言在 Android 开发者生态系统中越来越受欢迎，以及 Android Jetpack 工具和 API 的新功能，以及 Android Studio IDE。然而，最大的惊喜可能是应用内更新 API 的推出。

虽然这个名字听起来不像是一个突破性的特性，但它实际上是一件大事。有了这个新的 API，开发者现在有两种新的方式来推动用户更新他们的应用程序。

“这是开发人员长期以来一直问我们的问题——假设你拥有一个应用程序，你想确保用户运行的是最新版本，”谷歌 Android 产品管理和开发人员关系高级总监斯蒂芬妮·萨阿德·卡斯伯特森告诉我。"这是开发商真正担心的事情."

假设您的应用程序带有一个主要的 bug(这种情况时有发生…)，并且希望确保每个用户都能立即升级；你很快就可以向他们显示全屏阻止消息，该消息将在他们第一次再次启动应用程序和应用更新时显示。这显然只适用于主要错误。第二个选项允许更大的灵活性，并允许用户在下载更新时继续使用应用程序。开发人员可以完全定制这些更新流程。

目前，新的更新 API 正在与一些合作伙伴进行早期测试，计划很快向更多的开发者开放。

正如卡斯伯特森所强调的，该团队近年来的重点是给开发者他们想要的东西。她指出，典型的例子是科特林语。“这不是谷歌设计的语言，也许不是显而易见的选择——但它确实是最好的选择，”她告诉我。“回顾过去几年，你会发现这是一项始于 IDE 的投资。它实际上只有五年的历史，从那以后，我们一直在完全基于开发者的反馈来构建它。”

今天，该公司宣布 46%的专业开发人员现在使用 Kotlin，仅上个月就有超过 118，000 个新的 Kotlin 项目在 Android Studio 中启动(这还只是来自选择与谷歌共享指标的用户)，因此投资肯定是有回报的。

然而，开发者最近抱怨的一件事是 Android Studio 的构建时间变慢了。“我们在内部看到的是构建时间变得越来越快，但我们从外部开发者那里听到的是他们变得越来越慢，”卡斯伯特森说。“因此，我们开始进行基准测试，不仅在内部可控的情况下，而且对于任何选择参与的人，我们开始对整个生态系统进行基准测试。”该团队发现，Android Studio 构建系统的核心 Gradle 变得快了很多，但你构建的系统和平台也有重大影响。卡斯伯特森指出，例如，Spectre 和 Meltdown 补丁对 Windows 和 Linux 用户产生了重大影响，定制插件也是如此。因此，展望未来，该团队正在构建新的分析和分析工具，以使开发人员能够更深入地了解他们的构建时间，谷歌将构建更多自己的插件来加速性能。

其中大部分还没有出现在当前的 Android Studio 3.3 测试版中(3.3 的测试版 3 也将在今天发布)，但 Android Studio 用户可能会很高兴听到的一件事是，Chrome OS 将在明年年初获得对 IDE 的官方支持，使用 Chrome OS 运行 Linux 应用程序的新能力。

该公司今天宣布的其他更新包括新的 Jetpack 架构组件库，用于导航和工作管理器，使开发人员可以更容易地将 Android 的导航原则添加到他们的应用程序中，并执行后台任务，而不必编写大量的样板代码。Android 应用捆绑包允许开发人员模块化他们的应用程序，并按需提供部分应用程序，也有一些更新，即时应用程序也是如此，用户无需安装即可运行。使用即时应用程序的网址现在是可选的，在 Android Studio 中创建它们变得更加容易。
