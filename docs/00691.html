AWS 为 EC2 TechCrunch 发布基于 Arm 的服务器

原文：https://web.archive.org/web/https://techcrunch.com/2018/11/27/aws-launches-arm-based-servers-for-ec2/

在拉斯维加斯的 re:Invent 会议上，AWS 今天宣布为其 EC2 云计算服务推出基于 Arm 的服务器。不过，这些不是普通的 Arm 芯片。AWS 采用标准 Arm 内核，并对其进行定制以满足其需求。该公司表示，其所谓的 AWS Graviton 处理器已针对性能和成本进行了优化，重点是可分布在多个较小实例中的横向扩展工作负载(想想集装箱化微服务、网络服务器、缓存车队等)。).

第一组实例被称为 A1，现在已在美国和欧洲的许多 AWS 地区上市。它们支持所有 AWS 的标准实例定价模型，包括按需、预留实例、现货实例、专用实例和专用主机。

目前，您只能使用亚马逊 Linux 2、RHEL 和 Ubuntu 作为这些机器的操作系统，但是 AWS 承诺未来会推出额外的操作系统支持。

因为这些都是 ARM 服务器，所以在运行应用程序之前，您显然必须为它们重新编译任何本机代码。然而，几乎任何用脚本语言编写的应用程序都可能在没有任何修改的情况下运行。

对于具有 1 个 CPU 和 2 GiB RAM 的 a1.medium 机器，这些实例的价格从 0.0255 美元/小时开始，对于具有 16 个 CPU 和 32 GiB RAM 的机器，价格高达 0.4080 美元/小时。考虑到基于 X86 的 t3.nano 服务器的起价为 0.0052 美元/小时，这可能没有您预期的那么便宜，但是当然，通过使用 spot 实例，您总是可以节省相当多的费用。不过，在我们看到一些基准测试之前，很难比较这些不同的机器类型。

正如亚马逊的杰夫·巴尔在今天的公告中指出的，该公司转向所谓的 Nitro 系统现在允许它以更快的速度推出新的实例类型。Nitro 本质上提供了创建新实例类型的构建块，团队可以根据需要混合和匹配这些类型。

值得注意的是，AWS 也在本月早些时候推出了对 AMD EPYC 处理器的 T4 支持。
