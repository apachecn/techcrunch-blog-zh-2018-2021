谷歌跨平台 Flutter UI 工具包推出 1.0 版 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2018/12/04/googles-cross-platform-flutter-ui-toolkit-hits-version-1-0/

Google 用于构建移动 Android 和 iOS 应用程序的 UI 工具包 Flutter 今天发布了 1.0 版本。此外，谷歌今天还宣布了一系列与 Square 等公司的新第三方集成，以及一些使 Flutter 更容易与现有应用程序集成的新功能。

开源的 Flutter 项目在谷歌 2017 年 I/O 开发者大会上首次亮相。自那以后，它的受欢迎程度迅速增长，Groupon、Philips Hue、腾讯、阿里巴巴、Capital One 和其他公司已经用它开发了应用程序，尽管它还没有推出 1.0 版本，而且开发者必须用 Dart 语言编写他们的应用程序，这是一个额外的准入门槛。

谷歌表示，总的来说，开发者已经向苹果和谷歌应用商店发布了“数千个”Flutter 应用。

“Flutter 是我们的便携式 UI 工具包，用于在单一代码基础上为 iOS 和 Android 创建美丽的原生体验，”谷歌 Dart 集团产品经理 Tim Sneath 解释道。“我们正在解决的问题是当今大多数移动开发者面临的问题。作为一名开发人员，你不得不做出选择。无论是构建 iOS 应用还是 Android 应用，您都可以使用 platform SDK 原生构建应用。然后你必须建造它们两次。”

Sneath 在 2017 年加入谷歌之前也是微软 Silverlight 团队的一员，所以他在学习这个跨平台开发领域的什么是行不通的方面有一些经验。不过，众所周知，脸书正试图用 React Native 解决一个非常类似的问题，React Native 也很受欢迎。

“我的意思是，React Native 显然是一项被证明非常受欢迎的技术，”Sneath 说。“React 本地开发人员面临的一个挑战，或者说是过去报道过的一个挑战是，React 本地代码是用 JavaScript 编写的，这意味着它是使用浏览器的 JavaScript 引擎运行的，这立即有点偏离了平台的本地模型。它们与生俱来的一点是，它们使用操作系统自己的控件。虽然从表面上看，这似乎是一件好事，但在实践中，这给开发人员带来了相当多的兼容性挑战。”

谷歌显然相信，它编译本机代码的能力——以及随之而来的速度提升——让它的平台在竞争中脱颖而出。在某种程度上，它通过使用硬件加速的 2D 引擎来实现这一点，当然，也通过将 Dart 代码编译为 iOS 和 Android 的原生 ARM 代码来实现。该公司还强调，开发者可以完全控制屏幕上的每个像素。

随着今天的发布，谷歌还宣布了新的第三方集成。第一个是 Square，它宣布了两个新的 Flutter SDKs，用于构建支付流，既用于应用内体验，也用于使用 Square 阅读器的面对面终端。其他的还有 2Dimensions ，用于构建矢量动画并将它们嵌入到 Flutter 中，还有 Nevercode ，它宣布了一个用于自动构建和打包 Flutter 应用程序的工具。

至于新的 Flutter 功能，谷歌今天宣布了“添加到应用程序”，这是一个新功能，可以让开发者更容易地将 Flutter 代码慢慢添加到现有的应用程序中。在早期，Flutter 的重点是从头开始构建新的应用程序，但随着它越来越受欢迎，开发人员现在希望在更新现有应用程序时将其用于现有应用程序的一部分。

另一个新功能是“平台视图”，它本质上与“添加到应用程序”相反，因为它允许开发者在他们的 Flutter 应用程序中嵌入 Android 和 iOS 控件。
