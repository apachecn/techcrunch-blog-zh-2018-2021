众议院报告 TechCrunch 称，如果 Equifax 采取了基本的安全措施，其漏洞是“完全可以避免的”

原文：https://web.archive.org/web/https://techcrunch.com/2018/12/10/equifax-breach-preventable-house-oversight-report/

众议院监督委员会周一发布的一份报告得出结论认为，Equifax 的安全实践和政策低于标准，其系统陈旧过时，并且困扰于基本的安全措施——如修补易受攻击的系统——本可以防止去年的大规模数据泄露。

一年多前，世界上最大的信用评级机构之一 Equifax(T3)证实其系统被黑客入侵,。全球约有 1.43 亿消费者受到影响，其中大部分在美国，但也有加拿大和英国的消费者，这一数字后来上升到 1.48 亿。然而，到目前为止，这家公司几乎没有受到任何影响，尽管一系列公司故障导致了历史上最大的数据泄露事件之一。

众议院的报告措辞严厉，批评了 Equifax 前首席执行官理查德·史密斯对黑客攻击的处理——他在黑客入侵后“退休”。

史密斯吹嘘说，这家信贷巨头每天持有的数据是国会图书馆的“近 1200 倍”，但众议院的报告称，Equifax“未能实施足够的安全计划来保护这些敏感数据。”

“这种违规行为是完全可以避免的，”报告称。

该报告证实了大部分已知的东西，但增加了以前没有报道过的新的色彩和见解。该信用机构未能 修补 Apache Struts 中的一个公开漏洞，这是一个常见的开源 web 服务器，国土安全部在 几个月前就已经发布了关于 的警告。未打补丁的 Apache Struts 服务器驱动着它 50 年前的(！)面向网络的系统，允许消费者从该公司的网站上查看自己的信用评级。众议院委员会发现，攻击者利用该漏洞在几周后在服务器上弹出了一个 web shell，并成功保持了两个多月的访问权限，并能够通过在一台服务器上获得一个未加密的密码文件来浏览该公司的各种系统，让黑客访问超过 48 个包含未加密消费者信贷数据的数据库。

在此期间，黑客对数据库发出了 9000 多次查询，在 265 个不同的场合下载了数据。

Equifax 的前老板 Smith 将未能修补 Struts 系统的责任推给了一名 IT 员工。众议院报告发现，事实上，这只是该公司对数据安全漫不经心态度的又一个例子。

“Equifax 没有看到数据泄露，因为由于安全证书过期，用于监控(易受攻击的服务器)网络流量的设备已有 19 个月没有使用，”报告称。Equifax 又花了两个月更新过期的证书，此时员工“立即注意到可疑的网络流量。”就连 Equifax 自己的前首席信息官大卫·韦伯(他也在事件发生后“退休”)也告诉众议院调查人员，如果该公司在补丁发布后两天内更新了易受攻击的 Struts 系统，整个事件本来是可以避免的。

“如果该公司在此次网络攻击之前采取行动解决其明显的安全问题，数据泄露本来是可以避免的，”报告称。

两个月后，Equifax 上市了。那也不轻松。

当 Equifax 的“你有风险吗？”网站没有崩溃，这是喷出不正确的结果。然后这个网站很快被假冒——并且被 Equifax 自己的社交媒体员工无意中链接到了。当忧心忡忡的消费者最终登陆该网站时，他们获得了 Equifax 自己的信用冻结服务，这种服务正在淘汰脆弱的 PIN 码——这是唯一一种保护消费者本已脆弱的信用的服务。在另一名安全研究员发现信用冻结网站的一个漏洞让攻击者窃取敏感的消费者数据后，该网站后来被关闭。与此同时其呼叫中心超负荷运转，许多人挣扎着寻找基本问题的答案。

总之，众议院的报告没有隐瞒其批评——抨击信用评级机构糟糕的安全做法，特别是考虑到所涉及的数据——报告指出，消费者“没有能力选择退出这一信息收集过程。”

Equifax 对众议院报告的回应？采取守势。

Equifax 发言人 Wyatt Jefferies 表示:“委员会选择不给我们足够的时间来审查和回应这份长达 100 页的报告，其中包含高度技术性和重要的信息，我们对此深感失望。”“在我们被给予进行初步审查的几个小时内，我们发现了重大的不准确之处，并不同意许多事实调查结果，”声明继续说。

“这是不幸的，破坏了我们协助委员会为那些希望从我们管理 2017 年网络安全事件的经验中学习的人提供可信和全面的公共资源的希望，”声明继续说。

当 TechCrunch 要求提供那些“重大错误”时，发言人返回了一份“事实错误”的列表，而不是指出与报告的实质性差异，包括 Equifax 提供两年的信用监控，而不是报告中所说的一年，以及报告提到与州司法部长的明显和解并没有发生。
