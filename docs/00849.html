Google+ bug 允许开发者访问 5250 万用户的非公开数据 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2018/12/10/google-security-bug-gave-developers-access-to-non-public-data-from-52-5m-users/

Google+在它还活着的时候对公司来说是一场灾难，现在它已经是行尸走肉了，它变得更像是一块套在它脖子上的石头。在 10 月份披露了一个影响不到 50 万用户的重大安全漏洞后，它宣布这项服务将于 2019 年 8 月关闭。但是事情越来越糟。今天，公司宣布上个月发现了一个新的隐私漏洞，使得大约 5250 万用户的一些数据被使用 Google+ API 的应用程序窃取。

因为每一个错误似乎都会提前 Google+的关闭日期，谷歌今天也宣布该服务将于 2019 年 4 月关闭。所有 Google+API 将在未来 90 天内关闭。

这个新的 bug 在 11 月初只存在了大约 6 天，它与 Google+ People API 有关。它允许申请权限查看用户个人信息(如姓名、电子邮件地址、职业、性别、生日、关系状态和年龄)的应用程序访问这些信息，即使这些数据被设置为非公开。

然而，情况变得更糟了。可以访问这些数据的应用程序也可以访问其他 Google+用户已经与该用户共享但没有公开共享的个人资料数据。

谷歌表示，它没有任何证据表明开发者曾经意识到他们可以访问这些数据(我猜，这是运行一个很少人仍在使用的社交网络的优势)，或者这些数据被以任何方式滥用。它还强调，这些应用程序只能在六天内访问这些数据。从今年 11 月 7 日到 13 日，在一周的时间内引入、检测并修复了该漏洞。

上一次，谷歌因等待太久才披露漏洞而受到严厉批评。这一次，公司内部知情人士告诉我，他们在经历了内部披露流程后，决定迅速做出反应，部分原因是公司希望表现出更大的透明度。

该公司今天在一篇博客文章中写道:“我们知道，我们制造保护您数据的可靠产品的能力会赢得用户的信任。”。“我们一直非常重视这一点，并继续投资于我们的隐私计划，以完善内部隐私审查流程，创建强大的数据控制，并与用户、研究人员和决策者合作，以获得他们的反馈并改进我们的计划。我们永远不会停止为每个人建立隐私保护的工作。”
