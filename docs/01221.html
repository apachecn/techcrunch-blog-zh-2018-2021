GitHub 免费用户现在可以获得无限的私有库

如果你是 GitHub 用户，但你没有付费，这是一个好的一周。历史上，GitHub 总是提供免费账户，但是警告是你的代码必须是公开的。要获得私人储存库，你必须付费。从明天开始，这个限制就没有了。免费的 GitHub 用户现在可以获得最多三个合作者的无限私人项目。

协作者的数量实际上是唯一的限制，服务处理公共存储库的方式没有变化，仍然可以有无限的协作者。

这感觉像是代表微软的善意信号，去年 10 月完成了对 GitHub 的收购，前 Xamarin 首席执行官 Nat Friedman 接任 GitHub 首席执行官。一些开发者对这次收购相当紧张(尽管感觉大多数人已经接受了)。假设 GitHub 的服务盈利模式与微软略有不同也是合理的猜测。微软不需要试图从小团队那里赚钱——那不是它大部分收入的来源。相反，该公司最感兴趣的是让大型企业使用这项服务。

谈到团队，GitHub 今天还宣布将 GitHub 开发者套件的名称改为“GitHub Pro”该公司表示，这样做是为了“帮助开发者更好地确定他们需要的工具。”

但可能更重要的是，GitHub 商业云和 GitHub 企业(现在称为企业云和企业服务器)已经合二为一，现在以“GitHub 企业”的标签销售，并按用户定价。

免费私有库的宣布可能让 GitHub 的一些竞争对手大吃一惊，但下面是我们从 GitLab 首席执行官 Sid Sijbrandij 那里听到的消息:“GitHub 今天宣布与多达三个合作者推出免费私有库。从一开始，GitLab 就在私有库上提供了无限的合作者。我们认为微软更关注 Azure 创收，而不是对 DevOps 软件收费。在 GitLab，我们相信组织使用多个公共云平台的多云未来。”
