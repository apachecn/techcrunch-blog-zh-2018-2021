Square 推出应用内支付 SDK

Square 今天宣布推出其应用内支付 SDK ，该 SDK 允许开发者将 Square 支持的支付嵌入他们的移动应用。虽然 Square 仍然以其离线支付解决方案而闻名，该解决方案为几乎所有独立的咖啡店和古怪的街角商店提供服务，但该公司长期以来一直提供用于在网络上进行在线支付和与其阅读器硬件配合使用的 API。

今天的发布将该公司的触角延伸到了移动应用，在这个领域，它面临着 Stripe、Adyen 等公司的激烈竞争。然而，Square 认为，鉴于它现在提供完整的在线和离线支付解决方案，这一推出使它领先于竞争对手。 “随着将应用内移动支付引入 Square 平台， 开发者 现在拥有了一个完整的全渠道支付解决方案，可以满足他们所有的支付需求，”Square 开发者负责人 Carl Perry 在今天的发布会上表示。“从软件到硬件再到服务， Square 在一个紧密结合的开放平台中提供完整的支付体验。更好的是， 开发者 和卖家可以在一个地方管理他们所有的店内、移动和在线支付。”

该 SDK 可用于 Android、iOS 和 Flutter，后者是谷歌用于构建跨平台应用的工具包。不过，目前只有美国、加拿大、英国、澳大利亚和日本的开发者能够使用它。该应用程序提供了一个默认的支付流程，但开发者也可以定制它来匹配他们的应用程序和需求。使用这项服务，移动应用程序开发者将能够通过通常的信用卡和借记卡以及 Apple Pay 和 Google Pay 进行支付。
