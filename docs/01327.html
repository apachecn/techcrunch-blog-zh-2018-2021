Unity 取消了云游戏初创公司的核选项，终止了游戏引擎许可

原文：https://web.archive.org/web/https://techcrunch.com/2019/01/10/unity-pulls-nuclear-option-on-cloud-gaming-startup-improbable-terminating-game-engine-license/amp/

一对资金雄厚的游戏独角兽正在公开交锋，这笔交易可能会对游戏开发商产生重大影响。

今天，总部位于英国的云游戏初创公司impossible宣布，非常受欢迎的游戏开发引擎 Unity 已经终止了他们的许可，实际上把他们从他们最大的客户来源之一拒之门外。如果是永久性的，许可终止将是对 Improbable 的重大打击，它使工作室能够在多个服务器上托管大型在线多人游戏。这家游戏初创公司已经从软银、安德森·霍洛维茨和地平线风险投资公司等顶级投资者那里筹集了超过 6 亿美元的资金。

究竟有多少不太可能的客户通过 SpatialOS GDK 将 Unity 作为他们的游戏引擎选择还不得而知，但这两个平台在寻求创新的小团队中确实有一些相似之处。“Unity 是一个受欢迎的引擎，这种受欢迎程度延伸到了使用我们(游戏开发工具包)的人，”一位不太可能的发言人告诉 TechCrunch。不太可能的 SpatialOS 平台也运行在虚幻引擎和 CryEngine 上，并且可以设计为与定制引擎一起工作。

这是怎么发生的？

不可思议的方式今天早上告诉它，Unity 上个月改变了他们的服务条款，然后在没有任何警告的情况下，把地毯从他们下面拉了出来。然而 Unity 并不这么认为，该公司写了一篇简短的博客文章回应，声称不可能很清楚他们违反了 ToS。

“一年多前，我们当面告诉 Improbable，他们违反了我们的服务条款或 EULA。六个月前，我们以书面形式通知了不可能的违规行为。最近的行动对“不可能”来说并不意外；事实上，他们已经知道这件事好几个月了，”帖子上写道。

使用 SpatialOS 的 Unity 开发人员花了一天时间抱怨这一举动，并想知道他们正在开发的项目是否必须彻底改造。虽然不可能的人似乎也不确定这个细节，但 Unity 在其博客帖子中澄清说，SpatialOS 项目仍将得到支持。

Unity 的服务条款并不完全是最清晰的阅读材料，但标题为流媒体和云游戏限制的部分似乎对不可能的事情进行了相当明确的指责。

未经 Unity 单独许可或授权，您不得通过流或广播的方式直接或间接分发 Unity 软件，包括 Unity 软件的运行时部分(“ Unity 运行时”)或您的项目内容(如果包含 Unity 运行时)，以便 Unity 软件的任何部分主要在云或远程服务器上执行或模拟，并通过互联网或其他网络传输到最终用户设备。

语言的模糊似乎给了 Unity 挥舞大锤的自由裁量权。

那么，问题是为什么“不可能”似乎成了目标。在被要求置评时，Unity 的发言人让我们参考他们的博客帖子。答案可能在于 Unity 和 impossible 暗示他们正在达成的“伙伴关系”,即如果他们要在 impossible 服务器上托管 Unity 运行时，Unity 可能希望 impossible 付钱，但两者无法达成协议。

Epic Games 首席执行官蒂姆·斯维尼的公司运营着竞争对手虚幻引擎，他似乎在 Twitter 上指责 Unity，建议引擎需要在其建立的管理规则中更加透明。

不管怎样，现在看起来不可能意识到他们可能惹错了强大的合作伙伴。

在今天下午晚些时候发表的一篇更加悔悟的博文中，该团队写道:

我们很抱歉，这个事件，我们煽动了这么多的不确定性，混乱和痛苦，为这么多的开发人员谁真的不值得这样…

作为一个平台公司，我们相信谦逊和自省是对你的社区的痛苦的重要回应，无论它是如何发生的。我们邀请参与今天讨论的每一家公司都为此做一点努力。

我们也邀请统一和我们一起参与到这个更广阔的思考中，不管我们误解的结果如何。你是一家非常重要的公司，一天的不顺并不能抵消你给我们的一切。让我们为我们的社区解决这个问题，你知道我们的号码。

听起来似乎不可能在一天中充分意识到他们将不得不在这里妥协。
