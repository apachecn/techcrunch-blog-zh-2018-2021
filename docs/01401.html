用这个可教的“寄生虫”从窥探的智能扬声器手中夺取控制权

原文：https://web.archive.org/web/https://techcrunch.com/2019/01/16/wrest-control-from-a-snooping-smart-speaker-with-this-teachable-parasite/

当你把一个联网设备放在另一个上面时，你会得到什么？比你在Alias“可教的‘寄生虫’”的情况下要多一点控制力，Alias是由两位设计师Bjorn Karmann和Tore Knudsen制作的物联网项目智能扬声器礼帽。

树莓Pi驱动、真菌启发的blob的任务是向亚马逊Alexa(或谷歌Home)的永远在线的耳朵低语甜言蜜语，这样它就不会意外地窥探你的家。

项目别名来自比约恩·卡曼在 Vimeo 上。

Alias只有在听到自己的唤醒命令时，才会停止向主机的扬声器输入噪音，唤醒命令可以是您喜欢的任何命令。

中间人物联网设备有自己的本地神经网络，允许其所有者通过配套应用程序中的训练界面，以他们选择的名称(或声音)命名它。

开源TensorFlow库用于构建名称训练组件。

因此，你不必说“Alexa”或“Ok Google”来与商业智能扬声器交谈——从而在自己家里鹦鹉学舌一个大的技术品牌，更不用说背负着一个总是容易受到语音恶作剧(更糟糕的是:意外窃听)的设备——你可以控制唤醒词是什么，从而收回对一项天生敌视隐私的技术的一点控制权。

这意味着你可以把Alexa重新命名为“Bezosallseeingeye”，或者把你的谷歌主页称为“Carelesswhispers”。不管什么能让你开心。

一旦Alias听到其自定义唤醒命令，它将停止向主机扬声器输入噪音，从而使底层智能助理能够正常听到并响应命令。

“我们研究了冬虫夏草真菌和病毒如何利用和控制昆虫来实现自己的议程，并受到启发，为智能家居系统创造了我们自己的寄生虫，”卡门和努森在这里的项目的书面报道中解释道。“因此，我们启动了项目Alias，通过将设计师的更多权力下放给产品的最终用户，展示如何利用创客文化来重新定义我们与智能家居技术的关系。”

如果你想知道在你与语音助手聊天后，你如何知道麦克风何时再次被安全阻止，卡门告诉我们:“由于Alexa和谷歌新的连续转换功能，在Alias再次返回阻止麦克风之前，有一个内置的30秒时间框架。一旦助手被激活，外壳内的明亮LED就会亮起，应用程序也会立即给出反馈。”

虽然物联网隐私盾是Alias的直接用例，但卡门还设想用户能够使用该设备创建其他语音快捷方式，并与底层技术建立更友好和舒适的关系。

“由于Alias本质上是一个中间人设备，它可以说不仅仅是唤醒词。我们可以想象用户写他们自己的回复和快捷方式。例如:说“天气”和别名这个词可以触发助手，询问它今天的天气预报。

Alias为物联网提供了一个富有创意和个性化的未来，因为生产定制但仍然强大的互联技术产品的方式变得更加实惠和容易获得。

对于那些不想完全放弃的人来说，这也可能是物联网隐私问题的部分答案。(尽管如此，在安全方面，更多定制和可控的物联网确实增加了可黑客攻击的表面积——所以这是另一个需要记住的因素；更多的自定义控制以获得更大的隐私并不一定与强大的设备安全性相匹配。)

“我们俩一开始就不会买智能音箱。但由于比约恩收到了一个开发设备，他很好奇，并将其视为一个研究的机会，最终导致沮丧和一个聪明的想法。卡门补充道:“今天，我很高兴地使用了一个完全改名为‘马文’的谷歌主页。

如果你渴望拥有自己的破坏Alexa的blob-topper，这两人已经上传了一个构建指南到 Instructables ，并将源代码放在 GitHub 上。所以穿上你的靴子。

Project Alias当然不是智能助手底层跟踪问题的解决方案——智能助手从语音命令中收集见解，以进一步充实用户的兴趣档案，包括用于广告定位目的。

这将需要适当的隐私监管，或者，呃，一种新的软件病毒渗透到主机系统，阻止它访问用户数据。而且，与这种创造性的物理物联网附加产品不同，这种技术根本不合法。
