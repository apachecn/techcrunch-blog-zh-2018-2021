BetterCloud 现在可以管理任何 SaaS 应用程序

BetterCloud 的诞生是为了给 G Suite 提供一个操作层。最近，在的一次平台革新之后，它开始在少数其他 SaaS 应用上分层。今天，该公司宣布，现在可以将任何 SaaS 应用程序添加到其运营仪表板，并通过 API 监控应用程序的使用情况。

正如创始人兼首席执行官 David Politis 解释的那样，像 Okta 这样的工具提供了一种验证你的 SaaS 应用程序的方法，但一旦员工开始使用它，BetterCloud 就会让你看到它是如何被使用的。

“首要问题是身份、权限和联系。我们正在做的是解决二阶问题，也就是相互作用，”Politis 解释道。在他看来，当人们在组织内外交互和共享信息时，公司缺乏监控和理解跨 SaaS 应用程序的交互的能力。他解释说，BetterCloud 旨在为 IT 部门提供对其环境中发生的事情的控制和安全性。

他说，他们可以根据公司的需要提供或多或少的控制，他们可以按应用程序或跨多个应用程序设置控制，而不会实际改变用户的体验。他们通过脚本库做到这一点。BetterCloud 附带了许多脚本，并提供了日志访问功能，以提供对脚本活动的可见性。

如果客户希望更有效地使用这些数据，该解决方案包括一个图形 API，用于接收数据并查看 BetterCloud 正在收集的数据之间的联系。客户还可以根据满足特定条件时收集的数据设置事件触发器或操作。

所有这些都是可能的，因为该公司去年对平台进行了彻底改革，允许 BetterCloud 超越 G Suite，并将其他 SaaS 应用程序插入其中。今天的发布是这种能力的最终体现。BetterCloud 没有构建连接器，而是提供了一个 API 让客户来做这件事。

据 Crunchbase 报道，该公司成立于 2011 年，已经筹集了超过 1.06 亿美元。
