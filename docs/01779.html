微软 Azure 着眼于更多的分析工作负载

原文：https://web.archive.org/web/https://techcrunch.com/2019/02/07/microsoft-azure-sets-its-sights-on-more-analytics-workloads/

企业现在从他们自己的工具和应用程序以及他们使用的 SaaS 应用程序中积累了大量的数据。在很长一段时间里，数据基本上是穷尽的。也许它被存储了一段时间以满足一些法律要求，但后来它被丢弃了。现在，数据是驱动机器学习模型的因素，数据越多越好。因此，大型云供应商很早就开始投资数据仓库和湖泊也就不足为奇了。但这只是第一步。之后，您还需要分析工具来使所有这些数据变得有用。

今天，轮到微软将聚光灯投向其数据分析服务。这里的实际消息非常简单。其中两项服务正在走向全面可用:用于大数据分析工作负载的第二代 Azure Data Lake Storage 和 Azure Data Explorer ，这是一项托管服务，可以更轻松地对海量数据进行即席分析。微软还预览了 Azure Data Factory 的一项新功能，这是一项用于构建数据转换的图形无代码服务。数据工厂现在具有映射数据流的能力。

如果你是一名用户，或者正在考虑将 Azure 用于大数据工作负载，这些单独的新闻片段会很有趣，但这里更重要的是，微软正试图提供一套全面的工具来管理和存储这些数据，然后用它来构建分析和人工智能服务。

(图片来源:乔希·埃德尔森/法新社/盖蒂图片社)

“人工智能是全球每家公司的重中之重，”微软 Azure 公司副总裁朱莉娅·怀特(Julia White)告诉我。“随着我们与客户在人工智能方面的合作，很明显，他们的分析往往不足以构建一个人工智能平台。”这些公司正在产生大量的数据，然后这些数据必须被拉入分析系统。她强调说，她不记得最近几个月有哪次客户对话不是围绕人工智能展开的。“实现人工智能梦想迫在眉睫，”怀特说，但数据的增长和多样性给许多企业带来了重大挑战。“他们认为这是一项独立于核心系统的技术。现在，面向客户的应用程序和业务线应用程序都需要它。”

数据湖存储有助于管理各种数据，因为它可以处理结构化和非结构化数据(并针对 Spark 和 Hadoop 分析引擎进行了优化)。这项服务可以接收任何类型的数据——然而微软仍然承诺它会非常快。“分析的世界往往被定义为必须预先决定，然后围绕它建立僵化的结构，以获得您想要的性能，”White 解释说。另一方面，数据湖存储希望提供两个世界的最佳选择。

同样，White 认为，虽然许多企业过去一直将这些服务保存在内部服务器上，但其中许多企业仍然基于设备。但是她相信云计算现在已经达到了性价比对其有利的地步。然而，这需要一段时间来达到这一点，并说服企业。White 指出，很长一段时间以来，查看他们的分析项目的企业认为 3 亿美元的项目永远不会结束，会束缚很多人，坦率地说有点可怕。“但是，直到最近的一些工作，我们在云中提供的东西才令人惊叹，”她说。“我们已经踏上了征程，其他云供应商也是如此，现在的性价比极具吸引力。”如果企业想要实现他们的人工智能目标，他们现在也必须处理这些工作负载，这肯定会有所帮助。
