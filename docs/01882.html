Figma 获得 4000 万美元用于将设计工具放到云中

原文：https://web.archive.org/web/https://techcrunch.com/2019/02/14/figma-gets-40-million-series-c-to-put-design-tools-in-the-cloud/

随着越来越多的行业和组织认识到设计是商业的支柱，设计工具制造商之间正在酝酿一场战争。有了新的 4000 万美元的 C 系列资金， Figma 准备战斗。

联合创始人兼首席执行官迪伦·菲尔德解释说，当他和联合创始人埃文·华莱士在 2012 年创办公司时，IBM 每 72 名工程师中就有一名设计师。如今，IBM 每名设计师有八名工程师，这一比例在移动设备上达到了 3:1。

这种变化在各个行业中得到了更广泛的反映，这使得他们组织中更多的人想参与设计过程。这意味着曾经为小型设计团队和在孤岛中工作的个体自由职业者“完成工作”的工具不再有用了。

菲尔德发现了对实时协作设计工具的需求，于是退出了布朗大学，加入泰尔研究团队来构建 Figma。自推出以来，该公司已发展到 100 万个签约，通过 4.4 亿美元的资金后估值共筹集了 8290 万美元。

Figma 提供了一个 freemium 模型，该产品最多可免费提供 3 名编辑。从那里你遇到了专业层，它提供了无限的版本历史和能力，以 15 美元/月/编辑器创建一个设计系统。组织层捆绑了一个额外的安全和内容控制层，费用为 45 美元/月/编辑。

Figma 与众不同的很大一部分是它在网络上的家。Figma 允许设计师和合作者在一个 web 应用程序中处理这个过程的每个部分——从初始设计到协作、存储到原型制作。

“我们开始制作这些传统设计工具的云版本，”菲尔德说。“我们意识到，一旦您将所有这些都放在云中，并使之成为贯穿设计、存储、原型制作、开发人员移交和版本控制的整个工作流程……一旦您将所有这些联系起来，您实际上并没有创建所有这些不同的产品。您正在创建一个集成系统。”

正因为如此，像文件版本控制和实时协作这样的常见设计问题对 Figma 来说并不是真正的问题。设计人员可以一起工作，或者自己进行更改，这些更改会实时反映在文件中，并带有完整的修订历史。要分享新的东西，他们可以简单地通过链接发送。

Adobe 和 InVision，这一领域的另外两家大公司，都开发了本地应用程序来处理捆绑设计工具、协作原型制作和文件版本控制的全栈问题。Adobe 通过其协作设计工具 Adobe XD 应对了日益激烈的竞争。2011 年作为一个协作原型开发平台成立的 InVision，要么是建造了要么是购买了其在工作流程中向上和向下扩展的产品。

看起来，对于一些大的 设计 团队，Figma 的网络应用占据了上风——这解释了为什么红杉的合伙人 Andrew Reed 改变了主意。Figma 实际上在 2018 年筹集 B 轮资金时去了红杉，风投公司错过了这个机会。

“当时，该产品很有趣，但我们谈论这些产品的人并没有指出 Figma 正在改变他们的公司，”Reed 说。“在过去的 12 个月里，事情发生了变化。我们给人们打电话询问他们的意见，人们主动给我们打电话，告诉我们这对他们的公司有多大影响。”

里德说，在查看数据后，他发现红杉投资组合中的一半公司都有 Figma 用户。他联系了 Field，向 Figma 发送了一份条款清单，不到一周，Figma 就完成了这一轮可能被视为机会主义的融资，考虑到 Figma 最近刚刚获得了 b 轮融资。

但这笔交易的一个额外好处是 Reed 投资 GitHub 的经验，这是设计工具公司寻求为分散的格局带来某种程度的凝聚力的一个很好的例子。

“协作将嵌入软件的未来，”Reed 说。
