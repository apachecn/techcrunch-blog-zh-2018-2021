谷歌收购云迁移平台 Alooma

原文：https://web.archive.org/web/https://techcrunch.com/2019/02/19/google-acquires-cloud-migration-platform-alooma/?utm_source=xp&utm_medium=blog

谷歌今天宣布打算收购 Alooma，这是一家允许企业将其所有数据源整合到谷歌的 BigQuery、亚马逊的 Redshift、雪花和 Azure 等服务中的公司。Alooma 的承诺是，它为用户处理和管理数据管道。然而，除了这项数据集成服务，Alooma 还帮助迁移到云，清理这些数据，然后将其用于人工智能和机器学习用例。

“在谷歌云，我们致力于帮助企业客户轻松、安全地将其数据迁移到我们的平台，”谷歌工程副总裁 Amit Ganesh 和谷歌云平台产品管理总监 Dominic Preuss 今天写道。“Alooma 的加入是一个自然的选择，它使我们能够为客户提供简化、自动化的谷歌云迁移体验，并让他们访问我们的全方位数据库服务，从托管开源数据库产品到 Cloud Spanner 和 Cloud Bigtable 等解决方案。”

在收购之前，Alooma 已经融资约 1500 万美元，包括 2016 年初由 Lightspeed Venture Partners 和红杉资本领投的 1120 万美元 A 轮。两家公司没有透露收购的价格，但考虑到 Alooma 之前筹集的资金，我们谈论的价格可能是适中的。

谷歌和 Alooma 都没有透露现有产品和客户会发生什么，以及它是否会继续支持向谷歌竞争对手的迁移。我们已经联系了谷歌，一旦有更多消息，我们会更新这篇文章。

更新。以下是谷歌关于平台未来的声明:

目前，Alooma 和谷歌云的业务一切正常，我们正在等待监管机构的批准并完成交易。交易结束后，该团队将加入我们在特拉维夫和桑尼维尔的办公室，我们将利用 Alooma 技术和团队在未来为我们的谷歌云客户提供出色的数据迁移服务。

关于支持竞争对手，是的，现有的 Alooma 产品将继续支持其他云提供商。我们将只接受将数据迁移到谷歌云平台的新客户，但现有客户将继续访问其他云提供商。

因此，今后，Alooma 不会接受任何想要将数据迁移到任何竞争对手的新客户。这并不令人惊讶，很高兴看到谷歌将继续支持 Alooma 的现有用户。那些将 Alooma 与 AWS、Azure 和其他非谷歌服务结合使用的人可能会很快开始寻找其他解决方案，因为这也意味着谷歌不太可能为他们开发超出当前状态的服务。

不过，Alooma 的联合创始人强调，“旅程还没有结束。他们写道:“Alooma 一直致力于提供最简单、最有效的途径来标准化各种来源的企业数据，并将其转化为可操作的情报。”。“加入谷歌云将使我们更接近提供完全自助的数据库迁移体验，这种体验由他们的云技术的力量支持，包括分析、安全、人工智能和机器学习。”
