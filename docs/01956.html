戴姆勒为何将其大数据平台移至云端

原文：https://web.archive.org/web/https://techcrunch.com/2019/02/20/why-daimler-moved-its-big-data-platform-to-the-cloud/

与几乎所有大型企业一样，几年前，德国汽车巨头戴姆勒(Daimler)决定投资建设自己的内部数据中心。虽然这些不会很快消失，但该公司今天宣布，它已经成功地将其内部大数据平台迁移到微软的 Azure 云。这个新平台被该公司称为 eXtollo，是戴姆勒第一个在自己的数据中心之外运行的主要服务，尽管它可能不会是最后一个。

正如戴姆勒公司高级分析和大数据卓越中心的负责人 Guido Vetter 告诉我的那样，该公司大约在五年前开始对大数据感兴趣。“我们投资了技术——传统的方式，内部部署——并让几个人参与进来。我们正在研究我们可以用数据做什么，因为数据也正在改变我们的整个业务，”他说。

到 2016 年，组织的规模已经发展到需要一个更正式的结构来使公司能够在全球范围内处理数据的地步。当时，流行的说法是“数据湖”，该公司开始建立自己的数据湖，以增强其分析能力。

“迟早，我们会达到极限，因为运行这些大环境不是我们的核心业务，”维特尔说。“灵活性和可扩展性是人工智能和高级分析所需要的，而我们的整个运营并不是为此而建立的。我们的后端运营旨在保持工厂正常运转，确保一切安全无虞。”但是在这个新的企业 IT 世界中，公司需要能够灵活地进行实验——并且，如果必要的话，快速地抛弃失败的实验。

因此，大约一年半前，维特尔的团队启动了 eXtollo 项目，将该公司围绕高级分析、大数据和人工智能的所有活动纳入 Azure 云，就在两周前，该团队在缓慢开启其在欧洲、美国和亚洲的数据中心的解决方案后，关闭了其最后的内部服务器。总而言之，内部数据中心和 Azure 云之间的实际过渡花费了大约九个月的时间。这可能看起来不算快，但对于这样的企业项目来说，这已经是最快的速度了(有一段时间，它将所有新数据都输入了内部数据湖和 Azure)。

如果你为一家初创公司工作，那么所有这些可能看起来没什么大不了的，但对于像戴姆勒这样更传统的企业来说，即使只是放弃对数据所在的物理硬件的控制，也是一种重大的文化变革，需要相当多的说服力。最终，解决方案归结为加密。

“我们需要用我们自己的方法来保护微软数据中心的数据，确保只有我们能够访问原始数据并使用这些数据，”Vetter 解释道。最终，该公司决定使用 Azure Key Vault 来管理和轮换其加密密钥。事实上，Vetter 指出，知道公司完全控制自己的数据是让这个项目向前发展的原因。

Vetter 告诉我，该公司显然也关注了微软的竞争对手，但他指出，在功能和所需的安全特性方面，他的团队没有从其他供应商那里找到令人信服的报价。

如今，戴姆勒的大数据部门使用 HD Insights 和 Azure Databricks 等工具，覆盖了该公司 90%以上的当前用例。未来，维特尔还希望让经验较少的用户更容易使用自助服务工具来推出人工智能和分析服务。

虽然成本通常是不利于云的一个因素，因为租用服务器容量并不便宜，但维特尔认为，这一举措实际上会为公司节省资金，尤其是存储成本在云中会比在内部数据中心更便宜(鉴于戴姆勒作为客户的规模和声望，它可能不会像其他人那样为 Azure 服务支付相同的机架价格)。

正如许多大数据人工智能项目一样，预测是戴姆勒工作的重点。这可能意味着查看汽车的数据和错误代码，并帮助技术人员诊断问题或对商用车进行预测性维护。有趣的是，该公司目前没有将自己工厂的任何物联网数据带到云上。这些都在公司的内部数据中心进行管理，因为它希望避免因工具失去与数据中心的连接而不得不关闭工厂的风险。
