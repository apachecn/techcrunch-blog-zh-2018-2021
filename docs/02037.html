Cloudflare扩大其政府授权金丝雀

原文：https://web.archive.org/web/https://techcrunch.com/2019/02/26/cloudflare-warrant-canary/amp/

图片来源: Getty Images

当政府需要你的数据时，科技公司不可能总是告诉你。但是由于一个法律漏洞，公司可以说如果他们还没有访问。

这开启了一个有趣的条款，允许公司在政府秘密搜查其客户数据时悄悄警告客户，而不违反禁令。根据美国言论自由法，当没有数据需求时，公司可以公开说“政府不在这里”，但当搜查令到来时，他们可以删除声明，以警告任何关注的人。

这些所谓的“权证金丝雀”——以矿井中一只可怜的金丝雀命名，当有人类无法检测到的气体时，它就会死去——是一种关键的透明工具，主要关注隐私的公司使用它来让他们的客户了解幕后发生的事情。

在公司已经放弃他们的金丝雀或屈服于法律压力的地方，Cloudflare正在逆潮流而动。

这家网络和内容交付网络巨头本周在一篇博客文章中表示，T2正在扩展透明度报告，以包括更多的金丝雀。

迄今为止，该公司:

  • 从未将他们的SSL密钥或客户的SSL密钥交给任何人；
  • 从未在其网络上安装任何执法软件或设备；
  • 从未因政治压力而终止客户或删除内容；
  • 从未向任何执法机构提供通过其网络传输的客户内容。

这些要点对公司的业务至关重要。政府要求提供SSL密钥并在其网络上安装拦截设备将允许调查人员前所未有地访问客户的通信和数据，并破坏该公司的安全。类似的要求导致拉达·莱维森关闭了他的电子邮件服务Lavabit ,当他们寻找密钥以获取使用该服务的告密者爱德华·斯诺登的信息时。

现在，Cloudflare的授权金丝雀将包括:

  • Cloudflare从未应执法部门或其他第三方的要求修改客户内容。
  • Cloudflare从未应执法部门或其他第三方的要求修改DNS响应的预期目的地。
  • Cloudflare从未应执法部门或其他第三方的要求削弱、损害或破坏其任何加密技术。

它还扩大并替换了它的第一只金丝雀，以确认该公司“从未向任何人移交我们的加密或认证密钥或我们客户的加密或认证密钥。"

Cloudflare表示，如果它被要求做上述任何事情，该公司将“用尽所有法律补救措施”来保护客户数据，并从其网站上删除这些声明。

网络和内容交付网络是几年来使用权证金丝雀的少数几家大公司之一。有报道称，美国国家安全局(National Security Agency)正在批量从主要电信巨头那里收集通话记录，苹果在最近的透明度报告中发表声明称，该公司迄今为止“没有收到任何批量数据订单”Reddit在2015年移除了的权证金丝雀，表明它收到了不允许披露的国家安全命令。

Cloudflare的扩展版canaries包含在该公司本周发布的最新透明度报告中。

根据其涵盖2018年下半年的最新数据，Cloudflare仅回应了19项请求中的7项传票，影响了12个账户和309个域。该公司还回应了55项请求中的44项法院命令，涉及134个账户和19，265个域名。

该公司在此期间收到了0-249份国家安全请求，在此期间没有处理任何窃听或外国政府的请求。
