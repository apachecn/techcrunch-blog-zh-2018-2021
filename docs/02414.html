脸书承认它明文存储了“数亿”账户密码

原文：https://web.archive.org/web/https://techcrunch.com/2019/03/21/facebook-plaintext-passwords/

将“自上次脸书安全事件以来的天数”归零。

受网络安全记者 Brian Krebs 在的一篇报道的推动，脸书周四在的一篇博文中证实，该国多年来以明文形式存储了“数亿”账户密码。

脸书的 Pedro Canahuati 说，这一发现是在一月份的一次例行安全检查中发现的。他说，脸书以外的人看不到任何密码。几个月后，脸书承认了安全漏洞，此前克雷布斯表示，大约 2000 名工程师和开发人员可以访问日志。

克雷布斯说，这个 bug 可以追溯到 2012 年。

“这引起了我们的注意，因为我们的登录系统被设计成使用使密码不可读的技术来掩盖密码，”卡纳华提说。“迄今为止，我们没有发现任何证据表明任何人在内部滥用或不当访问它们，”但没有说明该公司如何得出这一结论。

脸书表示，它将通知“数以亿计的脸书 Lite 用户”，这是一个针对网速慢、带宽贵的用户的脸书的简化版本，以及“数以千万计的其他脸书用户”该公司还表示，“成千上万的 Instagram 用户”将收到曝光通知。

Krebs 表示，多达 6 亿用户可能会受到影响，约占该公司 27 亿用户的五分之一，但脸书尚未证实这一数字。

脸书也没有说这个 bug 是怎么来的。以可读的明文存储密码是一种不安全的存储密码的方式。像脸书这样的公司，使用哈希和盐密码——进一步打乱密码的两种方法——来安全地存储密码。这使得公司可以在不知道用户密码的情况下验证用户的密码。

Twitter 和 GitHub 去年也遭到了类似但独立的 bug。两家公司都表示，密码是明文存储的，没有加密。

这是该公司一系列令人尴尬的安全问题中的最新一起，促使国会调查和政府调查。据上周报道，脸书允许其他科技公司未经同意访问账户数据的交易正在接受刑事调查。

目前还不清楚为什么脸书花了几个月的时间来确认这一事件，也不知道该公司是否根据美国违规通知和欧洲数据保护法通知了州或国际监管机构。我们询问了脸书，但是一位发言人除了这篇博文之外没有立即发表评论。

负责脸书欧洲业务的爱尔兰数据保护办公室表示，该公司“向我们通报了这个问题”，监管机构“目前正在寻求进一步的信息”
