厨师走向 100%开源

原文：https://web.archive.org/web/https://techcrunch.com/2019/04/02/chef-goes-100-open-source/

广受欢迎的自动化服务 Chef 今天宣布在 Apache 2 许可下开源其所有软件。到目前为止，Chef 使用的是一个开放的核心模型，其中有许多专有产品来补充其开源工具。这些专有工具大多专注于企业用户及其安全性和部署需求。现在，所有这些工具，代表了 Chef 全部代码库的三分之一到一半，也是开源的。

“我们正在远离我们的开放核心模式，”产品和工程的厨师 SVP·科里·斯科比告诉我。“我们现在正转向专门的开源软件开发。”

他补充说，这也包括开放的产品开发。展望未来，该公司计划分享更多关于其路线图、功能积压和其他产品开发细节的细节。Chef 的所有商业产品也将基于相同的开源代码构建，每个人现在都可以访问这些代码。

斯科比指出，公司这样做有很多原因。例如，他认为构建软件的最佳方式是与实际使用它的人公开合作。

Scobie 说:“有了这种理念，我们就很容易证明如何将我们生产的软件的剩余部分开源。”“我们相信这是为人们——真实世界中的真实人物——开发软件的最佳方式。”

Scobie 说，另一个原因是 Chef 越来越难解释软件的哪些部分是开源的，哪些不是。“说实话，我们想让这种对话变得更容易。”

Chef 的决定是在开源世界的混乱时期做出的。Redis、MongoDB 和 Elastic 等许多公司最近已经转向许可，明确禁止 AWS 等大型云供应商对其开源产品的商业使用，除非他们也购买了商业许可。

但这里是厨师，开源一切。Chef 联合创始人兼董事会成员亚当·雅各布认为这不是问题。“在开放核心模型中，你是说价值在这个专有的小块中。你付给我的那部分是它的一小部分。我认为这是不正确的，”他说。"我认为，事实上，价值总是存在于产品的整体中."

雅各布还认为，那些转向这些新的、更具限制性的许可证的公司只会伤害他们自己。“事实证明，产品才是最重要的，”他说。“他们继续为客户生产优秀的企业软件，他们的客户继续感到满意并继续购买，这是他们一直以来的做法。”他还指出，他不认为 AWS 在经营 Elasticsearch 方面会比 Elastic 更好，或者在经营 Chef 方面比 Chef 更好。

值得注意的是，Chef 今天还宣布推出其企业自动化堆栈，该堆栈将所有 Chef 工具(Chef Automate、Infra、InSpec、Habitat 和 Workstation)集中在一个统一的保护伞 下。

Chef 首席执行官 Barry Crist 表示:“Chef 完全致力于帮助组织消除所有应用程序生命周期中的摩擦，确保无论他们是从我们的开源代码构建解决方案还是许可我们的商业分发，他们都可以从代码协作中受益。”。“Chef Enterprise Automation Stack 让团队能够为任何应用程序建立和维护一致的生产路径，以提高速度和效率，从而使任务关键型软件的部署和更新变得更容易、移动速度更快且工作顺畅。”
