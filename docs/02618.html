Rippling 以 2.7 亿美元融资 4500 万美元，成为商业应用身份层 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2019/04/03/rippling-45-million/

帕克·康拉德的上一个创业公司 Zenefits 淹没在繁忙的工作中。现在随着荡漾，他想把那片海洋煮沸。我们没有试图固定一件事情然后扩张，“与传统智慧相反，我们做了一件更广泛、更有雄心的事情。”这意味着花两年时间，让 40 名工程师秘密工作，与几乎所有流行的业务工具建立集成，将人力资源、IT 和单点登录服务结合起来。结果是，当你雇佣一名员工时，只需点击一下鼠标，就能让他们使用所有这些服务。再见，工作忙。你好，通往企业应用生态系统的门户。

在过去的几年里，涌现出了一大批为办公效率和协作开发专业软件的创业公司。但是这使得客户很难在所有这些分散的工具上建立他们的团队。因此，荡漾有一个非常好的第一年在市场上与快速增长的收入。所以当 Rippling 出去筹集资金时，Conrad 在一周多一点的时间里就签署了投资意向书。

四千五百万美元。“我知道现在的轮数更多了，但对于 A 系列来说，这已经相当可观了，”康拉德在旧金山的 Four Barrel 喝咖啡时咧嘴笑着告诉我。“我们希望继续在工程、投资和研发方面加倍努力，这样我们就拥有真正的产品优势和技术优势，尽管他们中的许多人比我们存在的时间要长得多。”消息的佐伊·伯纳德报道说，涟漪已经筹集了至少 3000 万美元。

碧波荡漾的一轮由凯鹏华盈(Kleiner Perkins)及其企业大师马蒙·哈米德(Mamoon Hamid)领衔。正如康拉德告诉我的，“你用来评估 SaaS 公司的许多指标都是马穆恩发明的。他确实知道他的东西。他也是一个真正伟大的人。”凯鹏华盈是他的梦中情人。“我记得我上高中时，凯鹏华盈是我唯一听说过的风险投资公司。当我还是个孩子的时候，我想‘哦，有一天那会很酷。’该轮投资由 Initialized Capital、Threshold Ventures(前身为 DFJ)和 Y Combinator 参与。

一位消息人士证实，该轮融资的估值达到了惊人的 2.7 亿美元。哈米德还对 Rippling 在推出之前试图与所有人融合持怀疑态度。但是，他说，“几年前担心的事情现在变成了我们喜欢公司的事情。”在推销了这么多零碎的企业解决方案后，Hamid 突然明白了为什么客户想要“一站式服务”。你需要一个独立的政党来充当粘合剂。”

典型地，企业软件是一团混乱。应用程序不会相互交流，所以当你雇用一名新员工时，你必须手动将他们、他们的角色、他们的团队、他们的经理、他们的权限等添加到你的团队使用的每一个工具中。有控制工资和福利的人力资源系统，决定给你发放什么设备的 IT 系统，Slack 和 Dropbox 等生产力和协作应用程序，以及 Salesforce 或 GitHub 等部门专用工具。Conrad 认为，每次招聘、解雇或晋升时手动更新这些信息是公司几乎所有行政工作的来源。

Zenefits 之所以发展如此之快，然后突然碰壁，是因为它愿意努力完成办公室杂务，而不是战略性地让它们无效。在 50 名员工身上可以勉强推行的东西，在 500 名员工身上就变得不可能管理了。他说，这就是为什么“我们不想在产品中包含任何非软件的东西。”如果它需要客户打电话给 Rippling 的运营团队寻求帮助，它可以建得更好。这种狂热的专注实际上让 Conrad 暂时担任了 Rippling 的唯一角色，回应用户投诉，他也认为这推动了快速迭代。这位首席执行官希望保持这种心态，所以他仍然在 LinkedIn 上把他的职位列为“客户支持”。

康拉德似乎已经说服投资者，尽管他被赶出了估值 45 亿美元的人力资源初创公司 Zenefits ，但他对该公司的崛起而非衰落负有更大的责任。康拉德编写了一个脚本，允许 Zenefits 的员工登录到他们保险考试的学习部分。康拉德坚持认为，它没有帮助他们学习或通过认证考试。尽管如此，监管机构介入其中，导致他离职，并对他和 Zenefits 处以总计100 万美元的 SEC 罚款。加快速度的愿望是繁忙的工作耗尽公司时间的另一个症状。

还有文化问题，Zenefits 曾经不得不告诉员工不要在办公室楼梯井做爱。更有分寸的步伐和对多样性更深的承诺是避免康拉德上一次冒险的文化问题的其他几种方式。

当一年前脱离秘密行动时，Rippling 才真正开始雇佣超过 100 名工程师。现在，这家初创公司已经建立了两种利润丰厚的商业模式。首先，当人们通过 Rippling gateway 购买时，它从其他企业工具制造商那里赚取转让费。任何一个有知名品牌的开发商，都成为整合荡漾的合作伙伴。它不会试图超越 Zoom 或 Mailchimp。“随着 bring 的成功，我认为它可以为这些其他业务带来大量客户。如果你能降低添加 N+1 业务系统的边际成本，那么在添加产品时就会少很多犹豫。”客户想要更多的效用，只是没有头痛。

与此同时，Rippling 开发了自己内部版本的人力资源和 IT 堆栈的无差别部分，如 PTO 管理或通勤福利。在这些地区，消费者还没有对一个品牌产生忠诚度，所以涟漪很容易乘虚而入。它可以收取类似的费用，但在便利性上胜过竞争对手，因为它自己开发的系统直接与 Rippling 的员工详细信息的真实来源相集成。Okta 和 LastPass 等单点登录领域的新贵声称是身份层，但实际上只是密码管理器。他们早期的发展刺激了 SaaS 公司建立 API 端点，而 Rippling 的 RPass 版本可以搭载在这些端点上。

有一段时间，我认为 Slack 将成为企业身份提供商，因为聊天是一种无处不在的需求，它可能是跨应用程序档案的开始。但人力资源和 IT 是更基础的一层，Slack 不像 Rippling 那样是收集员工详细信息的自然场所。哈米德说:“对于 slack 来说，沟通和协作是一个足够大的机会，不要让身份妨碍那里的核心业务。”

现在，随着碧波集团业务的加速发展和大量现金的投入，康拉德告诉我，他最关心的是雇佣合适的人。“在公司里，真正具有挑战性的事情是员工人数增长过快。我确保我们不会在 12 个月内裁员超过一倍，”他告诉我。虽然 Zenefits 是一场疯狂的大规模闪电战，但 Conrad 试图在不冲动到让公司犯错的情况下，让 T2 倾向于行动。“这从来都不容易，但我们还没有达到让事情变得真正可怕的程度。我们还有一点时间来达到里程碑。我们正以健康的速度增长，但没有任何东西以任何方式让事情变得紧张，我们看到这一点是因为我们非常密切地跟踪我们的 NPS，”他在谈到试图以更宜居的速度经营企业时表示，同时也是一名积极的父亲。

幸运的是，泽内菲斯教会了他如何避免创业的许多陷阱。康拉德得出结论，他很高兴已经从“玩电子游戏的不可能模式对媒体模式。”
