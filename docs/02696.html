谷歌云以新的开源集成挑战 AWS 技术危机

原文：https://web.archive.org/web/https://techcrunch.com/2019/04/09/google-gives-aws-the-open-source-middle-finger/amp/

Google today 宣布已经与许多顶级开源数据管理和分析公司合作，将他们的产品集成到其 Google 云平台中，并作为由合作伙伴运营的托管服务提供。这里的合作伙伴有 Confluent、DataStax、Elastic、InfluxData、MongoDB、Neo4j 和 Redis Labs。

谷歌表示，这里的想法是为用户提供无缝的用户体验，以及在谷歌云中轻松利用这些开源技术的能力。但是这里有更多的东西在起作用，尽管谷歌从来没有这么说过。这是因为谷歌此举显然是为了将其开源生态系统的方法与亚马逊进行对比。众所周知，亚马逊的 AWS 云计算平台以采用一些最好的开源项目，然后将这些项目分支并包装在自己的品牌下而闻名，通常不会返还给原始项目。有一些迹象表明这种情况正在改变，但是一些公司最近已经采取行动，改变了他们的开源许可证，以明确防止这种情况发生。

这就是事情变得有趣的地方，因为这些公司包括 Confluent，Elastic，MongoDB， Neo4j 和 Redis Labs——这些公司都与谷歌在这个新项目上合作，尽管值得注意的是 InfluxData 并没有采取这种新的许可方式，尽管 DataStax 使用了许多开源技术，但它的重点是它的企业版。

谷歌云基础设施合作伙伴关系负责人曼文德·辛格(Manvinder Singh)在新闻发布会上说，“正如你所知，业界对在云中提供这些开源技术的最佳方式有很多争论。“鉴于谷歌的 DNA 和我们对开源模式的信念，如 Kubernetes、TensorFlow、Go 等项目，我们认为解决这个问题的正确方法是与那些投入资源开发这些开源技术的公司密切合作。”

因此，虽然 AWS 接受这些项目，然后将它们变成自己的项目，但谷歌已经决定与这些公司合作。虽然谷歌及其合作伙伴拒绝对这些交易背后的财务安排发表评论，但我们在这里谈论的可能是某种程度的利润分成。

“每一个主要的云玩家都在试图区分它为客户带来了什么，尽管我们与微软和亚马逊有着强大的合作伙伴关系，但很高兴看到谷歌选择深化与 Atlas 的合作伙伴关系，而不是推出模仿服务，”MongoDB 的云产品高级副总裁 Sahir Azam 告诉我。“MongoDB 和 GCP 多年来一直密切合作，可以追溯到 2017 年初 GCP Atlas 的开发。在过去两年在 GCP 运行 Atlas 的过程中，我们的联合团队发展了强大的工作关系和支持模式，为客户的关键任务应用提供支持。"

至于实际的功能，这里的核心原则是，谷歌会将这些服务深度整合到其云控制台中；例如，类似于微软在 Azure 上对数据块所做的。这些将是托管服务，谷歌云将处理发票，账单将计入用户的谷歌云支出承诺。谷歌也将提供支持，因此用户可以使用单一服务来管理和记录所有这些服务的门票。

Redis Labs 首席执行官兼联合创始人 Ofer Bengal 对此表示赞同。他说:“通过这种合作关系，Redis 实验室和谷歌云将这些创新带给企业客户，同时让他们选择在云中的哪里运行他们的工作负载。“客户现在可以使用 GCP 上完全集成的托管服务，通过 Redis Enterprise 灵活地开发应用程序。这将包括从 GCP 控制台管理 Redis Enterprise 的能力、供应、计费、支持以及与 GCP 的其他深度集成。"
