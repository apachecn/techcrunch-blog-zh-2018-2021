Google Cloud Data Fusion 让您可以在 big query TechCrunch 中获取所有数据集之前查看它们

原文：https://web.archive.org/web/https://techcrunch.com/2019/04/10/google-improves-data-migration-and-integration-for-bigquery/

今天，谷歌在其 Google Cloud Next 开发者大会上宣布了其数据分析产品的几项更新。该公司希望更容易地将数据转移到谷歌的云，操纵数据，并将这些数据转化为见解。

首先，谷歌希望从谷歌的平台上更容易地访问你的所有数据。该公司正在推出测试版的云数据融合。顾名思义，该服务允许您在一个界面中聚合所有数据集。

即使您的数据托管在其他云服务上，您也可以从云数据融合界面查看它。然后你可以计算出你需要在 BigQuery 中处理哪个数据集——big query 是谷歌的基于云的服务，用于使用 SQL 查询分析大量数据。

选择数据集后，您需要确保 BigQuery 能够接收所有这些数据。你可能需要将一些数据转移到谷歌的云端。

即使你已经在谷歌云存储上存储了数据，你的数据也有可能分散在多个系统和数据格式中。例如，你可能已经上传了万亿字节的原始数据到谷歌的服务器上，但是如果你不能用 Salesforce 或 Workday 中的数据来处理它，那就没有用了。

该公司已经允许你使用 BigQuery 数据传输服务自动传输数据。谷歌表示，这项服务现在支持 100 多个 SaaS 应用，包括 Salesforce、Marketo、Workday 和 Stripe。

该服务已经允许您传输数据仓库数据。所以如果你想从 Amazon Redshift 或者 Teradata 切换，你可以迁移一切。

现在所有的数据都可用了，您需要将其可视化。谷歌已经允许你在谷歌数据工作室创建交互式仪表盘和报告。该服务由 BigQuery BI 引擎提供支持。谷歌表示，现在许多复杂的查询处理时间不到一秒钟。

很快，谷歌将向其他数据可视化服务开放 BigQuery BI Engine，如 Looker 和 Tableau。BigQuery BI 引擎将在后台运行，处理您的数据。

如果你不想学习 SQL 查询，你可以使用微软的 Excel 或者谷歌的工作表来排序和处理数据。谷歌已经让你将 BigQuery 与谷歌表单连接起来。该公司更进一步，创建了一种新型的无限电子表格——连接表。即使您有数十亿行数据，Google Sheets 现在也可以显示您的 BigQuery 支持的电子表格。

去年，谷歌推出了 BigQuery ML 。该服务允许客户使用 SQL 在其数据仓库的基础上构建模型。Google 正在测试版中添加 AutoML 表格。它让您只需点击几下鼠标，而不是几行代码，就可以构建机器学习模型。

最后，谷歌推出了名为数据目录的元数据管理服务。我相信电信公司会喜欢这个的。客户将能够使用谷歌的云 IAM 界面限制对敏感数据资产的访问。

这就是谷歌云下一期的谷歌数据分析新闻。
