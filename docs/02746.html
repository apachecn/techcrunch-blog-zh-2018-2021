谷歌云 2019 年最重要的 6 项公告

原文：https://web.archive.org/web/https://techcrunch.com/2019/04/10/the-6-most-important-announcements-from-google-cloud-next-2019/

谷歌本周将在旧金山举办其年度云下一代开发者大会。虽然活动仍在如火如荼地进行，今晚的神秘音乐会结束了大部分节目，但大量的新闻现在已经平息，所以这里是我们关于活动的最重要公告的列表。

安索斯

什么事？ Anthos 是谷歌云服务平台的新名称，谷歌的托管服务允许企业在其私有数据中心和谷歌云中运行应用程序。谷歌决定给这项服务起一个新名字，Anthos，但也扩大了对其竞争对手的云平台 AWS 和 Azure 的支持。这将允许企业使用运行在谷歌云上的单一平台，在任何云上部署和管理他们的应用程序。企业将获得一份账单，并拥有一个控制面板来管理他们的应用程序。不出所料，所有这些都是由集装箱和 Kubernetes 驱动的。

这有什么关系？对于大型云竞争对手来说，推出允许用户在其他平台上运行应用程序的产品仍然是极不寻常的。毕竟，这笔钱是通过收取计算时间和存储分配费用获得的。谷歌辩称，这是它的客户想要的，而且它解决了一个真正的问题。然而，谷歌也在追赶它的竞争对手，并寻找不同于他们的方法。我不认为我们会看到 AWS 和 Azure 用类似的工具做出反应，但如果他们这样做了，这对他们的客户来说是一件好事。

与谷歌云控制台的开源集成

什么事？谷歌宣布，它将把许多开源公司的产品深度整合到它的云中，并从本质上使它们成为第一方服务。这些合作伙伴是 Confluent、DataStax、Elastic、InfluxData、MongoDB、Neo4j 和 Redis Labs，随着时间的推移，其他合作伙伴可能会跟进。

为什么有关系？这些集成对于可能已经在使用其中一些服务的谷歌云客户来说是一个福音。他们将从这些公司获得一份账单和支持，同时从一个控制台管理服务。然而，这里的潜台词有点复杂，揭示了谷歌的开源方法，并将其与 AWS 进行了对比。许多参与其中的公司都对 AWS 对待开源的方式持高度批评态度，并对此相当公开。谷歌正在与他们合作，而人们的感觉是 AWS 只是简单地使用代码，并没有回馈。

谷歌的人工智能平台

什么事？谷歌认为其人工智能实力是其与 AWS、Azure 和 Co .竞争的主要优势之一。该公司已经提供了广泛的人工智能工具，从面向高级数据科学家的开发工具和服务到 AutoML，这是一种可以自动训练模型且不需要博士学位的服务。新的人工智能平台为更高级的开发人员提供了端到端的解决方案，允许他们从摄取数据到训练和测试他们的模型，再到将它们投入生产。该平台还可以使用预先构建的模型。

这有什么关系？人工智能(和机器学习)是所有大型云提供商的主要关注点，但开发人员的体验还有很大的改进空间。拥有一个端到端的解决方案显然是向前迈出的一大步，并向更广泛的潜在用户展示了机器学习的前景。

您的 Android 手机现在是一把安全钥匙

什么事？您现在可以使用任何 Android 7+手机作为安全密钥，而不是使用物理安全密钥来实现双重身份验证。你在你的谷歌账户中设置它，然后你的手机将使用蓝牙(但没有创建蓝牙连接的麻烦)来提供你的第二个因素。目前，这仅适用于 Chrome，但谷歌希望将其转化为其他浏览器和移动操作系统供应商也将支持的标准。谷歌还建议你还是使用普通钥匙作为备份，以防万一你丢失了手机。

为什么很重要？与仅使用登录名和密码相比，双因素身份验证本质上更安全。使用短信和推送通知的系统仍然容易受到网络钓鱼攻击，而安全密钥——这种新的基于 Android 的系统使用与现有密钥相同的标准——通过确保您在合法的网站上来防止这种情况。这个新系统消除了使用物理密钥的麻烦，可能会说服更多人使用双因素身份认证。

谷歌云代码

什么事？云代码是一组流行 IDE 的插件和扩展，如 IntelliJ 和 VS 代码。这里的总体思想是为开发人员提供构建云原生应用程序的所有必要工具——所有这些都不需要处理随之而来的任何管道工作和配置。使用云代码，开发人员可以像以前一样简单地编写他们的应用程序，然后将它们打包为云原生应用程序，并将其发送到 Kubernetes 集群进行测试或生产。

为什么有关系？编写云原生应用程序非常复杂，通常需要编写复杂的配置文件。理想情况下，云代码使所有这一切变得如此简单，以至于开发人员——以及雇佣他们的公司——更容易迁移到现代基础设施。

谷歌云瞄准零售商

什么事？这里的消息是，谷歌正在推出一个直接针对零售商的垂直解决方案。这听起来没什么惊天动地的，对吧？但谷歌云计划随着时间的推移提供更多这种专业化的解决方案。

这有什么关系？ Google Cloud 首席执行官 Thomas Kurian 告诉我们，客户正在寻求这种集成解决方案，将一些公司现有的工具打包成这些企业可以部署的集成解决方案。这基本上是它第一次这样做(也许医疗保健是个例外)，但随着时间的推移，它可能会提供更多这样的服务，它们可能会成为平台用户群增长的主要因素。
