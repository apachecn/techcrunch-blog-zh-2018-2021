甲骨文转向创新中心，推动文化和业务向云转变

原文：https://web.archive.org/web/https://techcrunch.com/2019/04/23/oracle-turns-to-innovation-hubs-to-drive-cultural-and-business-shift-to-cloud/

甲骨文成立于 1977 年。虽然通用电气和 IBM 的历史分别可以追溯到 19 世纪末和 20 世纪初，但它的历史已经足够长，可以凭自己的力量经历相当多的颠覆。在其存在的大部分时间里，它向世界上一些最大的公司出售数据库。但是今天，随着市场的变化和从内部数据中心向云的转移，像 Oracle 这样的公司如何实现这种转变呢？

当然，甲骨文在过去几年里一直在向云转移，但公平地说它来晚了。此外，要改变一家像甲骨文这样规模的公司，光靠建设一些数据中心和推出一些产品是不够的。公司领导层认识到了这一点，并一直在组织的最高层思考如何从文化和业务的角度成功转型为云公司。

为此，甲骨文在过去几年里开设了五个创新中心，分别位于德克萨斯州的奥斯汀；弗吉尼亚州雷斯顿；马萨诸塞州伯灵顿。；印度班加罗尔和加州圣莫尼卡。这些中心希望实现什么目标，如何将学到的经验推广到公司的其他部门？这些都是甲骨文要想在云市场取得进展必须回答的大问题。

理解问题

甲骨文似乎明白，它必须做一些不同的事情来改变市场看法和其萎靡不振的市场地位。追踪云市场份额的公司 Synergy Research 报告称，该公司正在苦苦挣扎。

Synergy 首席分析师兼董事总经理 John Dinsdale 告诉 TechCrunch:“对于云基础设施服务(IaaS、PaaS、托管私有云服务)，甲骨文拥有 2%的份额。他补充道，“这是一家排名前十的公司，但其规模远不及领先的云提供商；它的市场份额一直在稳步下降。”

SaaS 传来的消息稍微好一点。“与 SAP 一样，Oracle 也是 ERP 领域的领导者之一。但 SaaS 企业比 ERP 更广泛，在整个 SaaS 企业中，它是排名第四的提供商，仅次于微软、Salesforce 和 Adobe。丁斯代尔说:“第四季度甲骨文全球市场份额为 6%。

该公司知道，要从一个主要销售软件许可和维护协议的组织转变过来，将需要一个巨大的转变。它把那些人逼得很紧，有时太紧了，以至于让专业人士吃了一口酸味。如今，有了云，销售格局发生了巨大变化，变成了合作模式。该公司知道它也必须改变。问题是，怎么做？

这将对产品开发、销售和营销采取全新的方法；创新中心已经成为一种实验室，工程师们可以在这里试验更专注的项目，并学习展示他们的想法，而不是告诉客户他们能做什么。

年轻人将会领导

改变这种文化的一个方法是给它注入思维新颖、聪明的年轻人，而这正是甲骨文试图在这些中心做的事情，他们在那里雇用年轻的工程师，其中许多人刚从大学毕业，在更有经验的甲骨文高管的帮助下领导变革。

他们正在寻找方法来重新思考甲骨文的云产品，将服务整合到有用的工具包中，帮助解决具体的业务问题，从处方阿片滥用到预测鳄梨产量。这个想法不仅仅是在公司的某个部门让人们从事梦想的项目。他们希望它们与真正的商业问题相关，最终导致实际的销售和可衡量的结果。

甲骨文负责云解决方案中心的集团副总裁 Hamza Jahangir 表示，他们寻找的是那些希望钻研新解决方案的人，但他们希望他们的创新中心员工有实践经验。“我们不仅仅想要修补匠。如果你解决的唯一问题是你自己的无聊，那就不是我们要找的那种人，”他说。

高管买入

据贾汉吉尔说，创新中心的想法实际上始于首席执行官马克·赫德。几年来，他一直致力于改变销售队伍的性质，通过用全新的方法雇用大学毕业生，这支队伍以对新一代 IT 专业人士强制武装而闻名。

然而，赫德并不想停止销售。他开始考虑采用同样的想法，雇佣更年轻的员工来推动工程领域的文化转变。贾汉吉尔说:“大约两年前，马克要求我们思考，随着商业模式向云转移，我们如何才能改变面向客户的技术员工。”

赫德给了他一些预算，在奥斯汀和莱斯顿开设了前两个中心，他开始尝试，试图找到合适的员工和项目。这笔资金没有附带任何条件。赫德希望看到，如果他们释放新一代员工，给他们一定的自由，让他们以不同于甲骨文传统的工作方式工作，会发生什么。

变化的期望

贾汉吉尔在评估客户对甲骨文迁移到云的期望时非常坦率。有很多怀疑，创新中心的部分原因是找到实用的解决方案，可以向客户展示他们实际上有现代计算方法，如果有机会的话。

他说，一般客户的立场是，“我们不相信你有任何真实的东西，我们需要在付钱给你之前看到我们实现的真正价值。”这从根本上转变为关注实际解决方案。它始于这样一个前提，即顾客不应该相信任何营销的东西。相反，它会向他们展示。

“不要看幻灯片演示。请让我们向您展示我们已经解决了真实材料问题的真实解决方案和使用案例，然后我们可以进行讨论。”

就连董事长兼公司创始人拉里·埃里森也认识到，随着公司转向云计算，这种关系和销售模式也需要改变。贾汉吉尔转述了他在最近的一次内部会议上所说的话，“在云中，我们现在不再销售巨型单片软件。相反，我们卖的是苹果的小块。卖家和买家之间的关系越来越像一种消费者模式。”这反过来需要一种销售和交付解决方案的新方式，这正是他们在创新中心试图解决的问题。

将想法付诸实践

一旦你有了一种新的思维方式，你就必须把它运用到工作中，随着公司创建了这些不同的中心，这就是方法。例如，一个不一定是原创的，但以实用的方式将 Oracle 功能组合在一起的例子是关联患者。患者佩戴类似 Fitbit 的监护仪，使用智能血压袖带和智能药盒。

然后，患者可以在一个整合的移动应用程序中使用这些工具来监控自己的健康状况，该应用程序使用物联网云服务、Oracle 移动云以及 Oracle 集成云将这些数据整合在一起。更重要的是，患者的药房和医生可以共享这些信息，他们可以监控患者的健康状况，并在出现严重问题(如危险的高血压)时发出警告。

另一个项目涉及与 Waypoint Robotics 的合作，在那里他们展示了一个与人类工人一起工作的机器人。人类与机器人互动，但机器人将货物从一个工作站移动到另一个工作站，在整个过程中充当质量控制代理。如果它发现缺陷或问题，它会通过设备侧面的屏幕传达给工人，并传送到云端。人、货物和机器人之间的每一次互动都在甲骨文云中更新。

Waypoint Robotics 机器人正在检查 iPhones。显示屏上的信息显示它正在与 Oracle 云通信。照片:罗恩·米勒

另一个项目与农民和经销商合作，帮助商店保持鳄梨库存，这肯定是你可能找到的最好的 Z 世代项目。该工具查看天气数据、历史销售额和来自农场传感器的信息，并结合所有这些数据来预测鳄梨产量，利用 Oracle 自治数据仓库、Oracle 分析云以及 Oracle 云堆栈中的其他服务。

超越中心

这种类型的创新中心近年来变得很受欢迎，作为一种帮助避免中断的方式，甲骨文的方法实际上符合这一趋势。虽然公司有时会隔离这些创新中心，以保护它们免受组织中消极和反对者的影响，但让它们隔离往往会阻止将学到的经验应用到更广泛的组织中，从根本上违背了最初创建它们的目的。

贾汉吉尔说，他们试图通过与公司其他人会面，分享他们的学习成果以及他们在创新中心用来衡量成功的各种指标来避免这个问题，这些指标可能与公司其他部门不同。

他说，要把甲骨文放在客户议程上，他们必须把话题从宗教斗争中转移出来，他称之为人们如何支持或谴责某些公司的技术。“我们必须克服宗教斗争和观念。我不喜欢用更多的宗教来对抗宗教。我们需要退出这场对话。我们看到的吸引开发人员社区的最佳方式是向他们展示如何构建真正酷的东西，然后我们可以雇佣开发人员来做这件事，并向社区展示这一点，以表明这不只是口头上的服务。”

诀窍就在于这样做，或许创新中心会有所帮助。截至今天，该公司还没有分享其云收入，所以很难衡量这对公司整体成功的贡献有多大。但甲骨文显然还有很多工作要做，以改变企业买家对其云产品和服务的看法，并增加其在不断增长的云蛋糕中的份额。它希望这些创新中心将引领这一方向。

贾汉吉尔意识到他必须不断调整方法。“枢纽模式仍在不断成熟。我们正在发现和解决组织中需要新工具和参与模型的新问题。我们仍在学习和发展，”他说。
