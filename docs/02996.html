Docker 依靠合作伙伴和软件包来简化容器实施 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2019/04/30/docker-looks-to-partners-and-packages-to-ease-implementation/

Docker 似乎正在寻找简化公司核心价值主张的方法——创建、部署和管理容器。虽然大多数人都认为它革新了软件开发，但像许多技术解决方案一样，它需要一定水平的专业知识和人员来完成。在本周于旧金山举行的 DockerCon 公司客户会议上，Docker 宣布了几种方法，可以帮助客户解决实施集装箱化解决方案的困难部分。

首先，T2 公司今天早上宣布了 Docker Enterprise 3.0 的测试版。这次更新是为了让开发者的生活更简单。随着公司转向集装箱化环境，除了像谷歌、亚马逊和脸书这样的大型组织，这是一个挑战，所有这些组织都有大量的资源需求和相应的大型工程团队。

然而，大多数公司没有这种奢侈，Docker 认识到，如果它想将集装箱化带给更多的客户，它必须创建更容易实现的包和程序。

Docker Enterprise 3.0 向提供一个解决方案迈出了一步，该解决方案让开发人员专注于开发方面，同时使用模板和其他工具来简化部署和管理。

该公司看到客户在实施以及如何配置和建立一个容器化的工作流程方面存在困难，因此它正在与系统集成商合作，以帮助解决困难的部分。今天，该公司宣布 Docker Enterprise as a Service，目标是使用 Docker stack 和 Kubernetes 等相邻工具，帮助公司建立和管理容器化环境。

服务提供商将负责运营细节，如管理升级、推出补丁、进行备份和进行容量规划，所有这些运营任务都需要对企业容器堆栈有很高的了解。

凯捷将成为首家上市合作伙伴。Docker 首席产品官斯科特·约翰斯顿(Scott Johnston)对 TechCrunch 表示:“凯捷拥有自动化、技术工具以及后端服务的组合，可以管理企业平台本身的安装、供应和管理，以防客户不想这样做，而他们希望花钱找人代劳。”。

该公司过去曾发布过一些工具，帮助客户轻松地将遗留应用程序迁移到容器中。今天，该公司宣布了一个名为 Accelerate Greenfield 的解决方案包，这是一套旨在帮助客户成为容器优先开发公司的工具。

“这是为那些可能走得稍远的组织准备的。Johnston 解释说:“他们全力以赴，致力于采用容器优先的方法来开发新的应用程序。他说，这可能是云原生微服务，甚至是 LAMP stack 应用程序，但关键是他们希望将所有东西放在容器平台上的容器中。

加速绿地就是为此而设计的。“他们得到的好处是，他们知道从开发人员到生产终端都是安全的。在整个生命周期中，他们只有一种方式来定义它。他们可以确保它快速移动，并且他们将这种可移植性嵌入到容器格式中，因此他们可以[在他们希望的任何地方]部署，”他说。

这些计划和产品都是为了提供一定程度的帮助，或者扮演直接顾问的角色，与系统集成商合作，或者提供一套工具和技术来帮助客户完成集装箱化生命周期。随着这些程序的成熟，我们将逐渐了解它们是否能提供客户所需的足够水平的帮助。
