Docker更新专注于简化开发者的容器化技术

原文：https://web.archive.org/web/https://techcrunch.com/2019/04/30/docker-updates-focus-on-simplifying-life-for-developers/

在过去的五年里， Docker 已经成为软件容器的同义词，但这并不意味着每个开发者都理解构建、管理和部署它们的技术细节。在本周于三藩市举行的客户会议上，该公司宣布了新的工具，这些工具旨在使那些可能不是Docker专家的开发者更容易使用容器。

随着技术的成熟，该公司看到了市场的扩大，但为了利用这一点，它需要提供一套工具，使其更容易使用。“我们发现，客户通常只有一小群Docker专家，但也有成百上千的开发人员想要使用Docker。我们想，我们如何帮助他们非常、非常快速地提高工作效率，而不需要他们成为Docker专家，”Docker首席产品官斯科特·约翰斯顿(Scott Johnston)告诉TechCrunch。

为此，它宣布了Docker Enterprise 3.0的测试版，其中包括几个关键组件。首先，Docker Desktop Enterprise允许IT部门使用适合每个客户的安全和部署模板来建立Docker环境。然后，开发人员可以选择对他们的实现有意义的模板，同时符合公司的法规遵从性和治理规则。

“这些模板已经具有IT认可的容器映像，并且具有IT认可的配置设置。这意味着它可以通过这些可视化工具提供这些模板，使开发人员能够快速行动并选择他们想要的模板，而不必回去获得批准，”Johnston解释说。

这个想法是让开发人员专注于构建应用程序，模板提供了所有预先构建好的Docker工具，因此他们不必担心所有这些。

另一个部分是Docker应用程序，它允许开发人员将复杂的容器化应用程序构建为单个包，并将其部署到他们希望的任何基础架构中——本地或云中。五年前，当Docker真正开始使用容器时，它们是一个更简单的想法，通常只涉及一个单独的容器，但随着开发人员将这些较大的应用程序分解为微服务，这带来了新的困难，特别是对于必须部署这些越来越大的应用程序容器集的运营来说。

“操作现在可以根据环境以编程方式改变容器的参数，而不必进入并改变应用程序。所以你可以想象，这种能力降低了首先必须管理所有这些文件的摩擦，”他说。

最后一个部分是编排层，目前流行的处理方式是使用Kubernetes。Docker基于开源工具创建了自己的Kubernetes风格。Johnston说，与其他两个部分一样，这里的目标是采用一个像Kubernetes这样强大的工具，并降低运行它的总体复杂性，同时使它完全兼容Docker环境。

为此，Docker宣布了Docker Kubernetes服务(DKS)，该服务是为Docker用户设计的，包括对Docker Compose的支持，这是一个深受Docker用户欢迎的脚本工具。虽然你可以随意使用任何风格的Kubernetes，但Docker为开发者提供了DKS这个Docker友好版本。

所有这些组件除了都是Docker Enterprise 3.0的一部分之外，还有一个共同点。他们试图降低与部署和管理容器相关的复杂性，并抽象出最困难的部分，以便开发人员可以专注于开发，而不必担心连接到构建和部署容器的技术基础。与此同时，Docker正试图让运营团队更容易管理这一切。至少这是目标。最后，一旦这些工具在今年晚些时候普遍可用，DevOps团队将成为Docker表现的最终评判者。

Docker Enterprise 3.0测试版将于本季度晚些时候推出。
