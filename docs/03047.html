微软通过个性化服务、手写识别 API 等扩展了其认知服务

原文：https://web.archive.org/web/https://techcrunch.com/2019/05/02/microsoft-extends-its-cognitive-services-with-personalization-and-handwriting-recognition-apis/

作为下周旗舰 Build 开发者大会之前相当奇怪的新闻的一部分，微软今天宣布了一系列新的预建机器学习模型，用于其认知服务平台。其中包括用于构建个性化功能的 API、用于自动化数据输入的表单识别器、手写识别 API 和专注于转录对话的增强语音识别服务。

也许这些新服务中最重要的是个性化服务。毕竟，很少有应用程序和网站不希望为用户提供个性化的功能。这很难，部分是因为它通常涉及到基于位于各种筒仓中的数据来构建模型。通过 Personalizer，微软正在押注强化学习，这是一种不需要机器学习中通常使用的那种带标签的训练数据的机器学习技术。相反，强化代理不断地试图根据用户的行为找到实现给定目标的最佳方式。微软辩称，它是第一家提供此类服务的公司，该公司本身一直在 Xbox 上测试这些服务，在实施这项服务后，其内容的参与度增加了 40%。

手写识别 API，或官方称为墨迹识别器，可以自动识别手写、常见形状和文档。这是微软在开发 Windows 10 墨迹功能时长期关注的问题，所以现在它也将这一功能包装为认知服务也就不足为奇了。事实上，微软 Office 365 和 Windows 已经在使用这项服务，所以我们在谈论一个非常强大的系统。有了这个新的 API，开发人员现在也可以将这些相同的功能带到他们自己的应用程序中。

对话转录顾名思义:它转录对话，是微软认知服务产品线中现有语音到文本功能的一部分。它可以标记不同的发言者，实时转录对话，甚至处理串扰。它已经集成了微软团队和其他会议软件。

还有一个新特性是表单识别器，这是一个新的 API，可以更容易地从业务表单和文档中提取文本和数据。这听起来可能不是一个非常令人兴奋的功能，但它解决了一个非常常见的问题，该服务只需要五个样本就可以理解如何提取数据，用户不必做任何在构建这些系统时经常涉及的艰苦的手动标记。

表单识别器也将进入认知服务容器，它允许开发人员将这些模型带到 Azure 之外的边缘设备上。对于现有的语音到文本和文本到语音服务以及现有的异常检测器来说也是如此。

此外，该公司今天还宣布，其神经文本语音转换、计算机视觉阅读和文本分析命名的实体识别 API 现已正式推出。

一些现有的服务也在进行一些功能更新，神经文本到语音转换服务现在支持五种声音，而计算机视觉 API 现在可以理解超过 10，000 个概念，场景和物体，以及 100 万名名人，而之前的版本只有 20 万名(有那么多名人吗？).
