微软和GitHub越来越密切

微软以75亿美元收购GitHub 的交易于去年10月完成。今天，在其年度构建开发者大会上，微软宣布了其现有服务和 GitHub 之间的大量新集成。这些都不是惊天动地的，也没有改变GitHub任何基本功能的本质，但它们确实表明了微软是如何开始将GitHub更紧密地融入进来的。

值得注意的是，微软并没有在Build中宣布GitHub的任何主要功能，尽管就在几周前，该公司做出了一项重大改变，给予GitHub免费用户无限制访问私有库的权利。对于主要功能发布，GitHub有自己的发布会。

那么有哪些新的集成呢？其中大多数都围绕着身份管理。这意味着GitHub企业用户现在可以使用Azure Active Directory访问GitHub。开发者也将能够使用他们现有的GitHub帐户登录Azure门户和Azure DevOps等Azure功能。“这一更新使GitHub开发人员只需使用他们的GitHub帐户就可以从一个存储库到另一个部署库，”微软在其发布公告中称。

就出售GitHub而言，微软今天还宣布了一项新的Visual Studio订阅，微软的企业协议客户可以访问GitHub Enterprise。鉴于Visual Studio的企业客户和GitHub企业用户之间肯定有很多重叠，这一举措是有意义的。对于当前的Visual Studio用户来说，这也有可能让迁移到GitHub Enterprise变得更有吸引力。

最后， Azure Boards 应用提供了看板和sprint规划工具等功能，现在也可以在GitHub市场上使用。
