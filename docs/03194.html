GitHub 获得一个包注册表 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2019/05/10/github-gets-a-package-registry/

GitHub 今天宣布推出 GitHub 包注册表的有限测试版，这是一种新的包管理服务，开发者可以在源代码旁边发布公共和私有包。

需要澄清的是，GitHub 并没有推出像 npm 或 RubyGems 这样的工具的竞争对手。然而，该公司正在推出的是一项与这些工具兼容的服务，允许开发人员使用他们用于代码的相同 GitHub 接口来查找和发布他们自己的包。新服务目前兼容 JavaScript (npm)、Java (Maven)、Ruby (RubyGems)、。NET (NuGet)和 Docker images，以及对其他语言和工具的支持。

GitHub 产品管理总监 Simina Pasat 在今天的公告中解释道:“ GitHub Package Registry 与常见的包管理客户端兼容，因此您可以使用自己选择的工具发布包。“如果您的存储库更复杂，您将能够发布不同类型的多个包。此外，通过 webhooks 或 GitHub 操作，您可以完全定制您的发布和后期发布工作流程。” 有了这个，企业还可以为他们的员工提供一套凭证来管理他们的代码和软件包——这一新功能还可以轻松创建一套经过批准的软件包。用户还可以在 GitHub 上获得下载统计数据和访问软件包的整个历史。

大多数开源软件包在将代码发布到公共注册表之前，已经使用 GitHub 开发了它们的代码。例如，GitHub 认为这些开发者现在也可以使用 GitHub 包注册表来发布预发布版本。

开发者已经经常使用 GitHub 来托管他们的私有库。毕竟，将包和代码放在同一个地方是有意义的。在某种程度上，GitHub 正在做的是将这种实践形式化，并围绕它包装一个产品。
