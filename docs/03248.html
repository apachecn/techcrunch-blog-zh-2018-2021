New Relic对平台改造采取了谨慎的方法——TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2019/05/14/new-relic-takes-a-measured-approach-to-platform-overhaul/amp/

图片来源: OstapenkoOlena / Getty Images

SaaS应用性能管理平台New Relic 今天宣布对该平台进行重大更新。该公司决定采取更有分寸的方式进行变革，给消费者一个放松的机会，而不是一下子撕掉创可贴。

新平台名为New Relic One，旨在取代过去十年开发的原始平台。该公司表示，通过慢慢转移到新平台，客户将能够利用在旧平台上无法构建的新功能，而不必学习新的工作方式。

New Relic的首席产品官Jim Gochee表示，所有现有的工具和功能最终都将移植到New Relic One上，或者在其上重新设计。“对我们来说，它是一个新的技术堆栈和一个新的产品平台。我们仍在使用现有产品运行现有技术堆栈。所以我们(本质上)在两个堆栈中并行运行两个平台，但随着时间的推移，所有的新东西都将建立在New Relic One上，”他解释道。

通过从零开始重新设计现有平台，New Relic创建了一个新的、现代的、更具可扩展性的模型，这将允许它随着时间的推移更容易地插入新功能，最终甚至允许客户做同样的事情。目前，它是关于改变引擎盖下发生的事情，并在重新设计的用户界面中提供新的用户体验。

“New Relic One是非常可插拔和可扩展的，这使得我们自己的团队更容易构建、扩展和扩展，并且我们最终将达到合作伙伴和客户能够自己扩展我们的UI的地步，这是我们非常兴奋的事情，”他说。

新功能中包括对该公司无服务器产品AWS Lambda的支持。它还使用户能够跨多个帐户进行搜索。客户监控多个账户和子账户并不罕见。有了New Relic One，客户现在可以跨这些帐户进行搜索，并更容易地发现问题是否已经级联。

在一篇介绍新平台的博客文章中，首席执行官Lew Cirne承认监控领域日益复杂，这是新平台专门设计来解决的问题。

Cirne写道:“与今天的分散工具不同，这些工具可以提供一袋图表和指标，其中有一堆看似无关的数字，New Relic One旨在消除复杂性，提供背景，让你看到人工组织边界，以便你可以快速找到并解决问题。”

451 Research的高级分析师南希·戈林(Nancy Gohring)表示，这种灵活性是新方法的关键优势。“这里最重要的更新之一是重新设计的数据模型，它允许New Relic为客户提供更大的灵活性，让他们能够搜索正在收集的运营数据并构建仪表板。这种灵活性在比过去更加复杂和动态的现代应用环境中更加重要。Gohring告诉TechCrunch:“每个人的环境都不一样，挖掘问题的原因比过去更复杂。跨账户搜索的新功能应该对此有所帮助。

她承认拥有并行平台并不理想，但明白为什么公司选择走这条路。“拥有两个ui从来都不是什么好事。但是New Relic采取的方法让他们一次就能得到全新的东西，而不是花时间逐步引入它。它将让顾客按照自己的节奏尝试新的东西，”她说。

新遗迹一号将于明天上线，并将免费提供给新遗迹的订户。
