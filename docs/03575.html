随着反垄断调查的临近，苹果公司逆转了禁止家长控制应用程序的进程

原文：https://web.archive.org/web/https://techcrunch.com/2019/06/04/with-antitrust-investigations-looming-apple-reverses-course-on-bans-of-parental-control-apps/

随着反垄断调查的临近，苹果公司逆转了禁止家长控制应用程序的进程

随着国会调查和联邦监管机构更严格的审查即将到来，苹果突然改变了在 T2 禁止其应用商店提供家长控制应用的做法。

据《T4》《纽约时报》报道，苹果悄悄更新了应用商店指南,撤销了禁止某些家长控制应用的决定。

苹果和某些应用程序开发者之间的斗争可以追溯到去年，当时 iPhone 制造商首次通知公司如果他们不改变他们的监控技术，它将切断他们对应用程序商店的访问。

苹果公司在今年早些时候的一份声明中表示，问题的核心是苹果公司从应用商店中移除的家长控制应用程序中使用了移动设备管理(MDM)技术。

这些设备管理工具让第三方能够控制和访问设备的用户位置、应用程序使用、电子邮件帐户、相机权限和浏览历史。

该公司表示:“我们早在 2017 年初就开始探索非企业开发人员对 MDM 的这种使用，并在 2017 年年中根据这项工作更新了我们的指南。”

苹果承认，在企业希望监控和管理企业设备以控制专有数据和硬件的背景下，该技术具有合法用途，但该公司表示，这“明显违反了应用商店的政策——一家以消费者为中心的私人应用企业在客户的设备上安装 MDM 控件。”

上个月，这些父母监控工具的开发者联合起来提供了一个解决方案。在一份由包括 OurPact、Screen Time、Kidslox、Qustodio、Boomerang、Safe Lagoon 和 FamilyOrbit 在内的应用开发者发布的联合声明中，这些公司简单地说，“苹果应该发布一个公共 API，允许开发者访问苹果原生‘Screen Time’使用的相同功能。”

通过提供对其屏幕时间应用程序的访问，苹果将不再需要开发者为绕过苹果的限制而设置的那种控制。

这里提出的 API 建议概述了开发有效的屏幕时间管理工具所需的功能。它是由一群领先的家长控制提供商开发的，”这些公司说。“它允许开发人员创建超越 iOS 屏幕时间功能的应用程序，以解决父母对社交媒体使用、儿童隐私、跨所有浏览器和应用程序的有效内容过滤等问题的担忧。这鼓励了开发者的创新，也有助于苹果支持他们的主张，即“竞争让一切变得更好，为我们的客户带来最好的应用。”"

现在，苹果公司已经改变了其指导方针，表明使用 MDM " 的应用程序必须请求移动设备管理功能，并且只能由商业企业提供，如商业组织、教育机构或政府机构，以及在有限的情况下，将 MDM 用于父母控制的公司。MDM apps 不得出于任何目的向第三方出售、使用或披露任何数据，并且必须在其隐私政策中对此做出承诺。”

从本质上来说，它只是扭转了公司的政策，而没有授予访问屏幕时间，正如公司财团所建议的那样。

“这是地狱般的过山车，”OurPact 的高级产品经理达斯汀·戴利告诉《纽约时报》。OurPact 在 2 月份被下架之前，一直是 App Store 中排名第一的家长控制应用。一位女发言人告诉《纽约时报》,该公司估计苹果此举花费了大约 300 万美元。
