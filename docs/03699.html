RapidAPI 获得由微软牵头的 2500 万美元，其 API 市场突破了 10K API 和 100 万用户 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2019/06/11/rapidapi-nabs-25m-led-by-microsoft-as-its-api-marketplace-cracks-10k-apis-and-1m-users/

API——开发人员用来集成其他应用程序或帮助他们的应用程序与其他应用程序集成的轻量级编程接口——已经成为我们以软件为动力的世界的重要组成部分，大多数组织都在使用它们进行构建和工作，据估计目前有超过 50，000 个 API 在流通。

然而，这些数字和受欢迎程度会导致在可发现性方面的大量分裂和混乱。现在，一家正在建立市场来帮助解决这些问题的初创公司已经筹集了资金来利用这个机会。

开发者用来搜索、支付和连接公共 API 的 RapidAPI 已经完成了 2500 万美元的 B 轮融资。

首席执行官兼联合创始人 Iddo Gino 在接受采访时表示，这家初创公司计划利用这笔资金继续扩大其市场规模，并为开发者提供各种工具。

首先将是一个针对开发者群体的新产品，恰当地命名为 RapidAPI for Teams ，这将帮助他们不仅管理公共 API 的使用，还可以组织和使用他们自己的内部 API 和微服务。(吉诺说，该产品将对最多五名开发者免费，此后每个用户每月收费 10 美元，不限次数。)

这笔资金是在这家初创公司发展良好的时候获得的。该公司目前在其市场上有 10，000 种 API，估计覆盖了全球所有公开可用 API 的 33%(仍有很大的增长空间)；使用 RapidAPI 的开发人员现在有 100 万，他们现在每月从大大小小的各种公司总共进行 5000 亿次 API 调用，包括微软、SendGrid、Nexmo、Telesign、谷歌、Skyscanner 和 Crunchbase (TC 的堂兄弟)。

“除此之外，我们还有 20，000 多个 API 要么是过时的(未使用，大多是私有的)，要么是仅由一个用户在私有模式下使用，因此我们不将它们纳入我们的统计，”Gino 说。

同时，客户包括一些 API 提供商和其他企业，包括 Cisco、Hyatt、SAP、Delta 和 Reddit。

RapidAPI 今天发布的统计数据都是基于它去年透露给我的数据，当时我写了这家初创公司的第一轮融资。(当时它有 8000 个 API，50 万开发者，4000 亿次调用。)

它现在还有大约 50 名员工，分布在旧金山、特拉维夫和乌克兰。

B 轮融资——使 RapidAPI 的融资总额达到约 3800 万美元——由微软的风险投资部门 M12 牵头，DNS Capital 和以前的投资者 Andreessen Horowitz、Green Bay Capital 和 Grove Ventures 也参与了。

微软的参与是战略性的，值得注意的:该公司长期以来一直将自己打造为万物开发者的首选平台，这一战略可以追溯到多年前，但最近已经扩展到其对 Azure 云平台的巨额投资，以及收购代码库 GitHub，以及其他许多举措。你可以很容易地看到像 RapidAPI 这样的东西可以很好地补充那些现有的工具。

“RapidAPI 正在改变企业利用 API 和微服务进行扩展的方式，”M12 总经理兼董事总经理 Mony Hassid 在一份声明中表示。“我们对 RapidAPI 在提高开发人员生产力、简化重复工作以及帮助合并 API 合同方面的潜力感到兴奋。“Hassid 现在也加入了 RapidAPIs 的董事会。

据我们所知，微软并不是此轮投资 RapidAPI 的唯一战略公司。但该公司年轻的创始人 Iddo Gino 在接受采访时表示，这笔交易的条款非常明确，该公司可以非常自由地寻求与他人合作，包括微软的竞争对手。

在另一方面，竞争对手也是 RapidAPI 的首选。正如你所想象的，API 使用的普及——不仅扩展到软件，还扩展到软件支持的许多硬件设备和物联网设备——导致了许多公司都在追求相同的业务。

像 Zapier 和 T2 IFTTT 这样的公司(同样由安德森支持)提供可以连接在一起的应用程序目录；ProgrammableWeb 也提供了一个历史悠久的 API 目录，尽管 Gino 认为它包含的 RapidAPI 提供的工具较少。他指出，他看到了多元化，这是一个更大的云服务市场，一个更直接的竞争对手。

开发人员工具的激增，尤其是那些旨在帮助开发人员使用 API 的工具，将不可避免地导致更多的整合，大鱼吃掉一些小鱼，或者小鱼聚集在一起向市场提出更强的主张。RapidAPI 已经成为这一趋势的受益者:它在 2017 年收购了 Mashape，导致其用户数量大幅增加。可能还会有更多这样的事情发生。

虽然 RapidAPI 迄今为止一直专注于公共 API，但团队转向私有 API 将为一系列全新的服务和用例打开大门，表明公司的另一种增长。RapidAPI 引用 Ping Identity 的一项调查指出，25%的公司通常使用 1000 个以上的内部 API 或微服务，而另外 35%的公司有 400-1000 个内部 API。API 的日子似乎又来了。
