Twitter 开发者实验室向所有人开放，发布首批 API TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2019/06/11/twitter-developer-labs-opens-to-all-with-release-of-first-apis/

今年 5 月，Twitter 宣布计划推出 Twitter 开发者实验室计划，这是一种供应用开发者注册以试验预发布测试版 API 的方式。该公司当时解释说，这个想法是为了让开发者能够尽早测试新的 API 产品并提供反馈。

今天，Twitter 表示将推出其首个 Twitter 开发者实验室端点:GET/users 和 GET/tweets。这些允许开发者通过 ID 查询推文和用户。

Twitter 早些时候证实，这些将是通过新计划推出的第一批 API 产品。随后会有过滤推文，提供实时但经过过滤的推文流；以及最近搜索，提供了搜索过去七天的推文的方法。在开发者实验室网站上，后面这些 API 仍然被列为“即将推出”。

除了作为开发者测试和反馈 API 产品的一种方式，新程序的推出是 Twitter 重新设定与更广泛的开发者社区关系的最新尝试。

该公司在 API 平台方面经历了一段坎坷的道路，有时会中断对关键 API 的访问，并在此过程中摧毁了应用程序制造商的业务。Twitter 从未完全接纳它的开发者社区。几年前，它从那些构建另类 Twitter 客户端的人脚下拔掉了地毯。T4 欺骗了自己的合作伙伴。它主办了然后又取消了自己的开发者大会。它卖掉了它的 Fabric developer 工具集。最近，2018 年的一个变化打破了许多第三方 Twitter 阅读客户端。

然而，该公司现在希望更直接地与开发者合作将是另一种前进的方式。通过让开发人员在 API 变化方面有发言权，它甚至可能对其付费 API 产品产生更多兴趣，如高级(每月 339 美元至 2899 美元)API 或甚至企业 API 层。

Twitter 今天提醒开发者，它的新端点仍然是早期版本，仍然可能发生变化。

“由于我们将在实验室发布的端点是早期预览，在我们广泛发布之前，它们可能会发生变化，”Twitter 在其 Twitter Labs 社区网站上的一份声明中写道。“我们鼓励您在与他们合作时考虑这一点。例如，这可能意味着它们不是用于大规模生产应用程序的好选择，而可能更适合原型开发(如果你运行多个版本的应用程序)，”该公司说。

当发布新版本的端点时，开发人员会提前得到通知，因此他们有时间进行更新。和实验室版本也将在过渡期间得到支持。

感兴趣的开发者必须创建一个开发者账户，获得批准，然后加入实验室访问新的 API。
