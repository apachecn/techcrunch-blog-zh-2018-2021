数以百万计的 Venmo 交易因隐私设置遭到警告

原文：https://web.archive.org/web/https://techcrunch.com/2019/06/16/millions-venmo-transactions-scraped/

一名计算机科学学生收集了 700 万条 Venmo 交易，以证明用户的公共活动仍然可以很容易地获得，一年前，一名隐私研究员以类似的方式下载了数亿条 Venmo 交易。

丹·萨蒙说，他在六个月的时间里收集了这些交易，以提高人们的意识，并警告用户将他们的 Venmo 支付设置为私人支付。

去年，在前 Mozilla 研究员 Hang Do Thi Duc 下载了 2.07 亿笔交易后，P2P 移动支付服务受到了批评。刮擦努力是可能的，因为用户之间的 Venmo 支付默认是公开的。这些可废弃的数据激发了几个新项目——包括一个每当有人购买毒品就发出推特的机器人。

一年过去了，Salmon 显示几乎没有什么变化，仍然可以很容易地通过该公司的开发者 API 下载数百万笔交易，而无需获得用户许可或需要该应用程序。

使用这些数据，任何人都可以查看整个用户的公共交易历史，他们与谁分享了金钱，何时分享，在某些情况下是出于什么原因——包括非法商品和物质。

“真的没有理由让这个 API 对未经认证的请求开放，”他告诉 TechCrunch。“该 API 的存在只是为了为应用程序的主页提供公共交易的滚动提要，但如果这是你的目标，那么你应该在每个请求中要求一个令牌来验证用户是否登录。”

他在自己的 GitHub 页面上发布了抓取的数据。

自从一年前刮刮乐爆发以来，Venmo 几乎没有为其 4000 万用户解决隐私问题。Venmo 的反应是改变其隐私指南，后来更新了其应用程序，当用户将默认隐私设置从公开更改为私人时，删除了警告。

相反，Venmo 一直致力于使数据更难抓取，而不是潜在的隐私问题。

当 Dan Gorelick 在 2016 年首次对 Venmo 的公共数据发出警报时，对 API 的限制很少，这意味着任何人都可以批量和快速地收集数据。像 Johnny Xmas 这样的其他研究人员已经说过【Venmo 限制了它的 API 来限制可以收集的历史数据。但是 Venmo 最近的限制仍然允许 Salmon 每分钟吐出 40 个交易。他说，这相当于每天约有 5.76 万笔废弃交易。

去年，拥有 Venmo 的 PayPal 就侵犯隐私和安全问题与美国联邦贸易委员会达成和解。该公司因在隐私设置上误导用户而受到批评。美国联邦贸易委员会表示，用户没有被适当告知一些交易将被公开分享，Venmo 谎称该应用程序的安全性，称其为“银行级”，美国联邦贸易委员会对此提出质疑。

PayPal 的发言人 Juliet Niczewicz 没有回复置评请求。
