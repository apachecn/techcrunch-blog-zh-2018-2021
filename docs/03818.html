MongoDB 获得了数据湖、新的安全特性等等

原文：https://web.archive.org/web/https://techcrunch.com/2019/06/18/mongodb-gets-a-data-lake-new-security-features-and-more/

今天，MongoDB 将举办其开发者大会，不出所料，该公司有几项声明要宣布。有些是直截了当的，比如推出具有一些重要的新安全特性的 MongoDB 4.2，而另一些，比如推出该公司的 Atlas Data Lake，则表明该公司超越了其核心数据库产品。

“我们的新产品从根本上扩展了开发人员使用 MongoDB 更好地处理数据的方式，”MongoDB 首席执行官兼总裁 Dev Ittycheria 说。“我们努力帮助开发人员提高工作效率，并消除基础架构方面的难题——提供额外的功能以及附加功能，如全文搜索和数据湖。IDC 预测，到 2025 年，全球数据将达到 175，其中 49%将驻留在公共云中。我们的使命是为开发人员提供更好的方法来处理数据，无论数据位于何处，包括公共云和私有云。”

今天这组公告的重头戏大概就是 MongoDB Atlas 数据湖的推出了。Atlas Data Lake 允许用户使用 MongoDB 查询语言在 AWS S3 上查询数据，无论其格式如何，包括 JSON、BSON、CSV、TSV、Parquet 和 Avro。开始时，用户只需要将服务指向他们现有的 S3 桶。他们不必管理服务器或其他基础设施。Google 云存储和 Azure 存储对数据湖的支持正在进行中，将在未来推出。

同样新的是全文搜索，它给用户提供了基于开源 Apache Lucene 8 的高级文本搜索功能。

此外，MongoDB 现在也开始整合 Realm，它在今年早些时候收购的移动数据库产品和它的产品阵容。利用 Realm 品牌，Mongo 正在合并其无服务器平台 MongoDB Stitch 和 Realm 的移动数据库和同步平台。Realm 的同步协议现在将连接到 MongoDB Atlas 的云数据库，而 Realm Sync 将允许开发人员将这些数据带到他们的应用程序中。

MongoDB 的首席技术官兼联合创始人艾略特·霍洛维茨(Eliot Horowitz)解释说:“通过将 Realm 广受欢迎的移动数据库和同步平台与 Stitch 的优势相结合，我们将使在堆栈的每一层处理数据变得自然而简单，并在边缘设备和核心后端之间无缝移动数据，从而消除开发人员的大量工作。

至于 MongoDB 的最新版本，该版本的亮点是一组新的安全功能。在这个版本中，Mongo 实现了客户端字段级加密。传统上，数据库安全一直依赖于服务器端的信任。这通常使管理员可以访问数据，即使他们没有客户端访问权限。如果攻击者破坏了服务器，那几乎必然会是一个灾难性的事件。

有了这个新的安全模型，Mongo 将访问权转移到了客户端和本地驱动程序。它提供了多种加密选项；为了让开发人员利用这一点，他们将使用一个新的“加密”JSON scheme 属性。

这确保了所有应用程序代码通常可以不加修改地运行，即使是管理员也不能访问数据库或其日志和备份，除非他们自己获得客户端访问权限。因为逻辑驻留在驱动程序中，所以加密的处理也完全独立于实际的数据库。

MongoDB 4.2 中的其他新特性包括对分布式事务的支持，以及从单个 Kubernetes 控制平面管理 MongoDB 部署的能力。
