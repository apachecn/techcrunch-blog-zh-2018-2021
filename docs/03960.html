我们将在 TC 会议上谈论更多的 Kubernetes:微软的 Brendan Burns 和谷歌的 Tim Hockin 的企业

如今，参加企业会议不能不谈论容器——尤其是 Kubernetes 容器管理系统。因此，毫不奇怪，我们将在 9 月 5 日于旧金山举行的首届 TC Sessions: Enterprise 活动中做同样的事情。正如我们已经在上周宣布的，Kubernetes 的联合创始人 Craig McLuckie 和谷歌的 Kubernetes 产品管理总监 Aparna Sinha 将加入我们，谈论企业中容器的过去、现在和未来。

此外，我们现在可以宣布，Kubernetes 的另外两位联合创始人将加入我们:谷歌首席软件工程师蒂姆·霍金，他目前从事 Kubernetes 和谷歌容器引擎的工作，以及微软杰出的工程师布伦丹·伯恩斯，他在谷歌期间是 Kubernetes 的首席工程师。

T2】

因此，我们将邀请 Kubernetes 的四位联合创始人中的三位上台谈论这个历时五年的项目。

在加入 Kuberntes 项目之前，Hockin 参与了 Google 的内部项目，如 Borg 和 Omega，以及 Linux 内核。在 Kubernetes 项目中，他致力于核心功能和早期设计决策，涉及网络、存储、节点、多集群、资源隔离和集群共享。

当他的同事 Craig McLuckie 和 Joe Beda 决定将他们在 Kubernetes 上的工作转化为一家初创公司 Heptio，然后以大约 5.5 亿美元的价格成功出售给 VMware 时，Burns 走了一条不同的道路，并于三年前加入了微软 Azure 团队。

我想不出更好的专家组来谈论 Kubernetes 在重塑企业构建软件的方式中所扮演的角色。
