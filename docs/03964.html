Cloudflare 将今天的宕机归咎于“糟糕的软件”部署

如果你整个早上都在经历“502 坏网关”通知，不管是好是坏，你并不孤单。Cloudflare 今天早上经历了一些重大的停机，许多站点都受到了影响。事实上，该公司收集全球事件的系统状态页面，读起来就像是全球各大城市的洗衣单。

Cloudflare 已经承认了这个看起来极其普遍的问题，并且似乎正在努力解决这个问题。该公司写道:“Cloudflare 已经针对该问题实施了一个修复程序，目前正在监控结果。”“一旦问题得到解决，我们将更新状态。”我们已经联系了该公司以获取更多信息，我们也会这样做。

现在，也许去附近走走。外面天气很好。

更新: Cloudflare 联合创始人兼首席执行官马修·普林斯提供了一些关于这里发生的事情的见解。“CPU 使用量的巨大峰值导致主系统和备份系统崩溃。影响了所有服务。没有证据表明与袭击有关。关闭导致 CPU 峰值和流量恢复到正常水平的服务。挖掘根本原因。”

一切似乎都恢复了正常。该公司现在对所发生的事情提供了更多的见解，指责自己的软件，而不是一些人最初猜测的攻击。

“在今天大约 30 分钟的时间里，访问 Cloudflare 网站的访问者收到了 502 个错误，这些错误是由于我们网络上的 CPU 利用率大幅上升而导致的，”该公司在博客文章中写道。“此 CPU 峰值是由回滚的错误软件部署导致的。回滚后，服务恢复正常运行，所有使用 Cloudflare 的域恢复正常流量水平。这不是一次攻击(正如一些人猜测的那样),我们对这一事件的发生感到非常抱歉。在我撰写本文时，内部团队正在开会进行全面的事后分析，以了解这种情况是如何发生的，以及我们如何防止这种情况再次发生。”

在进一步的帖子中，该公司就此事道歉，写道，“我们认识到这样的事件对我们的客户来说是非常痛苦的。在这种情况下，我们的测试流程是不够的，我们正在审查并改变我们的测试和部署流程，以避免将来发生类似事件。”
