如果你整个早上都在经历“502 坏网关”通知，不管是好是坏，你并不孤单。Cloudflare 今天早上经历了一些大规模的停电，让许多站点都受到了影响。事实上，该公司收集全球事故的系统状态页面读起来就像是全球每个主要城市的洗衣单。

Cloudflare 已经承认了一个看起来非常普遍的问题，并且似乎正在努力解决这个问题。该公司写道:“Cloudflare 已经对这个问题进行了修复，目前正在对结果进行监控。”。"一旦问题解决，我们将更新状态."我们已经向该公司寻求更多信息，并将继续努力。

现在，也许去街区里散散步。外面很好。

更新: Cloudflare 的联合创始人兼首席执行官马修·普林斯提供了一些关于这里发生的事情的见解。“CPU 使用率的大幅飙升导致主系统和备份系统崩溃。影响了所有服务。还没有证据表明与袭击有关。关闭造成 CPU 峰值的服务，并将流量恢复到正常水平。挖掘根本原因。”

一切似乎都恢复了正常。该公司现在对发生的事情提供了更多的见解，指责自己的软件，而不是一些人最初猜测的攻击。

“今天，在大约 30 分钟的时间里，Cloudflare 网站的访问者收到了 502 个错误，这些错误是由我们网络上的 CPU 利用率大幅飙升造成的，”该公司在博客文章中写道。“CPU 峰值是由回滚的错误软件部署引起的。回滚后，服务恢复正常运行，所有使用 Cloudflare 的域恢复到正常流量水平。 这不是袭击(正如一些人猜测的)，我们对此次事件的发生深感抱歉。当我写这篇文章时，内部团队正在开会进行全面的事后分析，以了解这是如何发生的，以及我们如何防止这种情况再次发生。”

在进一步的帖子中，该公司就此事道歉，写道，“我们认识到这样的事件对我们的客户来说是非常痛苦的。在这种情况下，我们的测试流程是不够的，我们正在审查并改变我们的测试和部署流程，以避免将来发生类似事件。”
