互联网集团指责 Mozilla 支持 DNS 隐私功能是“互联网恶棍”

原文：https://web.archive.org/web/https://techcrunch.com/2019/07/05/isp-group-mozilla-internet-villain-dns-privacy/

一个互联网服务提供商的行业组织称火狐浏览器制造商 Mozilla 为支持 DNS 安全标准的“互联网恶棍”。

英国互联网服务提供商的行业组织英国互联网服务提供商协会(ISPA)提名浏览器制造商推出安全功能，他们说这将允许用户“绕过英国的过滤义务和家长控制，破坏英国的互联网安全标准。”

去年底，Mozilla 公司称T3 公司正计划对一小部分用户测试 HTTPS 域名系统。

每当您访问网站时，即使它启用了 HTTPS，将网址转换为电脑可以读取的 IP 地址的 DNS 查询通常是不加密的。安全标准是在应用程序级别实现的，这使得 Mozilla 成为第一个使用 HTTPS 域名系统的浏览器。通过加密 DNS 查询，它还可以保护 DNS 请求免受中间人攻击，中间人攻击允许攻击者劫持请求并将受害者指向恶意页面。

HTTPS DNS 也提高了性能，使 DNS 查询和整体浏览体验更快。

但是 ISPA 认为 HTTPS 域名系统与英国目前的网站屏蔽制度不兼容。

根据英国法律，如果网站被认为包含恐怖主义材料或虐待儿童的图像，则可能会因侵犯版权或商标材料而被屏蔽。在加密 DNS 查询方面，据称这将使互联网提供商更难过滤其用户的互联网接入。

ISPA 并不孤单。英国间谍机构 GCHQ 和维护英国互联网黑名单的互联网观察基金会(Internet Watch Foundation)批评了向浏览器推出加密 DNS 功能的举措。

ISPA 的提名很快引起了安全界的愤怒。在社交媒体的强烈反对声中，ISPA 加倍了自己的立场。“默认引入 DNS-over-HTTPS 将对在线安全、网络安全和消费者选择有害，”但表示它鼓励“进一步的辩论”

安德鲁斯&阿诺德公司(Andrews & Arnold)是一家互联网供应商，它向 Mozilla 捐赠了 2940 英镑(约合 3670 美元)，以支持这家非营利组织。“选择这个数额是因为如果我们是会员，这就是我们的 ISPA 会员费，”在该公司的一条推特上说。

Mozilla 发言人贾斯汀·奥凯利(Justin O'Kelly)告诉 TechCrunch:“我们感到惊讶和失望的是，一个互联网服务提供商行业协会决定歪曲几十年来旧的互联网基础设施的改进。”

“尽管有相反的说法，一个更加隐私的 DNS 不会阻止在英国使用内容过滤或家长控制。HTTPS 域名系统(DoH)将给英国公民带来真正的安全利益。我们的目标是建立一个更安全的互联网，我们将继续与英国可信的利益相关者就如何实现这一目标进行认真、建设性的对话，”他说。

“我们目前没有计划在英国默认启用 HTTPS 域名系统。但是，我们目前正在欧洲寻找潜在的 HTTPS 域名系统合作伙伴，以便将这一重要的安全功能更广泛地推广到其他欧洲国家，”他补充道。

Mozilla 并不是第一个在 HTTPS 推广 DNS 的公司。去年，Cloudflare 发布了其专注于 1.1.1.1 隐私的 DNS 服务的移动版本，包括 HTTPS 域名系统。几个月前，谷歌旗下的 Jigsaw 发布了其旨在防止 DNS 操纵的突破审查的应用 Infra。

Mozilla 尚未确定在 Firefox 中全面发布 HTTPS 域名系统的日期。
