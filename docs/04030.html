Mozilla 阻止间谍公司 DarkMatter 使用 Firefox，理由是对用户有“重大风险”

原文：https://web.archive.org/web/https://techcrunch.com/2019/07/09/darkmatter-firefox-certificates/

Mozilla 阻止间谍公司 DarkMatter 使用 Firefox，理由是对用户有“重大风险”

Firefox 制造商 Mozilla 表示，它不会信任来自监控制造商 DarkMatter 的证书，从而结束了长达数月的被这款流行浏览器列入白名单的努力。

几个月前，总部位于阿联酋的 DarkMatter 要求 Mozilla 正式信任其在 Firefox certificate store 中的根证书，Firefox certificate store 是浏览器中为受信任并被批准颁发 HTTPS 证书的证书颁发机构保留的位置。Mozilla 和其他浏览器制造商使用这个商店来了解哪些 HTTPS 证书值得信任，从而有效地允许这些证书管理机构确认网站的身份，并证明进出网站的数据是安全的。

但是流氓或恶意的认证机构可以通过伪造或假冒网站来拦截加密的互联网流量。

DarkMatter 有一段充满争议和可疑的操作历史，包括开发用于监视行动的恶意软件和间谍软件，以及据称针对批评该公司的记者。就在几周前，路透社报道称，这家雇佣前美国国家安全局黑客的阿联酋公司在阿拉伯君主制的授意下，将几名媒体人士和持不同政见者作为攻击目标。

但该公司作为认证机构没有不良记录，这让 Mozilla 处境艰难。

Mozilla 要么接受 DarkMatter 作为认证机构的记录，要么基于可感知的风险拒绝它。

事实证明，后者赢了。

“我们的首要责任是保护依赖 Mozilla 产品的个人，”Mozilla 认证中心项目经理韦恩·塞耶(Wayne Thayer)周二在的一个讨论组帖子中说。他补充说，暗物质给“我们的用户带来了巨大的风险。”

“我相信这一框架有力地支持了撤销对暗物质中间证书信任的决定，”他写道。

塞尔补充说，尽管考虑了 DarkMatter 业务的两个方面，但浏览器制造商引用了 Mozilla 的一个核心原则——“个人在互联网上的安全和隐私是根本的，不能被视为可有可无”——作为拒绝该提议的理由。

Mozilla 表示，同时也不信任六个中介证书。

DarkMatter 周二没有回应置评请求。
