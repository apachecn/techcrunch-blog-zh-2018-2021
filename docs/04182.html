高空天使发布安全无人机飞行 API

原文：https://web.archive.org/web/https://techcrunch.com/2019/07/17/altitude-angel-launches-an-api-for-safer-drone-flights/

为无人机提供安全、数据和交通管理系统的英国初创公司 Altitude Angel 正在推出无人机飞行的冲突消除服务——可通过其开发者 API 平台获得。

“动态系统将持续监控飞机周围的空域，以发现‘意外’情况，如其他飞行器或空域变化(如临时飞行限制/警方事件周围的动态地理围栏)，”它对新服务写道。

“在识别出潜在冲突后，CRS 将进行必要的航线调整，允许无人机在其他空域用户之间保持适当的分离标准，或者绕过受限空域飞行，以便它可以安全(高效)地继续飞往目的地。”

全球冲突解决服务(CRS)有两个组成部分:战略去冲突，将于 7 月 23 日首先推出，让无人机运营商向初创公司提交飞行计划，以确定是否与其他先前提交的飞行计划有任何冲突，或者与在海拔 天使的全球数据反馈中可用的地面和空域地理围栏区域有任何冲突。

如果发现冲突，高空天使公司表示，其 CRS 将建议更改起飞时间和/或路线，以“消除冲突”——正如其所说，这意味着“微创变化，以允许任务继续顺利进行。”

该服务还支持“私人”模式，适用于只想检查与自己的无人机或客户发生冲突的车队运营商。

第二部分——将于 9 月下旬推出——被称为战术去冲突。这将为无人机飞行员或无人机本身提供信息，以确保在飞行阶段保持分离。

“我们带来了当今所有载人空中交通的商业数据。所以这是每一次商业飞行，在某些情况下是警用直升机，医疗直升机等。等。首席执行官理查德·帕克说:“因此，战术服务将在无人机碰撞数据(来自统计 CRS)上补充无人机与有人驾驶飞机的碰撞数据。”。

帕克说，这家英国初创公司还为无人机的地理围栏服务提供数据(无人机制造商 DJI 是其客户之一 )正在将其软件和服务业务定位为 无人驾驶交通管理(UTM)公司、国家组织和车队运营商的支持层，以嵌入他们自己的产品。

他告诉 TechCrunch:“我们所做的已经超越了一家典型的 UTM 公司眼中的自己的客户，然后向所有人提供我们收到的飞行计划。”。“因此，举例来说，优步可能会使用[CRS]服务来注册优步的所有航班，而亚马逊可能会使用该服务来注册亚马逊的所有航班——但通过 API，它们可以有效地避开彼此。

“因此，这是一种将所有人联系在一起的服务，只有当预计会发生冲突时，它才会告诉你。”

显然，随着用户越来越多，战略冲突消除组件的效用将会增加，这使它能够增加它可以提供的关于什么在何时何地飞行的可见性。

高空天使并没有假装它能够提供天空中所有人造物体的全景。

帕克说:“我们认为，如今在 UTM 行业中普遍存在的一件事就是虚假声明。”。“我们很容易错误地推广这项服务——我们可以这样说，这项服务保证无人驾驶飞机永远不会坠毁。那根本不是真的。然而，它保证的是，任何向我们提交计划的无人机都将被提前告知是否可能与另一架无人机发生冲突。

“当战术服务上线时，我们将再次非常明确——如果你可能与之冲突的其他一切都在使用该服务，那么我们就可以提供这种分离。”

他指出，即使是空中导航服务提供商(ANSPs)也不能一直看到所有的空中交通。因此，CRS 被定位为无人机操作员提高对附近可能飞行的其他物体的意识的一种方式，从而降低碰撞或安全事故的风险。

至于 CRS 的动态战术冲突消除组件，该组件旨在提醒无人机操作员注意附近的意外飞行器，Altitude Angel 表示，这是基于“可靠的安全技术”。

根据帕克的说法，支撑它的核心平台自 2016 年以来一直在运行——最初由通用航空飞行员用来请求进入过境 D 类空域，这意味着它“积累了数千个请求”，并在全球范围内接受了“大量审查”，包括来自国家空中交通服务的审查。

“这是一项极其可靠和强大的服务，”他声称。

高度天使也在其 GuardianUTMS 空域管理平台上分层。Parker 指出，该公司的企业背景是大规模分布式云系统——因此，它习惯于每月处理大约 700 万到 1000 万个 API 请求。

“所以我们认为我们有一个相当稳健可靠的系统，”他说。“它还能容忍失败，并能自我修复。从基础设施的角度来看，它非常强大；从应用的角度来看，它已经为世界上最值得信赖和尊重的 ANSPs 之一做了大量的运营使用案例和负载。”

“使用率仍在增加。我们仍在从中学习。但同样，我们的主要目标是把它拿出来，使用它，监控它，确保我们随着时间的推移不断改进它。这是一种爬行、行走和奔跑类型的服务，”他补充道。

所有 Altitude Angel 目前的客户都已签约使用 CRS——帕克表示，这将转化为每月约 5000 至 6000 次航班，为消除冲突服务提供信息。

他继续说道:“然后，我们将连接与我们共享的额外航班，所以我认为我们谈论的是今天与任何 UTM 共享的所有航班中相当大的一部分。”。“我们接下来要做的是与我们的 ANSP 客户合作，看看他们当前管理的许可请求是否也可以连接到该网络。我认为这是一个非常有趣的探索领域。

“因为我们这样做只是因为，最终，该行业的每个人都希望超越视线，每个人都希望能够有一个更加自动化的飞行系统。但现实是，在 ANSP 和监管方面没有基础设施，从安全管理的角度来看，在商业方面也没有技术。

“这就是我们试图填补的空白，以便更多的人能够参与进来。”

虽然由国家机构而不是商业实体来运营无人机冲突消除平台可能更有意义，但帕克并不担心监管机构会突然介入并占据这一空间，因为该业务将自己定位为扮演多重角色:帮助无人机运营商整合并适应多变的监管规定，同时确保如果政府决定监管机构应该提供 UTM，它可以为 ANSPs 提供网关服务。

“我们提供给客户的技术是在我们自己的开发者平台上提供给商业行业使用的，但我们也向 ANSPs 提供同一系统的一个版本，以便能够在全国范围内提供该服务，”他说。

“我认为重要的是要认识到，许多 ANSPs 还没有被要求这样做。因此，他们不一定要部署这些基础…一个有趣的角度是，我们对这些开发客户和使用我们商业技术的人的承诺是，让他们远离当地法规和国际差异。”

“在英国，如果政府突然转向(英国空中交通运营商)NATS，说‘嘿，你们必须为整个国家提供 UTM 服务’，运营这项服务的将不是我们，但我们非常希望有机会向 NATS 提供技术，实际上向该行业的其他人提供这种能力，”他补充道，并指出高空天使已经在向 NATS 提供空域用户门户技术。

“所以，再一次，我们有这个业务的商业方面——这都是为了让那些人能够与受监管的社区整合，然后我们有一项技术能力[卫报 UTMS],这就是我们正在推动 ANSPs 使他们能够开放天空，在他们的空域范围内与无人机合作并拥抱无人机。”
