GitHub 获得 CI/CD 服务

原文：https://web.archive.org/web/https://techcrunch.com/2019/08/08/github-actions-is-now-a-ci-cd-service/

微软 GitHub 今天发布了新版 GitHub Actions 的测试版，该服务内置了完整的持续集成和交付(CI/CD)功能。计划于 11 月 13 日正式上市。

该公司今天还宣布，其平台上现在有超过 4000 万开发者。

十个月前，GitHub 推出了其工作流自动化平台 行动。开发人员已经可以采取措施来触发各种事件，并使用它们来构建定制的 CI/CD 管道。在发布时，GitHub 团队强调，行动允许建立这些管道，但它远不止于此。尽管如此，开发人员显然对使用 CI/CD 的动作很感兴趣。

GitHub 首席执行官 Nat Friedman 在今天的公告中写道:“自从我们去年推出 GitHub Actions 以来，反响非常显著，开发人员已经创建了数以千计的灵感工作流。”“但我们也听到了几乎所有人的明确反馈:您需要 CI/CD！这就是我们今天要宣布的。”

有了这个更新版本的 Actions，开发人员现在可以在任何平台上构建、测试和部署他们的代码，并在容器或虚拟机中运行他们的工作流。开发人员还可以并行测试多个版本的应用程序，这要归功于一项名为“矩阵构建”的新功能，例如，它可以让你在 Linux、Windows 和 MacOS 上同时测试三个不同版本的 Node.js。因为 GitHub 动作是在一个基本的 YAML 文件中定义的，所以做这些修改只需要在文件中添加几行代码。

支持的语言和框架包括 Node.js、Python、Java、PHP、Ruby、C/C++、。NET，Android 和 iOS。Actions 还集成了 GitHub 包注册表。

在构建应用程序时，您还可以将实时日志流式传输到操作控制台，并且很容易链接到日志文件中的任何一行，以便与团队的其他成员讨论问题。

这些新特性在测试期间是免费的，并且将对所有公共存储库保持免费。

GitHub Enterprise Server 的 Actions 将于明年推出，并将包括一个混合选项，允许您将代码保存在私有数据中心，同时仍然使用 GitHub 来编排工作流。

“GitHub Actions 是 CI/CD 和软件自动化的民主化。GitHUb 产品设计高级总监 Max Schoening 表示:“开发人员可以编写对任何 GitHub 平台事件做出反应的工作流，并引用开源 GitHub 动作——可重复使用的代码片段——来加速他们的软件生命周期，就像他们习惯于编写应用程序代码一样。”"它真正是社区驱动的 CI/CD，其定价模式适用于所有人."

随着这一推出，GitHub 现在也与一些在该平台上建立业务的 CI/CD 初创公司进行更直接的竞争。这可能会引起一点摩擦。

CircleCI 首席执行官吉姆·罗斯在一份声明中说:“GitHub 已经承诺向所有合作伙伴开放他们的平台，但只有时间才能证明一切。”“最终，开发人员是聪明的，他们会选择市场上最好、最强大的工具，我们相信 CircleCI 将继续这样做。[……]circle CI 在团队如何从创意到交付方面拥有超过九年的数据和经验，是 CI/CD 领域的领导者，我们有信心为开发人员提供最佳解决方案。”

我预计 Rose 的评论将与其他 CI/CD 播放器的评论相呼应，尽管也值得注意的是，正如 Rose 所做的那样，Actions 可以与其他持续集成服务集成，以允许开发人员在其平台上触发构建。这些提供者也可以在 GitHub 上提供他们自己的操作。

“我们认为 GitHub 的行动是对 Codefresh 的补充。这是用户可以利用 Codefresh 以可伸缩的方式构建健壮管道的另一种方式。一件有趣的事情是，GitHub 在设计动作时遵循了我们的领导。实际上，您可以使用 GitHub 操作作为 Codefresh 管道中的步骤。所以你看，我们实际上是非常一致的，”CI/CD 平台 Codefresh 的首席技术传道者丹·加菲尔德说。“开发者可以在 GitHub 上找到全新的动作！”

当我就此询问 GitHub 时，Schoening 提供了以下声明:“GitHub 和我们的社区相信选择和开放的生态系统。这是我们认真对待的事情，并融入我们所做的一切。GitHub Actions 允许开发人员集成他们所有现有的工具，混合和匹配新的开发人员产品，并与软件生命周期的所有部分挂钩，包括现有的 CI/CD 合作伙伴。”
