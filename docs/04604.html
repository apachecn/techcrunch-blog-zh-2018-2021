自动驾驶卡车初创公司 Starsky 推出 Hutch、其 API 和神经中枢 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2019/08/14/self-driving-truck-startup-starsky-launches-hutch-its-api-and-nerve-center/

三年来，Starsky Robotics 一直在建立其无人驾驶和人类驾驶的卡车业务。它终于准备好展示它的第一个产品了。

它并不像人们想象的那样是一辆自动驾驶卡车。这款名为 Hutch 的产品提供了 Starsky 认为更有价值的东西:自动获取其货运能力，无需电话或纸质日志。

卡车运输业是一个庞大而分散的系统，包括托运人、货运公司、经纪人，当然还有司机。该系统几乎总是与超过可用容量的容量需求(以及驱动因素)不协调。

传统上，大型托运人与 C.H. Robinson 等货运经纪人合作管理运力。包括 corry、Loadsmart、Transfix 和优步货运在内的初创公司旨在通过开发将卡车司机和托运人匹配起来的应用程序，打破卡车运输世界的货运经纪业务。

与此同时，像施耐德这样的传统卡车运输公司已经建立了他们自己的经纪公司，因为——再一次——销售通常会超过运输能力。

Starsky 认为它可以通过自动驾驶卡车解决容量问题。但是 Starsky 没有将其自动驾驶卡车出售给 Schneider 等卡车运输公司，也没有与主要承运商签订合同，而是建造了 Hutch API 向市场投放其产能。

Hutch 是一个 API，它让货运经纪人和主要托运人能够访问 Starsky 的货运能力，而无需联合创始人兼首席执行官 Stefan Seltz-Axmacher 所描述的“后台办公室干预”。该 API 允许合作经纪人和选定的托运人直接与 Starsky 的业务整合。使用 Hutch API，经纪人可以在不打电话给 Starsky 公司的情况下，竞标 Starsky 的自动驾驶和人类驾驶卡车的空间。

“这很简单，我们正在出售我们自动驾驶卡车的能力，”Seltz-Axmacher 说，并承认在大多数行业，像 Hutch 这样的 API 将无处不在。

“如果这是一个寻找最近的踏板车的 API，我们就不会说话，因为我认为有 25 个这样的踏板车存在，”Seltz-Axmacher 说。“在卡车运输领域，我们惊讶于这个行业的手工程度——所有你认为存在的技术都不存在。”

今天，Starsky 的卡车大部分由人驾驶，每天运送 40 个货物。Starsky 通过经纪人 Schneider Logistics 和 C.H. Robinson 运输大部分货物。该公司希望其大部分负载都通过 API 代理，最终没有负载不通过 Hutch。

塞尔茨-阿克斯马赫认为，哈奇对斯塔奇公司的成长至关重要。有了 Hutch，Starsky 的员工可以专注于制造一个可以安全驾驶的机器人，进而为运力有限的卡车运输市场带来一些缓解。

Seltz-Axmacher 告诉 TechCrunch:“我不想通过给经纪人打一大堆电话、漫长的谈判以及等等来出售这种能力。”"我想通过一个 API 让经纪人可以使用它，这一切都是程序化的."

更重要的是，它让 Starsky 可以控制货运地点。Starsky 一直在经营传统的卡车运输业务，雇佣人类司机为客户拖运货物。它还拥有几辆由少数试驾者驾驶和支持的自动驾驶卡车。

Starsky 可以将 API 仅限于其自动驾驶卡车的最安全、最容易和合法的路线。这意味着斯塔奇将错过全国数百甚至数千条路线。

即使有这些限制，斯塔奇也看到了充足的机会。Seltz-Axmacher 说:“我们相信，在我们目前的系统看来容易行驶的路线上，有一个 500 多亿美元的货运市场。”目前，这意味着在阿拉巴马州，佛罗里达州，路易斯安那州和得克萨斯州的路线。

哈奇已经开始行动了。就在几周前，Hutch API 与 Loadsmart 的自动调度 API 集成，以调度一辆自动驾驶卡车。Hutch API 也在内部用于更有效地测试其自动驾驶卡车车队。
