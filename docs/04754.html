网络主机 Hostinger 称 1400 万人受到数据泄露的影响

原文：https://web.archive.org/web/https://techcrunch.com/2019/08/25/web-host-hostinger-data-breach/?utm_source=feedburner&utm_medium=feed&utm_campaign=Feed%3A+Techcrunch+%28TechCrunch%29

网络主机 Hostinger 称数据泄露可能影响 1400 万用户

Hostinger 表示，在检测到未经授权访问包含数百万客户信息的数据库后，它已重置用户密码作为“预防措施”。

据说这次泄密事件发生在周四。该公司在的一篇博客文章中表示，它收到了一个警报，称其一台服务器被不当访问。利用服务器上的访问令牌，黑客无需用户名或密码就可以访问系统，从而进一步访问该公司的系统，包括 API 数据库。该数据库包含用 SHA-1 算法加扰的客户用户名、电子邮件地址和密码，在研究人员发现 SHA-1 容易受到欺骗后，已经弃用了更强的算法。此后，该公司将其密码散列法升级为更强的 SHA-2 算法。

霍斯汀格说，API 数据库存储了大约 1400 万条客户记录。该公司拥有 2900 多万客户。

该公司表示，它正在“与各自的主管部门联系”

来自 Hostinger 的一封解释数据泄露的电子邮件(图片:已提供)

泄密的消息在一夜之间传开了。根据该公司的状态页面，受影响的客户已经收到了重置密码的邮件。

该公司表示，财务数据没有受到损害，客户网站文件或数据也没有受到影响。

但一位受到此次泄密事件影响的客户指责该公司在泄密范围上存在潜在的“误导”。

TechCrunch 看到的一份聊天记录显示，一名客户支持代表告诉客户，API 可以检索客户的财务数据是“正确的”，但该公司“不存储任何支付数据”。该代表告诉客户，Hostinger 使用多个支付处理器，但没有说出它们的名称。

首席执行官 Balys Kriksciunas 告诉 TechCrunch，客户支持代表的言论是“误导”，并否认任何客户财务数据受到损害。然而，公司对违规事件的调查仍在进行中。
