新的开源项目希望将无服务器愿景扩展到功能之外

无服务器技术为开发人员提供了一种无需考虑运行程序所需的基础设施资源的开发方式，但迄今为止，它主要局限于函数驱动的编程。来自 Lightbend 的新开源项目 CloudState 希望通过超越功能来改变这种情况。

Lightbend 首席技术官 Jonas Bonér 认为，这种抽象基础设施的能力可以超越功能和触发器，扩展到更广泛的开发人员体验中。“我认为人们有时[不区分]无服务器和功能即服务。我认为这实际上缩短了技术。Bonér 告诉 TechCrunch:“通过尽可能多地实现自动化，无服务器真正带来的是全新的开发人员体验和运营体验。

他说，当他与客户交谈时，他们渴望更完整的无服务器开发体验，包括程序的所有部分。“很多人说，我有这个优秀的无服务器和服务功能的当前化身用例，但我的应用程序的其余部分并没有真正在那里运行，”他说。这正是 CloudState 试图解决的问题。

Bonér 谨慎地指出，他并不打算取代函数驱动编程。他只想增加它。CloudState 利用了一些现有的技术，如试图将无服务器和容器化结合在一起的开源项目 KNative ，以及 Kubernetes 上的 gRPC、Akka Cluster 和 GraalVM。

他承认 CloudState 仍然是一项正在进行的工作，但他已经有了基本的构建模块，他希望利用开源的力量来推动这个早期项目的发展。今天，它包括几个关键部分——一个概述项目目标的规范，一个开始实现它的协议和一个测试包。

这里的目标是实现无服务器意味着什么的更广阔的视野，开发人员只需编写代码，而不必担心程序运行的底层基础设施。这是一个大胆的方法，但正如 Bonér 所说，这仍处于早期阶段，需要时间和社区来真正实现这一点。
