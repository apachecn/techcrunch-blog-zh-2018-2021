随着 Kubernetes 的赌注得到回报，Cloud Foundry 加倍重视开发人员的体验

超过 50% 的财富 500 强公司正在使用开源云铸造平台即服务项目——直接或通过 Pivotal 等供应商——来构建、测试和部署他们的应用程序。像许多其他项目一样，包括 OpenStack 之类的项目，Cloud Foundry 近年来经历了一点转变，因为越来越多的开发人员开始将容器——尤其是 Kubernetes 项目——作为开发平台。然而，现在这个项目已经准备好把重点放在它与闭源和开源竞争者的区别上:开发者体验。

然而，早在 Docker 普及应用程序部署的容器之前，Cloud Foundry 就已经在容器上下了赌注，并编写了自己的编排服务。然而，在 Kubernetes 背后的所有动力下，毫不奇怪，云铸造厂中的许多人开始考虑这个新项目来取代现有的容器技术。
