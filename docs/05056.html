GitHub 收购代码分析工具 Semmle

原文：https://web.archive.org/web/https://techcrunch.com/2019/09/18/github-acquires-code-analysis-tool-semmle/amp/

微软的 GitHub 今天宣布已经收购了 Semmle ，这是一个代码分析工具，可以帮助开发者和安全研究人员发现他们代码中的潜在漏洞。Semmle 从安全测试中去除了大量的手工工作，而是提供了一种查询语言，允许研究人员使用该服务的分析引擎来测试他们的代码。随着时间的推移，GitHub 团队计划将 Semmle 紧密集成到 GitHub 工作流中。

GitHub 没有透露收购的价格，但 Semmle 最初是从牛津大学的研究中剥离出来的，去年正式由推出，由 Accel 领投了 2100 万美元的 B 轮融资。在此次收购之前，该公司总共筹集了 3100 万美元。

GitHub 的产品 Shanku Niyogi 在今天的声明中写道:“正如关系数据库使提出非常复杂的数据问题变得简单一样，Semmle 使研究人员更容易快速识别大型代码库中的安全漏洞。”“许多漏洞的根本原因都是相同类型的编码错误。使用 Semmle，您可以找到错误的所有变体，消除一整类漏洞。此外，这种方法使 Semmle 更加有效，可以发现更多的问题，并大大减少误报。”

目前 Semmle 的用户包括优步、美国宇航局、微软和谷歌等公司，以及该公司的核心分析平台，该平台具有自动代码审查、项目跟踪，当然还有安全警报，可免费用于开源项目。

Semmle 首席执行官兼联合创始人 Oege De Moor 表示:“GitHub 是一个社区聚会的地方，是安全专家和开源维护者合作的地方，也是开源用户寻找构建模块的地方。“GitHub 最近采取的保护生态系统的措施(维护者安全建议、自动安全修复、令牌扫描和安全开发中的许多其他进步)都是同一个难题的一部分。Semmle 的愿景和技术属于 GitHub。”

GitHub 首席执行官 Nat Friedman 在今天的一篇博客文章中回应了这一点，并指出他认为 GitHub 有“独特的机会和责任来提供工具、最佳实践和基础设施，以使软件开发安全。”

作为这一整体任务的一部分，GitHub 今天还宣布，它现在是一个公共漏洞和暴露(CVE)编号机构。有了这个，维护者现在将能够从他们的库报告漏洞，GitHub 将处理分配 id 并将问题添加到国家漏洞数据库(NVD)。理想情况下，这意味着开发人员将会披露更多的漏洞(因为现在变得容易多了)，并且使用这些代码的其他人将会更快地收到警报。
