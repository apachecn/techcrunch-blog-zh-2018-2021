Passbase 获得 360 万美元支持隐私保护在线身份检查 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2019/09/24/passbase-grabs-3-6m-to-power-privacy-preserving-online-id-checks/

数字身份初创公司 Passbase 已经完成了一轮 360 万美元的种子轮投资，由 Cowboy Ventures 和 Eniac Ventures 领投，Seedcamp 和其他欧洲投资者也参与了投资。

这家 2018 年创立的初创公司今年早些时候在获得了 60 万美元的种子前融资，用于其带有隐私扭曲的全栈身份引擎。

联合创始人 Mathias Klenk 表示，最新一笔资金将用于在美国和欧洲发展团队和销售渠道。“我们的目标是建立一家以 API 为先的公司，因此建立一个强大的核心组织对我们来说至关重要，这样我们才能专注于确保与补充服务的合作关系，”他告诉 TechCrunch。

“到明年年底，我们的目标是推出我们的消费者应用程序，以便个人可以利用我们服务的核心价值主张，企业可以获得无缝重新认证的回报，”他补充说。在客户方面，我们的目标是扩大规模，与目标细分市场中的一些较大的参与者进行试点。"

根据 Klenk 的说法，Passbase 在 5 月份推出了公开测试版，并在整个夏天一直在进行测试，他说第轮 15 家公司一直在积极测试该平台——声称有 300 多家企业对该产品“表示了兴趣”。

Klenk 表示，早期的测试者来自医疗保健、零工经济和移动等行业，“从招聘到金融服务的令人兴奋的用例即将推出”。

产品是什么？Passbase 将其称为“身份验证条纹”——这意味着它提供 API，使开发人员可以轻松地将一系列消费者友好的身份检查插入和集成到他们的数字服务中。例如自拍视频扫描和身份证件扫描。(Passbase 本身正在插入来自一系列合作伙伴的 ID 文档验证服务，并添加了诸如活跃度检查之类的插件。)

它吹嘘“NIST 认证的面部识别、法医身份真实性分析和正在申请专利的零知识共享架构”是其堆栈的一部分。

总体目标是通过构建一个开发人员平台来支持验证技术集成到 web 服务中，同时在消费者端允许 web 用户限制谁可以访问他们的实际数据，从而成为企业和最终用户之间的可信中介交换，也就是“同意层”。因此，隐私的承诺就产生了。

“我们的愿景是建立一个开放的身份系统，鼓励服务持有更少的信息，但要确保他们收到的结果的质量，”Klenk 补充说。

消费者可以提交个人数据来验证他们的身份，例如通过他们的网络摄像头进行面部生物特征扫描和身份证件扫描，而不必依赖他们的数据被暴露给非专业人士并可能被他们错误处理——相反，他们必须信任 Passbase 的技术架构。

它还计划在明年年初推出一款(免费)消费者应用，该应用将为最终用户提供对他们为身份验证而分享的信息的控制，并提供关于这些信息如何被使用的见解——正如 Klenk 所说，为人们提供“他们在线数据暴露的整体视图和分析”。

尽管它不需要终端用户高度参与，通过下载应用程序来“要求他们的数字身份”。

“我们的目标是将你的数字身份整合到验证流程中，”他说，并补充道:“如果你不太关心你的数字足迹，你就不必申请你的数字身份，可以像其他身份验证提供商一样，通过交易关系进行处理。 但是，通过结合您的生物特征和唯一标识符，我们拥有了创建通用数字身份的第一个构件。”

Klenk 表示，他预计访问管理和账户恢复将成为 Passbase 的一个重要领域，因为——或者，如果——消费者接受他们可以控制的“验证数字身份”的想法。

“就企业接受这一点而言，当然有网络效应在起作用，”他继续说道。“也就是说，身份是一个堆栈，如果我们设法将根身份与背景调查、信用评分等其他凭证(通过合作关系)联系起来，就很难使用这样的系统。所以说到底，这取决于谁能提供最全面的解决方案。”

数字身份管理领域的竞争越来越激烈——包括现在的隐私保护登录系统苹果公司也参与其中——所以 Passbase 肯定有自己的工作要做。尽管它的目标是更全面的身份检查，但认为用户名和密码不足以满足数字服务现在要求的许多认证检查，因为现在有一个平台可以让你与几乎任何人联系起来，无论是医疗专业人员、保姆、出租车司机、清洁工、送货司机还是潜在的生活伴侣。

Klenk 表示，Passbase 的防御性“来自 B2B2C 方式，即我们从第一天起就为企业创造有用的服务，同时让消费者拥有数据所有权，以创造一个更加安全和保护隐私的数字未来”。

它在美国也有专利申请。

“对于市场上的一些现有企业来说，彻底转变商业模式很复杂，而对于新的竞争对手来说，这归结于运营模式和执行，”他还谈到了竞争格局。

如果 Passbase 可以让他们的全栈坚持下去，计划是通过开发者平台赚钱，他们将免费为企业提供前 50 次验证。

“之后，我们的定价包含平台访问费和每次验证成本。原因是，随着我们建立越来越多的模块(身份证件验证、电话号码、居住地址、电子邮件、工作许可证)，我们计划转向 SaaS 模式，以可预测的成本为企业提供各种身份识别服务，”他说。“这就是为什么随着销量的增长，我们的定价也反映了较低的可变成本和增加的订阅费。”

一种自助式 b2b 产品将于下月推出——这意味着任何企业都将能够利用 Passbase 的 API 并集成其验证服务。消费者 app 自然会跟在后面。

“对于消费者来说，产品将永远是免费的，因为我们相信数据需要被回馈并属于消费者，”Klenk 补充道。
