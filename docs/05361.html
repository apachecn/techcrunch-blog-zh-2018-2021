Arm 为其嵌入式 CPU 带来定制指令

原文：https://web.archive.org/web/https://techcrunch.com/2019/10/08/arm-brings-custom-instructions-to-its-embedded-cpus/

在圣荷西举行的年度 TechCon 活动上， Arm 今天发布了定制指令，这是其 Armv8-M 架构针对嵌入式 CPU 的一项新功能，顾名思义，它使其客户能够编写自己的定制指令，以加速他们在嵌入式和物联网应用中的特定用例。

“我们已经有了增加加速的方法，但还没有深入到 CPU 的核心。ARM 汽车和物联网业务高级总监 Thomas Ensergueix 在今天的发布会之前告诉我:“我们在这里给(我们的客户)的是编写自己的指令、定义自己的指令的灵活性，并让它们由 CPU 执行。”。

他指出，Arm 一直有一系列的加速选项，首先是通过总线连接 GPU 和今天的神经处理器单元的内存映射架构。这允许 CPU 和加速器并行运行，但是总线是瓶颈。客户也可以选择直接连接到 CPU 的协处理器，但今天的新闻基本上允许 Arm 客户创建自己的加速算法，然后直接在 CPU 上运行。这意味着延迟很低，但它不是像内存映射解决方案那样并行运行。

正如 Arm 所言，这种设置为集成客户工作负载加速提供了最低成本(和风险)的途径，因为不会中断现有的 CPU 功能，并且仍然允许客户使用他们已经熟悉的现有标准工具。

目前，定制指令将仅可在 Arm Cortex-M33 CPU 中实现，从 2020 年上半年开始。默认情况下，它也将适用于所有未来的 Cortex-M 处理器。Arm 的客户无需支付额外费用或购买新的许可证。

Ensergueix 指出，随着我们进入一个联网设备越来越多的世界，越来越多的 Arm 客户将希望针对他们通常非常具体的使用情况优化他们的处理器，例如，他们通常会希望这样做，因为通过创建定制指令，他们可以从这些设备中获得更长的电池寿命。

Arm 已经找到了许多支持定制指令的合作伙伴，包括 IAR Systems、恩智浦、硅实验室和意法半导体。

“Arm 的新定制指令功能允许像恩智浦这样的芯片供应商为其客户提供新程度的特定应用指令优化，以提高新的和新兴嵌入式应用的性能、功耗和静态代码大小，”恩智浦微控制器总经理杰夫·利斯(Geoff Lees)写道。此外，所有这些改进都是在广泛的 Cortex-M 生态系统中实现的，因此客户的现有软件投资得到了最大限度的利用。"

在相关的嵌入式新闻中，Arm 今天还宣布，它正在为运行 Arm Cortex-M 芯片的嵌入式设备的开源操作系统 Mbed OS 建立一个治理模型。Mbed OS 一直是开源的，但 Mbed OS 合作伙伴治理模式将允许 Arm 的 Mbed 芯片合作伙伴通过每月产品工作组会议等工具，在如何开发操作系统方面拥有更多发言权。ADI 公司、Cypress、Nuvoton、恩智浦、瑞萨、Realtek、
三星和 u-blox 等合作伙伴已经加入该小组。
