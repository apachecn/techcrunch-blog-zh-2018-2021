Databricks 将其三角洲湖项目引入 Linux 基金会 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2019/10/15/databricks-brings-its-delta-lake-open-source-project-to-the-linux-foundation/

由 Apache Spark 的原始开发者创建的大数据分析服务 Databricks 今天宣布，它将在开放的治理模式下，将其用于构建数据湖的 Delta Lake 开源项目带到 Linux 基金会。该公司今年早些时候宣布推出 Delta Lake，尽管它仍然是一个相对较新的项目，但它已经被许多组织采用，并得到了英特尔、阿里巴巴和 Booz Allen Hamilton 等公司的支持。

“2013 年，我们有一个小项目，我们在 Databricks […]将 SQL 添加到 Spark，并将其捐赠给 Apache 基金会，”Databricks 首席执行官兼联合创始人 Ali Ghodsi 告诉我。“这些年来，人们慢慢改变了他们实际利用 Spark 的方式，直到去年左右，我们才真正开始意识到，一种新的模式正在出现，Spark 的使用方式可能与我们最初的计划完全不同。”

他说，这种模式是公司将所有数据放入数据湖，然后用这些数据做一些事情，机器学习和数据科学是显而易见的。但是他们也在做一些传统上与数据仓库相关的事情，比如商业智能和报告。对于这种用法，Ghodsi 使用的术语是“湖边小屋”Databricks 越来越多地看到 Spark 正被用于此目的，而不仅仅是取代 Hadoop 和执行 ETL(提取、转换、加载)。“我们看到这种湖边小屋模式越来越多，我们想加倍努力。”

Spark 3.0 将于今天发布，除了推出一个新功能，使您能够向 Spark 添加可插拔的数据目录之外，还支持更多这些用例，并显著加快它们的速度。

Ghodsi 说，三角洲湖本质上是湖屋模式的数据层。例如，它支持数据湖的 ACID 事务、可伸缩的元数据处理和数据版本控制。所有数据都以 Apache Parquet 格式存储，用户可以强制执行模式(如果需要，可以相对容易地更改模式)。

看到 Databricks 为这个项目选择 Linux 基金会很有意思，因为它的根基是 Apache 基金会。“我们非常高兴能与他们合作，”Ghodsi 谈到公司选择 Linux 基金会的原因。“他们运行着这个星球上最大的项目，包括 Linux 项目，但也有许多云项目。云原生的东西都在 Linux 的基础上。”

“将 Delta Lake 纳入 Linux 基金会的中立之家将有助于依赖该项目的开源社区开发解决如何在本地和云中存储和处理大数据的技术，”Linux 基金会战略计划副总裁 Michael Dolan 表示。“Linux 基金会帮助开源社区利用开放的治理模式来实现广泛的行业贡献和共识建设，这将提高数据存储和可靠性的艺术水平。”
