新发现的早期无服务器监控初创公司 io pipe TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2019/11/01/new-relic-snags-early-stage-serverless-monitoring-startup-iopipe/

新遗迹阻碍了早期无服务器监控初创公司 IOpipe

随着我们从一个由虚拟机主导的世界转向一个无服务器的世界，它改变了监控的性质，像 New Relic 这样的供应商肯定认识到了这一点。今天早上，该公司宣布收购位于西雅图的早期无服务器监控初创公司 IOpipe ，以帮助加强其无服务器监控能力。交易条款没有披露。

New Relic 得到了它所谓的“团队关键成员”，至少包括联合创始人艾丽卡·温迪施和亚当·约翰逊，以及 IOpipe 技术。新员工将从西雅图搬到 New Relic 的波特兰办公室。

该公司在宣布收购的博客文章中写道“这笔交易使我们能够立即投资 onboarding，这将使客户更快更简单地将他们的[无服务器]功能与 New Relic 集成，并充分利用我们的仪器和用户界面，允许快速排除整个应用程序堆栈中的复杂问题。”。

它补充说，最初 IOpipe 团队将专注于将 AWS Lambda 功能，如 Lambda 层移入新的 Relic 平台。随着时间的推移，该团队将致力于增加对无服务器功能监控的支持。New Relic 希望通过将 IOpipe 团队和解决方案与自己的结合，可以加快其无服务器监控印章的速度。

Bold Start 的投资者艾略特·德宾(Eliot Durbin)在 2018 年领导了该公司的 200 万美元种子轮，他表示，两家公司都在这笔交易中获胜。他告诉 TechCrunch:“New Relic 对无服务器有着巨大的承诺，因此将 IOpipe 的产品带到他们市场领先的客户群的机会对所有相关人员都很有吸引力。”

这家初创公司一直在帮助运行 AWS Lambda 的公司监控无服务器操作。重要的是要明白，无服务器并不意味着没有服务器，而是云供应商(在这种情况下是 AWS)提供了完成一项操作的确切资源，仅此而已。

一旦行动结束，资源可以简单地重新部署到其他地方。这使得为这种短暂的资源建立监控工具成为一个巨大的挑战。New Relic 也一直致力于解决这个问题，并在今年早些时候为 AWS Lambda 发布了 New Relic Serverless。

正如 TechCrunch 的 Frederic Lardinois 在他关于该公司 2017 年 250 万美元种子期的文章中指出的,温迪施和约翰逊带来了令人印象深刻的资历:

IOpipe 的联合创始人亚当·约翰逊(首席执行官)和埃里卡·温迪施(首席技术官)在这一领域也非常有经验，他们之前曾在 Docker 和 Midokura 等公司工作过(亚当是 Midokura 的第一名员工，埃里卡创建了 Docker 的安全团队)。他们最近从纽约 Techstars 项目毕业。

IOpipe 成立于 2015 年，正好是亚马逊宣布 Lambda 的时间。种子期时，公司有八名员工。根据 PitchBook 的数据，它目前有 1 到 10 名员工，自成立以来已筹集了 707 万美元。

据 Crunchbase 称，新遗迹成立于 2008 年，在 2014 年上市之前，已经筹集了超过 2 . 14 亿美元。截至发稿时，其股价为 65.42 美元，上涨了 1.40 美元。
