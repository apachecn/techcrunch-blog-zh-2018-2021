微软的Azure Synapse Analytics在数据湖和数据仓库之间架起了一座桥梁

在佛罗里达州奥兰多举行的年度Ignite大会上，微软今天宣布一项面向企业的重要新Azure服务: Azure Synapse Analytics ，微软将其描述为“Azure SQL数据仓库的下一次进化”像SQL数据仓库一样，它旨在弥合数据仓库和数据湖之间的鸿沟，它们通常是完全分离的。Synapse还利用了其他各种各样的微软服务，包括Power BI和Azure机器学习，以及包括Databricks、Informatica、Accenture、Panoply、Talend、Attunity、practical Works和Adatis在内的合作伙伴生态系统。它还集成了Apache Spark。

这里的想法是，Synapse允许任何在不同地方处理数据的人从单一服务中管理和分析数据。使用标准SQL，它可以用来分析关系数据和非结构化数据。

微软还强调了Synapse与Power BI的集成，这是一款易于使用的商业智能和报告工具，以及用于构建模型的Azure机器学习。

通过Azure Synapse studio，该服务为数据专业人员提供了一个单一的工作空间，用于准备和管理他们的数据，以及他们的大数据和人工智能任务。还有一个管理数据管道的无代码环境。

正如微软所强调的，希望采用Synapse的企业可以通过Synapse继续在生产中使用他们现有的工作负载，并自动获得该服务的所有好处。Azure Data的微软CVP Rohan Kumar写道:“企业可以将所有数据源、数据仓库和大数据分析系统的见解汇集在一起，更快、更高效、更安全地让数据发挥作用。

在Ignite的演示中，Kumar还将Synapse与Google的BigQuery进行了对比。Synapse在1pb的数据上运行相同的查询所需的时间减少了75%。他还指出，Synapse可以处理成千上万的并发用户——不像微软的一些竞争对手。
