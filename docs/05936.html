GitHub 推出了一款移动应用、更智能的通知和改进的代码搜索

原文：https://web.archive.org/web/https://techcrunch.com/2019/11/13/github-launches-a-mobile-app-smarter-notifications-and-improved-code-search/amp/

图片来源: TechCrunch

在今天的年度 Universe 大会上，微软旗下的 GitHub 宣布了几款新产品，以及开发人员在过去几个月已经可以测试的一些工具的正式可用性。开发者可能最感兴趣的两个公告是 GitHub 首个原生移动应用的发布和改进的通知体验。但除此之外，它还将 GitHub Actions，该公司的工作流自动化和 CI/CD 解决方案，以及 GitHub 软件包退出测试。GitHub 也在改进其代码搜索，添加预定提醒，并推出一个预发布程序，允许用户在准备更大范围的推广之前尝试新功能。

GitHub 还将其赞助商计划扩展到项目级别，该计划直到现在还允许您为个人开源贡献者的工作支付小费。有了 GitHub 赞助商，任何人都可以资助一个项目，项目成员可以选择如何使用这笔钱。这些项目必须是开源的，并有一个公司或非盈利实体(和一个银行账户)。

“开发者是我们的驱动力，我们正在打造工具和体验，帮助他们一起创造世界上最重要的技术，并在开放的平台和生态系统上实现这一点，”产品 Shanku Niyogi 的 GitHub SVP 告诉我。他说，今天的声明是由公司改善开发者体验的使命所驱动的。Niyogi 强调，在过去的一年中，该公司推出了 150 多项新功能和增强功能。不过，在宇宙秀上，该公司决定重点展示新的移动应用和通知增强功能。

这款新的移动应用目前正在 iOS 上进行测试，很快将支持 Android，它提供了你想从这样的移动应用中获得的所有基本功能。该团队决定直接专注于对移动开发人员最有意义的移动用例，因此您可以分享讨论反馈，查看几行代码并合并更改，但这并不意味着是一个复制完整 GitHub 体验的工具，尽管至少在 iPad 上，您可以获得更多的屏幕空间。

“当你开始审视平板电脑体验时，这种体验会延伸出去，因为你现在有了更多的空间，”Niyogi 解释道。“你可以查看代码，浏览其中的一些内容，我们支持一些与 github.com 相同的键盘快捷键，以便能够查看更多的内容和代码。因此，我们的想法是，这种体验可以随着你拥有的移动设备而扩展，但它也是为你不使用电脑时可能会做的事情而设计的。”

当然，其他人以前也为 GitHub 开发过移动应用程序，事实证明， GitHawk 的开发者最近加入了 GitHub，帮助该公司开发这款新应用程序。

第二个主要的新特性是改进的通知体验。即使是中等规模团队的每个 GitHub 用户都知道，GitHub 当前的通知集会很快变得令人不知所措。GitHub 团队也敏锐地意识到了这一点，因此该公司决定建立一个大大改进的系统，包括过滤器，以及 GitHub 中所有通知的收件箱。

GitHub 战略和产品管理副总裁凯利·施蒂曼说:“今天开发人员的体验可能会导致 Gmail 收件箱或你使用的任何电子邮件客户端收到大量通知，最终可能很难知道什么是重要的，什么只是噪音。”“在过去的一年里，我们做了很多事情来改善通知，但我们已经迈出了一大步。我们重新构想了通知应该是什么样子。”

使用过滤器和规则，开发人员可以专注于对他们重要的通知，而不会用不必要的噪音淹没您的收件箱。开发者可以根据自己的喜好定制这些过滤器。这也是新的移动体验非常适合的地方。“很多时候，当你不在电脑前，当你不在桌面上时，通知会被发送给你，”Stirman 指出。该通知可能是有人请求您帮助解除封锁。因此，我们很自然地认为，我们需要将 GitHub 体验从桌面扩展到移动体验。”

谈论通知:GitHub 今天也在有限的预览版中宣布了一个新功能，在你的收件箱中增加了一些通知。您现在可以为待定的代码评审设置预定的提醒。

在今天发布的其他内容中，改进的代码搜索最为突出，因为这绝对是一个需要改进的领域。这种新的代码搜索目前处于有限的测试阶段，但应该会在未来几个月内向所有用户推出。该公司表示，这将引入一种全新的搜索体验，可以匹配特殊字符和大小写等。

还有新的代码审查任务，现在是公开测试版，以及一种在 GitHub 上导航代码的新方法。
