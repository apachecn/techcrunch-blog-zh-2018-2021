米兰蒂斯收购 Docker 企业

原文：https://web.archive.org/web/https://techcrunch.com/2019/11/13/mirantis-acquires-docker-enterprise/

Mirantis 今日宣布收购 Docker 的企业业务和团队。 Docker Enterprise 在很大程度上是 Docker 产品系列的核心，所以这次出售让 Docker 变成了以前高高在上的独角兽。Docker 本身在今年早些时候任命了一位新的首席执行官，它表示将继续专注于推进开发人员工作流程的工具。不过，米兰蒂斯将保留 Docker Enterprise 品牌，这肯定不会造成任何混乱。

通过这笔交易，Mirantis 正在收购 Docker 企业技术平台和所有相关的 IP: Docker 企业引擎、Docker 可信注册中心、Docker 统一控制平面和 Docker CLI。它还将继承所有 Docker 企业客户和合同，以及其战略技术联盟和合作伙伴计划。Docker 和 Mirantis 表示，他们将继续致力于 Docker 平台的开源部分。

两家公司没有透露收购的价格，但肯定远远低于 Docker 在最近几轮融资中的估值。事实上，众所周知，Docker 的命运多年来发生了很大的变化，从领导容器革命到在 Google 开源 Kubernetes 和行业的其他公司围绕它联合起来之后变得有点事后诸葛亮。不过，它的企业业务依然健康，在大型企业中拥有大量大客户。该公司表示，约有三分之一的财富 100 强企业和五分之一的全球 500 强企业使用 Docker Enterprise，这是大多数公司都希望能够强调的统计数据——这使得这笔交易从 Docker 的角度来看有点令人困惑，除非该公司认为这些客户中很少有人会继续押注于其技术。

更新 : 由于只有 Docker 的沟通团队知道的原因，我们事先没有被告知此事，但该公司今天也宣布它已经从 Benchmark 筹集了 3500 万美元的资金。 这并没有改变下面故事的整体主旨，但确实凸显了公司的新方向。

以下是 Docker 自己不得不说的话。“Docker 通过专注于推进开发人员在构建、共享和运行现代应用程序时的工作流程，开创了一个回归本源的新时代。作为这一调整的一部分，米兰蒂斯宣布收购了 Docker 企业平台业务，Docker 在被问及这一变化时发表声明说。展望未来，我们将扩大 Docker Desktop 和 Docker Hub 在现代应用开发人员工作流程中的角色。具体来说，我们正在投资扩展我们的云服务，以使开发人员能够快速发现构建应用程序时使用的技术，轻松地与队友和社区共享这些应用程序，并在任何 Kubernetes 端点上流畅地运行应用程序，无论是在本地还是在云中。"

米兰蒂斯本身也经历了起起落落。虽然它最初是一个资金充足的 OpenStack 发行版，但今天的 Mirantis 专注于提供以 Kubernetes 为中心的内部云平台和应用交付。正如该公司首席执行官阿德里安·艾奈尔在今天的声明之前告诉我的那样，今天可能是该公司最重要的一天。

那么米兰蒂斯会拿 Docker Enterprise 怎么办？“Docker Enterprise 是绝对一致的，是我们已经开始的方向的加速器，”Ionel 告诉我。“我们在很大程度上转向 Kubernetes 和 containers，旨在针对多云和混合及边缘用例，这些目标是为任何地方的任何基础架构(公共云、混合云、多云和边缘用例)的开发人员提供一致的体验，并使其非常简单、按需，并消除开发人员或基础架构所有者的任何运营顾虑或负担。”

米兰蒂斯此前拥有约 450 名员工。通过这次收购，它又获得了 300 名需要整合到其组织中的前 Docker 员工。然而，Ionel 说，在整合之前，Docker 的现场营销和销售团队将在一段时间内保持独立。“我们最重要的目标是不给客户造成干扰，”他指出。“因此，我们将保持出色的客户体验，同时将团队凝聚在一起。”

这也意味着，对于目前的 Docker 企业客户来说，在不久的将来什么都不会改变。Mirantis 表示，它将加速该产品的开发，并将 Kubernetes 和生命周期管理技术融入其中。随着时间的推移，它还将为 Docker Enterprise 提供托管服务解决方案。

虽然 Mirantis 和 Docker Enterprise 的客户群之间已经有一些重叠，但通过此次收购，Mirantis 将获得大约 700 个新的企业客户。

Ionel 认为，凭借这一点，Mirantis 可以与 VMware 和 IBM/Red Hat 等大型公司竞争。“我们是一家真正的云原生厂商，拥有有意义的规模，可以为他们提供替代方案，而不会局限于传统或现有的技术体系。”

虽然这显然是米兰蒂斯团队庆祝的一天，但很难不将这视为 Docker 一个时代的结束。该公司表示，今天将分享更多关于其未来计划的信息，但在这一宣布之前没有任何发言人。
