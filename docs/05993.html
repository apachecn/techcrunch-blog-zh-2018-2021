Gremlin 将混沌工程作为一项服务引入 Kubernetes

十年前，亚马逊和网飞开发了混沌工程,帮助那些网络规模的公司在最坏情况发生之前测试他们的复杂系统。 Gremlin 是由这两家公司的一名前雇员创建的，目的是在没有现场可靠性工程师团队的情况下更容易执行这种类型的测试。今天，该公司宣布，它现在支持在 Kubernetes 集群上进行混沌工程风格的测试。

该公司在本周在圣地亚哥举行的 Kubernetes 会议 KubeCon 开始时宣布了这一消息。

Gremlin 的联合创始人兼首席执行官 Kolton Andrus 表示，这个想法是为了能够测试和配置 Kubernetes 集群，以便它们不会失败，或者至少降低可能性。他说，要做到这一点，关键是要在真实环境中运行混沌测试(在极端胁迫下测试关键任务系统)，无论你是测试 Kubernetes 集群还是其他任何东西，但这样做也有点危险。他说，为了降低风险，最佳实践建议你将实验限制在能给你提供最多信息的最小测试范围内。

“我们可以进来说，我将只处理这些集群。我想在这里引起失败，以了解当这些作品失败时，在库伯内特发生了什么。例如，能够看到当您暂停调度程序时会发生什么。我们的目标是能够帮助人们理解爆炸半径的概念，并安全地指导他们进行实验，”安德鲁斯解释道。

此外，Gremlin 正在帮助客户强化他们的 Kubernetes 集群，以通过一套最佳实践来帮助防止故障。“我们显然拥有人们(进行这种类型的测试)所需的工具，但我们也通过许多许多客户互动和实验了解到，可以帮助他们真正调整和配置他们的集群，使其具有容错和弹性，”他说。

Gremlin 界面旨在促进这种有针对性的实验。您可以检查您想要应用测试的区域，并且您可以图形化地看到系统的哪些部分正在被测试。如果事情失去控制，有一个切断开关来停止测试。
