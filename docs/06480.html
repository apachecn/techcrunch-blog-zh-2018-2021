GitHub、Mozilla和Cloudflare呼吁印度公开其中介责任规则的变化

原文：https://web.archive.org/web/https://techcrunch.com/2020/01/07/github-mozilla-and-cloudflare-appeal-india-to-be-transparent-about-changes-in-its-intermediary-liability-rules/amp/

图片来源:桑吉特达斯/彭博/盖蒂图片社

微软的GitHub、Mozilla和Cloudflare敦促印度对即将出台的法律进行透明修改，该法律可能会影响大量公司和超过5亿人在线获取信息的方式。

2018年12月，印度政府提议修改其中介规则 (PDF)，该规则将要求任何促进两个或两个以上用户之间通信的服务，以及在印度拥有超过500万用户的服务，设立一个当地办事处，并在该国有一名高级管理人员负责任何法律问题。

该提案还建议，这些服务必须能够在24小时内删除有问题的内容，并在72小时内分享用户数据。

脸书和谷歌等科技巨头迄今享受着所谓的“安全港”法律。这些法律目前在美国适用于《通信行为规范法》,在印度适用于《2000年信息技术法》,法律规定技术平台不对用户在平台上分享的内容负责。

几个组织分享了反馈，表达了对印度中介规则变化的担忧。在周二致印度IT部长拉威·香卡·普拉萨德的一封公开信中，GitHub、Mozilla和Cloudflare要求印度政府对其为即将到来的法律起草的最终修正案更加透明。

印度政府此前曾表示，将在1月15日前向该国最高法院提交该提案的最终草案。但该提案的一个令人担忧的问题是，除了政府官员之外，没有人知道最终草案的内容。

“我们理解并尊重确保互联网是一个安全空间的需要，大型平台在其中承担适当的责任。然而，在公共领域可获得的这些修正案的最新版本表明，这些规则将促进自动化审查，使竞争环境向有利于大型参与者的方向倾斜，大幅增加监控，并促进印度互联网的分裂，这将损害用户，同时无法增强印度人的权能，”这些组织写道 (PDF)。

这些组织认为，安全港责任保护是印度互联网发展的基础，印度已经成为全球互联网公司最后一个巨大的增长市场。实施这些新规则中提出的义务将“给许多在线中介机构，尤其是新的组织和公司带来巨大的，在许多情况下是致命的负担”。他们说:“一个新的社区或初创公司将面临巨大的挑战，需要建立昂贵的过滤基础设施，并雇佣一大批律师。

批评者说，草案的最后一个公开版本对“中介”一词的定义也非常宽泛，这可能会导致许多意想不到的当事人受到影响。此外，他们认为，主动监控平台上的用户内容，并为“所有非法内容”带来“可追溯性”，将导致过度监管。这些组织今天表示，24小时的内容下架时间和72小时的用户数据共享时间这一短暂时限，对实施和言论自由构成了重大挑战。

“鉴于贵国政府向印度最高法院承诺在2020年1月15日之前公布这些规则，让公众有机会看到这些修正案的最终版本至关重要，这有助于确保这些修正案缓解各种利益相关方在公众咨询期间表达的担忧。我们呼吁增加透明度，我们将继续致力于与你们合作，以实现这些修正案的更广泛目标，同时让印度人从全球互联网中受益，”他们补充说。
