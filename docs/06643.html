Canonical 的 Anbox Cloud 将 Android 放在云端 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2020/01/21/canonicals-anbox-cloud-puts-android-in-the-cloud/

广受欢迎的 Ubuntu Linux 发行版背后的公司 Canonical 今天宣布推出 Anbox Cloud ，这是一个允许企业在云中运行 Android 的新平台。

在 Anbox Cloud 上，Android 成为运行容器化应用的访客操作系统。这开启了一系列用例，从定制企业应用到云游戏解决方案。

结果类似于谷歌在 Chrome OS 上对 Android 应用程序所做的事情，尽管实现非常不同，并且基于 LXD 容器管理器，以及许多规范项目，如用于提供容器和自动化部署的 Juju 和 MAAS 。该公司在公告中指出:“LXD 容器是轻量级的，与虚拟机中的 Android 仿真相比，容器密度至少是两倍——取决于流质量和/或工作负载复杂性。”。

值得注意的是，Anbox 本身是一个开源项目，它来自 Canonical 和更广泛的 Ubuntu 生态系统。Anbox 由 Canonical 工程师 Simon Fels 在 2017 年推出，在一个容器中运行完整的 Android 系统，这反过来允许你在任何基于 Linux 的平台上运行 Android 应用程序。

这一切有什么意义？Canonical 认为，它允许企业将移动工作负载卸载到云，然后将这些应用程序传输到员工的移动设备上。但 Canonical 也押注于 5G，以支持更多的用例，这不是因为可用的带宽，而是因为它支持的低延迟。

Canonical 产品总监 Stephan Fabel 在今天的声明中表示:“在新兴 5G 网络和边缘计算的推动下，数百万用户将受益于在他们选择的平台上访问超丰富的按需 Android 应用程序。“企业现在能够以更低的功耗和更经济的方式向任何远程设备提供高性能、高密度的计算。”

在企业之外，Canonical 关注的一个用例是游戏和游戏流。毕竟，云中的服务器通常比智能手机更强大，尽管这种差距正在缩小。

Canonical 还将应用测试作为另一个用例，因为该平台将允许开发者在数千台 Android 设备上并行测试应用。然而，鉴于 Android 生态系统的碎片化，大多数开发者更喜欢在真实设备上测试他们的应用程序，而不是模拟设备。

Anbox 云可以在公共云中运行，尽管 Canonical 专门与边缘计算专家 Packet 合作，在边缘或内部托管它。该项目的硅合作伙伴是安培和英特尔。
