一群前国家安全局和亚马逊的工程师正在建立一个“数据 GitHub”

原文：https://web.archive.org/web/https://techcrunch.com/2020/02/20/gretel-nsa-amazon-github-data/

大约六个月前，一群来自国家安全局、谷歌和亚马逊网络服务的工程师和开发人员有了一个想法。

数据对于帮助开发人员和工程师构建新功能和更好地创新非常有价值。但这些数据通常高度敏感，遥不可及，被繁文缛节和合规部门牢牢锁住，可能需要数周时间才能获得批准。因此，工程师们创办了 Gretel，这是一家早期创业公司，旨在帮助开发人员安全地实时共享和协作敏感数据。

联合创始人之一亚历克斯·沃森说，这并不像你想的那样是个小问题。他说，任何公司的开发人员都可能面临这个问题。通常，开发人员不需要完全访问用户数据库，他们只需要一部分或一个样本。在许多情况下，开发人员可以满足于看起来像真实用户数据的数据。

“首先要让数据安全共享，”沃森说。“人们已经能够用数据做所有这些非常酷的用例。”他说，像 GitHub 这样的公司，一个广泛使用的源代码共享平台，有助于使源代码易于访问和协作。“但是 GitHub 在数据方面没有对等物，”他说。

这就是沃森和他的联合创始人约翰·梅尔斯、阿里·高尔杉和拉兹洛·博克想出格莱特的方法。

“我们现在正在开发软件，使开发者能够自动检查数据集的匿名版本，”沃森说。这种所谓的“合成数据”本质上是人工数据，看起来和工作起来就像常规的敏感用户数据。Gretel 使用机器学习对数据进行分类，如姓名、地址和其他客户标识符，并尽可能多地对数据进行分类。一旦标记了数据，就可以对其应用访问策略。然后，该平台应用差分隐私——一种用于匿名化大量数据的技术——以便它不再与客户信息联系在一起。“这是机器学习产生的完全虚假的数据集，”沃森说。

这是一个已经引起关注的项目。这家初创公司已经从 Moonshots Capital、Greylock Partners、Village Global 和几家天使投资者那里筹集了 350 万美元的种子资金，以使该平台启动。

“在谷歌，我们不得不建立自己的工具，让我们的开发人员能够安全地访问数据，因为我们需要的工具并不存在，”前谷歌高管、现为 Greylock 合伙人的斯里达尔·拉马斯瓦米说。

Gretel 表示，它将根据消费向客户收费——这与亚马逊对其云计算服务的定价结构类似。

“现在，这是非常低调和建设，”沃森说。他说，这家初创公司计划在未来几周内加强与开发者社区的合作，着眼于在未来六个月内推出 Gretel。
