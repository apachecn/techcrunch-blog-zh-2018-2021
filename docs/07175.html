Firefox 默认为美国用户启用 HTTPS 域名系统

原文：https://web.archive.org/web/https://techcrunch.com/2020/02/25/firefox-dns-https-default-united-states/amp/

图片来源: Getty Images

浏览器制造商 Mozilla 证实，在未来几周内，默认情况下，Mozilla 将向美国所有 Firefox 用户推出其新的 DNS-over-HTTPS 安全功能。

在此之前，该公司花了一年时间测试新的安全功能，旨在使浏览网页更加安全和隐私。

每当您访问网站时，即使它启用了 HTTPS，将网址转换为电脑可以读取的 IP 地址的 DNS 查询通常是不加密的。HTTPS 域名系统(DoH)对请求进行加密，这样就不会被拦截或劫持，从而将用户发送到恶意网站。

这些未加密的 DNS 查询也可用于窥探用户访问的网站。

DoH 工作在应用级，被烘焙到 Firefox 中。该功能依赖于向第三方提供商发送 DNS 查询，如 Cloudflare 和 NextDNS 等，这两家公司都将他们的 DoH 产品嵌入 Firefox，并将处理 DoH 查询。

但此举并非没有争议。去年，一个互联网行业组织给 Mozilla 贴上了“互联网恶棍”的标签，因为它推进了安全功能。该贸易组织声称，这将使发现恐怖主义材料和虐待儿童的图像变得更加困难。但即使是安全社区中的一些人也存在分歧，有人警告说，这可能会使事件响应和恶意软件检测更加困难。

默认启用 DoH 的举措无疑会面临阻力，但浏览器制造商认为这不是浏览器制造商回避的技术。Firefox 成为第一个实现 DoH 的浏览器，其他浏览器，如 Chrome、Edge 和 Opera，也迅速跟进。

Firefox 表示，美国以外的用户也可以启用 DoH，就像美国境内的用户可以选择禁用它一样。Mozilla 还表示，它计划扩展到其他 DoH 提供商和地区。
