Twitter向开发者开放“隐藏回复”功能

原文：https://web.archive.org/web/https://techcrunch.com/2020/02/26/twitter-opens-its-hide-replies-feature-to-developers/

去年11月，Twitter 向全球所有用户推出了隐藏回复功能。这项功能主要是为了减少网络钓鱼者破坏对话的能力，让用户决定将他们推文的哪些回复放在额外点击的后面。Twitter表示，今天，Twitter正在向其开发人员社区提供隐藏回复，允许创建工具来帮助人们更快更有效地隐藏对他们推文的回复。

这些工具将会引起那些保持Twitter存在的企业和品牌的特别兴趣，但是他们的账户经常会收到太多的Twitter回复，以至于无法在个人的基础上进行适当的管理。现在，Hide Replies作为一个新的API端点可用，开发人员可以创建工具，根据对客户重要的因素自动隐藏破坏性的推文，例如，包括某些禁止关键字或因有毒而得分高的推文。

在今天的发布会之前，Twitter与少数开发人员合作，这些开发人员正在发布利用新增功能的工具。

Alphabet旗下的公司Jigsaw 致力于解决网络最糟糕的问题，它已经将Twitter的新端点与它的视角API 集成在一起，后者使用人工智能根据推文的毒性对推文进行评分。该集成将自动隐藏超过特定毒性阈值(. 94)的回复，从而节省了手动梳理回复所需的时间。

一个用于业务工作流的脚本平台 reshuffle ，已经使用端点来开发脚本，该脚本基于关键词或者甚至由用户来检测和隐藏回复。

流行应用 QuotedReplies 的创造者达拉·奥拉多苏也使用端点构建了一个名为隐藏不想要的回复的新应用。这款应用如今会自动隐藏关键字或Twitter句柄的回复。很快，它将支持隐藏来自可能的troll或bot帐户的回复，包括来自最近创建的用户帐户或关注者很少的帐户的推文。

隐藏回复是Twitter迄今为止最具争议的推出之一，因为它可能允许用户在有正当理由的情况下压制批评者或压制异议——例如，在反驳错误信息或宣传的情况下。其他人认为这并没有真正帮助解决网上滥用；虐待仍在发生，但在暗处。一个组织甚至最近利用隐藏回复开展了一个聪明的在线活动，关于家庭暴力是如何被忽视的，这进一步说明了这个问题。

然而，隐藏回复的采用正在增长，像 CIA这样的组织甚至在一些推特上利用它。

Twitter表示，新的Twitter API endpoint for Hide Replies今天以生产就绪的形式向所有开发人员提供，最初通过Twitter开发人员实验室提供。该项目于去年启动，旨在为开发者提供一种途径，在Twitter的最新API大范围发布之前进行测试，并提供反馈。参与 Twitter开发者实验室是免费的，但是感兴趣的开发者必须使用一个被认可的开发者账户注册。Twitter还邀请开发人员通过社区论坛与该公司合作。

基于第一批测试人员的早期反馈，Twitter表示已经对端点进行了一些更改，包括支持通过端点取消隐藏回复，支持高容量用例的更高速率限制，以及检索回复列表的方法，该列表指示它们是否被隐藏。
