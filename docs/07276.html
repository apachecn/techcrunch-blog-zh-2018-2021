卡珊德拉专家数据税收购最后一个泡菜

原文：https://web.archive.org/web/https://techcrunch.com/2020/03/03/datastax-acquires-the-last-pickle/

数据管理公司Datastax,Apache Cassandra项目的最大贡献者之一，今天宣布它已经收购了 Last Pickle(不，我也不知道这个名字是怎么回事)，这是一家总部位于新西兰的 Cassandra 咨询和服务公司，它是许多流行的分布式 NoSQL 数据库开源工具的幕后支持者。

正如 Datastax 首席战略官 Sam Ramji 告诉我的那样，你可能还记得他最近在 Apigee、Cloud Foundry Foundation、Google 和 Autodesk 的任期，最后一个 Pickle 是 Apache Cassandra 咨询和服务公司之一。自 2012 年成立以来，该团队一直在为 Spotify、T Mobile 和美国电话电报公司等公司开发基于 Cassandra 的开源解决方案。虽然最后一个 Pickle 位于新西兰，但该公司在世界各地都有工程师从事繁重的工作，并帮助这些公司成功实施 Cassandra 数据库技术。

值得一提的是，上一任 Pickle 首席执行官亚伦·莫顿第一次发现卡珊德拉是在他为 WETA Digital 制作《阿凡达》特效时，当时该团队利用卡珊德拉让 VFX 的艺术家们存储他们的数据。

“他们的工作分为两部分，”拉姆奇解释道。“一个是非常明显的咨询，这使他们成为运作卡桑德拉的世界专家。因此，随着我们将 Cassandra 自动化，随着我们提高项目对企业的可操作性，他们关于如何操作和扩展 Apache Cassandra 的智慧越来越好，是世界上最好的。”最后一个 Pickle 在构建具有成千上万个节点的系统方面的经验——以及其客户所面临的挑战——也是 Datastax 可以提供给其客户的。

当然，Datastax 还计划将最后一个 Pickle 的开源工具产品化，如自动修复工具 Reaper 和 Medusa 备份和恢复系统。

正如 Ramji 和 Datastax 的工程副总裁 Josh McKenzie 所强调的那样，Cassandra 近年来经历了很多商业开发，例如 AWS 现在提供托管 Cassandra 服务，但这个项目已经没有那么多宣传了。但是他们认为这是一件好事。现在已经十多年了，卡珊德拉已经身经百战。Ramji 认为，在过去的十年中，业界一直试图找出横向扩展计算的 de 因子标准应该是什么。到 2019 年，Kubernetes 显然是这个问题的答案。

“未来十年，横向扩展数据的事实标准是什么？我们认为这有一定的启示性，一定的结构性需求，我们认为卡珊德拉花了几十年时间让哈登成为那一波浪潮的数据。”

McKenzie 还指出，Cassandra 为用户提供了许多内置功能，如支持多数据中心和地理复制、滚动更新和实时扩展，以及跨编程语言的广泛支持，这使它比竞争对手的数据库具有许多优势。

“很容易忘记 Cassandra 仅仅基于它的架构就给了你多少免费的东西，”他说。“整个数据中心断电，升级数据库版本，每天都有硬件故障？没问题。群集始终 100%保持运行和可用。上一个泡菜的工具和专业知识确实有助于将所有这些分布式和弹性的力量带到大众手中。”

两家公司没有透露收购价格。
