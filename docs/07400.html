脸书的照片传输工具向欧洲、拉美和非洲的更多用户开放 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2020/03/10/facebooks-photo-transfer-tool-opens-to-more-users-in-europe-latam-and-africa/

脸书继续开放其于 12 月在爱尔兰推出的数据移植工具。该工具允许其网络用户通过加密传输将他们存储在服务器上的照片和视频直接传输到另一个照片存储服务，如谷歌照片。

脸书的一位发言人向 TechCrunch 证实，转移工具的访问权限将于今天推广到英国、欧盟其他国家以及拉丁美洲和非洲的其他国家。

据这位发言人说，上个月末，脸书还开放了 APAC 和黎巴嫩的多个市场。科技巨头 此前表示，该工具将于 2020 年上半年在全球上市。

通过您的脸书信息设置菜单访问“传输您的照片和视频副本”设置。

该工具基于脸书参与数据传输项目 (DTP)开发的代码，该项目始于 2018 年，由苹果、脸书、谷歌、微软和 Twitter 等公司支持，致力于使用开源代码建立一个通用框架，用于连接任何两个在线服务提供商，以支持“两个平台之间无缝、直接、由用户发起的数据可移植性”

近年来，科技巨头的主导地位导致竞争投诉增加，引起了政策制定者和监管者的关注。

例如，在欧盟，竞争监管机构正在关注科技巨头的数据做法，包括亚马逊、脸书和谷歌。而在美国，包括谷歌、脸书、亚马逊、苹果和微软在内的科技巨头，也面临着反垄断审查。随着越来越多的反垄断问题被提出，大型科技公司一直面临着回应的压力——因此集体推进便携性。

去年 9 月，脸书还发布了一份白皮书，阐述了其对数据可移植性的想法，试图将其描述为对隐私的挑战——这看起来像是试图游说监管护城河，以限制用户积累的个人数据山的可移植性。

与此同时，可移植性工具的发布让脸书在监管机构打电话来时有所指——尽管该工具只允许用户移植该服务持有的一小部分个人数据。这类工具也只可能被少数更懂技术的用户寻找。

脸书的传输工具目前还只支持直接传输到谷歌的云存储，这为用户将他们的面部生物特征副本从一个科技巨头传递到另一个科技巨头提供了便利。

然而 发言人暗示更广泛的实用程序可能会到来——他说DTP 项目更新了 SmugMug(其拥有 Flickr )的照片 API 的适配器；并为音乐流媒体服务 Deezer 增加了新的集成；去中心化社交网络乳齿象；和蒂姆·伯纳斯·李的权力下放项目。

他说，适配器是基于每种数据类型的，注意到开源贡献者正在为一系列数据类型(如照片、播放列表和联系人)开发适配器，并指向 GitHub 上可用的正在开发的项目列表。

虽然还不完全清楚为什么在脸书还没有提供直接连接到这些服务的选项。据推测，第三方仍然需要额外的开发工作来实现直接数据传输。当被问及此事时，发言人证实谷歌照片是目前唯一的选择，称这是“第一步”，他声称“在其他公司加入 DTP 时，为利益相关者提供了一个切实的评估工具，我们致力于转移到不同的服务和数据类型。”

发言人说，DTP 的目标是开发一个标准化版本，使其他人更容易加入，而不必“每次他们想构建可移植性工具时都要重新创建轮子”，他补充说:“我们在当前 DTP 合作伙伴的支持下构建了这个工具，希望将来会有更多的公司和合作伙伴加入我们。”

他还强调代码是开源的，并声称对于一个希望将其服务插入框架的公司来说，这“相当简单”,尤其是如果他们已经有了公共 API 的话。

“他们只需要针对那个公共 API 编写一个 DTP 适配器，”他建议道。

这位发言人补充说:“现在这个工具已经推出，我们期待着与更多的专家和公司合作——特别是那些希望为这种类型的服务提供入口的初创公司和新平台。”
