Spotify 向第三方应用开放其播客目录，但不适用于流媒体

原文：https://web.archive.org/web/https://techcrunch.com/2020/03/20/spotify-opens-up-its-podcast-catalog-to-third-party-apps-but-not-for-streaming/

Spotify 正在向第三方应用开放其播客目录。该公司本周推出了新版本的 Podcast API,该公司表示，这使得第三方应用程序可以连接到 Spotify，以管理用户的播客库，搜索其播客目录，获取详细信息以及播客节目和剧集。

此次发布意义重大，因为它利用了更广泛的开发者社区来帮助 Spotify 扩大其播客用户群。更重要的是，它首次在 Spotify 自己的应用程序之外提供了 Spotify 的独家节目。

像许多流媒体提供商一样，Spotify 已经开始不严格地使用“播客”这个术语。对 Spotify 来说，这个名字仅仅意味着一个音频节目，以插曲的形式呈现。但最初，这个词的意思是指使用开放格式 RSS 在网络上发布的音频片段。

苹果自己的播客应用程序，尽管其在的市场份额占多数，但从未通过将精选节目置于某种付费墙、会员或付费订阅之后来改变播客的本质。

但是 Spotify(以及其他新来者，如杰出人物)正是这么做的。在 Spotify 的案例中，它收购了旨在帮助人们创建和管理播客的技术初创公司，以及许多播客网络，包括制作一些行业顶级节目的 The Ringer、 Gimlet 和 Parcast 。Spotify 业务的播客流媒体方面的投资帮助其服务的播客收听量去年增长了 200%,并为该公司通过更有针对性的广告创造额外收入铺平了道路。

今天，Spotify 的 70 多万个播客中有许多是独家服务。这意味着如果你想听他们，你必须加入 Spotify。

不幸的是，对于播客听众来说，这也意味着你必须使用 Spotify 应用程序来播放这些节目，而不是你喜欢的第三方播客应用程序，如 covery、Pocket Casts、Breaker、Castro 或应用程序商店中的许多其他应用程序。

遗憾的是，Spotify 的新播客 API 并没有改变这一点。

相反，新的 API 专注于播客发现、搜索和管理节目，而不是通过第三方应用体验流媒体 Spotify 的播客、独家或原创节目。

Spotify 预计，现在可用的新功能将有助于应用程序将节目导入用户的 Spotify 库，或与日历应用程序集成，或帮助 Spotify 用户与朋友分享他们正在听的内容的社交播客体验。

“现在推出这个播客 API 对 Spotify 来说非常有意义，因为我们继续深入研究用新的方式为播客的听众创造价值，”该公司在公告中写道。“我们很高兴释放开发者社区的创造力，并允许 Spotify 扩展到我们尚未探索的领域，”它说。

在推出其新的播客 API 之前，Spotify 与一组精选的合作伙伴合作，这些合作伙伴正在构建他们的外部集成，以获得反馈。Spotify 对 API 的设计进行了一些修改，结果改善了开发人员的入职体验。然而，该公司表示，它计划在接下来的六个月里继续密切关注该项目，随着它的广泛应用，进一步完善它。

接下来，它希望在其开发者网站的上突出新应用。
