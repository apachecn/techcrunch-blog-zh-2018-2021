谷歌云推出托管 Memcached 服务

谷歌今天宣布了 Memcached 的 Memorystore 测试版，这是一项提供完全托管的内存数据存储的新服务，与开源的 Memcached 协议兼容。它将加入 2018 年首次推出的 Memorystore 家族中的 Redis。

正如谷歌 Memorystore 产品经理 Gopal Ashok 在今天的公告中指出的那样，Redis 仍然是会话存储、游戏排行榜、流分析、威胁检测和 API 速率限制等用例的热门选择，而 Memcached 通常用作数据库的缓存层。开发人员也经常使用 Memcached 作为会话存储，有了这项新服务，开发人员可以将其集群扩展到每个实例 5TB 的内存。

因为该服务与 Memcached 完全兼容，开发者应该能够将任何使用该协议的应用程序迁移到 Google Cloud 及其 Memorystore 平台上。作为一项完全托管的服务，谷歌将处理所有的日常任务，如监控和修补。然而，计算缓存的正确大小仍然是一门艺术，但谷歌云认为，其详细的指标将允许开发人员根据需要轻松地扩大和缩小其实例，以优化其特定用例的服务。该公司指出，这些指标暴露在云监控、谷歌云的集中监控仪表板和云控制台中。

目前，Memcached 的 Memorystore 可用于在计算引擎、谷歌 Kubernetes 引擎(GKE)、App Engine Flex、App Engine Standard 和云功能上运行的应用程序。

值得注意的是，亚马逊和像 T4 MemCachier 这样的专业初创公司也提供类似的服务，亚马逊为 Memcached 提供了 ElastiCache。Redis Labs 也在提供完全托管的 Memcached 服务，可以在 AWS、Azure 和 Google Cloud 上运行。
