欧盟隐私专家推动新冠肺炎接触者追踪的分散方法

原文：https://web.archive.org/web/https://techcrunch.com/2020/04/06/eu-privacy-experts-push-a-decentralized-approach-to-covid-19-contacts-tracing/

一群欧洲隐私专家提出了一个基于蓝牙的分散式新冠肺炎联系人追踪系统，他们认为该系统比将数据放入集中罐中的应用程序更能防止人们的数据被滥用和误用。

该协议被称为分散隐私保护邻近追踪(DP-PPT)，由来自欧洲至少七个研究机构的约 25 名学者设计，包括瑞士联邦理工学院、苏黎世联邦理工学院和比利时鲁汶大学。

他们发布了一份白皮书，详细介绍了他们的方法。

关键要素是，该设计需要在用户设备上本地处理联系人跟踪和风险，基于设备生成和共享短暂的蓝牙标识符(在本文中称为 EphIDs)。

后端服务器用于将数据推送到设备，即当被感染的人被诊断患有新冠肺炎时，卫生当局将批准从该人的设备上传感染期内的 EphIDs 的压缩表示，该压缩表示将被发送到其他设备，以便他们可以在本地计算是否存在风险并相应地通知用户。

在这种设计下，不需要对假名 id 进行集中，因为集中的数据会带来隐私风险。这反过来应该更容易说服欧盟公民信任该系统，并自愿下载使用该协议的联系人跟踪应用程序，因为它的设计可以防止被重新用于个人级别的国家监控。

该小组确实讨论了其他一些潜在的威胁，例如精通技术的用户可能会窃听本地交换的数据，并反编译/重新编译应用程序以修改元素，但最重要的争论是，与创建集中的数据库相比，这种风险较小，更易于管理，这种风险为“监控蔓延”铺平了道路，即，如果各州将公共健康危机作为建立和保留公民级跟踪基础设施的机会。

一旦公共卫生危机结束，DP-PPT 的设计就考虑到了其自身的有限拆卸目的。

“我们的协议证明了这样一个事实，即隐私保护的近距离追踪方法是可行的，国家或组织不需要接受支持风险和滥用的方法，”EPFL的 Carmela Troncoso 教授写道。“在法律要求严格的必要性和相称性，并且社会支持支持邻近追踪的情况下，这种分散的设计提供了一种防止滥用的方法来实现它。”

最近几周，欧洲各国政府一直依赖数据控制器来移交用户数据，用于各种冠状病毒跟踪目的。应用程序也在被私营部门争相推向市场——包括声称可以帮助研究人员对抗疾病的症状报告应用程序。虽然科技巨头们发现公关机会为声称的公共医疗事业重新包装对互联网用户的持续跟踪，但实际效用却很模糊。

冠状病毒的下一个重大技术推动看起来可能是接触追踪应用程序:即使用近距离追踪蓝牙技术来绘制受感染个人和其他人之间接触的应用程序。

这是因为，如果没有某种形式的接触追踪，通过限制人们的流动来降低感染率的来之不易的成果就有被逆转的风险，即一旦经济和社会活动再次开放。尽管联系人追踪应用程序是否能像政策制定者和技术专家希望的那样有效地帮助遏制新冠肺炎病毒仍是一个悬而未决的问题。

然而，现在非常清楚的是，如果没有一个经过深思熟虑设计的协议，通过设计联系人来烘烤隐私，追踪应用程序会给隐私带来真正的风险——如果存在的话，也会给来之不易的人权带来风险。

支持 DP-PPT 协议的团体发出的信息是，以打击新冠肺炎的名义剥夺权利既不好也没有必要。

“围绕集中的一个主要问题是，该系统可以扩展，各国可以重建一个谁与谁关系密切的社会图，然后可能在此基础上扩大特征分析和其他规定。伦敦大学学院的迈克尔·维尔博士解释说，“这些数据可以被执法和情报部门用于非公共卫生目的，”他是分散设计的另一位支持者。

“虽然一些国家可能能够通过在欧洲建立一个集中协议来制定有效的法律保障措施，但邻国被迫与它进行互操作，并使用集中而非分散的系统。反之亦然:一个分散的系统通过确保其他国家使用保护隐私的方法，对新冠肺炎蓝牙追踪在全球范围内的监控滥用施加了严格的技术限制。”

“这也是完全没有必要的，”他补充说，集中邻近数据。“数据保护的设计要求将数据减少到必要的程度。从技术上来说，收集和集中数据对于蓝牙联系追踪来说是不必要的。”

上周我们报道了欧盟的另一项努力——由不同的技术专家和科学家组成的联盟，由德国弗劳恩霍夫·海因里希·赫兹电信研究所 (HHI)领导——该研究所表示正在为新冠肺炎联系人追踪制定“隐私保护”标准，他们称之为:泛欧洲隐私保护邻近追踪( PEPP-PT )。

当时还不清楚这种方法是否局限于处理伪匿名 id 的集中模型。PEPP-PT 项目的联合发起人之一汉斯-克里斯蒂安·布斯(Hans-Christian Boos)今天在接受 TechCrunch 采访时证实，标准化工作将支持集中和分散的接触追踪处理方法。

这项努力遭到了欧盟隐私社区一些人的批评，因为他们似乎更喜欢集中而不是分散的方法——因此，批评者认为，这破坏了保护用户隐私的核心主张。但是，根据 Boos 的说法，事实上它将支持这两种方法——为了在世界范围内最大限度地吸收。

他还表示，无论数据是集中还是分散，它都将是可互操作的。(在集中的情况下，他说希望正在建立的监督 PEPP-PT 的非营利组织能够自己管理集中的服务器，等待适当的融资——例如，这一步骤旨在进一步降低缺乏人权框架的地区的数据集中风险。)

“我们将有两种选择——集中和分散，”Boos 告诉 TechCrunch。“我们将提供两种解决方案，这取决于谁想使用什么，我们将使它们具有可操作性。但是我要告诉你的是，这两种解决方案各有千秋。我知道在加密社区有很多人想要去中心化——我可以告诉你，在卫生社区有很多人讨厌去中心化，因为他们害怕太多人有关于感染者的信息。”

“在一个分散的系统中，你有一个简单的问题，你会向每个人广播受感染者的匿名身份——所以一些国家的卫生立法绝对禁止这样做。即使你有一种加密方法，你也是在把 id 广播到所有的地方——这是你的本地电话能够发现我是否有联系的唯一方法，”Boos 继续说道。

“这就是分散解决方案的缺点。除此之外，这是一件非常好的事情。集中式解决方案的缺点是，只有一个操作员，您可以选择信任或不信任他，他可以访问匿名 id，就像广播一样。所以问题是，你可以让一方访问匿名 id，还是让所有人都可以访问匿名 id，因为最终你会在网络上传播它们(因为)这是欺骗性的。”

“如果你的假设是有人可以黑掉中央服务器…那么你也必须假设有人可以黑掉路由器，这些东西会通过，”他补充说。“同样的问题。

“这就是我们同时提供这两种解决方案的原因。我们不信教。这两种解决方案都提供了良好的隐私。你的问题是你更信任谁，更不信任谁？你会更信任你向其广播内容的用户，还是更信任操作服务器的人？还是你更相信有人能黑掉路由器还是有人能黑掉服务器？两者都有可能，对吧。这两个选项都是完全有效的选项——这是加密人员之间的宗教讨论……但我们必须在加密想要的和医疗保健想要的之间进行平衡。因为我们无法做出这样的决定，所以我们最终会提供两种解决方案。

“我认为必须有选择，因为如果我们试图建立一个国际标准，我们应该努力不成为宗教战争的一部分。”

Boos 还表示，该项目旨在对各自的协议(集中式与分散式)进行研究，以比较和进行基于访问各自数据的风险评估。

“从数据保护的角度来看，数据是完全匿名的，因为它不依赖于位置、时间、电话号码、MAC 地址、SIM 卡号码等等。你唯一知道的是有联系——两个匿名 id 之间的相关联系。这是你唯一拥有的东西，”他说。“我们给计算机科学家和黑客的问题是，如果我们给你这个列表，或者给你这个图表，你能从中得出什么？在图中它们只是互相连接的数字，问题是你如何从中推导出任何东西？他们正在努力——让我们看看会有什么结果。”

“有很多人试图证明这场讨论是正确的。这不是正确与否的问题；这是关于做正确的事情——我们将主动提供任何好的选择。如果他们中的每一个都有缺点，我们会将这些缺点公之于众，我们会尽可能地得到更多的证实和研究。我们会把它公布出来，这样人们就可以选择他们想要的系统类型。

“如果结果是一个可行，一个完全不可行，那么我们会放弃其中一个——但从‘隐私保护’的角度来看，到目前为止两个都是可行的，所以我们会两个都提供。如果一种方法因为可被黑客攻击而变得不可行，或者你可能会以不可接受的风险获得元信息，那么我们会完全放弃它，不再提供这种选择。”

在互操作性方面，Boos 将其描述为“一个挑战”,他说这可以归结为系统如何计算它们各自的 id——但他强调这是正在进行的工作，是一个重要的部分。

“没有这些，整个事情就没有意义，”他告诉我们。“这是一个挑战，为什么选项还没有出来，但我们正在解决这个挑战，它肯定会工作…有多种想法如何使它工作。”

“如果每个国家都这样做，我们将不会再有开放的边界，”他补充说。“如果一个国家有多个应用程序不共享数据，那么我们就没有足够多的人参与进来，让感染追踪成为可能——如果没有一个地方可以让我们讨论如何正确对待隐私，那么可能每个人都会做其他事情，其中一半人会使用电话号码和位置信息。”

PEPP-劳工党联盟尚未公布其协议或任何守则。这意味着外部专家想要对与提议的标准相关的具体设计选择提供有根据的反馈，但他们无法获得必要的数据来进行审查。

Boos 说他们打算本周在 Mozilla 许可下开放源代码。他还表示，该项目愿意接受“任何好的建议”作为贡献。

“目前只有测试版成员可以访问它，因为那些已经向我们承诺，他们将更新到最新版本，”他说。“我们希望确保当我们发布第一个代码版本时，它应该已经通过了数据隐私验证和安全验证——所以我们尽可能确定没有开源系统上的人可能会跳过的重大更改。”

该协议缺乏透明度引起了隐私专家的担忧，并导致呼吁开发商在更多细节出台前暂停支持。甚至有人猜测，欧洲各国政府可能会进行干预，推动这一努力朝着集中化模式发展，并背离欧盟的核心数据保护原则。

目前，欧盟由来已久的数据保护法遵循了数据最小化等原则。透明度是另一个核心要求。就在上周，欧盟的主要隐私监管机构 EDPS 告诉我们，它正在监控新冠肺炎联系人追踪应用的发展。

“EDPS 支持开发用于抗击疫情冠状病毒的技术和数字应用，并正在与其他国家数据保护监督机构合作，密切监测这些发展。一位发言人告诉我们:“我们坚定地认为，GDPR 不会成为处理个人数据的障碍，卫生当局认为处理个人数据是抗击疫情所必需的。”

“所有目前致力于对抗疫情冠状病毒的有效措施的技术开发人员应该从一开始就确保数据保护，例如通过应用应用设计数据保护原则。EDPS 和数据保护界随时准备在这一集体努力中帮助技术开发者。数据保护机构的指导可在此处获得:关于第 25 条设计和默认数据保护的 EDPB 指南 4/2019；和 EDPS 关于隐私设计的初步意见

我们还了解到，欧盟委员会正在关注突然出现的冠状病毒应用程序和工具，其有效性和符合欧洲数据标准。

然而，与此同时，欧盟委员会一直在推动大数据议程，作为欧盟工业战略重启的一部分，该战略将数字化、数据和人工智能置于核心地位。就在今天euractiv报道了欧盟理事会泄露的文件，该文件称欧盟成员国和委员会应该“彻底分析从新冠肺炎疫情获得的经验”，以便为整个数字领域的未来政策提供信息。

因此，即使在欧盟，也存在对数据的高度偏好，这些数据有可能与冠状病毒危机交叉，推动事态朝着可能损害个人隐私权的方向发展。因此，某些支持隐私的人士强烈反对分散联系人追踪——以防止任何国家数据被窃取。

布斯认为，最佳实践“数据最小化”可以归结为你更信任谁的观点。“你可以为这两种(去中心化和中心化的方法)提出论据，它们都是数据最小化——仅仅因为在某一点上有数据最小化并不意味着在一个去中心化的系统中有整体的数据最小化，”他建议道。

“你信任谁是个问题？你更信任谁——这才是真正的问题。我认为数据的关键点不是匿名联系人的列表，关键数据是确认的感染。

“这在很大程度上是集权和分权之间的一种古老的、宗教式的讨论，”他补充道。“一般来说，它在这些工具之间摇摆不定；完全分布，完全集中…因为没有一个是完美的解决方案。但在这种情况下，我认为两者都提供了有效的安全选项，而且它们对您愿意或不愿意对医疗数据做什么都有不同的影响。然后你必须做出决定。

“我们必须做的是，我们必须确保选项是可用的。我们必须确保在重量级讨论中有可靠的研究，而不仅仅是猜测:什么是如何工作的，它们如何比较，以及有什么风险？”

就世卫组织参与 PEPP-过渡阶段的讨论而言，除了直接的项目参与者之外，Boos 说，政府和卫生部参与的实际原因是他们“必须将此纳入其卫生流程”。“许多国家现在都创建了自己的官方追踪应用，当然，这些应用应该与 PEPP-PT 连接，”他说。

“我们还与卫生系统中的人进行了交谈——无论各个国家的卫生系统是什么——因为最终这需要与卫生系统进行互动，需要与测试进行互动……它应该与传染病法律进行互动，这样人们就可以与当地的疾控中心联系，而不会向我们透露他们的隐私或联系信息，所以这也是我们正在进行的对话。”

拥有早期(测试版)访问权限的开发者已经在测试这个系统了。当被问及第一批使用 PEPP-PT 技术的应用程序何时能在大众中流通时，嘘声表示最快也要几周。

“他们中的大多数人只需将这些信息放入他们的追踪层，我们已经给了他们足够的信息，让他们知道如何将这些信息与他们的健康流程联系起来。我不认为这需要很长时间，”他说，并指出该项目还提供了一个追踪参考应用程序，以帮助那些没有开发人员资源的国家。

“对于用户参与，你必须做的不仅仅是跟踪——你必须包括，例如，来自疾病预防控制中心的信息……但我们将提供一个应用程序的框架实施，使这成为一个项目(更容易)，”他说。

“如果自上周以来所有给我们发邮件的人都把它放在他们的应用程序中(我们会得到广泛的接受)，”Boos 补充道。“假设有 50%的可能性，我认为我们会有一个非常好的开端。我想说的是，来自各国的涌入，我想说的是公司，尤其是那些想要回他们的劳动力的公司，尤其是在允许国际交换和互操作性的系统上有很大的压力。”

在接触追踪应用程序是否是一种有用的工具来帮助控制这种新型冠状病毒的传播这一更广泛的问题上——这种病毒已经显示出高度传染性，例如比流感更强——Boos 说:“我认为隔离感染很重要这一点没有太多争议，这种疾病的问题是，当你已经具有传染性时，却没有任何症状。这意味着你不能只是去测量人的体温就没事了。你真的需要回顾过去。我认为如果没有数字的帮助，这是不可能准确完成的。

“因此，如果你需要隔离感染链的理论是真的，许多疾病已经表明它是真的——但每种疾病都是不同的，所以没有 100%的保证，但所有的数据都证明了这一点——那么这肯定是我们需要做的事情……争论[归结为]如果我们有这么多的感染者，这有意义吗——我们不会很快结束，因为这个世界是如此相互联系，具有相同类型的锁定机制？

“这就是为什么只有当你突破了这些 R0 值(即一个感染者可以感染多少其他人)时，推出这样的应用才有意义——一旦你把它控制在 1 以下，并且把你的国家的病例数量降低到一个良好的水平。我认为，用一个传染病患者的话来说，这意味着回到遏制疾病的方法，而不是减轻疾病——我们现在正在做的。”

“接触链评估的方法允许你在测试上设置更好的优先级——但目前人们没有真正的优先级问题，他们有一个关于测试的资源问题，”他补充道。“测试和跟踪是相互独立的。两者你都需要；因为如果你在追踪接触者，而你不能接受检测，那有什么用呢？所以，是的，你肯定[也]需要测试基础设施。”
