Google Cloud 使得在 Bigtable 上运行较小的工作负载更加便宜

Cloud Bigtable 长期以来一直是 Google Cloud 完全托管的 NoSQL 数据库，用于大规模、Pb 级的分析和运营工作负载。每小时每节点 0.65 美元，这是一项不便宜的服务，尤其是因为 Google Cloud 强制要求每个集群至少有三个节点用于生产工作负载。然而，今天是改变了，您现在可以在单个节点上运行大表生产工作负载。

“我们希望 Bigtable 成为您所有键值和宽列用例的优秀之家，无论大小，”谷歌云 Bigtable 产品经理 Sandy Ghai 在今天的声明中表示。“无论您是刚刚起步的开发人员，还是正在为自我管理的 HBase 或 Cassandra 集群寻找落脚点的成熟企业，都是如此。”

有了这个，Google Cloud 现在还能够使用复制来提高这些小型集群的可用性，并且能够根据需要轻松地将单节点开发实例切换到单节点生产实例。此外，该服务的 SLA 现在还涵盖了所有 Bigtable 实例，无论其大小如何。

看到 Google Cloud 推动将较小的工作负载放到 Bigtable 上是很有趣的，特别是考虑到该组织当前对大型企业客户及其特定需求的关注。但是，现在只需要单个节点的公司很可能在未来需要大规模集群，Bigtable 的最小值一直是较小公司的准入门槛，一旦公司在给定的数据库服务上下了赌注，就不太可能随时改变。
