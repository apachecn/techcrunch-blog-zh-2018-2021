GitHub 现在对所有团队都是免费的

原文：https://web.archive.org/web/https://techcrunch.com/2020/04/14/github-is-now-free-for-all-teams/

GitHub 今天宣布它的所有核心功能现在都可以免费在对所有用户开放，包括那些目前免费的用户。这意味着所有人都可以免费使用无限的私人存储库和无限的合作者，包括将该服务用于商业项目的团队，以及每月最多 2000 分钟的免费访问 GitHub Actions，该公司的自动化和 CI/CD 平台。

想要更高级功能(如代码所有者)或企业功能(如 SAML 支持)的团队仍然需要升级到付费计划，但现在团队计划和用户计划的起价为每月 4 美元，而不是以前的 9 美元，企业计划的起价为每月 21 美元。

当我本周早些时候与 GitHub 首席执行官 Nat Friedman 交谈时，他强调这一举措早就在路线图上，并不是由当前的新冠肺炎·疫情推动的有限推广。“这是我们计划要做的事情，也是我们很久以来一直想做的事情——因为本质上我们做了收购——直到现在才做到这一点，当时它成为一个高度优先事项，”弗里德曼说。“但这肯定是我们想做的事情，我的意思是，这是一笔大买卖。"

该公司显然一直采取免费增值的定价模式，但自从被微软收购后，它开始扩大其免费账户的功能数量。举例来说，不久前，为最低级别的个人 GitHub 账户付费的主要原因之一是为了访问私人存储库。但是去年一月，该公司决定向所有免费用户开放无限制的私人项目,尽管每次仅限三名合作者。

“我们正在将 GitHub 从隐私付费模式转变为功能付费模式，也就是通常所说的免费增值——你可能听说过，”弗里德曼说。“我认为我们希望地球上的每个开发者和团队都能够使用 GitHub 进行开发，无论是私人开发还是公共开发。”

现在，GitHub 上有超过 4000 万的开发者，Friedman 说这个团队预计到 2025 年会达到 1 亿。

“这是 GitHub 业务架构的根本性变化。这是我们可以在内部思考的事情，”弗里德曼说。“其他人可以考虑的事情是:他们现在可以使用 GitHub，不管是出于什么原因。如果你正在创业，或者你是一家大公司的团队成员，你只想使用 GitHub，不需要信用卡，不需要预算，你可以组建你的团队。”

Friedman 认为团队没有做出这些改变是因为来自其他玩家的竞争压力，尽管值得一提的是，例如 GitLab 提供了一个内置 CI/CD 功能的竞争免费计划，而 Atlassian 的 BitBucket 现在提供了一个免费服务，鉴于 GitHub 的变化，看起来有点受限。

“这种变化使我们成为迄今为止价格最优惠、最容易被开发者获得的软件开发场所，”他说。我认为比我们几乎所有的竞争对手都要多。因此，这并不是真正的竞争驱动，而是通过进入更多的市场，看看人们在那里说些什么来获得更多的信息。"
