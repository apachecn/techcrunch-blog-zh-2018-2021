一个由欧盟科学家和技术专家组成的联盟正在为基于蓝牙的近距离追踪开发一种被称为“隐私保护”的标准，作为新冠肺炎感染风险的代理，该联盟希望苹果和谷歌对他们为相同的首要目的开发的 API 进行修改。

泛欧洲隐私保护邻近追踪(PEPP-PT)于 4 月 1 日解禁，呼吁联系人追踪应用程序的开发者支持处理智能手机用户数据的标准化方法，以协调跨境数字干预，并减少过度侵入的位置追踪工具因疫情而获得动力的风险。

PEPP-PT 今天表示，已经有七个政府签署了将其方法应用于国家应用程序，并声称还有 40 个政府正在讨论加入。(更新:尽管至少有两个已确认的政府支持者明确表示他们希望分散实施，包括瑞士和西班牙。)

“我们现在有很多政府在互动，”PEPP-PT 的汉斯-克里斯蒂安·布斯在一次记者网络研讨会上说。“一些政府公开宣称他们的本地应用程序将建立在 PEPP-PT 的原则以及该计划中提供的各种协议之上。

“据我们所知，有七个国家已经承诺这样做，我们目前正在与 40 个国家进行对话，这些国家正处于不同的入职状态。”

Boos 说，政府名单将与记者分享，尽管在撰写本文时，我们还没有看到它。但是我们已经向 PEPP-PT 的公关公司询问了信息，当我们得到信息时，我们会更新这份报告。

“泛欧洲的方法已经奏效，”他补充道。“各国政府以前所未有的速度做出了决定。但随着 40 多个国家加入进来，我们肯定已经超越了欧洲的关注范围，对我们来说，这表明隐私作为一种模式和一个讨论点……是一种声明，是我们可以出口的东西，因为我们在这方面是可信的。”

意大利政府创新技术和数字转型部的首席技术官保罗·德·罗萨(Paolo de Rosa)也参加了此次网络研讨会，并证实其国家应用程序将基于 PEPP-PT 构建。

“我们很快就会有一个应用程序，显然它将基于这种模式，”他说，但没有提供更多细节。

PEPP-PT 的核心“隐私保护”主张在于使用不需要收集位置数据的系统架构。相反，彼此靠近的设备将共享假名 id——如果系统计算出感染风险已经发生，这可以在以后用于向个人发送通知。被感染个体的联系方式将在诊断时被上传——允许通知被发送到接触过的其他设备。

Boos 是 PEPP-PT 的发言人和协调员，他在本月早些时候告诉 TechCrunch该项目将支持集中式和分散式方法。前者意味着 id 被上传到可信的服务器，例如由卫生机构控制的服务器；后者意味着 id 保存在本地设备上，感染风险也在本地设备上计算——后端服务器只是将信息转发到设备的一个环节。

苹果和谷歌正在合作支持这样一个分散的联系人跟踪系统——上周通过宣布一项计划，通过即将推出的 API 进行跨平台的新冠肺炎联系人跟踪，然后在系统范围内(选择加入)基于蓝牙的邻近跟踪，快速跟进 PEPP-PT 。

当目标是主流采用时，只有两个重要的智能手机平台进行了干预，这是一个重大发展——为分散式接触追踪提供了动力，以数字化方式应对西方世界的冠状病毒危机，当然是在平台层面。

在今天通过的一项决议中，欧洲议会也呼吁对新冠肺炎邻近追踪采取分散的方法。

欧洲议会议员正在推动委员会和成员国“在联系追踪应用程序的功能上完全透明，以便人们可以验证安全和隐私的基础协议，并检查代码本身，以了解应用程序是否像当局声称的那样发挥作用。”(委员会此前也表示倾向于权力下放。)

然而，PEPP-PT 的支持者，包括至少七个政府(以及更多政府的主张)，并没有放弃“保护隐私”的集中选项——他们阵营中的一些人将其称为“伪分散化”——今天，嘘声声称正在与苹果和谷歌讨论改变他们的方法。

目前，不使用分散基础设施的联系人跟踪应用程序将无法在 Android 或 iOS 上的后台进行蓝牙跟踪——因为这些平台限制了一般应用程序访问蓝牙的方式。这意味着这类应用程序的用户必须一直打开并激活应用程序，才能进行邻近跟踪，这对电池寿命和设备可用性产生了相关的(负面)影响。

由于苹果-谷歌联合模型中部署了中继服务器模型，因此对如何集中联系人跟踪数据也有(有意的)限制。

“我们非常欣赏谷歌和苹果正在加紧提供操作系统层，或者把应该是操作系统的东西放在那里，这就是蓝牙测量和加密处理，以及这些任务的后台运行，这些任务必须始终保持弹性运行——如果你看看他们的协议，如果你看看他们是由谁提供的，移动生态系统中的两个主导者， 我认为，特别是从政府的角度来看，或者从许多政府的角度来看，有许多开放的问题需要讨论。

“从 PEPP-PT 的角度来看，有几点需要讨论，因为我们希望在模型方面进行选择和实施选择——在他们的协议之上进行分散或集中实际上会产生最坏的结果——所以有很多点需要讨论。但与我们许多在科技公司工作的人习惯的行为相反，谷歌和苹果在这些讨论中非常开放，没有必要大动干戈，因为这些讨论正在进行，看起来可以与他们达成协议。”

目前还不清楚 PEPP-PT 希望从苹果和谷歌那里得到什么具体的改变——我们在网上研讨会期间要求更多的细节，但没有得到回应。但该组织及其政府支持者可能希望淡化科技巨头的立场，以便更容易地创建蓝牙联系人的集中图表，以支持国家冠状病毒应对措施。

就目前情况来看，苹果和谷歌的 API 旨在阻止服务器上的联系人匹配——尽管政府(和其他人)可能仍然有办法部分解决这些限制并集中一些数据。

我们联系了苹果和谷歌，询问他们声称与 PEPP-PT 进行的讨论。在编写本报告时，双方都没有答复。

除了意大利，德国和法国政府也表示支持 PEPP-PT 的国家应用程序——这表明，如果调整 API 的压力失败，强大的欧盟成员国可能会像苹果和联邦调查局一样，与科技巨头展开斗争。

这个故事的另一个关键点是，PEPP-PT 继续在自己的后院面临隐私和安全专家的严厉批评——包括在它删除了对的引用后，这是另一个由隐私和安全专家组成的欧洲联盟正在开发的新冠肺炎联系人追踪的分散协议，名为 DP-3T。

Coindesk 昨天在 PEPP-PT 网站上报道了无声编辑。

DP-3T 的支持者也一再质疑为什么 PEPP-PT 至今还没有公布代码或协议以供审查——甚至将这一努力称为“特洛伊木马”

苏黎世联邦理工学院的 Kenneth Paterson 博士既是 PEPP-PT 计划的一员，也是 DP-3T 的设计者，当我们询问时，他无法解释联盟希望从“Gapple”中获得的确切变化。

他在一封电子邮件中告诉我们，“他们还没有确切地说他们的系统将如何工作，所以我不能说他们需要什么(就苹果和谷歌系统的变化而言)”。

今天，嘘声认为 PEPP-PT 网站删除提及 DP-3T 是一个错误——他将其归咎于“糟糕的沟通”他还声称，该联盟仍然有兴趣将前者的分散协议纳入其标准化技术捆绑包中。因此，阵营之间已经有时模糊的界线继续被重新划分。(有趣的是，媒体发给老板的电子邮件现在被一家提供包括危机公关在内的宣传服务的通讯公司进行筛选。)

“我们真的很抱歉，”Boos 谈到 DP-3T 切除手术时说。“实际上，我们只是想把现有的各种选择放在同一水平上。仍然有所有这些选择，我们非常赞赏同事和其他人正在做的工作。

“你知道在加密社区中有一个关于这个的热烈讨论，我们实际上鼓励这个讨论，因为改进协议总是好的。我们不能忽视的是…我们这里不是在谈论加密，我们是在谈论疫情管理，只要底层传输层能够确保隐私就足够了，因为政府可以选择他们想要的任何东西。”

Boos 还表示，PEPP-PT 将最终在今天下午发布一些技术文件——选择在公开发布约三周后的一个周五晚上发布信息(一份七页的“高水平概述”已被放在他们的 GitHub 上这里【此链接已被删除——编辑。]——但离审查代码还很远)——同时让 同时呼吁记者们关注抗击冠状病毒的“大局”，而不是纠结于技术细节。

在今天的网络研讨会上，一些支持 PEPP-PT 的科学家谈到了他们如何测试蓝牙作为跟踪感染风险的代理的功效。

“我们一直在研究的算法着眼于个人在彼此附近度过的累计时间，”牛津大学大数据研究所病原体动力学高级小组组长、纳菲尔德医学院教授 Christophe Fraser 说，他提供了使用蓝牙邻近数据跟踪病毒传播的一般入门知识。

“目的是从电话邻近数据中预测传播的概率。因此，理想的系统会将隔离请求减少到那些最有可能被感染的人，而不会向那些没有被感染风险的人发出通知——即使记录了一些接近事件。”

“显然，这将是一个不完美的过程，”他继续说道。“但关键的一点是，在这种创新的方法中，我们应该能够审核这些信息和通知的正确程度——所以我们需要实际看到，在收到通知的人中，有多少人实际上受到了感染。在那些被确定为接触者的人中，有多少人不是。

"对每个系统来说，审计可以有许多不同的方式，但这一步是至关重要的."

per Fraser 认为，评估数字干预的有效性至关重要，他的陈述可以被解释为公共卫生当局可以更全面地访问联系人图表。但重要的是要注意到， DP-3T 的分散协议明确规定应用程序用户可以选择自愿与流行病学家和研究小组共享数据，使他们能够重建受感染和有风险用户之间的交互图(也就是访问邻近图)。

弗雷泽补充说:“如果你要进行一项将影响数百万人的干预，就这些[隔离]请求而言，这一点非常重要，即在你发出通知时，这些信息应该是最好的科学或最好的证据表示。”。“因此，随着我们向前推进，证据——我们对病毒传播的了解——将会改善。事实上，对应用程序的审计可以改善这一点，因此反馈这些信息似乎是必不可少的。”

根据 Boos 的说法，目前用于测试或参考的 PEPP-PT-aligned 应用程序没有一个与国家卫生当局的系统相接口——尽管他引用了意大利的一项测试，该测试被插入到一家公司的卫生系统中进行测试。

“我们为应用构建者提供了后端，我们为他们提供了样本代码，我们为他们提供了协议，我们为他们提供了测量科学，等等，等等。我们有一个工作应用程序，只是没有集成到一个国家的卫生系统中——在 Android 和 iOS 上，”他指出。

在它的网站上，PEPP-PT 列出了许多支持这项努力的企业“成员”——包括沃达丰这样的公司——以及几个研究机构，包括德国的弗劳恩霍夫海因里希赫兹电信研究所 (HHI)据报道是这项努力的领导者。

HHI 的执行董事 Thomas Wiegand 也参加了今天的电话会议。值得注意的是，他的名字最初出现在 DP-3T 白皮书的作者名单上。然而，4 月 10 日，根据 GitHub 文档历史，他被从自述和作者名单中删除。没有解释这一变化。

在今天的新闻发布会上，韦根做了一次干预，似乎不太可能让他受到更广泛的加密和数字权利社区的喜爱——他将围绕新冠肺炎联系人追踪使用哪种加密系统的辩论描述为一场“表演”，并表示担心他所谓的欧洲“公开的公共讨论”可能会“破坏我们作为欧洲人摆脱这一问题的能力”

“我只是想让每个人都意识到这个问题的难度，”他还说。“密码术只是系统中 12 个构件之一。所以我真的希望大家回去重新考虑一下我们现在的问题。我们必须战胜这种病毒……否则我们将面临另一次封锁，否则我们将面临许多大问题。我希望每个人都考虑这一点，因为如果我们齐心协力，真正战胜病毒，我们就有机会。”

在 Zoom 通话被聊天领域的种族主义垃圾邮件打断后，新闻发布会有了一个更不吉利的开始。就在那之前，Boos 开始了电话，他说他从“一些更懂技术的人那里听到，我们不应该使用 Zoom ，因为它不安全——对于一个想要安全和隐私的倡议来说，它是错误的工具。”

“不幸的是，我们发现我们的许多国际同事只在他们的公司电脑上安装了这个，所以随着时间的推移，要么 Zoom 必须改进，要么我们需要在那里安装更好的设备。我们当然不想泄露这次变焦的数据，”他补充道。
