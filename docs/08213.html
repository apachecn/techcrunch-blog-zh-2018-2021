AWS推出亚马逊AppFlow，这是其新的SaaS集成服务TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2020/04/22/aws-launches-amazon-appflow-its-new-saas-integration-service/

AWS today 发布了 亚马逊AppFlow ，这是一项新的集成服务，使开发者能够更容易地在AWS和SaaS应用程序之间传输数据，如谷歌分析、Marketo、Salesforce、ServiceNow、Slack、Snowflake和Zendesk。例如，像类似的服务，包括微软Azure的 Power Automate ，开发者可以基于特定事件、在预设时间或按需触发这些流。

然而，与一些竞争对手不同的是，AWS将这项服务更多地定位为数据传输服务，而不是自动化工作流程的一种方式，尽管数据流可以是双向的，但AWS的公告主要集中在将数据从SaaS应用程序移动到其他AWS服务以供进一步分析。为此，AppFlow还包括许多工具，用于在数据通过服务时对其进行转换。

“开发人员花费大量时间编写定制集成，这样他们就可以在SaaS应用程序和AWS服务之间传递数据，以便进行分析；AWS的主要倡导者Martin Beeby在今天的声明中说:“这些可能很昂贵，而且往往需要几个月才能完成。“如果数据需求发生变化，那么就必须对集成进行昂贵而复杂的修改。没有工程资源的公司可能会发现自己需要手动从应用程序中导入和导出数据，这非常耗时，有数据泄露的风险，并且有可能引入人为错误。”

每个流(AWS定义为调用源应用程序将数据传输到目的地)每次运行的成本为0.001美元，不过，在典型的AWS方式中，还有与数据处理相关的成本(每GB 0.02美元起)。

“我们的客户告诉我们，他们喜欢能够在AWS中存储、处理和分析他们的数据。他们还使用各种第三方SaaS应用程序，他们告诉我们很难管理AWS和这些应用程序之间的数据流，”AWS副总裁Kurt Kufeld说。“亚马逊AppFlow为客户提供了一种直观、简单的方式来合并来自AWS和SaaS应用程序的数据，而无需在公共互联网上移动数据。借助Amazon AppFlow，我们的客户可以汇集和管理分布在其所有应用程序中的数Pb甚至数EB的数据，而无需开发定制连接器或管理底层API和网络连接。”

在这一点上，支持的服务数量仍然相对较低，只有14个可能的来源和4个目的地(亚马逊红移和S3，以及Salesforce和雪花)。有时，根据你选择的来源，唯一可能的目的地是亚马逊的S3存储服务。

随着时间的推移，集成的数量肯定会增加，但就目前而言，AppFlow团队仍有相当多的工作要做，以扩展支持的服务列表。

AWS长期以来一直将这个市场留给竞争对手，尽管它有像 AWS Step Functions 这样的工具来构建跨AWS服务的无服务器工作流，还有 EventBridge 用于连接应用程序。有趣的是，EventBridge目前支持更广泛的第三方来源，但顾名思义，它的重点更多地是在AWS中触发事件，而不是在应用程序之间移动数据。
