Google Cloud 的完全托管 Anthos 现已面向 AWS TechCrunch 全面开放

原文：https://web.archive.org/web/https://techcrunch.com/2020/04/22/google-clouds-fully-managed-anthos-is-now-generally-available-for-aws/

一年前，回到面对面会议的时代，谷歌在其云下一次会议上正式宣布推出其 Anthos 云计算应用现代化平台。Anthos 的承诺一直是，它将允许企业编写一次应用程序，将它们打包到容器中，然后管理他们在 GCP、AWS、Azure 及其内部数据中心的多云部署。

到目前为止，对 AWS 和 Azure 的支持仅在预览版中可用，但今天，该公司正在全面提供对 AWS 和内部部署的支持。不过，微软 Azure 支持仍在预览版中。

“现在作为 AWS 客户，或者 GCP 客户，或者多云客户，[……]你现在可以在这些环境中以一致的方式运行 Anthos，所以你不必学习任何专有的 API，也不必受限制，” Eyal Manor，负责 Anthos 的总经理兼工程副总裁告诉我。“我们第一次实现了不同基础架构环境之间的可移植性，而不是像过去那样局限于一组 API。”

Manor 强调 Anthos 从第一天起就被设计成多云。至于为什么 AWS 支持先于 Azure 推出，Manor 说只是对它有更多的需求。“我们调查了客户，他们说，‘嘿，除了 GCP，我们还想要 AWS，’”他说。但对 Azure 的支持将在今年晚些时候到来，该公司已经有了一些预览版客户。此外，Anthos 未来还会来到裸机服务器。

展望未来，Manor 还指出，对机器学习工作负载的更好支持正在进行中。毕竟，许多企业都希望能够在数据所在的地方更新和运行他们的模型，不管是什么样的云。同样，Anthos 的承诺是开发者只需编写一次应用程序，就可以在任何地方运行。

“我认为很多最初的反应和兴奋来自于开发者观众，”谷歌云产品管理副总裁詹妮弗·林告诉我。“Eric Brewer 在我们撰写的白皮书中指出，许多 Anthos 架构在某种程度上分离了开发者和运营商利益相关方的顾虑。以前没有一个多云共享软件架构可以让我们做到这一点，并且仍然可以用一个通用的共享软件堆栈来驱动新兴的和现有的应用。”

她还指出，许多谷歌云的生态系统合作伙伴很早就认可了 Anthos 的整体架构，因为他们也希望能够编写一次并在任何地方运行——他们的客户也是如此。

Plaid 是这些新功能的发布合作伙伴之一。“我们的客户依赖我们始终可用，因此我们有非常高的可靠性要求，”Naohiko Takemura 说，格子的工程负责人。“我们采用了多云策略来确保我们关键的 KARTE 服务的冗余性。谷歌云的 Anthos 在 GCP 和我们的其他云提供商之间无缝运行，防止任何业务中断。多亏了 Anthos ，我们避免了供应商锁定，避免了管理特定于云的基础设施，我们的开发人员也不受云提供商的约束。”

随着这一版本的发布，Google Cloud 也为 Anthos 带来了对虚拟机的更深层次的支持，以及改进的策略和配置管理。

在接下来的几个月里，Anthos 服务网格还将增加对运行在传统虚拟机上的应用程序的支持。正如林告诉我的，“这在很大程度上是为了提高灵活性，减少复杂性，以便我们拥有可以在任何环境下工作的抽象，无论是传统环境、新环境、本地环境、AWS 环境还是环境。”
