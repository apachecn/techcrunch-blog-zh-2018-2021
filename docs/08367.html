Cloud Foundry 将目光投向企业之外，重新关注开发者体验 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2020/04/30/cloud-foundry-renews-its-focus-on-developer-experience-as-it-looks-beyond-the-enterprise/

云铸造基金会 (CFF)刚刚经历了一次重大的领导层变动，执行董事 Abby Kearns 在五年后离职(并成为 Puppet 的首席技术官)，CFF 的首席技术官 Chip Childers 接任该组织的最高领导职位。不过，在很大程度上，这些变化只是加速了该组织在过去几年中已经采取的一些战略举措。

如果你不熟悉开源的 Cloud Foundry 项目，这是一个被大多数财富 500 强企业使用的平台即服务。在经历了大量的技术变革之后，其中主要包括构建对容器的支持，并将 Kubernetes 作为容器编排的一个选项添加到早在 Google 开源工具兴起之前构建的容器工具 Cloud Foundry 旁边，该项目的技术基础现在已经稳定了。正如 Childers 之前提到的，这使得项目可以重新关注开发人员的体验。

毕竟，这一直是 Cloud Foundry 的卖点。开发人员遵守的一些规则，作为回报，他们可以通过一个命令(“cf push”)轻松地将他们的应用程序推送到 Cloud Foundry，并知道它会运行，而雇用他们的企业则获得了更快开发周期的好处。

然而，另一方面，实际管理 Cloud Foundry 安装并不容易，需要内部基础架构团队的大力支持，或者 Pivotal、IBM、SAP、Suse 等外部公司的帮助来运行和管理平台。这很大程度上排除了小公司，尤其是初创公司，使用该平台。正如 Childers 所指出的，有些人仍然在使用它，但这从来都不是这个项目的重点。

现在，有了 Kubernetes 的支持，他相信非企业用户也可以更容易地开始使用这个平台。像【】KubeCF 和 CF for K8s 这样的项目现在为 Kubernetes 提供了完整的云铸造发行版，这使得在现代基础设施之上使用该平台变得相对容易。

为了强调其中的一些变化，CFF 今天发布了其新的教程中心,它不仅会解释什么是 Cloud Foundry，还会提供入门教程。其中一些将由基金会自己主持和编写，而社区成员将贡献其他内容。

“我们的社区创建了一个学习中心，由 Cloud Foundry 基金会管理，提供开源教程，供人们学习 Cloud Foundry 和相关的云原生技术，”Childers 说。“该中心包括一个交互式动手实验室，供首次使用 Cloud Foundry 的用户体验该平台如何轻松地将应用部署到 Kubernetes，并开放给社区贡献。”
