苹果和谷歌发布新冠肺炎曝光通知应用的样本代码、用户界面和详细政策 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2020/05/04/apple-and-google-release-sample-code-and-detailed-policies-for-covid-19-exposure-notification-apps/

苹果和谷歌正在为开发第一版暴露通知 API 的开发人员提供额外的资源，这是两家公司已经创建并正在开发的开发工具，旨在为公共卫生机构提供一种跨平台的方式，以通知个人与确诊新冠肺炎病例的潜在接触。

曝光通知 API 的第一个版本上周与 iOS 和 Xcode 的测试版更新一起向开发者发布，苹果和谷歌从“联系人跟踪 API”更名而来，以更准确地反映其实际用途和目的。今天，苹果和谷歌正在为开发者提供新的示例资源，包括示例 UI 资产，以及 iOS 和 Android 的示例代码。这些被设计为起点，代表公共卫生机构工作的开发人员可以使用它们来启动他们的应用程序开发过程。

这两家公司还发布了新的政策，任何使用该 API 的开发人员都必须遵守这些政策，才能让他们的应用程序获得使用许可。其中包括以下要求:

  • 它们必须由官方政府公共卫生机构制作或供其使用，并且只能用于应对新冠肺炎病毒。
  • 他们需要征得用户的同意才能实际使用该 API。
  • 他们要求用户同意在向运行该应用程序的公共卫生机构传播任何此类信息之前分享阳性检测结果。
  • 他们应仅收集风险通知所需的最少量信息，并且仅在新冠肺炎响应时使用这些信息。换句话说，这些应用程序被明确禁止将你的信息用于广告或其他目的。
  • 他们无法访问甚至无法获得访问设备定位服务的许可，该服务提供特定的地理位置数据。谷歌和苹果指出，公共卫生部门已经提供的利用位置数据的应用程序将继续提供，但没有利用这些信息的应用程序也将获得新的暴露通知 API。
  • 每个国家只能有一个应用程序，旨在避免分散，从而提高效率，尽管苹果和谷歌表示，如果一个国家依赖于基于区域或州的方法，他们愿意与当局合作，以尽可能最好的方式支持他们。这基本上意味着，如果一个国家通知苹果，它将在每个州使用不同的应用程序，它将解锁多个应用程序出现在该国商店的能力，并且它可以在每个州内的暴露通知机制是否相互作用方面与他们灵活合作。

这些公司表示，在本月晚些时候向消费者发布 API 的公共版本之前，他们还将继续为他们的软件和软件开发工具包发布更新。苹果和谷歌都将面向消费者的 API 发布定在“5 月中旬”，最终计划在今年晚些时候将暴露通知作为系统级功能发布。

苹果和谷歌开始了这一前所未有的联合努力，以回应一些公共卫生机构的外联活动，这些机构正在开发自己的接触者追踪应用程序，并希望获得 iOS 和 Android 的特定方面，以使这些工作得以进行。两家公司决定合作开发一种基于蓝牙标识符而非地理定位数据的标准，以此来保护用户身份，并确保该系统可以在各种环境下工作，包括地理定位卫星服务不可用的室内。

卫生当局还可以要求用户输入与他们进行的测试相关的唯一代码，这可以帮助他们确保阳性结果实际上来自经过验证和授权的测试，而不是可能只是自我报告，或者是根据未经卫生当局批准的新冠肺炎诊断测试报告的。

值得注意的是，谷歌和苹果提供的示例参考应用程序实际上永远不会提供给用户；它们严格来说是为开发者准备的，但这些公司正在向开发者提供它们的全部内容，包括完整的源代码，以便帮助他们自己开发应用程序，及时响应新冠肺炎。
