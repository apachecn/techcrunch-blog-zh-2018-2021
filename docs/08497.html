企业发现 MLOps 对可靠性和性能至关重要

企业初创公司 UIPath 和 Scale 近年来吸引了寻求自动化工作流程的公司的巨大关注，从 RPA(机器人流程自动化)到数据标签。

在这种特定于工作流的工具出现之后，被忽视的是企业用来构建其机器学习(ML)工作流核心的产品的基本类别，以及关注点向 ML 工作流的自动化部署和治理方面的转移。

这就是 MLOps 的用武之地，它的流行是由核心 ML 工作流平台的兴起推动的，比如波士顿的 DataRobot 。该公司已经筹集了超过 4.3 亿美元的资金，并在去年秋天达到了 10 亿美元的估值，以满足企业客户的这一需求。DataRobot 的愿景很简单:使企业内的一系列用户(从业务和 IT 用户到数据科学家)能够收集数据并快速构建、测试和部署 ML 模型。

该公司成立于 2012 年，已经悄悄地积累了一个客户群，拥有超过三分之一的财富 50 强，自 2015 年以来每年增长三位数。DataRobot 排名前四的行业包括金融、零售、医疗保健和保险；其客户通过 DataRobot 的平台部署了超过 17 亿个模型。该公司并不孤单，竞争对手如 H20.ai 去年 8 月筹集了由高盛牵头的7250 万美元的 D 轮融资，提供了类似的平台。

为什么这么兴奋？随着人工智能进入企业，第一步是从数据到工作的 ML 模型，这从数据科学家手动完成开始，但今天越来越自动化，并被称为“自动 ML”。像 DataRobot 这样的 auto-ML 平台可以让企业用户根据他们的数据快速自动选择功能，并自动生成许多模型，以查看哪些模型效果最好。

随着自动 ML 变得越来越流行，改进 ML 工作流的部署阶段对于可靠性和性能变得至关重要——因此进入了 MLOps。这与 DevOps 改进应用程序源代码部署的方式非常相似。DataRobot 和 H20.ai 等公司，以及其他创业公司和主要的云提供商，正在加大力度为客户提供 MLOps 解决方案。

我们与 DataRobot 的团队坐在一起，了解他们的平台如何帮助企业构建 auto-ML 工作流，MLOps 是什么，以及是什么推动客户现在采用 MLOps 实践。
