杜比推出 Dolby.io，这是其新的自助式开发者平台 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2020/05/12/dolby-launches-dolby-io-its-new-self-service-developer-platform/

以其各种音频和视频处理工具而闻名的杜比实验室(Dolby Laboratories)今天宣布推出 Dolby.io ，这是其第一个自助服务平台，面向希望将杜比的一些功能引入自己的应用的软件开发人员。目前，Dolby.io 的阵容包括两组 API，一组用于快速分析和改善现有音频文件的声音，另一组用于为交互式体验带来新的音频功能。随着时间的推移，该公司计划为这项服务增加额外的功能。

传统上，希望与杜比合作的硬件和软件开发人员必须经历一个相当复杂的手动过程。然而，这个新平台看起来和感觉上像任何其他以 API 为中心的开发者服务，提供免费试用和按使用计费。

“在这种情况下，杜比没有办法让开发者或客户直接与他们互动，”杜比负责开发者关系的副总裁亚伦·廖(Aaron Liao)说，他在担任 AWS 全球开发者营销主管一段时间后，最近加入了该公司。“如果你现在去杜比网站，你不会看到注册页面。传统上，客户通过一系列许可讨论之类的事情与杜比互动。因此，我们有效地为开发者引入了一种与杜比互动的新方式。一种是自助服务——如果你不想和一个人说话，你可以不说话。”

至于实际的 API，媒体文件 API 将允许开发人员分析或增强他们现有的音频文件。通过分析 API，您将获得关于音轨响度、咝咝声、均衡的数据——所有这些都在一个标准的 JSON 文件中。不过，更有趣的服务肯定是 enhance API，它汇集了杜比围绕改善音频的大量现有技术，并使其可供任何开发者使用。

“你可以给我们一个媒体文件，我们会尽最大努力为你清理，让它听起来更好，”廖说。这意味着 API 可以降低播客录制的背景噪音，或者均衡不同扬声器的音量。

至于交互 API，廖指出，它使用 WebRTC，所以你可以在你的应用程序中进行多播和视频交互。

在过去的几个月里，杜比开始与一些合作伙伴一起测试这项服务，其中包括远程健康理疗公司 Physitrack。Physitrack 的联合创始人兼首席执行官亨里克·莫林说:“这不仅仅是技术的问题，而是你如何与它互动的问题。”。“借助 Dolby.io，我们的 telehealth 平台为医疗保健客户提供了与患者沟通的能力，同时展示和覆盖重要的物理康复治疗工具和材料，同时每周提供数百万分钟清晰的通话时间。”

该公司还与前扰乱战场参赛者 发言人合作，使用这些新的 API 为其客户提供更一致的音频体验。
