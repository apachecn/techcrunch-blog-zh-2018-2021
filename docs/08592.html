微软与Redis实验室合作，为Redis TechCrunch改进其Azure缓存

原文：https://web.archive.org/web/https://techcrunch.com/2020/05/12/microsoft-partners-with-redis-labs-to-improve-its-azure-cache-for-redis/

微软与Redis实验室合作，改进其用于Redis的Azure缓存

几年来，微软已经为Redis 提供了 Azure Cache，这是一个完全托管的缓存解决方案，建立在开源的Redis项目之上。今天，它正在通过在其平台上添加Redis Enterprise(Redis Lab的商业产品)来扩展这项服务。它是与Redis实验室合作进行的，虽然微软将为这项服务提供一些基本支持，但Redis实验室将自己处理大部分软件支持。

微软开发工具部门的企业副总裁Julia Liuson 告诉我，该公司希望被视为Redis Labs等开源公司的合作伙伴，Redis Labs是首批改变许可证的公司之一，以防止云供应商在不对社区做出贡献的情况下商业化和重新打包他们的自由代码。去年，Redis Labs与Google Cloud合作，将自己的完全托管服务引入其平台，所以我们现在看到微软采取类似的行动也就不足为奇了。

Liuson告诉我，借助Redis的Azure缓存新层，用户将获得单一账单和原生Azure管理，以及在SSD闪存上原生部署的选项。原生Azure集成也应该使Azure上的开发人员更容易将Redis Enterprise集成到他们的应用程序中。

同样值得注意的是，微软将支持Redis实验室自己的Redis模块，包括Redis支持的搜索引擎RediSearch，以及RedisBloom和RedisTimeSeries，它们为Redis中的新数据类型提供支持。

“多年来，开发人员利用Redis的速度和吞吐量在其应用程序中产生无与伦比的响应能力和规模，”Liuson说。“我们已经看到Azure Cache for Redis(我们基于开源Redis的托管解决方案)被大量采用，因为Azure客户已经将Redis的性能用作分布式缓存、会话存储和消息代理。Redis Labs Redis Enterprise技术的引入扩展了开发人员可以利用Redis的用例范围，同时提供了增强的操作弹性和安全性。”
