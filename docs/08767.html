RapidAPI 再融资 2500 万美元扩大其 API 市场 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2020/05/21/rapidapi-api-marketplace-funding/

在筹集了由微软牵头的 2500 万美元用于建设 API 市场后不到一年， RapidAPI 迅速跟进了另一笔资金的注入，数百万开发人员在其市场上跟踪、集成和使用了 20，000 个 API。今天，这家初创公司宣布从现有投资者安德森·霍洛维茨、DNS Capital、Green Bay Ventures、M12(微软的风险基金)和 Grove 那里又筹集了 2500 万美元。

这是 RapidAPI B 系列的第二次结算，我们去年首次报道了这一轮，使这一轮的总收入达到 5000 万美元，总收入为 6250 万美元。PitchBook 指出这家初创公司之前的估值为 8000 万美元，这将使其现在的估值超过 1.05 亿美元，但考虑到该公司的规模已经相当大，这个数字可能会更高。联合创始人兼首席执行官 Iddo Gino 本周在接受采访时不愿透露实际金额。

API 是当今数字世界的基石:开发人员使用它们快速将特性、数据、服务和功能集成到自己的应用程序中，无需从头开始构建和扩展所有这些元素。但是，尽管使用 API 的最大卖点是它们允许开发人员只使用几行代码进行集成，但这并不能说明全部情况。问题是许多 API 接口都不是统一的，因此寻找和使用各种接口会变得非常耗时，并且总体上比 API 的基本概念要困难得多。

联合创始人 Iddo Gino 曾经对我说“你不能从头开始构建一切，使用 API 会让工作更有效率。但是每个 API 都有不同的格式和认证策略。你必须会说很多不同的语言才能使用它们。”

RapidAPI 的方法是创建一个框架，它不仅可以帮助您找到您正在寻找的 API，还可以让您通过单个 API 键和 SDK 更容易地集成它们。它涵盖了免费和付费 API，以及公共和“私有”API。如果您的公司是 RapidAPI for Teams 产品的订阅者，它还可以帮助您跟踪自己组织的 API 工作。

这一方案取得了成功。在使用该平台的 100 多万开发人员中，现在有 18，000 个团队在使用 teams 产品。

RapidAPI 最初于 2015 年在以色列成立，现在总部也在旧金山，它表示，自 1 月份以来，它增加了 30 万名新开发人员，比 2019 年同期的 5 个月增长了 6 倍。市场本身现在有 20，000 个 API，去年翻了一番，每月增加 1，000 个。其市场的贡献者包括微软、Twilio、SendGrid、Nexmo、Skyscanner 和(我们以前的 stablemate) Crunchbase。

RapidAPI 不对使用已经可以免费使用的 API 的人收费。相反，它通过订阅其 API 管理服务以及提供付费 API 来赚钱。它表示，付费订阅也增长了 3 万，使用企业级的用户(在这里，你可以为自己的员工和客户开发自己的白标、内部版本的市场)正在增加，金融服务、保险公司、运营商和医疗保健公司也在 RapidAPI 的 rails 上建立市场。

吉诺指出，虽然许多企业，甚至包括科技初创公司，都不得不做出重大调整，以适应我们的新环境，以及它对社交距离的关注，以帮助管理新冠肺炎的传播，但 RapidAPI 却没有这样做。该公司已经有了远程团队——这是在一个国家成立的结果，现在基本上有了两个引力极 RapidAPI 的 75 人团队及其客户在他们的文化中跨不同的环境工作，包括虚拟化环境。

然而，目前的形势表明，RapidAPI 是那种能够从其他组织应对数字化转型的方式中受益的公司，它帮助开发人员提供他们可以随时随地使用的库。

在当前的环境下，另一件有趣的事情是它对 API 调用最多的影响。除了包括通信、支付和其他金融服务在内的最受欢迎的应用编程接口的常规名册之外，吉诺告诉我，与新冠肺炎数据相关的应用编程接口已经成为流量最大的一些应用编程接口，这与许多人正在努力理解正在发生的事情以及它们可能如何帮助我们其他人的情况相符。

这些包括对数据集和地理位置以及其他统计数据的 API 调用，其中一些是免费的，一些是付费的。RapidAPI 表示，在 3 月 1 日至 5 月中旬期间，前五大 COVID APIs 有超过 2.24 亿次调用，单日峰值接近 450 万次。
