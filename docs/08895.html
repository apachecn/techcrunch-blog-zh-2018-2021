新闻中的新冠肺炎联系人追踪应用

  • 拉脱维亚 : 路透社本周报道拉脱维亚旨在成为首批推出智能手机应用 Stop Covid 的国家之一，该应用使用苹果和 Alphabet 的谷歌创造的新工具包来帮助追踪冠状病毒感染。
  • 澳大利亚:澳大利亚的 Covidsafe 应用在经济复苏中的作用似乎微不足道，《卫报》报道。在该网站发布后的一个月里，据报道只有一个人的身份被确认。一项调查甚至发现，澳大利亚人更支持使用电信元数据来跟踪密切接触者(79%)，而不是下载应用程序(69.8%)。在第二次调查中，他们对该应用的支持下降到 64%。这款应用受到了公众对它和技术问题的争论的诽谤。
  • 法国:法国数据保护监管机构 CNIL审查了其联系追踪应用 StopCovid，发现围绕 StopCovid 的技术实现和法律框架没有重大问题，但有一些警告。法国没有使用谷歌和苹果的联系人追踪 API，而是使用了一个有争议的集中式联系人追踪协议，名为 ROBERT。这依赖于中央服务器来分配永久 ID 并生成附加到该永久 ID 的临时 ID。CNIL 表示，该应用程序最终将是开源的，它将创造一个 bug 赏金。周三，该应用通过了第一次投票支持其发布。
  • 卡塔尔:大赦国际发现卡塔尔强制联系人追踪应用存在严重安全漏洞。大赦国际安全实验室 的一项调查发现了卡塔尔 EHTERAZ 联系人追踪应用程序配置中的一个致命弱点。该漏洞现已修复，将允许网络攻击者访问高度敏感的个人信息，包括超过 100 万用户的姓名、身份证、健康状况和位置数据。
  • 印度 : 据印度电子和信息技术部秘书 Ajay Prakash Sawhney 周二表示，印度的联系人追踪应用 Aarogya Setu 将开源。该代码正在 GitHub 上发布。该应用的 1.14 亿多用户中，近 98%使用安卓系统。政府还将向发现漏洞的安全专家提供 1325 美元的现金奖励。
  • 瑞士 : 数千人正在测试瑞士联系人追踪应用的试用版 SwissCovid。和 Lativia 一样，这款应用也是首批使用苹果和谷歌联系人追踪 API 的应用之一。在计划于 6 月中旬公开发布之前，EPFL 大学、苏黎世联邦理工学院、军队以及部分医院和政府机构的员工将率先测试这款瑞士应用。
  • 中国:据路透社报道，中国流行的微信和支付宝智能手机应用中嵌入的健康追踪二维码正在引发隐私担忧。要想自由行走，人们必须获得绿色评级。他们现在还必须出示他们的健康二维码才能进入餐馆、公园和其他场所。这些努力几乎没有遇到什么阻力。但东部城市杭州此后提议，根据用户的医疗记录和生活习惯，向他们发放彩色编码的健康徽章，包括他们的运动量、饮食习惯、是否吸烟以及前一天晚上睡了多长时间。这一建议在类似 Twitter 的中国微博平台上引发了一场批评风暴。

App Annie 关于冠状病毒对 App 经济影响的新数据

App Annie 本周举办了一场网络研讨会，详细介绍了冠状病毒爆发对移动经济的一些主要影响。

以下是几个亮点:

  • 与 2019 年 Q2 相比，2020 年 Q1 的全球平均周应用时间同比增长 20%
  • 从 2019 年第四季度到 2020 年 Q1，平均每周花费在非游戏应用上的时间增加了 10%
  • 从 2019 年第四季度到 2020 年 Q1，游戏的平均周下载量增长了 30%以上
  • 从 2019 年第四季度到 2020 年 Q1，全球游戏平均周下载量增长 50%以上
  • 超休闲类、益智类和皇家战役类游戏需求旺盛；琐事和文字游戏也激增，多人游戏也是如此
  • 从 2019 年第四季度到 2020 年 Q1，美国花在视频流应用上的时间增加了 20%以上
  • 从 2019 年第四季度到 2020 年 Q1，教育应用的全球下载量平均增长 90%
  • 谷歌教室，提醒和课堂道场在美国飙升。
  • 全球消费者在健身应用上的支出创下 3600 万美元的新纪录
  • 3 月 22 日至 28 日，消费者在全球范围内创造了 1.13 亿小时的家庭健身/健康新纪录
  • 在美国，新冠肺炎期间食品和饮料应用的每周会话增长了 15%以上，杂货交付增长了 510%以上
  • 从 2020 年 1 月 26 日的一周到 3 月 22 日的一周，花在美国股市/交易应用上的时间增加了 80%
  • Robinhood 占美国前五大交易应用花费时间的 40%
  • 商业应用每周都打破下载记录，超过 7200 万(比 2019 年的平均水平增长了 90%以上。)
  • 与 2019 年 12 月 29 日相比，2020 年 3 月 22 日这一周美国社交应用增长了 25%以上

HBO Max 的应用首日安装 87K

根据 Sensor Tower 的数据，华纳媒体在网飞的新竞争对手 HBO Max在推出当天只有大约 87，000 次安装。这远远低于 Disney+的 400 多万首日安装量，甚至低于移动流媒体 Quibi 的 30 万安装量。但公平地说，这不是苹果之间的比较。许多 HBO 的客户已经安装了 HBO NOW 应用程序，该应用程序升级为 HBO Max。Sensor Tower 表示，较旧的应用程序平均每天有 16K 个新安装。然而，下载量的激增将 HBO Max 推上了 App Store 免费应用排行榜的第二名。

抖音有新的竞争对手吗？

本周，一款名为 Zynn 的中国新视频应用登上了美国应用商店排行榜的榜首。该应用程序是抖音体验的虚拟克隆，屏幕右侧有一组相同的交互按钮，还有一个用户界面，可以滑动观看视频或创建自己的视频。

抖音现在还不应该太担心。Zynn 通过内置的金字塔计划成功登上排行榜榜首，奖励用户加入并邀请朋友——就像亚马逊、沃尔玛和苹果的礼品卡——邀请他们的朋友加入应用程序。对于现有应用的潜在竞争对手来说，这种增长方式并不总是好的。(问问任何一家“我们是下一个脸书”的失败创业公司。)说到底，用户不想要钱，他们想去每个人都去的地方——包括他们最喜欢的抖音明星，他们不会为了一个可疑的新人而跳槽。

但是 Zynn 似乎得到了中国抖音的主要竞争对手快手的支持，Gelt VC 的 Turner Novak 说，这意味着它可能有很多现金可以利用。

安卓漏洞让恶意软件伪装成应用程序窃取用户数据

TechCrunch 的 Zack Whittaker 报道了几乎每个版本的 Android 都有一个新的漏洞，这个漏洞被命名为 StrandHogg 2.0，在挪威语中是恶意收购的意思。该漏洞让恶意软件模仿合法应用程序来窃取应用程序密码和其他敏感数据。StrandHogg 2.0 的工作原理是欺骗受害者认为他们正在合法的应用程序上输入密码，而不是与恶意覆盖进行交互。它还可以劫持其他应用程序的权限，窃取敏感的用户数据，如联系人、照片和跟踪受害者的实时位置。据说这种病毒比它的前身更危险，因为它几乎无法被检测到。但是没有证据表明黑客已经使用了它。此外，Android 设备内置的应用筛选服务 Google Play Protect 会阻止利用 StrandHogg 2.0 漏洞的应用。

亚马逊推出 Amplify iOS 和 Amplify Android 库和工具

亚马逊本周宣布推出 Amplify iOS 和 Amplify Android 库和工具，以帮助移动应用程序开发者轻松构建安全和可扩展的云应用程序。

“直到今天，当你开发云驱动的移动应用程序时，你一直在使用工具和 SDK 的组合:Amplify CLI 来创建和管理你的后端，一个或多个 AWS 移动 SDK 来访问后端，”Sébastien Stormacq，在博客文章中解释道。“一般来说，AWS Mobile SDKs 是围绕 AWS 服务 API 的低级包装器。它们要求你理解 API 的细节，而且大多数时候，你需要编写许多行无差别的代码，比如对象(反)序列化、错误处理等。”

Amplify iOS 和 Amplify Android 就是为了简化这一点而创建的，它们提供了面向用例(如身份验证、数据存储和访问、机器学习预测)的原生库；通过提供一个声明性接口，使您能够以编程方式应用抽象的最佳实践，并通过提供与您的本机 IDE 工具链集成的工具:iOS 版的 Xcode 和 Android 版的 Gradle 。

苹果修复了阻止 iOS 应用打开的 bug

苹果用户在过去的这个周末面临一个错误，导致一些 iPhone 和 iPad 应用程序根本无法启动。该问题与苹果家庭共享系统的一个漏洞有关，因为用户报告了错误信息，称“该应用程序不再与您共享”，并指导他们从 App Store 购买该应用程序，以便仍然使用它。在这个问题之后，用户说他们看到了几十个 iOS 设备的应用程序更新，其中一些甚至可以追溯到一周前的最后一次更新。用户在论坛上报告说看到多达 10 个、20 个、50 个甚至 100 多个新的更新需要安装。这表明修复工作正在进行中，因为这些不是全新的更新——应用程序已经是最新的了。相反，这些重新发布的更新似乎是对家庭共享问题的修复的一部分，因为后来错误被解决了。苹果证实已经修复了漏洞，但拒绝谈论其原因。

Niantic 开始从 Pokémon GO 玩家那里收集 3D 视觉数据

本周，Niantic 表示，它将开始允许 Pokémon GO 玩家支持该公司的 3D 数据收集。这款游戏现在可以让用户分享真实世界的神奇宝贝和健身房位置的视频，Niantic 将众包这些视频来创建丰富的 3D 地图。该功能将被玩家选择加入，至少最初是这样。该公司表示，上传到 Niantic 服务器的数据也将被匿名化，包括人脸和车牌在内的视觉数据将被自动模糊化。自该游戏 2016 年首次亮相以来，炒作可能已经消退，但它在 2019 年取得了有史以来最好的收入，近 9 亿美元，并继续领先于 AR 领域。

Quibi 的广告商现在不想付费

Quibi 是一款用于“移动”流媒体的移动应用，在疫情中部推出，用户被要求呆在家里。这款应用不可避免地未能蓬勃发展。本周，鉴于其首次亮相的时机，Quibi 推出了至少一个它从一开始就应该在应用中拥有的关键功能:支持 AirPlay。但是对于一些广告商来说，Quibi 迎合家庭用户的迟来的尝试可能来得太晚了。据《华尔街日报》报道，一些广告商——包括百事可乐、塔可钟、百威英博和沃尔玛——已经寻求推迟付款并修改协议。这些公司表示，他们担心这项服务的低收视率以及冠状病毒对 Quibi 商业模式的影响。

谷歌在套现多年后将 QAnon 应用从 Google Play 中踢出

谷歌已经从它的安卓应用商店中移除了几个散布阴谋的应用。本周，该公司推出了至少四款应用——QMAP、Q Alerts！Q Alerts LITE 和一个西班牙语的 QAnon 应用程序——列举了它在有害信息传播方面的政策。谷歌没有必要庆祝这一姗姗来迟的举措。相反，谷歌花了这么长时间才感到羞耻。在某些情况下，这些现在启动的应用程序已经在 Google Play 上运行了多年。据 CNET 报道，一些网站甚至通过付费下载来获取收入。这意味着谷歌通过对这些违反其政策的阴谋论应用视而不见来获利。

Media Matters 今年早些时候报道了谷歌如何继续从这些与白人至上主义恐怖主义有联系的危险应用的分发中获利，包括 2019 年德克萨斯州埃尔帕索的大屠杀。它还指出，一些卡农的追随者也与暴力、谋杀和企图绑架的行为有关联，导致联邦调查局将卡农标记为潜在的国内恐怖威胁。

苹果曾在 2018 年禁止 QAnon 应用，迫使 iOS 用户想出变通办法。

谷歌删除数百万条抖音负面评论

抖音本周在一个关键市场印度经历了重大反弹。批评者对该平台上似乎宣扬家庭暴力、虐待动物、种族主义、虐待儿童和物化妇女的视频感到愤怒，导致该应用在 Google Play 上遭到强烈反对并获得大量一星评级。谷歌随后介入，删除了数百万条负面评论，这些评论是批评者反对该应用的运动的一部分。谷歌表示，这一行动是为了遏制垃圾邮件的滥用。这些问题引发了人们对抖音在国内的温和努力的质疑。

据 TechCrunch 的马尼什·辛格(Manish Singh)本周的一篇报道，印度政治家马内卡·桑贾伊·甘地表示，抖音没有听取反馈意见，并拒绝删除贬损视频，也拒绝追究发布这些视频的人的责任，尽管她的报告如此。

抖音已经从疫情冠状病毒中获得了经济上的利益，其应用内收入在封锁期间飙升了十倍。(中国版的豆音贡献了其中的 86.6%，美国紧随其后，贡献了 8.2%)。母公司字节跳动去年收入 170 亿美元，净利润 30 亿美元。

Trove 为人工智能项目提供了更好的贡献方式

微软内部的 R&D 集团微软车库(Microsoft Garage)发布了 Trove ，这是一款面向 iOS 和 Android 的移动市场应用，允许人们向涉及机器学习模型训练的人工智能项目贡献照片。

“开发人员通常通过寻求 gig 工作人员的帮助来提交照片，从而众包这些大批量的照片(用于人工智能项目)，但通常情况下，这些照片征集感觉像是一个黑盒，”莱妮·休斯顿在微软车库博客上解释道。“参与者对他们提交照片的原因知之甚少，当他们的提交被毫无理由地拒绝时，他们会觉得自己的时间被浪费了。与此同时，开发人员会发现，由于质量较低、投入较少，这些采购项目需要很长时间才能完成。”

相反，Trove 允许人们看到可用的项目，谁在背后支持他们，然后在做出贡献之前了解他们的数据将如何被使用。他们甚至可以与人工智能开发人员交流，以了解项目的进展。该应用首先在美国微软员工中进行了内部测试，但现在只对受邀的公众开放。开发者可以在这里索取邀请函。

ChatableApps 将 AirPods Pro 变成助听器

ChatableApps 本周在 iOS 上发布了其听力辅助应用。该应用由马克·库班支持，基于听觉神经信号处理研究员安迪·辛普森博士的工作。为了工作，该应用程序几乎实时地消除背景噪音，以便一对一的对话可以听得更清楚。这款应用的独特之处在于它没有使用噪音过滤或其他 DSP 技术。相反，它使用一种深度学习神经网络方法来分离语音和噪声，这种方法不会对原始音频应用过滤器，而是近乎实时地聆听和重新打印一个全新的音频流，这是对原始音频中人声成分的模仿，创始人解释道。

但 ChatableApps 也很有趣，因为它不需要你佩戴它自己的设备。用户可以从像苹果 AirPods Pro、Bose Quietcomfort 20 或余震 z 这样的耳塞中进行选择，所有这些耳塞都被指定为与应用程序一起“工作最佳”。
