法国发布联系人追踪应用 StopCovid TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2020/06/02/france-releases-contact-tracing-app-stopcovid-on-android/

经过数周的激烈辩论，法国政府发布了联系追踪应用 StopCovid。虽然它应该在六个小时前在 iOS 和 Android 上发布，但现在只在 Play Store 上有售。

目前还不清楚是什么导致了 iOS 上的瓶颈，因为政府上周在今天最终发布之前向我发送了 iOS 应用的测试版。我已经联系了法国政府，但还没有回音。

更新:安卓发布几小时后，这款应用已经在应用商店发布。

法国不依赖苹果和谷歌的联系人追踪 API。一群研究人员和私人公司已经在研究一个独立的集中式架构。服务器给你分配一个永久 ID(化名),并向你的手机发送一个从该永久 ID 派生的临时 ID 列表。

像大多数联系人追踪应用程序一样，StopCovid 依靠蓝牙低能耗来建立一个你与之互动超过几分钟的其他应用程序用户的综合列表。如果你正在使用这个应用程序，它会收集你周围其他应用程序用户的临时 id。

实现与这里不同。如果你正在使用 StopCovid，并且你被诊断为新冠肺炎阳性，你的医生、医院或检测机构会给你一个二维码或一串字母和数字。你可以选择打开应用程序，输入代码来分享你在过去两周内互动过的人的临时 id 列表。

然后，服务器后端将所有这些短暂的 id 标记为属于可能暴露于冠状病毒的人。在服务器上，每个用户都与一个风险分值相关联。如果超过某个阈值，用户会收到通知。该应用程序然后建议你进行测试，并遵循官方指示。

从一开始，法国的联系追踪协议罗伯特就提出了一些隐私问题。数百名法国学者在一封要求保护措施的信上签名。特别是，他们表示，尚不清楚接触追踪应用程序在抗击冠状病毒爆发时是否有用。

法国数据保护监管机构 CNIL 声明该应用符合法国的监管框架。但 CNIL 也添加了一长串建议，以保护应用程序用户的隐私。

上周，议会投票赞成发布联系追踪应用程序。但是这是一场复杂的辩论，有很多误解和一些有趣的反对意见。

现在，让我们看看人们对该应用程序的发布有何反应，以及他们是否真的下载了该应用程序。政府表示，如果你不使用 StopCovid，不会有任何负面后果，如果你使用它，也不会有任何特权。
