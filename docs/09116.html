微软将其视窗10内部程序从铃声转移到发布频道

原文：https://web.archive.org/web/https://techcrunch.com/2020/06/15/microsoft-moves-its-windows-10-insider-program-from-rings-to-release-channels/

在过去的几年里，微软给了视窗发烧友选择早期发布“铃声”的能力，让他们可以在“快”和“慢”铃声之间做出选择，以及相对稳定的“发布预览”选项。今天，该公司宣布对该程序进行重大修改，将发布频道，类似于大多数浏览器制造商可能熟悉的频道。

“我们正在转变和转换我们目前的环模型，基于构建的频率，到一个新的渠道模型，这种模型以构建的质量为中心，更好地支持并行编码工作，”微软首席项目经理阿曼达·兰果夫斯基在今天的博客文章中写道。

她指出，基于环的系统的结果是，例如，在2019年年中，视窗内部人员运行三个不同版本的构建，这取决于他们选择的环。

图像信用:微软

“随着我们发布Windows 10方式的不断发展，我们所服务的内部人员比以往任何时候都更加多样化，内部人员有一个可以根据他们的需求定制的飞行方案是至关重要的，”她补充道。“我们认为最好的方法是将注意力从频率转移到质量上。”

因此，从本月晚些时候开始，“快”环将成为开发通道，“慢”环将成为测试通道，“发布预览”将被称为发布预览通道。

Dev Channel是为那些希望很早就接触到新特性的用户设计的，这和fast rings没有太大的不同，但是这里重要的是这个Channel不依赖于任何特定的版本。这一渠道中的新功能一旦准备就绪就会进入发布版，无论是作为主要更新还是服务发布的一部分。由于其不稳定的性质，微软表示，这个版本主要是为高技术用户准备的。

至于测试版渠道，这里的主要区别在于它实际上是一个特定版本的测试版，是为早期采用者准备的。发布预览正是您所想的那样，旨在测试相对稳定的版本，然后再将其发送给更广泛的Windows 10用户群(这样，IT管理员也可以在向公司员工发布之前测试这些版本)。

如果你是 Windows Insider程序的一部分，这些改变将会是自动的，并从本月晚些时候发布的版本开始。
