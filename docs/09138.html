DeepSource 宣布启动 260 万美元的种子期，用于自动化静态代码分析

原文：https://web.archive.org/web/https://techcrunch.com/2020/06/16/deepsource-announce-2-6m-seed-round-to-automate-static-code-analysis/

2020 年冬季 Y Combinator 队列成员 DeepSource 今天宣布了 260 万美元的种子投资。该公司正在构建一个解决方案，以帮助开发人员自动进行静态代码分析，在进行人工代码审查之前发现某些错误。

在 Y Combinator、FundersClub、先锋基金、Liquid 2 Ventures 和一系列个人投资者的帮助下，645 Ventures 领投了这轮融资。该公司此前筹集了 14 万美元的种子前资本。

DeepSource 正在利用一种叫做静态分析的过程。公司联合创始人 Jai Pradeesh 和 Sanket Saurav 是软件工程师，他们认为 Python 等语言包附带的静态分析工具过于复杂，难以使用。Pradeesh 说，结果是大多数开发者都没有接受它们。

“我们试图做的是使用静态分析来确定我们是否可以自动化代码审查的客观部分。[…]每当软件开发人员提交或引入任何代码更改时，DeepSource 都会自动对该代码进行分析。然后，它会标记诸如安全问题、反模式、进度性能问题之类的问题，”Pradeesh 解释道。

通过自动化部分评审过程，它允许人类代码评审员专注于评审中不太客观的部分，并在进入人类评审之前发现无数的问题。DeepSource 向开发人员提供了一份错误报告，并解释了如何修复错误。

但是创始人们认识到了一遍又一遍地不断修复相同错误的苦差事。因此，他们创建了一个名为 Autofix 的自动修复工具，可以为开发人员自动检查并修复一系列常见错误。

行动中的自动修复。图片来源 : 深源

虽然 Pradeesh 说，到目前为止，自动化工具只能覆盖大约 12%的错误，但部分投资将用于帮助建立额外的覆盖范围。目前，该公司支持 Python 和 Go 编程语言，但计划随着时间的推移添加其他语言。他说，几个月前它已经发布了 Ruby on Rails 测试版，对 JavaScript 的支持也在进行中。

该公司目前有 12 人，包括两名联合创始人，总部设在印度的班加罗尔，但计划在疫情允许时最终搬到美国。该公司还计划在未来 12-18 个月内再招聘 10-15 人，主要是工程师。这家初创公司正在与 YC 顾问公司合作建立一个多元化计划，明白自己正处于发展的关键时刻。

他说，YC 的经历教会了他和他的联合创始人如何为设计和营销目的构建更精细的用户角色。不要像所有开发人员一样广泛地看待市场，他们可以开始关注该池的不同方面，就像一个大型团队与一个工程经理。他说，他们还学会了探索使用数据，以了解最常见的错误类型，这有助于为自动修复功能的创建提供信息。

今年早些时候，由于疫情，他们不得不离开办公室去偏远地区，成为 YC 的一部分也帮助了他们。Pradeesh 承认他没有管理远程团队的经验，他的 YC 顾问帮助他适应了这个过程，现在他们已经有了很好的系统。

“我们实施了这些独立的流程，这样我们就可以进行两周一次的冲刺，并学会了确保团队中每个人都能沟通的方法。现在我们已经开始实施这些流程，团队越来越习惯了，”他说。
