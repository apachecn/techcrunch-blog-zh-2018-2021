苹果将允许你把谷歌浏览器扩展移植到 Safari 上

原文：https://web.archive.org/web/https://techcrunch.com/2020/06/25/apple-will-let-you-port-google-chrome-extensions-to-safari/

苹果本周早些时候发布了 macOS 11 Big Sur，并谈到了 Safari 的一些改进。除了原生扩展，苹果正在增加对网络扩展的支持。这将使从 Chrome、Firefox 或 Edge 移植现有扩展变得更加容易。

该公司在 WWDC 会议上分享了更多关于它将如何运作的细节。Safari 已经支持扩展，但是如果你正在使用 Safari，你会知道外面没有太多的扩展。

在 iOS 和 macOS 上，您可以安装内容拦截器和具有共享扩展功能的应用程序。内容拦截器允许您在加载网页时提供要拦截的内容列表，如跟踪器和广告。

共享扩展可让您在 Safari 的“共享”菜单中添加功能。例如，Pocket 或 Instapaper 利用 share 扩展在网页上运行 JavaScript，并将结果返回给应用程序。

在 macOS 上，开发者也可以利用应用扩展。1Password 使用它来将其密码管理器与 Safari 集成。

“如果你是一个已经熟悉 Swift 或 Objective-C 的原生应用程序开发人员，这些都很棒，”Safari 工程师 Ellie Epskamp-Hunt 说。

其他浏览器采取了不同的方法。他们利用网络技术，如 JavaScript、HTML 和 CSS。这就是为什么苹果公司正在为 Safari Web 扩展添加另一种类型的扩展。

像其他 Safari 扩展一样，为 Safari 设计的 web 扩展与原生应用打包在一起。这意味着开发者将向应用商店提交扩展。用户将下载一个带有扩展的应用程序。app 什么都不用做，做个占位就可以了。

苹果推出了一个扩展转换器，让你可以快速移植你的扩展。当你运行它的时候，它会告诉你是否一切都像预期的那样工作。然后，您可以将其打包到 Xcode 项目中，签名并提交到 App Store。

一些扩展需要大量的权限。他们基本上可以查看你访问的所有网页。这就是为什么苹果让你限制扩展到一些网站，或只是活跃的标签。您也可以选择激活一天的扩展，这样它就不会永远保持活动状态。

当一个扩展首次试图访问一个网站时，用户会收到一个警告标志，在你激活一个可以访问你所有浏览数据的扩展之前，Safari 设置中会有一个大的警告横幅。

这一变化可能意味着未来会有更多的 Safari 扩展。许多 Chrome 用户不想离开 Chrome，因为他们找不到相同的扩展。如果开发者选择将其扩展移植到 Safari，苹果可以说服更多用户转向 Safari。
