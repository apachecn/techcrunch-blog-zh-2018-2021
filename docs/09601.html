Macro 刚刚筹集了 430 万美元，让您永无止境的 Zoom 电话变得更加有用 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2020/07/14/macro-just-raised-4-3m-to-make-your-never-ending-zoom-calls-more-useful/

在这个疫情世界，面对面的会议已经成为过去。如今，大多数会议都是通过视频会议进行的，没有哪家公司像 Zoom 一样充分利用了这种转变。

Macro 是一家由 FirstMark 支持的新公司，它正寻求利用这一资本。敬资本主义！

抱歉。让我们回到正轨。Macro 是一个本机应用程序，它采用 Zoom SDK 为您的日常工作会议增加深度和分析。

有两种模式。第一个主要集中在协作上，它将通常的缩放会议变成了一个灯光叠加，人们被显示在屏幕顶部的小圆形气泡中。这种模式是用于当人们工作在同一个项目，如线框或协作文件。用户界面应该淡出背景，允许用户点击其他与会者气泡后面的物体。

另一种模式是竞技场或体育场模式，用于实际操作会议和演示。它有两个明显的特征。第一个是广播时间功能，它显示在过去的五分钟、三十分钟或会议期间总共有多少不同的参与者“发言”。第二个是位于用户界面右侧的文本输入系统，让用户输入问题、要点、行动项目和对通话的见解。

Macro 自动将该文本添加到 Google 文档中，并将其格式化为可立即共享的内容。

在启动和运行宏时没有额外的麻烦。当用户在他们的计算机上安装宏时，每当他们点击缩放链接时，无论是在电子邮件中，日历邀请中，还是在 Slack 中，他们都会立即加载到宏中。

Macro 的联合创始人 Ankith Harathi 和 John Keck 向 TechCrunch 解释说，这不是你通常的企业游戏。该产品可以免费使用，而且通过谷歌文档导出，即使作为单机版产品也仍然有用。谷歌文档是用宏消息自动格式化的，解释说它是由该公司编译的，带有该产品的链接。

换句话说，Harathi 和 Keck 希望看到组织内的个人为自己获得宏，并让产品在组织内有机增长，而不是试图立即销售给大型团队。

Harathi 说:“许多协作生产力 SaaS 应用程序需要您的整个团队进行切换，才能从中获得任何价值。“这是一个相当大的障碍，尤其是因为有如此多的新产品问世，团队不断更换，这就产生了很多噪音。所以我们的计划是确保一个人可以使用它并从中获得价值，而其他人不会受到影响。他们获得了更好的界面，其他团队成员会希望在没有任何要求的情况下进行切换。”

这在很大程度上是因为 Zoom SDK 的成本，它是 0 美元。音频和视频的繁重工作由 Zoom 处理，因为计算成本很高。这意味着 Macro 可以在公司努力发展的过程中以相对较低的成本免费提供其产品。

当然，在现有平台上构建会有一些风险。也就是说，一个缩放平台的变化可能会对宏的产品或模型造成严重破坏。然而，该团队计划将 Zoom 扩展到其他视频会议平台，如谷歌、蓝战士、WebEx 等。Roelof Botha 在 5 月份告诉 TechCrunch，当整个行业都有平台时，建立在其他平台上的企业有更大的成功机会，就像这里一样。

尤其是 Macro 似乎存在一些竞争——例如，微软团队刚刚在其视频会议用户界面中添加了一些新功能以缓解大脑疲劳， Hello 正在寻求通过浏览器提供无应用视频聊天。

Macro 还希望为该平台添加额外的功能，例如将议程整合到会议中的能力，以及按议程项目分解附带的 Google doc 的能力。

自成立以来，该公司已筹集了 480 万美元，包括来自 FirstMark Capital、General Catalyst 和下划线风投的 430 万美元新一轮种子资金。其他投资者包括 NextView Ventures、Jason Warner(GitHub 首席技术官)、Julie Zhuo(前设计副总裁)、Harry Stebbings(20 minvc 的创始人/主持人)、Adam Nash (Dropbox、Wealthfront、LinkedIn)、Clark Valberg(in vision 首席执行官)等。

Macro 拥有 25，000 多名用户，迄今为止已经参加了 50，000 次会议。
