Twitter 引入了一个全新的、完全重建的开发者 API，将于下周发布

原文：https://web.archive.org/web/https://techcrunch.com/2020/07/16/twitter-introduces-a-new-fully-rebuilt-developer-api-launching-next-week/

Twitter 仍在从昨天高调账户遭到大规模攻击的余波中恢复，但它继续推进推出新版本开发者 API 的计划。今天，该公司宣布其全新的 Twitter API v2，从头开始重建。新的基础自 2012 年以来首次重建，包括早期 API 中缺失的功能，如对话线程、推文中的投票结果、固定推文、垃圾邮件过滤以及更强大的流过滤和搜索查询语言。该公司声称，它还被设计成允许 Twitter 比过去几年更快地发布新功能。

虽然 Twitter 表示，没有证据表明昨天的安全事件与其 API 有关，但实际上，按照计划，今天打开它必须让位于确保 Twitter 及其账户安全的重点。该公司计划现在推出 API v2 和其他内容，如其新的支持中心、文档和其他详细的博客帖子。

Twitter 的 API v2 将引入多个访问级别，以取代当前 API (v1.1)中早期的三层系统。

今天，Twitter 的 API 分为三个平台:标准(免费)、高级(自助付费)和企业。但是这使得开发人员很难从一个层迁移到另一个层。新的 API 最终将完全取代这三个产品，而是服务于三个不同产品领域的所有用户，旨在适应不同的开发人员群体。这些路线包括今天发布的标准路线，而商业和学术/研究路线将很快推出。在每条赛道内，还将提供基本、提升或定制的通行级别。

Twitter 开发者平台产品负责人 Ian Cairns 解释道:“我们当然知道一个尺寸并不适合所有人，我们希望让更多的开发者能够更容易地与我们一起成功构建。其中一首歌将永远免费，他补充道。“Twitter 的存在是为了服务公众对话，保持一个免费、开放的 API 对我们来说非常重要，”他说。

Twitter 表示，标准赛道的基本访问级别将永远免费，是为刚刚起步的开发者设计的。

图片来源:推特

该公司尚未最终确定其他层级的定价，但表示通过与开发商的交谈，它已经了解到其定价和费率限制模式是如何限制开发商的，特别是研究人员和那些为了好玩而开发的人。新的定价预计将考虑不同类型的开发者需求，并将在学术研究轨道的提升水平内提供免费和付费等级。

标准产品轨道可以启用常见的 Twitter 工具、实用程序和有趣的机器人，如 BlockParty 、 TweetDelete 、 Tokimeki Unfollow、 HAM: Drawings bot 、 Hansard House of Lords bot 和 Emoji Mashup bot 。

图片来源:推特

与此同时，商业 API 将支持“服务于创新用例”的企业，Twitter 表示。

这是一个 Twitter 有着复杂历史的领域，因为在过去，它已经从开发替代 Twitter 客户端的开发者脚下拔掉了地毯，甚至 T2 欺骗了自己的合作伙伴。Twitter 今天将它旨在支持的用例定义为那些提供“未来产品趋势的社交预测、人工智能驱动的消费者洞察和金融科技市场情报”的用例，如黑天鹅、 Spiketrap 和社交市场分析。

然而，Twitter 在与 press 的电话会议中澄清说，它已经与第三方客户端的制造商进行了交谈，以确定未来如何更好地与他们合作，并指出目前不会改变任何与其支持相关的政策。这些客户端也将能够使用 v2 中的新特性。该公司仍未明确表示，它没有计划改变这些业务目前的运营方式。

相反，Twitter 向 TechCrunch 解释说，它认为这些客户“应该清楚如何操作”新的 API。但这种清晰性可能需要 Twitter 重新审视其政策和产品访问细节，Twitter 表示，并补充说，它正在向前看，以确定如何最好地与这一群体合作。鉴于 API 已经开发了一年多，对于喜欢第三方客户端的 Twitter 高级用户来说，这是一个令人失望的答案，比如 Tweetbot、Twitterific、Echofon 等。Twitter 有足够的时间来采取这种“新的面貌”，但似乎仍未做出决定。

此外，业务 API 将服务于 Twitter 官方合作伙伴，如 Brandwatch 、 Sprinklr 和 Sprout Social ，以及 Twitter 的企业数据客户。未来的这一路线将包括对相关端点的提升和定制访问。

与此同时，即将到来的学术/研究轨道将允许合格的研究人员了解 Twitter 的公共对话中发生了什么。

开发人员今天使用 Twitter 数据研究一系列主题，如人们对新冠肺炎的态度，T2 洪水和气候变化的社会影响，以及 T4 网上仇恨言论的流行。它还表示，这将在以后增加对相关终端的提升和定制访问，并将是 Twitter 首次为研究人员的特定需求量身定制产品。

在所有这些产品中，只有标准 API 产品准备在下周发布，并在基础级别免费提供一组新功能。它的发布之后将是学术/研究产品跟踪，Twitter 将在未来几个月继续发布新的 API。然而，将开发人员从 1.1 版转移到 v2 版需要一些时间，所以 Twitter 的 API 路线图和文档可以帮助指导他们何时会发生变化。

Twitter 的“消防水管”数据(完整流)将继续只在有限的合作伙伴中可用，就像今天一样。Twitter 表示，大多数开发人员不希望这样，即使他们有很高的数据访问需求，因为 firehose 数据很难处理。

图片来源:推特

该公司表示，重建其开发者平台的决定是因为 Twitter 需要更容易地扩展大量 API 端点，以实现未来计划的和新的功能。(或许有关联:Twitter 的一份工作描述中提到计划“建立一个订阅平台”。这可能需要新的 API？)

在 API 的当前版本中，端点是由大量 HTTP 微服务实现的——这是 Twitter 在 2013 年从 Ruby 重新架构时做出的决定。这最终创造了一个脱节的产品，其中独立的团队在他们的端点上工作，而没有与其他人协调。

图片来源:推特

Twitter 已经测试新的 API 特性一年多了，作为其 Twitter 开发者实验室计划的一部分，这是向公开构建的转变。这一变化允许公司从开发人员社区获得实时反馈，因为产品是在开放环境中构建的。开发人员告诉 Twitter，他们想要更好的文档，进入一个参与社区，一个测试沙箱，更容易的入职和其他功能。

Twitter 特别回应了这些要求，建立了一个新的开发者门户，该门户也经过了重新设计。门户将提供一个 onboarding 向导来简化获取 API 密钥。该门户还允许开发人员管理他们的应用程序，了解他们的 API 用法和限制，访问新的支持中心，查找文档等。开发人员还可以查看 Twitter 的公开路线图并通读即将发布的“TwitterAPI 未来指南”，该指南将于下周 v2 发布时发布，了解更多预期内容。

下周，Twitter 将推出首批新终端的“早期接入”。与 Twitter 开发者实验室不同，早期访问将是生产就绪和完全支持的。新的端点将允许开发人员实时传输推文，分析过去的对话，测量推文性能，收听重要事件，并探索来自任何帐户的推文。在接下来的几周，Twitter 将决定哪些新功能可能会转移到 API 中——例如语音推文或允许特定受众回复你的推文。

Twitter 表示，在贬低任何现有产品之前，它将继续分享 v2 的更新。

“我们的目的是在淘汰现有终端时，提供充足的迁移时间和资源帮助。我们知道迁移可能具有挑战性，我们致力于尽我们的努力使迁移到我们的新 API 尽可能容易，”一位发言人说。

然而，也有一些例外。例如，今年晚些时候，Twitter 将宣布缩短 1.1 版状态/样本和状态/过滤器端点的弃用窗口。他们的替代品将于下周在 v2 中推出。

当 API v2 发布时，开发人员可以通过开发人员门户提前开始访问。
