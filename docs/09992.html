黑客称“劫持”漏洞欺骗了受欢迎的自动提款机吐出现金

原文：https://web.archive.org/web/https://techcrunch.com/2020/08/06/hackers-atm-spit-cash/

2010 年，已故的世界知名安全研究员巴纳拜·杰克在黑帽大会上现场入侵了一台自动提款机，诱骗提款机吐出一大串美钞。这项技术被恰当地命名为“jackpotting”

在杰克的轰动性演示十年后，安全研究人员提出了鹦鹉螺 ATM 机的两个新漏洞，尽管是虚拟的，但这要归功于冠状病毒疫情。

总部位于纽约的安全公司 Red Balloon 的安全研究人员 Brenda So 和 Trey Keown 表示，他们的一对漏洞使他们能够欺骗商店而不是银行常见的流行独立零售 ATM 机，按照他们的命令分发现金。

黑客需要与 ATM 机在同一个网络上，这使得成功发动劫持攻击更加困难。但是他们的发现强调了自动取款机通常有潜伏多年的漏洞——在某些情况下从它们第一次被制造出来开始。

已故的安全研究员巴纳拜·杰克是第一个“劫持”ATM 机的黑客。现在，10 年后，两名安全研究人员发现了两种新的 ATM 吐钞攻击。鸣谢:YouTube

So 和 Keown 表示，他们的新漏洞针对 Nautilus ATM 的底层软件，这是一个十年前的 Windows 版本，不再受微软支持。首先，这对夫妇买了一台自动取款机来检查。但由于几乎没有文档，两人不得不对内部软件进行逆向工程，以了解其工作原理。

第一个漏洞是在一个被称为 XFS 的软件层中发现的，该软件层是金融服务的扩展，ATM 使用它来与各种硬件组件进行对话，如读卡器和现金分发单元。问题不在于 XFS 本身，而在于 ATM 制造商如何将软件层应用到 ATM 中。Keown 告诉 TechCrunch，研究人员发现，通过网络发送一个特制的恶意请求可以有效地触发 ATM 的取款机，并将现金倒入其中。

第二个漏洞是在自动取款机的远程管理软件中发现的，这是一个内置的工具，让主人通过更新软件和检查还剩多少现金来管理他们的自动取款机。触发这个漏洞会让黑客获得易受攻击的 ATM 设置。

So 告诉 TechCrunch，有可能用一个恶意的黑客控制的服务器来切换 ATM 的支付处理器，从而窃取银行数据。“通过将 ATM 指向恶意服务器，我们可以提取信用卡号码，”她说。

去年，当研究人员私下向鹦鹉螺公司报告他们的发现时，彭博首次报告了这些漏洞。据彭博报道，在修复之前，美国约有 80，000 台鹦鹉螺自动取款机易受攻击。鹦鹉螺公司的发言人不愿证实这一数字。

成功的劫持攻击很少见，但并非没有听说过。近年来，黑客使用了许多技术。2017 年，一个活跃的 jackpotting 集团被发现在欧洲各地活动，净得数百万欧元现金。

最近，黑客们从 ATM 制造商那里窃取专有软件，让制造他们自己的黑客工具。
