谷歌 Chrome 更新带来了更好的标签管理、二维码以及性能提升

原文：https://web.archive.org/web/https://techcrunch.com/2020/08/25/google-chrome-update-brings-better-tab-management-qr-codes-plus-performance-improvements/

谷歌今天将开始对其 Chrome 网络浏览器进行几项更新，目标是提高用户生产率，让浏览体验更快。具体来说，谷歌正在让 Chrome 的标签及其更新的标签组更容易、更快捷地使用。在引擎盖下，它承诺进行改进，将提供高达 10%的页面加载速度，并将减少闲置背景标签的影响。

总的来说，全套新功能解决了 Chrome power 用户的担忧，他们倾向于同时打开许多标签，并经常在浏览器中工作。

谷歌今年五月引入了标签组，最初处于测试阶段。该功能允许用户将他们打开的选项卡添加到一个组中，然后他们可以命名和标记该组，以保持他们的各种项目、任务、应用程序和其他在线研究有组织。在此版本中，选项卡组将向所有用户展开。

基于测试版用户的反馈，谷歌也在调整标签组的工作方式。

Chrome 现在允许用户折叠和展开他们的标签组，所以你可以专注于你现在需要访问的标签组。谷歌表示，这是它从那些在测试中使用标签组的人那里听到的最普遍的要求。此外，谷歌正在为平板电脑模式下使用的笔记本电脑设计的标签引入新的触摸屏界面，首先是 Chromebooks。这将使用户更容易翻阅标签。

图片来源:谷歌

在 Android 版本的 Chrome 上，如果你开始在地址栏中输入页面，你会看到一个建议，如果它已经打开，就切换到那个标签。Android 用户还将获得简化的 URL 共享，从而更容易复制链接并共享到其他设备或通过其他应用程序发送链接。

图片来源:谷歌

此外，他们可以打印页面，甚至生成二维码进行扫描或下载。

新的二维码功能也将出现在桌面上的 Chrome 浏览器中，可以通过 Chrome 地址栏中的新二维码图标访问。

图片来源:谷歌

谷歌还将在最新版本中推出可能会非常受欢迎的新功能——直接从 Chrome 填写和保存 PDF 表单的功能。你甚至可以再次打开这些文件，从你停下来的地方重新开始。然而，谷歌表示，这项功能将在未来几周内慢慢推出。

与此同时，Chrome 浏览器的测试版将引入一项功能，让你将鼠标悬停在一个标签上，就可以看到页面的缩略图预览。这可能是有用的，特别是当你有很多来自同一个域的标签页的时候，比如谷歌文档。

为了改善在 Chrome 中使用许多标签页的整体体验，这个版本(M85 版)将提供两项改进。谷歌解释说，第一个是 Profile Guided Optimization，这是一种编译器优化技术，其中代码中最关键的性能部分可以运行得更快。它表示，通过优先处理最常见的任务，这将使页面加载速度提高 10%。这一改进将通过 Chrome M85 在 Mac 和 Windows 上推广。(这项技术实际上最早是在 M53 中使用微软 Visual C++ [MSVC]引入的，这是 Chrome 以前的构建环境。现在使用 Clang 并将登陆 Mac 和 Windows。

在 Chrome Beta 频道中，谷歌也引入了标签节流，通过从已经在后台很长时间的标签中收回资源，为你当前使用的标签提供更多资源。这一改变应该会在加载速度、电池和内存节省方面带来改进。

图片来源:谷歌

Chrome 经常因慢和 Mac 上的资源贪婪而被批评。虽然用户的抱怨可以归因于各种各样的问题，但最终用户会责怪软件，而不是运行它的操作系统。这意味着谷歌必须努力解决它能够通过推出这样的优化来解决的问题。然而，这些改进在多大程度上实现了谷歌承诺的结果，还需要第三方测试来证实。
