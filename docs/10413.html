一个 SonicWall 云漏洞将企业网络暴露给黑客 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2020/09/02/sonicwall-bug-exposed-networks/

在用于管理 SonicWall 防火墙的云系统中新发现的一个漏洞可能会让黑客侵入数以千计的公司网络。

企业防火墙和虚拟专用网设备是至关重要的把关者，其任务是保护企业网络免受黑客和网络攻击，同时在疫情期间仍然允许员工在家工作。尽管大多数办公室都是空的，黑客们还是经常在关键的网络设备中寻找漏洞,以便闯入公司网络窃取数据或植入恶意软件。

安全公司 Pen Test Partners 的研究员范吉利斯·斯蒂卡斯(Andrew sty kas)在 SonicWall 的全球管理系统(GMS)中发现了新的漏洞，这是一个 web 应用程序，允许 IT 部门通过网络远程配置他们的 SonicWall 设备。

但是这个漏洞如果被利用，意味着任何访问 SonicWall 的 GMS 的现有用户都可以创建一个用户帐户，在未经许可的情况下访问任何其他公司的网络。

从那里，新创建的帐户可以远程管理该公司的 SonicWall 设备。

在与 TechCrunch 分享的一篇博客文章中，Stykas 说有两个进入壁垒。首先，潜在的攻击者需要一个现有的 SonicWall GMS 用户帐户。最简单的方法——也是 Stykas 独立测试 bug 的方法——是购买一个 SonicWall 设备。

第二个问题是，潜在的攻击者还需要猜测一个与另一家公司的网络相关的独特的七位数。但是 Stykas 说，这个数字似乎是连续的，可以很容易地一个接一个地计数。

一旦进入一家公司的网络，攻击者可以直接向受害者的内部系统发送勒索软件，这是一种越来越受经济利益驱动的黑客们欢迎的策略。

SonicWall 证实该漏洞现已修复。但斯蒂卡斯批评该公司花了两个多星期来修补漏洞，他形容这是“微不足道”的利用。

“就连汽车报警器供应商也在美国报告的三天内修复了类似的问题，”他写道。

SonicWall 的一位发言人辩护说，该修复程序在推出之前要接受“全面”的质量检查，并表示“不知道”有任何利用该漏洞的行为。
