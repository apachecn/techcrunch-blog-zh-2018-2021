Diffblue 发布了其自动化 Java 单元测试工具的免费社区版

原文：https://web.archive.org/web/https://techcrunch.com/2020/09/08/diffblue-launches-a-free-community-edition-of-its-automated-java-unit-testing-tool/

从牛津大学剥离出来的 Diffblue ，使用机器学习来帮助开发人员自动为他们的 Java 代码创建单元测试。因为很少有开发人员喜欢编写单元测试来确保他们的代码按预期工作，所以自动化程度的提高不仅有助于开发人员专注于编写真正有所作为的代码，还会导致代码出现更少的错误。目前 Diffblue 的客户包括高盛和 AWS 等公司。

Diffblue 以前只通过付费订阅提供服务，而且价格昂贵。然而，今天，该公司还推出了免费的社区版 Diffblue Cover: Community Edition，它不具备付费版本的所有企业功能，但仍提供 IntelliJ 插件和与付费版本相同的人工智能生成的单元测试。

该公司还计划为 Diffblue Cover 推出一个新的更低成本的“个人”计划，每月 120 美元起。该计划还将提供支持和其他高级功能。

在其核心，Diffblue 使用无监督学习来构建这些单元测试。大约一年前加入公司的 Diffblue 首席执行官 Mathew Lodge 解释说:“我们正在做的事情是独一无二的，因为之前已经有工具使用所谓的静态分析。”“他们看着程序，基本上理解了程序中的路径，并尝试从该路径开始逆向工作。所以如果路径到达这一点，我们需要在程序中输入什么才能到达这里？”然而，这种方法有其局限性，Diffblue 的强化学习方法旨在解决这一问题。

一旦这个过程结束，Diffblue 就会为开发者提供可读的测试。洛格强调说，这很重要，因为如果测试失败，而开发人员不知道发生了什么，开发人员几乎不可能修复这个问题。这是团队通过艰难的方式学到的东西，所以 Diffblue 使用了一种非常激进的算法，提供了很好的测试覆盖率(单元测试的关键指标)，但让开发人员很难弄清楚发生了什么。

社区版不提供 Diffblue 付费版的命令行界面(CLI ),开发人员可以像以前一样用 IntelliJ 编写代码，然后只需单击一个按钮，就可以让 Diffblue 为该代码编写测试。

“社区版的设计非常易于使用。在 IDE 中只需点击一下鼠标，就可以得到测试结果。CLI 版本更加复杂，它涵盖了更多案例，并为组织内部的团队和大型部署提供了解决方案，”Lodge 解释道。

该公司计划增加对其他语言的支持，包括 Python、JavaScript 和 C#，但正如洛格指出的那样，Java 长期以来一直是商业世界的主流，该团队认为这将是最好的开始语言。不过，正如洛格指出的，这项技术可以跨语言使用。

Diffblue 实际上已经存在了一段时间。该公司在 2017 年筹集了 2200 万美元的首轮融资，由高盛牵头，牛津科学创新和牛津技术与创新基金参与。显然，你不会筹集那么多钱来只关注 Java 代码的单元测试。除了支持更多的语言，单元测试只是该公司在人工智能的帮助下自动化更多编程过程的总体目标的第一步。

“我们从测试开始，因为这是一个重要而紧迫的问题，尤其是它对 DevOps 和采用更快速的软件周期的影响，”Lodge 说。下一个明显的步骤是采用类似的方法自动修复代码中的错误——尤其是安全错误。

“这个想法是，有这些机器编写越来越多代码的垫脚石，”他说。“坦率地说，这也是让开发人员习惯这一点的一种方式。因为开发人员的认可是成功的关键。”
