保持清醒

GitHub to sunset主/从术语

这对开发者来说可能不是什么新闻，但当我发现许多科技公司仍然在使用主从语言时，这对我来说却是新闻。现在， 微软旗下的GitHub正准备通过将主要代码库命名为“main”而不是“master”来删除这些对奴隶制的引用。这些变化将于10月1日生效。

GitHub早在6月份就谈到了做出这些改变，当时 首席执行官Nat Friedman在推特上说 这是公司已经在做的事情。但GitHub绝不是第一家考虑并做出这些改变的公司。2014年 ，开源平台Drupal将“主/从”改为“主/副本”

做出这一改变的原因之一是，“‘奴隶’一词有负面含义(尽管这可能与一个技术术语的命名有关，也可能无关)，包括为欧洲殖民势力利益的长达数世纪的奴隶制历史，今天的监狱劳动者被迫在类似奴隶制的条件下工作，今天世界许多地方的年轻女孩被卖为性奴隶。”

然后，2018年， 编程语言Python抛弃了种族主义术语 。与此同时， Twitter今年早些时候开始采取措施替换这些术语 ，并希望在2021年底前完成替换，根据CNET的说法。

奇怪的是，这些术语原本就存在，只是现在才被提出来。 虽然洛杉矶市官员早在2003年就要求其 制造商和供应商停止使用这一术语，但他们并没有要求这样做。

因此，难怪一些科技公司难以留住黑人员工。例如，2019年，谷歌报告了其黑人和拉丁裔人才的流失率——表明员工每年的离职率——高于全国平均水平。当种族主义被植入一家公司的技术框架时，它会延续一种错误的观念，即白人优于黑人。

零工

关于道具22和AB 5的最新战斗

与临时工有关的两件大事正在发生:由优步、Lyft、Instacart和DoorDash支持的加州法案Prop 22，该法案旨在将工人归类为独立承包商，以及源于AB 5的诉讼，AB 5是今年早些时候生效的加州法律，规定了如何正确划分临时工。

让我们从22号提案开始。加州大学伯克利分校政府研究所的一项新民调发现，这将是一场势均力敌的选举。加州大学伯克利分校的IGS对5，900名可能的选民进行了调查，发现39%的选民会对第22号提案投赞成票，36%的人说他们会投反对票。另外25%的人尚未决定。

正如我们上周提到的，赞成22的一方已投入约1.8亿美元，反对22的一方投入约460万美元。与此同时，我们在22个点播应用中看到了Yes的广告。

图片来源:来自TechCrunch的DoorDash应用截图

在事情的AB 5方面，优步和Lyft仍然在法庭上在加州司法部长泽维尔·韦塞拉，以及来自洛杉矶、圣地亚哥和旧金山的城市律师起诉这些公司，声称他们对他们的工人进行了错误的分类。上诉法院批准暂缓执行初步禁令，该禁令将迫使优步和Lyft立即对其司机进行重新分类，一些法庭之友的辩护状已经提交。

在国家就业法律小组、美国公民自由联盟和其他民权组织提交的一份简报中，他们称优步和Lyft将有色人种工人归类为独立承包商，从而伤害了他们:

许多贫穷的有色人种工人和移民被困在一个独立和不平等的经济中，他们工资过低，在工作中受到伤害，没有带薪病假、失业保险、工人赔偿和其他保护，只能自谋生路。通过坚持他们的司机不是雇员，Lyft和优步进一步疏远了有色人种工人与提供真正灵活性和经济安全的基本工作场所权利的距离。相反，他们的商业模式让贫困工人陷入了贫困和经济排斥的恶性循环。

如果优步和Lyft被迫对他们的司机进行重新分类，优步首席执行官达拉·科斯罗萨西和Lyft首席执行官洛根·格林本月早些时候提交了宣誓声明，证实他们都计划遵守要求他们重新分类各自劳动力的命令。

在Khosrowshahi的声明中，他只是说“优步已经制定了实施计划”在不超过30天内遵守命令。在格林的声明中，他说“这种实施可能包括停止加州所有或部分地区的拼车业务。”
