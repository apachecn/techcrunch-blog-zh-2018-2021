Cube Dev 为其开源数据平台筹集了 620 万美元

原文：https://web.archive.org/web/https://techcrunch.com/2020/09/29/cube-js-raises-6-5m-for-its-open-source-data-platform/

Cube Dev 是 Cube.js 背后的开源公司，我正在建立一个数据平台，帮助开发者为内部和外部用户编写分析应用程序，今天宣布它已经筹集了 620 万美元的种子资金，由贝恩资本风险投资公司领导。以前的投资者 Eniac Ventures、Betaworks、Innovation destinations 和 Slack Fund 也参加了，此外还有新的投资者 Uncorrelated Ventures 和 Overtime.vc。

两位联合创始人 Artyom Keydunov 和 Pavel Tiunov 实际上为他们在 2016 年创立的另一家公司 Statsbot 构建了现在成功的 Cube.js 项目的核心。Statsbot 是一个商业智能平台，帮助企业创建报告和仪表板——还有一个 Slack bot。

Keydunov 说:“在开发 Statsbot 时，我们现在构建了 Cube.js 来支持 Statsbot 应用程序。“但随着时间的推移，随着用户使用 Statsbot，我们开始看到他们询问我们如何使用这项技术来支持内部应用程序或面向客户的分析应用程序。[……]我们与几家公司合作，对使用 Cube.js 作为独立技术的概念进行了一些验证，我们得到了非常积极的反馈。”

Cube.js 背后的总体思想是取代构建后端基础设施以连接数据源和构建可视化的大量繁重工作。开源工具本质上是一个中间件层，位于数据库和前端之间，处理 SQL 生成、缓存、安全和编排，以便开发人员可以专注于编写他们的应用程序。得益于其缓存技术，也为开发者解决了很多性能问题。

图片来源: Cube.js

正如 Keydunov 所说，如果你是一名数据分析师或数据工程师，今天市场上已经有很多工具可以满足你的大部分需求。然而，通常需要构建定制应用程序的开发人员不得不依赖于各种非连接的库。“他们没有任何可以专门用于构建分析应用的解决方案，”他指出。

“我们经常投资开源公司。贝恩资本(Bain Capital Ventures)合伙人斯蒂芬科恩(Stefan Cohen)告诉我，我们非常清楚的一个痛点是，构建连接到大量不同数据源的应用程序是一项挑战。“公共云的出现和数据源的异构性，以及开发人员和工程师的快速消费，使得我们很难将所有这些东西整合在一起，为企业提供视觉上有吸引力且有用的应用程序。”

然而，企业需要这些应用程序，因为它们可以帮助他们获得新的收入并简化他们的工作流程。这使得 Cube Dev 正好处于这一趋势的中间。

像大多数开源公司一样，Cube Dev 团队正在考虑为企业提供商业云和 SaaS 服务，以及所有常见的企业装备，如额外的安全性和登录功能。

正如 Keydunov 告诉我的，该团队预计在 2020 年通过活动和会议围绕 Cube Dev 建立开源社区。虽然这显然变得有点困难，但团队仍然专注于尽可能多地与潜在客户和社区成员交谈。

“我认为最大的挑战——也是我们的机会——是从开源到商业产品的飞跃，”科恩指出。“很高兴看到这么多开发人员和组织使用 Cube.js 开源软件。但是，我们真正需要做的是获得功能全面的云产品，然后开始推动它的使用。甚至不一定要赚钱，但只是确保我们的企业功能是正确的市场，我们正在解决一个有意义的痛点。我认为，如果我们能做到这一点，世界就是我们的了——但我们必须推出这种产品，然后开始推动一些初步使用。”

不出所料，这也是该团队计划在银行新一轮融资中关注的问题。
