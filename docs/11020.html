孔推出其云原生连接平台TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2020/10/07/kong-launches-kong-konnect-its-cloud-native-connectivity-platform/

在其(虚拟) Kong Summit 2020 上，API平台 Kong 今天宣布推出Kong Konnect，这是其托管的端到端云原生连接平台。这里的想法是为企业提供单一服务，使他们能够管理其API和微服务之间的连接，并帮助开发人员和运营商管理他们在Kong的API网关、Kubernetes Ingress和Kong Service Mesh运行时之间的工作流。

“这是一个基于消费的通用控制平面交付云，您可以从一个平台管理和协调API网关运行时、服务网格运行时和Kubernetes入口控制器运行时，甚至是针对设计的失眠症，”Kong首席执行官兼联合创始人 Augusto "Aghi" Marietti 告诉我。

这项新服务目前处于私人测试阶段，将于2021年初全面推出。

图片来源:孔

该平台的核心是孔的新服务中心(ServiceHub)，它为管理公司在整个组织中的服务提供了单一平台(并使这些服务可以跨团队访问)。

正如Marietti指出的，组织可以选择他们想要使用的运行时，并且只购买他们当前需要的那些服务功能。该平台还包括内置的监控工具，并支持任何云、Kubernetes提供商或内部环境，只要它们是基于Kubernetes的。

这里的想法也是让所有这些工具对开发者开放，而不仅仅是架构师和操作者。“我认为这也是一个关键的优势，”马里蒂说。“我们正在降低障碍，让连接技术更容易被5000万开发人员使用，而不仅仅是为大公司制定宏伟计划的建筑师使用。”

为了做到这一点，Konnect将作为一个自助服务平台，减少采用该服务的摩擦。

图片来源:孔

这也是该公司超越其核心API管理服务的宏伟计划的一部分。这些服务不会消失，但它们现在是更大的Kong平台的一部分。通过其开源的 Kong API Gateway ，该公司建立了达到这一点的途径，但这是一个稳定的产品，现在它显然在扩展，通过云连接，利用该公司现有的运行时，将它们结合起来，提供更全面的服务。

“我们已经提升了真正成为端到端云连接公司的愿景，”Marietti说。“无论是API管理还是Kubernetes Ingress、[…]还是库马服务网格。是关于连通性的问题。因此，该公司将该解决方案提升到了企业层面。”
