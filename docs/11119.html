WordPress 现在可以自动将博客文章转化为推特风暴

原文：https://web.archive.org/web/https://techcrunch.com/2020/10/13/wordpress-can-now-turn-blog-posts-into-tweetstorms-automatically/

今年早些时候，WordPress.com推出了一种将你的 Twitter 帖子发布到你的博客的更简单的方法，也称为 tweetstorms，引入了 Twitter 嵌入的“展开”选项。今天，该公司正在解决 tweetstorm 出版物的另一面——它是,使只需点击几下就可以将你现有的 WordPress 博客文章变成 tweetstorm。

该公司表示，新功能将允许你在推特上发布帖子的每一个字，以及附带的图像和视频。这些将自动插入到线索中，它们应该在你的文本旁边。

要使用 tweetstorm 功能，WordPress 用户将首先点击页面右上角的 Jetpack 图标，然后将他们的 Twitter 帐户连接到他们的 WordPress 站点，如果还没有这样做的话。

如果你想在几个地方发布你的推特风暴，这个选项也支持多个推特账户。

一旦 Twitter 连接上，你将选择一个或多个你想发推文的账户，然后选择新添加的选项，将帖子作为 Twitter 线程共享，而不是带有链接的单个帖子。

图片来源:WordPress.com

在提供的框中，你将为你的 tweetstorm 写一条介绍性消息，这样 Twitter 用户将知道你的 Twitter 线程将讨论什么。

当你点击“发布”按钮时，这篇博客文章将被自动分享为 tweetstorm。

图片来源:WordPress.com

该功能还经过了一些深思熟虑的设计，使推特风暴感觉更自然，就像是直接在推特上写的一样。

首先，WordPress 表示，它将关注博客文章的格式，以确定在哪里分隔推文。例如，它不是在第一条推文中尽可能多地使用单词，而是将断点放在第一句话的末尾。当一个段落对于一条推文来说太长时，它会根据需要自动拆分成多个推文，而不是被截断。同时，列表块将被格式化为 Twitter 上的列表。

为了帮助作者制作一篇将作为 tweetstorm 的博客文章，你可以选择在社交预览功能中查看推文将被拆分的位置。这使得 WordPress 用户可以更好地修改帖子，以适应 Twitter 的字符限制。

图片来源:WordPress.com

在发布的 tweetstorm 结束时，Twitter 追随者将能够点击链接阅读 WordPress 网站上的帖子。

这解决了 Twitter 线程的一个常见问题。虽然在社交媒体上发布更长的想法很有用，但直接在 Twitter 上阅读段落内容可能很困难。但是随着 tweetstroms 越来越受欢迎，解决这个问题的工具出现了。最受欢迎的是一个名为@ThreadReaderApp 的 Twitter 机器人，它允许用户以长格式阅读一个线程，方法是在线程中通过名称以及关键字“unroll”来提及帐户

然而，随着新 WordPress 功能的推出，Twitter 用户不必求助于第三方工具——他们只需点击提供的链接，就可以阅读博客内容。这反过来有助于将 Twitter 追随者转化为博客订阅者，允许 WordPress 作者增加他们的整体影响力。

上个月宣布了 WordPress 引入 tweetstorm 功能的计划，该计划将于 10 月初在 Jetpack 9.0 版本中发布。

该公司表示，该功能现已公开。
