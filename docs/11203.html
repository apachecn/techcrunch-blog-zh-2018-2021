脸书推出支持 Instagram 的新 Messenger API

原文：https://web.archive.org/web/https://techcrunch.com/2020/10/19/facebook-introduces-a-new-messenger-api-with-support-for-instagram/

继更新 Instagram 和 Messenger 提供跨应用程序通信和其他功能后，今日脸书宣布其 Messenger API 也已更新，除 Messenger 外，允许企业管理他们在 Instagram 上的通信。

在今天之前，企业只能通过 Instagram 应用程序和脸书的统一业务收件箱来回复客户的询问。这可能对一些小企业有用，但对于拥有大量信息的大品牌来说，这种方式可能很难有效。

这一更新意味着企业现在也可以将 Instagram 消息集成到他们已经在内部使用的应用程序和工作流中，以管理他们的脸书对话。具体来说，他们将能够使用丰富的媒体，如照片、URL 链接等，并与开发人员合作，将 API 与他们的产品和客户数据库集成在一起，在 Instagram 上提供与今天在 Messenger 上相同的体验。

例如，集成了 CRM 系统的企业将能够查看客户忠诚度信息，并在他们做出响应时考虑这些信息。

脸书说，使用 API 的企业还可以管理他们的 Instagram，包括他们的个人资料、商店和故事。

图片来源:脸书

这一变化发生在 Instagram 正在推动购物成为 Instagram 上的核心活动之际，此前推出了 Instagram 商店和应用程序的视觉变化，以突出购物功能。

据脸书称，过去一年，人们和企业在 Messenger 和 Instagram 上的日常对话总量增长了 40%以上。

随着新 API 的推出，脸书还在 Instagram 上引入了新功能，允许企业使用自动化功能立即响应常见问题，同时仍可在需要时将客户连接到实时支持。与合作伙伴 Clarabridge 就该功能进行的 alpha 测试表明，客户品牌通过其平台管理 DMs，将 Instagram 上的回复率提高了 55%。

更新后的 Messenger API 正在与阿迪达斯、Amaro、Glossier、H&M、MagazineLuiza、迈克高仕、Nars、丝芙兰和 TechStyle Fashion Group 等消费者品牌进行测试。测试版也对数量有限的开发者伙伴开放。今天，其他的企业和的开发者可以加入一个等待列表，请求获得 API 后测试版。

跨应用程序通信是 Instagram 和 Messenger 最近更新的一个关键部分，但在发布时不在 API 中提供。目前，这些信息将出现在一个品牌的 Messenger 或 Instagram 标签上，这取决于他们的客户从哪里发信息。然而，脸书证实，它计划在以后的更新中“最终”为企业和开发者带来跨应用程序的通信。
