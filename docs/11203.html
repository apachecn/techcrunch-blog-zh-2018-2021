脸书推出支持Instagram的新Messenger API

原文：https://web.archive.org/web/https://techcrunch.com/2020/10/19/facebook-introduces-a-new-messenger-api-with-support-for-instagram/

继更新Instagram 和 Messenger 提供跨应用程序通信和其他功能后，今日脸书宣布其Messenger API也已更新，除Messenger外，允许企业管理他们在Instagram上的通信。

在今天之前，企业只能通过Instagram应用程序和脸书的统一业务收件箱来回复客户的询问。这可能对一些小企业有用，但对于拥有大量信息的大品牌来说，这种方式可能很难有效。

这一更新意味着企业现在也可以将Instagram消息集成到他们已经在内部使用的应用程序和工作流中，以管理他们的脸书对话。具体来说，他们将能够使用丰富的媒体，如照片、URL链接等，并与开发人员合作，将API与他们的产品和客户数据库集成在一起，在Instagram上提供与今天在Messenger上相同的体验。

例如，集成了CRM系统的企业将能够查看客户忠诚度信息，并在他们做出响应时考虑这些信息。

脸书说，使用API的企业还可以管理他们的Instagram，包括他们的个人资料、商店和故事。

图片来源:脸书

这一变化发生在Instagram正在推动购物成为Instagram上的核心活动之际，此前推出了Instagram商店和应用程序的视觉变化，以突出购物功能。

据脸书称，过去一年，人们和企业在Messenger和Instagram上的日常对话总量增长了40%以上。

随着新API的推出，脸书还在Instagram上引入了新功能，允许企业使用自动化功能立即响应常见问题，同时仍可在需要时将客户连接到实时支持。与合作伙伴Clarabridge就该功能进行的alpha测试表明，客户品牌通过其平台管理DMs，将Instagram上的回复率提高了55%。

更新后的Messenger API正在与阿迪达斯、Amaro、Glossier、H&M、MagazineLuiza、迈克高仕、Nars、丝芙兰和TechStyle Fashion Group等消费者品牌进行测试。测试版也对数量有限的开发者伙伴开放。今天，其他的企业和的开发者可以加入一个等待列表，请求获得API后测试版。

跨应用程序通信是Instagram和Messenger最近更新的一个关键部分，但在发布时不在API中提供。目前，这些信息将出现在一个品牌的Messenger或Instagram标签上，这取决于他们的客户从哪里发信息。然而，脸书证实，它计划在以后的更新中“最终”为企业和开发者带来跨应用程序的通信。
