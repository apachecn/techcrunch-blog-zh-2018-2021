总部位于奥斯汀的 verified 为其 API 工具包筹集了 300 万美元，用于验证医疗保健证书

原文：https://web.archive.org/web/https://techcrunch.com/2020/10/20/austin-based-verifiable-raises-3-million-for-its-api-toolkit-to-verify-healthcare-credentials/

在尼克·马卡里奥(Nick Macario)推出可验证的之前，他经营了一系列旨在为专业人士提供公共证书的其他业务，这家总部位于奥斯汀的公司刚刚为其验证医疗证书的 API 工具包筹集了 300 万美元。

他第一次涉足身份管理服务领域是个人网站建设商 branded.me 。该公司被出售后，马卡里奥推出了 Remote.com 的，这是一家外包的人力资源服务提供商，不断进行背景调查和核实员工的资格。

这就是 Macario 获得可验证的想法并抓住市场机会的原因，由于远程医疗和按需服务的激增，以及合格医疗候选人的短缺，以填补职位并满足不断增长的需求。

远程医疗服务的蓬勃发展是 Macario 与联合创始人兼首席技术官 Vivekanand Rajkumar 合作，从包括 Tiger Global、Liquid2 Ventures、strike Capital、Soma Capital、Jack Altman、Max Mullen 和 Sahil Lavingia 在内的投资者那里筹集到 300 万美元的原因之一。

“我们正处于医疗保健的转折点，”马卡里奥说。“有大量的医疗保健验证和认证需要手动验证……基础设施和认证的缺乏是阻碍医疗保健发展的一大瓶颈。”

Verifiable 正在与总部位于区块链的分类账公司 Dock 合作，该公司发布数字凭证并将其锚定到公共分类账。

Verifiable 提供了一个 API，它连接到数百个主要来源，以保存在美国工作的 1700 万个获得许可的医疗保健提供商的最新记录。

像 Talkspace、Sesame 和 Verge Health 这样的公司已经在使用该 API 为超过 50，000 家医疗保健提供商自动进行实时验证。

“从更广的范围来看，我们正在自动化认证过程，但具体来说，我们正在自动化许可验证和监控，”马卡里奥说。

这位可证实的首席执行官估计，由于医疗服务提供商没有跟上执业医师在美国工作所需的资格认证和许可，每年损失数十亿美元的收入和罚款。

“这不是一劳永逸的验证，”马卡里奥说。"你需要每月检查一次，以确保供应商是合规的."

可核实的管理服务可以从 2 美元到 10 美元不等，取决于潜在雇主希望深入到何种程度，以确认他们的从业者的地位和许可。价格基于验证的次数和需要验证的医疗保健提供者的数量。

虽然 Verifiable 是从具体的核查重点开始的，但该公司有更大的愿景。“身份和医疗保健提供商数据是我们最感兴趣的领域。它连接到医疗保健的许多不同领域，”马卡里奥说。
