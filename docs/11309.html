谷歌地图为按需乘车和送货公司推出新的开发者解决方案 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2020/10/22/google-maps-launches-a-new-developer-solution-for-on-demand-ride-and-delivery-companies/

谷歌地图的开发者方谷歌地图平台(Google Maps Platform)今天将为按需乘车和送货公司推出一项新服务，该服务将该平台的一些现有功能与寻找附近司机以及与客户分享行程和订单进度信息的新功能结合在一起。

这并不是谷歌地图平台第一次涉足这项业务。例如，早在 2018 年，该公司就为拼车公司推出了应用内导航解决方案。当时，该团队并没有真正专注于交付解决方案，但由于新冠肺炎疫情，这显然是目前为数不多的蓬勃发展的市场之一。

谷歌高级产品经理伊莱·丹齐格(Eli Danziger)在今天的公告中写道:“基于 15 年的全球地图绘制经验，按需乘车&交付解决方案帮助企业改善运营，并转变司机和客户从预订到抵达或交付的旅程——所有这些都具有可预测的每次行程价格。”。"

这项服务的核心是谷歌地图路线服务，开发者可以对其进行调整，例如通过自行车或摩托车送货，并找到最短或最快的优化路线。该团队指出，这种所谓的“优先路线”功能还可以预测对时间敏感的交付和价格估计的到达时间。

该平台的另一个新功能是使开发者能够快速建立一种帮助用户找到附近司机的体验。被形象地称为“附近的驱动程序”，这个想法就像你能想象的一样简单，允许开发者通过一个 API 调用就能找到最近的驱动程序。他们还可以根据自己的具体需求添加自定义排名，以确保正确的司机与正确的路线相匹配。

不出所料，该平台还支持应用内导航，这与其他功能集紧密相关。

开发人员还可以轻松集成谷歌的实时行程和订单进度功能，以“通过实时查看司机的当前位置、路线和预计到达时间，让客户从取车到送车或送货都了解情况。”

所有这些几乎都是任何用户对现代拼车或送货应用程序的期望，所以在很大程度上，这就是赌注。然而，其背后的技术却并非如此，许多快递公司已经建立了大型的技术运营来构建这些功能。他们不太可能转向谷歌的平台，但该平台可能会给较小的参与者提供一个更有效地运营或进入新市场的机会，而无需从头构建这一技术堆栈的额外费用——或从多家供应商那里拼凑。
