Render 为其 DevOps 平台筹集了 450 万美元 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2020/10/22/render-raises-4-5m-for-its-devops-platform/

Render ，我们 Disrupt SF 2019 初创公司战地的赢家，今天宣布在现有的种子资金轮上再增加 450 万美元，使该公司的总投资达到 675 万美元。

这轮融资由 General Catalyst 牵头，之前的投资者南方公园公共基金(South Park Commons Fund)和一群天使投资人参与，其中包括 Lee Fixel、Elad Gil 和 GitHub CTO(Heroku 的前工程副总裁)Jason Warner。

该公司将自己描述为“AWS、Azure 和谷歌云的零 DevOps 替代品”，最初于 2019 年 4 月筹集了 225 万美元的种子资金，但在赢得 Disrupt 战地后，它获得了大量的入站兴趣。不过，最终该团队决定从现有投资者那里筹集更多资金。

目前的渲染用户包括 Cypress.io ，Mux，Bloomscape，泽洛斯，99designs 和 Stripe。

Render 联合创始人兼首席执行官阿努拉格·戈尔解释说:“Disrupt 之后，我们和很多人谈过，包括阿什顿·库彻的公司，因为他是评委之一。”“最后，我们决定从现有投资者那里筹集更多资金，因为我们喜欢他们，这有助于我们从现有投资者那里获得更好的交易。他们都对继续投资非常感兴趣。”

让 Render 脱颖而出的是，它实现了 Heroku 的许多承诺，也许还有谷歌云的应用引擎。您只需告诉它您将部署哪种服务，它就会为您处理部署和管理基础架构。

“我们的客户都是写代码的人。他们只是想非常容易地部署这些代码，而不必担心服务器、维护或依赖 DevOps 团队，或者在许多情况下，雇用 DevOps 团队，”Goel 说。“雇佣 DevOps 工程师非常昂贵，而且非常难找，尤其是好的工程师。我们的目标是消除开发人员在每家公司做的所有工作，因为在每家公司都非常相似。”

该公司今天推出的一项新功能是预览环境。您可以将它们视为可任意处理的登台或开发环境，开发人员可以对其进行测试，并承诺测试环境将与您的生产环境看起来一样(或者您也可以指定更改)。然后，开发人员可以在这个环境中与 QA 或他们的产品和销售团队合作测试他们的更新。

Render 上的开发团队在 YAML 文件中指定他们的基础设施环境，打开这些新的预览环境就像在该文件中设置一个标志一样简单。

图片来源:渲染

“一旦他们这样做了，那么对于每一个拉取请求——因为我们已经与 GitHub 和 GitLab 集成了——我们会自动创建该环境的副本。这可以包括你在生产中拥有的任何东西，或者像 Redis 实例、托管 Postgres 数据库、Elasticsearch 实例，或者显然是 API、web 服务和静态网站，”Goel 说。每当您将变更推送到分支或拉取请求时，环境也会自动更新。一旦 pull 请求被关闭或合并，Render 会自动破坏环境。

该公司将使用新的资金来发展其团队和扩大其服务。Goel 告诉我，计划是在明年进行更大规模的首轮融资。
