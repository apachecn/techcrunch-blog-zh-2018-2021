Tailscale 为其基于 WireGuard 的企业 VPN 筹集了 1200 万美元

原文：https://web.archive.org/web/https://techcrunch.com/2020/11/10/tailscale-raises-12-million-for-its-wireguard-based-corporate-vpn/

Tailscale 已经筹集了 1200 万美元的资金。Accel 领投，Heavybit 和 Uncork Capital 参与。该公司正在通过利用现代协议并注重实施的简便性来构建一个更好的企业 VPN。

VPN 或虚拟专用网络是两台设备之间的加密隧道。许多公司依靠 VPN 为远程员工、多办公室设置和内部服务提供服务，而这些服务应该只对员工可见。例如，如果您正在远程工作，那么您可以通过从公司的笔记本电脑连接到 VPN 服务器来连接到公司的内部网和内部服务。

在过去的几年中，访问公司内部网络有多种趋势。一些公司依赖复杂的访问策略。谷歌已经通过其 BeyondCorp 零信任系统走上了这条路。

其他公司仍然依赖企业 VPN 和防火墙，因为它们易于实施。他们通常使用带有 VPN 网关的 IPsec 协议来处理与内部网络的连接。

如果你最近一直在远程工作，你可能已经注意到这种传统的 VPN 设置不能很好地扩展。网关是一个瓶颈，当有很多人同时连接时，你会经历很长的加载时间。

回到 Tailscale，这家初创公司正试图实现企业 VPN 的现代化。它从不同的 VPN 协议开始。Tailscale 选择了 WireGuard ，这是一种轻量级的 VPN 协议，它依靠公钥和私钥的组合在两个客户端之间建立加密隧道。

但是 Wireguard 本身只是一个协议。它没有告诉你应该如何处理公钥，如何向网络添加新设备等等。Tailscale 就像胶水一样把所有的碎片粘在一起。

“在架构上，我将 Tailscale 描述为控制平面，WireGuard 是数据平面，”联合创始人兼首席执行官 Avery Pennarun 告诉我。

图片来源: Tailscale

我们举个例子。您的公司有一个内部 Git 服务器和一个内部文档 wiki。您有一台公司笔记本电脑，并且想要访问这两项服务。您可以在三台不同的机器上安装 Tailscale 客户端——您的笔记本电脑、Git 服务器和 wiki 服务器。

当你想连接内部服务时，Tailscale 要求你使用你公司的身份提供者登录，如 G Suite、Okta、Active Directory 等。

所有 Tailscale 客户端检查一个协调服务器，以查看连接是否得到授权。“这是一个公共密钥的投件箱，”Pennarun 说。当有人离开公司时，公钥会从协调服务器中删除，Tailscale 不再工作。为了提高安全性，密钥会定期轮换。

然后在您的笔记本电脑和 Git 服务器或者您的笔记本电脑和 wiki 服务器之间建立连接。由于 VPN 网关没有瓶颈，因为 Git 服务器和 wiki 服务器充当它们自己的 VPN 网关。没有必要将你的文档 wiki 暴露在互联网上，因为员工首先使用 Tailscale 访问服务器。

您不必打开服务器上的 SSH 端口，因为 Tailscale 可以找到一种方法来通过防火墙建立连接。

这家公司仍然很小，但效率很高。Tailscale 拥有大约 20 名员工，每月安装数万台客户端。

您可以通过单个用户和多个设备免费开始使用。一些用户已经在家里用树莓派尝试过，这样他们就可以在旅途中连接到他们的本地网络。他们后来把它带到工作中。

通过保持非常低的客户获取成本，Tailscale 成功筹集了 1200 万美元。 Twingate 是另一家试图解决同样问题的公司，但它做出了不同的技术选择——他们依赖于 TLS 隧道和中继。
