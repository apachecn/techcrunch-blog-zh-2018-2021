Databricks 发布 SQL 分析

原文：https://web.archive.org/web/https://techcrunch.com/2020/11/12/databricks-launches-sql-analytics-builds-itself-a-lake-house/

人工智能和数据分析公司 Databricks 今天宣布推出 SQL Analytics，这是一种新的服务，使数据分析师可以更容易地直接在数据湖上运行他们的标准 SQL 查询。因此，企业现在可以轻松地将 Tableau 和微软的 Power BI 等商业智能工具连接到这些数据存储库。

SQL Analytics 将于 11 月 18 日公开预览。

在许多方面，SQL Analytics 是 Databricks 长期以来一直寻求打造的产品，它将“湖边小屋”的概念带入了生活。它将数据仓库的性能与数据湖的性能结合起来，在数据仓库中，您可以存储经过转换和清理的数据，在数据湖中，您可以以原始形式存储所有数据。数据湖中的数据是 Databricks 的联合创始人兼首席执行官阿里·古迪(Ali Ghodsi)长期倡导的概念，通常只有在被使用时才会被转化。这使得数据湖更便宜，但对用户来说也更难处理。

图片来源:数据块

“我们一直在说统一数据分析，这意味着将数据与分析统一起来。因此，数据处理和分析，这两个应该合并。但是没有人明白这一点，”古德西告诉我。但是“湖边小屋”作为一个术语流行了起来。

“Databricks 一直提供数据科学和机器学习。我们已经讨论这个问题很多年了。通过 Spark，我们提供了数据处理能力。可以做【提取，转换，加载】。这一直是可能的。SQL Analytics 使您现在能够直接处理数据仓库工作负载，具体来说，就是直接在数据湖上处理商业智能和报告工作负载。”

这里的总体想法是，只需一份数据副本，您就可以支持传统的数据分析师用例(想想 BI)和数据科学工作负载(想想 AI)。理想情况下，这使得两种用例都更便宜、更简单。

该服务位于 Databricks 开源 Delta Lake 存储层的优化版本之上，使该服务能够快速完成查询。此外，Delta Lake 还提供自动扩展端点，以保持查询延迟一致，即使在高负载下也是如此。

虽然数据分析师可以使用标准 SQL 直接查询这些数据集，但该公司还构建了一套连接 BI 工具的连接器。其 BI 合作伙伴包括 Tableau、Qlik、Looker 和 ThoughtSpot，以及 Fivetran、Fishtown Analytics、Talend 和 Matillion 等摄取合作伙伴。

图片来源:数据块

Tableau 首席产品官 Francois Ajenstat 表示:“现在，组织比以往任何时候都更需要一种能够适应速度和敏捷性的数据策略。“随着组织迅速将他们的数据转移到云中，我们看到人们对在数据湖上进行分析的兴趣越来越大。SQL Analytics 的引入为客户提供了一种全新的体验，让他们能够从海量的数据中获得所需的性能、可靠性和规模。”

在一个演示中，Ghodsi 向我展示了新的 SQL Analytics 工作区的外观。它本质上是标准代码密集型体验的精简版本，Databricks 用户对此很熟悉。不出所料，SQL Analytics 提供了更图形化的体验，更侧重于可视化而不是 Python 代码。

虽然 Databricks 平台上已经有一些数据分析师，但这显然为该公司开辟了一个巨大的新市场——这肯定会支持其明年的 IPO 计划。
