OpsLevel 筹集 500 万美元修复 DevOps

原文：https://web.archive.org/web/https://techcrunch.com/2020/11/18/opslevel-raises-5m-to-fix-devops/

DevOps 初创公司 OpsLevel 背后的团队认为,“DevOps”一词已经变得毫无意义，开发人员仍然无法获得正确的工具来将整体想法付诸实践。这家由 PagerDuty 最早的两名员工约翰·拉班(John Laban)和肯尼斯·罗斯(Kenneth Rose)共同创立的公司今天宣布，它已经筹集了 500 万美元的种子资金，由 Vertex Ventures 领投。S28 资本、Webb 投资网和 Union Capital 也参与了这一轮，还有多位天使，包括 PagerDuty 的三位联合创始人。

“[page duty]是 DevOps 运动的重要组成部分。让工程师随叫随到对于 DevOps 来说非常重要，但是随叫随到和收到关于事件和事情的传呼，本质上是非常被动的。这一切都是为了尽快修复事故，”拉班说。“Ken [Rose]和我看到了帮助公司采取更积极姿态的机会。首先，没有人真的希望有任何停机时间或任何安全漏洞。他们想防患于未然。”

图片来源: OpsLevel

铭记这一使命，该团队开始将工程组织带回 DevOps 的根源，让这些团队拥有对其服务的所有权，并创建 Rose 所说的“您构建它，您拥有它”的文化。他指出，服务所有权是该团队经常看到公司努力解决的问题。当团队为他们的系统迁移到微服务甚至无服务器架构时，谁拥有什么很快就变得不清楚了，结果，你最终得到了无人维护的孤立服务。这样做的自然结果是安全性和可靠性问题。同时，因为没有人知道哪些系统已经存在，所以其他团队重新发明轮子，重新构建相同的服务来解决他们自己的问题。

“我们在让 DevOps 实际工作的工具上投资不足，”该团队在今天的声明中说。“要帮助工程团队采用服务所有权并释放 DevOps 的全部力量，我们还有很多工作要做。”

因此，OpsLevel 的核心是团队所说的“服务所有权平台”，从工程组织当前运行的服务目录开始。

图片来源: OpsLevel

“我们试图做的是收回 DevOps 的含义，”拉班说。“我们认为它已经变得毫无意义，我们希望重新关注服务所有权。我们将投入巨资打造我们的产品，然后与我们的客户合作，让他们真正拥有自己的服务，并真正着手解决这个问题。”

OpsLevel 已经与之合作的公司有 Segment、Zapier、Convoy 和安德玛。正如该团队所指出的，一旦一家公司运行大约 20 或 30 种不同的服务，它的服务就变得最有用。在此之前，维基或电子表格通常足以管理它们，但在这一点上，这些系统往往会崩溃。

OpsLevel 给了他们不同的 onramps 来开始编目他们的服务。如果他们喜欢使用“配置代码”的方法，他们可以使用这些 YAML 文件作为他们现有 Git 工作流的一部分。但是 OpsLevel 提供了 API，如果团队已经有了现有的服务创建工作流，他们可以将这些 API 插入到他们的各种系统中。

该公司的融资轮于 9 月底结束。该团队表示，疫情并没有真正阻碍其筹资努力，这是我最近从许多公司那里听到的(尽管我与之交谈的公司显然倾向于那些最近筹集到资金的公司)。

拉班说:“(我们筹集资金的)原因是我们想真正投资建设我们的产品。”。“我们的客户一直在吸引我们，我们真的希望加倍努力，开发出大量产品，并投资于我们的上市团队，我们真的希望加快速度。”
