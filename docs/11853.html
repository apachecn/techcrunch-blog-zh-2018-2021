谢谢你，Chrome 团队

原文：https://web.archive.org/web/https://techcrunch.com/2020/11/19/thank-you-chrome-team/

自从 Chrome 于 2008 年问世以来，它一直是我生活中不可或缺的伙伴。事实上，Chrome 的发布帮助我第一次将我当时工作的初创公司推上了 TechCrunch】。

我们拍照庆祝。铬摇滚，我们是第一天的球迷。

但随着时间的推移，曾经的浪漫开始变得糟糕，因为 Chrome 变得越来越慢，越来越重，越来越差。

这种权力下放感觉有点像正在发生在谷歌搜索上的事情，一个非常好的想法慢慢变成了以功能、速度和用户满意度为代价赚更多钱的东西(更多关于这种进展的自然终点在这里)。

因为我是一个任性的孩子，我一直对 Chrome 的遭遇感到非常恼火，这个软件我从来没有花过一分钱去使用。为了证明这一点，我出去收集了一两条我自己抱怨 Chrome 这些年的推文，但自从 5 月份以来发现了至少 9 个例子后，我开始感觉不好(一个，两个，三个，四个，五个，六个，七个，八个，九个所以我们继续吧。

Chrome 出了什么问题？我不知道。随着时间的推移，它对内存、延迟和烦人的习惯越来越强烈。但是当我生活在一个 G 套件的世界里，坚持使用 Chrome 是有意义的——所以我忍受了。

现在，我可能再也不用了。本周，谷歌详细介绍了即将推出的一系列 Chrome 更新，这些更新令人惊叹，可以通读并想象其对现实世界的影响。Big Goog 似乎已经深入其浏览器的代码，找到了让它更快、更轻内存使用和更智能的方法。

我非常兴奋。

什么来了？从谷歌的 Chromium 博客中抽出，而不是它的更为用户友好的帖子(这非常感谢 The Verge 让我注意到这组更新)，这里是我所关注的亮点(粗体:每个区块引用中的 TechCrunch):

即使你打开了很多标签页，你也可能只关注其中的一小部分来完成任务。从这个版本开始，Chrome 会主动管理你电脑的资源，让你关心的标签页变得更快——，同时允许你保持数百个标签页打开——这样你就可以从你离开的地方重新开始。

在这个版本中，我们正在通过选项卡节流、遮挡跟踪和后退/前进缓存来改进 Chrome 理解和管理资源的方式，以便您可以在需要时快速获得您需要的内容。

谷歌这是字面上的我。我觉得被人看见了。谢谢你。

我们调查了后台标签如何使用系统资源，发现 JavaScript 计时器占后台标签工作的 40%以上。降低它们对 CPU 和电源的影响对于提高浏览器效率非常重要。从 M87 开始，我们将后台标签中的 JavaScript 定时器唤醒限制为每分钟一次。在我们的内部测试中，这将 CPU 使用率降低了 5 倍，并将电池寿命延长了 1.25 小时。

当世界再次运转时，我想为所有参与这项努力的人买午餐。

接下来，我们将把遮挡追踪——之前添加到 Chrome OS 和 Mac 上——带到 Windows 上，这使得 Chrome 可以知道哪些窗口和标签页实际上对你是可见的。有了这些信息， Chrome 可以优化你正在使用的标签页的资源，而不是你已经最小化的标签页，使 Chrome 启动速度加快 25%，加载页面速度加快 7%，同时使用更少的内存。

当然了。

有多少次你访问一个网站，点击一个链接去另一个页面，只是意识到这不是你想要的，然后点击后退按钮？[…] 在 Chrome 87 中，我们的后退/前进缓存将使 20%的后退/前进导航即时完成，并计划在不久的将来通过进一步的改进和开发者拓展将这一比例提高到 50%。

我甚至不知道我需要这个，但我需要。我迫不及待想拥有它。

总而言之，当我在 Chrome 内部写这篇短文给你时，我不禁对新的和改进的 Chrome 感到兴奋。更多以后我得到一些测试，但是，老实说，耶！
