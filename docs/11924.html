Splunk 收购网络观察服务 Flowmill

数据平台 Splunk 继续进行收购，努力构建其最近推出的观察平台。继上个月收购了 Plumbr 和 Rigor 之后，该公司今天宣布已经收购了 Flowmill ，一家位于帕洛阿尔托的网络观察初创公司。Flowmill 专注于帮助其用户实时发现其云基础设施中的网络性能问题，并按服务测量其流量，以帮助他们控制成本。

像现在这个领域的许多其他公司一样，Flowmill 利用 eBPF——Linux 内核相对较新的功能——在内核中运行沙盒代码，而无需更改内核或加载内核模块。这使得它非常适合监控应用。

“可观察性技术在复杂性和能力方面都在快速增长，帮助组织彻底改变他们监控基础架构和应用程序的方式。Splunk 首席技术官 Tim Tully 表示:“Flowmill 创新的 NPM 解决方案利用扩展的 Berkeley 包过滤(eBPF)技术，提供对分布式云应用的网络行为和性能的实时观察。“随着 Splunk 继续为我们的客户提供一流的观察能力，我们很高兴将 Flowmill 富有远见的 NPM 技术引入我们的观察套件。”

虽然 Spunk 已经进行了一些更大的收购，包括以 10.5 亿美元收购 SignalFx，但它正在通过收购提供非常特定功能的小型初创公司来建立自己的可观察性平台。它可能会在内部构建所有这些功能，但该公司显然认为，随着企业在更新其技术堆栈时寻求新的观察工具，它必须快速行动，才能在这个不断增长的市场中获得立足点。

“Flowmill 构建支持全保真、实时、高基数摄取和分析的系统的方法与 Splunk 的可观察性愿景非常吻合，”Flowmill 首席执行官乔纳森·佩里说。“我们很高兴加入 Splunk，并将新一代 eBPF 引入 Splunk Observability Suite。”

这些公司没有透露收购价格，但 Flowmill 之前从 Amplify、Felicis Ventures、WestWave Capital 和 UpWest 筹集了资金。
