加利福尼亚州的 CA Notify 应用程序使用苹果和谷歌的框架提供全州范围的风险通知

加利福尼亚州现已将其 CA Notify 应用程序扩展到该州所有人，此前该应用程序最初于 11 月在加州大学伯克利分校的试点项目中部署，后来扩展到加州大学其他校区。加州州长加文·纽瑟姆周一宣布在全州范围内推出该应用程序，这意味着从 12 月 10 日周四开始，任何拥有兼容 iPhone 或 Android 设备的人都可以下载和选择使用基于苹果和谷歌曝光通知 API 的工具。

苹果和谷歌联合开发的暴露通知 API 使用蓝牙来确定已确认的 COVID 阳性个体与其他人之间的接触，提醒用户潜在的暴露，而不存储或传输任何与其身份或位置相关的数据。该系统使用一个随机的滚动标识符来传达与其他设备的可能接触，各州卫生当局可以定制具体的细节，如个人需要接触多近和多长时间，才能符合接触风险的资格。

在加利福尼亚州，该州已将与确诊的新冠肺炎阳性个人在 6 英尺范围内接触 15 分钟或更长时间定为值得曝光通知。接受新冠肺炎检测呈阳性的用户将收到一条来自该州公共卫生部的短信，其中包含他们在 CA Notify 应用程序中输入的代码，以便在之前的 14 天(病毒可传播的时期)内向符合上述标准的任何手机触发警报广播。

如前所述，用户的设备不会通过通知系统传输任何个人信息，这是一种完全自愿的安排。其他州已经部署了基于苹果/谷歌 API 的暴露通知应用，世界上许多其他国家也是如此。它不能取代接触追踪系统，在接触追踪系统中，医疗专业人员试图确定新冠肺炎患者与谁接触，以了解他们如何感染病毒，以及他们可能将病毒传播给谁，但它是全面追踪计划的重要组成部分，可以提高其效率和成功率。
