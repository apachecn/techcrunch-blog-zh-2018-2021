Cloudflare 正在测试 Netlify 竞争对手托管 Jamstack 站点的能力

原文：https://web.archive.org/web/https://techcrunch.com/2020/12/07/cloudflare-is-testing-a-netlify-competitor-to-host-jamstack-sites/amp/

图片来源:迈克尔·肖特/彭博/盖蒂图片社

Cloudflare 正在开发一款名为 Cloudflare Pages 的新产品。新产品可以直接与 Netlify 和 Vercel 竞争，这两家云托管公司允许你使用 Jamstack 框架构建和部署网站。流行的 Jamstack 框架有 Gatsby、Jekyll、Hugo、Vue.js、Next.js 等。

这款新产品是由逆向工程师简·满春·王在周六发现的，她通过研究 Cloudflare 的代码找到了细节。

如果您不熟悉 Jamstack，这是一种大规模开发和部署网站的流行方式。它让您能够利用全球边缘网络，重点关注性能。

假设你是一个非常著名的流行歌手，明天你将在你的网站上发布你的新专辑。你期望获得巨大的流量峰值和大量订单。

如果你碰巧也是一名 web 开发人员，你可以开发一个 Jamstack 网站，以确保你的网站保持可用并快速加载。Jamstack 框架让您可以将前端与后端分离，而不是使用传统的内容管理系统来托管内容并将其交付给用户。

你可以在你的内容管理系统中写一些文章，然后部署你的网站更新。您的 Jamstack 应用程序将根据您刚刚编写的内容预构建静态页面。这些网页将被缓存在一个全球边缘网络上，并在几毫秒内送达世界各地。这就像复印一封信，而不是每次有人想看的时候都写一份新的副本。

但是如果有人想在购买页面上购买你的新专辑呢？在那个页面上，会有一个 checkout 模块，是不能缓存的动态内容。你可以利用支付 API，比如 Stripe，这样用户就不用从你的服务器上下载内容了。

这是一个简单的例子，但是很多公司一直在走这条路。对于静态内容，一切都是预先构建和缓存的。对于动态内容，公司构建按需加载且易于扩展的微服务。

根据简·满春·王的截图，Cloudflare Pages 允许你通过一个简单的 Git commit 来部署站点。如果您在平台上托管源代码，它可以直接与您的 GitHub 库集成。

您可以配置一个 Node.js 构建命令，该命令将在您每次更改代码时执行。一旦构建过程完成，您的最终用户就可以访问您的网站。

Cloudflare 页面将有一个免费层，让您每月生成 500 个构建。超过这个限额，Cloudflare 很可能会让您为更多的构建和更多的功能付费。

默认情况下，Cloudflare 会给你一个子域名来试用该服务——your-website-name . Pages . dev，当然，你也可以配置自己的域名，将 Cloudflare 页面与其他 Cloudflare 产品结合使用。

你已经可以阅读 Cloudflare Pages 的文档，由简·满春·王发现。因此，听起来 Cloudflare Pages 可以尽快推出。
