Cloudflare 让您可以控制数据的存储和访问位置

原文：https://web.archive.org/web/https://techcrunch.com/2020/12/07/cloudflare-lets-you-control-where-data-is-stored-and-accessible/amp/

图片来源:乔丹·哈里森/ Unsplash

Cloudflare 今天推出了一组新功能，称为数据本地化套件。企业计划中的公司可以选择通过附加组件来启用这些功能。

借助 Data Localization Suite，Cloudflare 可以更轻松地控制数据的存储位置，以及您是否有权根据访问位置查看数据。该功能使您可以利用 Cloudflare 的产品，如无服务器基础架构，同时遵守当地和行业特定的法规。

例如，在今年的欧盟裁决结束了隐私保护之后，数据本地化套件显得尤为重要。如果您在高度管控的行业中运营，如医疗保健和法律行业，您可能也有一些特定的数据要求。

假设您正在构建一个应用程序，该应用程序应该专门存储欧盟的数据。您可以选择在单个数据中心或单个云区域运行您的应用程序。但是，如果你希望在世界各地都有客户，那就不太好了。你也可能遭受停电。

使用 Cloudflare 的方法，所有内容在静态和传输过程中都进行了加密(如果您强制执行 TLS 加密)。您可以选择自己管理您的私钥，也可以选择为您的私钥设置不同的规则。

例如，让您检查流量的私钥可能只能从欧洲数据中心访问。既然隐私保护无效，这种设置更容易符合欧洲法规。

Cloudflare 会检查网络请求，以了解如何处理这些请求。例如，该公司试图自动拒绝恶意的机器人请求。您可以选择在特定区域检查这些请求。如果一个恶意机器人正在美国的服务器上运行，该请求将被发送到美国最近的 Cloudflare 数据中心，路由到欧洲的数据中心，然后进行检查。

至于流量日志和元数据，您可以使用 Edge Log Delivery 将日志从 Cloudflare 的边缘网络直接发送到存储桶或本地数据中心。它根本不经过 Cloudflare 的核心数据中心。

最后，如果你使用的是最近发布的cloud flare Workers 持久对象，你可以配置权限限制。如果您在 Cloudflare 的无服务器基础架构上运行应用程序，出于监管目的，您可以选择避免在某些位置存储持久对象。

如您所见，数据本地化套件中有几个工具和服务。其中一些已经上线，另一些则是全新的。但有趣的是，尽管 Cloudflare 认为无服务器计算和边缘数据中心是未来的趋势，但它也在考虑本地性。
