Adobe的文档服务使开发人员更容易使用pdf

在过去的一年里，Adobe悄悄地继续扩展它的工具，帮助开发者在他们的应用程序中使用pdf。今年4月，该公司推出了几个SDK，例如现在被称为PDF Embed API和PDF Tools API的，随着这一更新，该公司还推出了其 Adobe文档服务平台。这里的想法是为开发人员提供易于使用的工具，将pdf构建到他们的应用程序和工作流程中。今天，该公司宣布与微软建立新的合作伙伴关系，将文档服务引入微软的低代码工作流自动化平台Power Automate。

“大约一年半前，我们就有了这个愿景，当时我们说，‘把我们自己应用程序中最好的东西也带到第三方应用程序中，怎么样？’Adobe文档服务营销高级总监Vibhor Kapoor告诉我。“这是一种简单的思维模式，我们说:让我们将Acrobat的功能分解为微服务[和APIs，并将其提供给开发人员和发行商，因为坦率地说，开发人员和发行商的PDF一直是缺乏更好的词的痛苦。所以我们将这些服务带入生活。”

例如，该团队致力于将pdf更好地嵌入到web体验中(Kapoor坦率地指出，以前，开发人员的体验总是“非常次优”，用户体验也不总是直观的)。现在，有了文档服务和嵌入API，嵌入PDF只需要几行JavaScript代码。

图片来源:土坯

Kapoor承认在SDK和API中公开这些特性是一个挑战，因为团队最初不必担心这个用例。但是除了技术上的挑战，这也是一个改变整体心态的问题。“我们过去从来没有非常面向开发者的产品，这意味着我们需要建立一个了解开发者的团队，并弄清楚我们如何打包这些API并使它们可用，”他指出。

新的Power Automate集成将PDF Tools API中的20多个新的以PDF为中心的操作带到了微软的平台上。这将允许用户从OneDrive文件夹中的文档创建pdf，例如，将图像转换为pdf或对pdf应用光学字符识别。

自从Adobe推出这些平台以来，大约有6000名开发人员已经开始使用它，Kapoor告诉我，他看到了API调用数量的“显著增长”。从商业角度来看，添加Power Automate也可能成为吸引新开发人员的新渠道。
