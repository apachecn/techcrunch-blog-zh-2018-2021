Cloudflare 推出了 Cloudflare Pages，这是一个部署和托管 JAMstack 站点的平台

原文：https://web.archive.org/web/https://techcrunch.com/2020/12/17/cloudflare-launches-cloudflare-pages-a-platform-to-deploy-and-host-jamstack-sites/amp/

图片来源:迈克尔·肖特/彭博/盖蒂图片社

Cloudflare 今天推出一款名为 Cloudflare Pages 的新产品。它直接与 Netlify 或 Vercel 竞争，这两家云托管公司允许你使用 JAMstack 框架构建和部署网站。流行的 JAMstack 框架有 Gatsby、Jekyll、Hugo、Vue.js、Next.js 等。

如果这个声明听起来很熟悉，那是因为逆向工程师简·满春·王通过查看 Cloudflare 的代码泄露了关于即将推出的产品的细节。几周前，我报道了这次泄漏。

Cloudflare 联合创始人兼首席执行官马修·普林斯(Matthew Prince)告诉我说:“你看到了简的推特，她有点早泄露了秘密，我觉得这有点太棒了。”。

但现在，Cloudflare Pages 真正上线了。它与 Git 存储库和现有的 JAMstack 框架无缝集成。当您将站点的新版本推送到 Git 存储库时，Cloudflare 会自动开始在其网络上构建和部署您的站点。

如果您不熟悉 JAMstack，这是一种大规模开发和部署网站的流行方式。它让您能够利用全球边缘网络，重点关注性能。

JAMstack 框架可以让你把网站的前端和后端分离开来。每次你改变你的站点，整个前端都是预先构建的。您的页面被转换为优化的静态页面，可以在全球边缘网络上托管和缓存。通过这种方式，页面在几毫秒之内就可以送达世界各地。

JAMstack 网站不必是 100%静态的。您可以使用流行的基于 API 的服务中的 API，比如 Stripe。当网页在你的网络浏览器中呈现时，支付模块直接从 Stripe 的服务器加载。您还可以构建自己的微服务，按需加载并轻松扩展。

回到 Cloudflare Pages，如果您在该平台上托管源代码，它可以与 GitHub 存储库集成。一旦您的站点配置完成，您就可以从 Cloudflare 的界面预览每个提交—每个提交都有自己唯一的 URL，并且有一个预览环境。

您可以通过邀请其他 Cloudflare 用户加入您的 Pages 项目来与他们进行协作。当您的网站在预览分支中看起来不错时，您可以将其推送到生产分支。

当不认为自己是开发者的人想要使用 Cloudflare 时，Cloudflare 开始在页面上工作。“创意方面的人告诉我们，‘嘿，你有兴趣建一个网站吗？’”王子说。他认定 Wix 和 Squarespace 是网站建设者。

与此同时，Cloudflare 注意到 JAMstack 变得相当流行。“许多 Netlify 和 Vercel 的客户实际上都将 Cloudflare 放在他们的 Netlify 或 Vercel 站点前面，”他补充道。当涉及到成本和性能时，这实际上没有意义，因为您正在堆叠多个 cdn。

由于 Cloudflare 已经在世界各地拥有庞大的服务器网络，所以在其边缘网络上托管 JAMstack 网站并不需要花费太多的钱，这比缓存 Netlify 托管的内容更便宜。这就是为什么有一个慷慨的免费层，有无限的带宽和无限的请求。

“我们有意将发布时间定在每年的这个时候。我们真的想给开发者社区一份礼物，”普林斯说。

该公司希望通过今天的发布获得更多关注。到明年 1 月，这项服务每天将增加 1000 人。之后，它会向所有人开放这项服务。

Cloudflare 认为您会为 JAMstack 项目选择页面，因为它的免费层和健壮的基础设施。但该公司也希望用户在未来尝试其他 Cloudflare 服务。

例如，如果你想超越静态网站，Cloudflare 提供了一个名为 Cloudflare Workers 的无服务器平台。您可以在 Workers 上部署一些代码，并在您的 Pages 站点中使用它。有了 Workers KV 和持久对象，您也可以超越无状态函数。

同样，当您开始使用 Pages 时，您可能会对 Cloudflare 的高级安全功能和其他 Cloudflare 产品感兴趣。该公司一直押注于开发者的长尾来寻找下一个客户，Pages 是这种策略的另一个例子。
