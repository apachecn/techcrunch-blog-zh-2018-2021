Twitter的新API平台现在向学术研究人员开放

原文：https://web.archive.org/web/https://techcrunch.com/2021/01/26/twitters-new-api-platform-now-opened-to-academic-researchers/

Twitter今天在它的API平台上推出了一个新产品轨道，这是T2从头开始重建Twitter API的努力的一部分。该跟踪旨在服务于学术研究社区的努力，提供了更广泛的Twitter档案访问和更少的Twitter检索限制，因此研究人员可以在Twitter的平台上访问公共对话的整个历史。

除了获得迄今为止发布的所有Twitter API v2端点的访问权限和提升的访问权限，研究人员还将获得更精确的过滤功能。

具体来说，他们将能够访问全存档搜索端点，这提供了对Twitter上所说的一切的访问。他们可以使用开始时间和结束时间参数来缩小这些历史推文的搜索范围。

图片来源:推特

研究人员还将获得使用Twitter API v2的更高的每月推文数量上限。在API访问的基本级别上，这个上限设置为500，000。学术研究轨道的基本访问水平是每月最多1000万条推文。Twitter表示，这适用于最近搜索、过滤流、全存档搜索和用户推文，并提到了时间线端点。

学术研究轨道将获得某些运营商，否则也无法获得，目标是帮助他们拉更精确的用户数据。今天这些包括: $(又名cashtag)，bio，bio_name，bio_location，place，place_country，point_radius，bounding_box，-is:nullcast，has:cashtags和has:geo。

当使用过滤器流端点时，研究人员还可以添加1，000个并发规则，而不是标准跟踪中可用的25个限制。与标准跟踪中的512个字符相比，最近搜索端点中的查询可以长达1，024个字符。

由于访问级别提高，那些想要获得学术研究产品跟踪的人必须首先提交申请。

所有申请人必须是学术机构或大学的硕士生、博士生、博士后、教员或研究型员工。他们还需要有一个明确的研究目标，必须能够详细说明他们打算如何使用、分析和分享他们研究中的Twitter数据的具体计划。

此外，Twitter指出，学术研究产品跟踪中使用的数据不能用于任何商业目的。

图片来源:推特

自2006年首次推出以来，学术研究人员一直在利用Twitter API，并利用这些数据研究各种主题，Twitter表示，如 国家支持的扰乱公共对话的努力、洪水和气候变化、对新冠肺炎的态度和看法以及促进在线健康对话的努力。

然而，Twitter API的早期版本并不便于研究人员访问Twitter数据——该公司希望通过API v2来纠正这一点。

迄今为止，Twitter一直在以其他方式迎合研究社区，增加了专门从事学术研究的网站，更新了开发者政策，使其更容易复制和验证他人的研究，甚至还有特殊的端点，如2020年4月发布的新冠肺炎流端点。但直到API v2，它还没有完全考虑如何建立工具，实际上帮助研究人员做他们的工作，而不是研究人员必须找出解决Twitter限制的方法。

学术研究产品跟踪在2020年10月开始的私人测试中进行了测试，现在正在更广泛地开放，将免费提供。

Twitter表示，它计划在未来的所有产品轨道上增加更高级别的访问权限，包括这个。后面的水平将帮助那些需要比今天的发射所提供的更多数据的研究人员。Twitter还指出，它也在考虑增加灵活的访问，这将有助于考虑开发人员全年消耗或多或少数据的时间。
