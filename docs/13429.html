微软的 Dapr 开源项目帮助开发者构建云原生应用 hits 1.0

原文：https://web.archive.org/web/https://techcrunch.com/2021/02/17/microsofts-dapr-open-source-project-hits-1-0/

微软孵化的开源项目 Dapr ，旨在让开发者更容易构建事件驱动的分布式云原生应用，今天达到了 1.0 里程碑，标志着该项目为生产用例做好了准备。微软早在 2019 年 10 月就推出了分布式应用运行时(这就是“Dapr”的意思)。从那时起，该项目发布了 14 个更新，社区启动了与几乎所有主要云提供商的集成，包括 Azure、AWS、阿里巴巴和谷歌云。

微软 Azure CTO Mark Russinovich 告诉我，Dapr 的目标是为企业开发者实现云原生开发的民主化。

“当我们去看看企业开发人员被要求做什么时，他们传统上一直在做客户端、服务器、web 和数据库类型的应用程序，”他指出。“但现在，我们要求他们容器化，创建可横向扩展且无停机更新的微服务，而且他们必须与所有这些云服务集成。除此之外，许多企业还要求他们开发可在内部环境和云环境之间移植的应用程序，甚至能够在云环境之间移动。因此，大量的复杂性被抛给了他们，而这些复杂性与他们试图解决的业务问题并不相关或不特定。”

许多开发都涉及到重新发明轮子，以使他们的应用程序可靠地与各种其他服务对话。Dapr 背后的想法是为开发人员提供一个单一的运行时，开箱即用，提供开发人员构建事件驱动的微服务所需的工具。除此之外，Dapr 还为服务对服务通信、状态管理、发布/订阅和机密管理等提供了各种构建模块。

“Dapr 的目标是:让我们远离开发人员，关注编写这些云原生分布式、高度可用、可扩展、安全的云服务的所有日常工作，以便他们可以专注于自己的代码。实际上，我们从无服务器和功能即服务中吸取了经验教训，例如 Azure Functions 是事件驱动的，他们专注于自己的业务逻辑，然后 Azure Functions 附带的绑定之类的东西负责与其他服务连接，”Russinovich 说。

他还指出，这里的另一个目标是消除特定于语言的模型，并创建一个可以从任何语言中利用的编程模型。毕竟，企业倾向于在他们现有的代码中使用多种语言，他们中的许多人现在都在考虑如何最好地更新他们现有的应用程序——而不抛弃他们当前所有的代码。

正如 Russinovich 指出的，该项目现在有超过 700 名微软以外的贡献者(尽管核心通勤者主要来自微软)，许多企业在 1.0 版本之前就开始在生产中使用它。已经在使用它的一个较大的云提供商是阿里巴巴。“阿里云真的爱上了 Dapr，并正在大力利用它，”他说。为 Dapr 做出贡献的其他组织包括 HashiCorp 和早期用户，如 ZEISS、Ignition Group 和 New Relic。

虽然云提供商对其竞争对手已经在使用其创新感到高兴似乎有点奇怪，但 Russinovich 指出，这正是计划，该团队希望很快将 Dapr 纳入基础。

“几个月来，我们一直走在开放治理的道路上，目标是让这成为一个基础。[……]目标是打开这一局面。这不是微软的事情。这是一个行业的事情，”他说——但他还没准备好说出这个团队正在与哪个基金会进行谈判。
