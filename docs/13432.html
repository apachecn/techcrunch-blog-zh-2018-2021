Spectral 为其 DevSecOps 服务 TechCrunch 筹集了 620 万美元

原文：https://web.archive.org/web/https://techcrunch.com/2021/02/17/spectral-raises-6-2m-for-its-devsecops-service/

总部位于特拉维夫的 Spectral 今天将推出其新的 DevSecOps 代码扫描仪，并宣布获得 620 万美元的融资。这家初创公司的编程语言无关服务旨在自动化代码安全开发团队，帮助他们检测代码库和日志中的潜在安全问题。这些问题可能是硬编码的 API 密钥和其他凭证，也可能是安全错误配置和影子 IT 资产。

四人创始团队在构建 AI、监控和安全工具方面有着深厚的背景。例如，首席执行官多坦·纳胡姆是 Klarna 和 Conduit(现在是科莫，尽管你可能还记得 Conduit 在臭名昭著的工具栏，后来脱离)的首席架构师，也是科莫和 HiredScore 的首席技术官。其他创始人在 Elastic 和 HP 开发监控工具，在 Akamai 开发安全产品。正如纳胡姆告诉我的那样，Spectral 的想法是他和联合创始人首席运营官·伊丹·迪迪在移动应用 build Conduit/Como 的共享时间里产生的。

“我们基本上为我们的每个客户存储了证书，所以我们可以将他们的应用程序提交到各种市场，”Nahum 告诉我他在 Counduit/Como 的经历。“那个证书真的证明了你就是你，而且超级敏感。在这些公司的每个环节，我真的没有合适的工具来确保我们正在存储、处理、检测[这些信息]，并确保它不会泄露到任何地方。”

纳胡姆决定辞去目前的工作，开始构建一个原型，看看他能否构建一个可以解决这个问题的工具(他在这个原型上的工作很快发现了 Slack 的一个问题)。随着企业从内部软件转向云、微服务和开发运维，对更好的 DevSecOps 工具的需求只会越来越大。

“重点是创造一个伟大的开发者体验，”Nahum 指出。“因为那是我们的起点。我们并不是从自上而下的网络工具开始的。我们一开始是一个适度的 DevOps 友好、开发人员友好的工具。”

Spectral 方法的一个有趣方面是，它使用机器学习模型来检测跨编程语言的这些漏洞，它还扫描面向公众的系统。在后端，Spectral 与 Travis、Jenkins、CircleCI、Webpack、Gatsby 和 Netlify 等工具集成，但它也可以监控 Slack、npm、maven 和日志提供商——大多数公司在考虑威胁建模时不会真正考虑这些工具。

“我们的解决方案每天都能防止安全漏洞，”Spectral 联合创始人兼首席运营官·艾丹·迪迪说。“我们正在解决的痛点在每一家开发软件的公司中都引起了强烈的共鸣，因为随着他们从自有代码到胶合代码再到无代码方法的发展，他们允许他们的开发人员获得更快的速度，但他们也增加了大量的风险。 Spectral 让开发人员在保持公司安全的同时提高工作效率。”

该公司成立于 2020 年年中，但已经拥有约 15 名员工，客户中有许多大型上市公司。
