疫情已经迫使全球的组织关闭办公环境，采取远程优先的策略。出于需要，专业人员已经适应了远程工作。但是他们使用的系统仍然在追赶。

较难适应远程环境的一个领域是入职流程。鉴于这是新创业者与一家公司的第一次持续接触，远程优先战略取决于它的成功。当寻找新员工时，第一天的见面和问候、现场硬件设置和团队午餐等奢华不再可用。从面试到录用函，任何新员工的早期旅程对他们在公司的生活、工作满意度以及最终的生产力都至关重要。远程感应必须是一个平稳的过程，因此需要彻底的重新思考。

最重要的是，组织中的每个人都优先考虑文档；他们具体如何做是次要的。

公司的文化转变可能是必要的。组织需要通过转向“手册优先”的方法来接受知识共享和协作。几个简单的步骤就可以把他们带到那里。公司也需要分析他们的工作流程。是否有合适的系统来确保隐性和显性知识的无缝流动？

也许最重要的是，人工智能可以帮助将笨重的旧入职流程转变为复杂、平稳的旅程。自然，最好的人工智能模型将取决于所涉及的业务和部门。然而，通过一些指点，商业领袖可以开辟一条人工智能集成的道路。

让我们深入了解一下可以改变远程入职流程的细节，这对公司和新入职者都有好处。

如何手册

当谈到确保新来者能够在正确的时间访问正确的信息时，这可以说是拼图中最重要的一块；也是最难做对的。组织中所有级别的员工都应该思考如何在团队之间共享知识，以及围绕这种思想交流的过程。

最重要的是，组织中的每个人都优先考虑文档；他们具体如何做是次要的。你可以开发大量的免费和付费软件来创建一本手册。任何基于云的东西都是合适的，建议使用更复杂的付费选项，通过将文档分类到定义良好的层次结构中，使事情易于搜索，而不是在文件夹的海洋中丢失那些宝贵的信息。

然而，这一系统性挑战最好自上而下解决。这个过程应该包括一些检查和平衡，对于手册中应该保持静态的部分，如政策和SLP，许可是至关重要的。文档的其他部分应该保持灵活性，比如过程和团队层面的知识。手册的大部分内容必须尽可能民主化。

GitLab，一家全远程公司，首先创造了术语“手册——第一个”DevOps软件提供商是一家通过记录和编纂内部知识而生存和发展的公司的典范。组织中的每个人都相信记录他们所知道的东西，并指派主题专家来管理知识库内容。保持公司文档的更新是一项协作任务，对公司的生存至关重要。软件伸出援助之手，推动贡献者保持信息更新。

git lab的远程负责人Darren Murph 说，他们的文档策略，与合作方法相结合，有助于与新创业者建立信任。“当一名新员工需要知道的一切都写下来时，就不会有歧义或怀疑是否遗漏了什么。我们将文档与一个入职伙伴——一个负责指导关键利益相关者对话并确保适应顺利进行的合作伙伴——结合在一起。”
