微软 Azure 通过 Apache Cassandra 托管实例扩展其 NoSQL 产品组合

在今天的 Ignite 大会上，微软宣布推出 Apache Cassandra 的 Azure Managed Instance，这是其最新的 NoSQL 数据库产品，也是 Datastax 等以 Cassandra 为中心的公司的竞争对手。微软将新服务描述为“半托管服务，将帮助公司将更多基于 Cassandra 的工作负载引入其云。”

“客户可以轻松承担本地 Cassandra 工作负载并添加无限的云规模，同时保持与最新版本的 Apache Cassandra 的完全兼容性，”微软在其新闻材料中解释道。"他们的部署获得了改进的性能和可用性，同时受益于 Azure 的安全性和合规性功能."

像它的对手 Azure SQL 管理实例一样，这里的想法是让用户访问可扩展的、基于云的数据库服务。以前要在 Azure 中使用 Cassandra，企业要么必须转移到支持 Cassandra、MongoDB、SQL 和 Gremlin APIs 的高度可扩展的数据库服务 Cosmos DB T1，要么管理自己在 T2 的虚拟机或内部基础设施。

Cassandra 最初是在脸书开发的，然后在 2008 年开源。一年后，它加入了 Apache Foundation，今天它被广泛应用于整个行业，例如，苹果和网飞等公司将其用于一些核心服务。AWS 在 2019 年的 re:Invent 大会上推出了托管的 Cassandra 兼容服务(今天称为亚马逊 Keyspaces )，微软在 2018 年 9 月推出了 Cosmos DB 的 Cassandra API。不过，随着今天的发布，该公司现在可以为希望将这些工作负载迁移到其云的企业提供全系列基于 Cassandra 的服务器。
