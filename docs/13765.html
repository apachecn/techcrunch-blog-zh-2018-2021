谷歌 Flutter 工具包的第二版增加了对桌面和网络应用的支持

原文：https://web.archive.org/web/https://techcrunch.com/2021/03/03/version-2-of-googles-flutter-toolkit-adds-support-for-desktop-and-web-apps/

在的在线活动上，谷歌今天宣布了 Flutter 2 ，这是其用于构建便携式应用的开源 UI 工具包的最新版本。虽然 Flutter 在两年前首次推出时专注于移动应用，但近年来它展开了翅膀，随着版本 2 的推出，Flutter 现在支持开箱即用的 web 和桌面应用程序。有了这个，Flutter 用户现在可以使用相同的代码库来为 iOS、Android、Windows、MacOS、Linux 和 web 构建应用程序。

Flutter 产品负责人 Tim Sneath 告诉我:“证明主要版本号转移的最重要的事情当然是网络和桌面支持的可用性。“这只是一个相当深刻的转变。产品很少会突然拥有所有这些额外的端点。”

图片来源:谷歌

他指出，由于 Flutter 的开源特性，网络和桌面支持已经“公开化”了一段时间，所以这些端点的加入并不令人惊讶。让这些新平台为 2.0 版本做好准备的大量工作包括让性能达到这些新平台的标准。

不过，值得注意的是，在 Flutter 的稳定发布渠道中，Flutter 桌面支持仍然是早期发布的标志，谷歌表示，开发者应该将其视为“测试版快照”。然而，Web 支持已经从测试版过渡到稳定版，并且已经成为用 Flutter 构建应用程序的另一个目标。

图片来源:谷歌

具体来说，在 web 平台上，Sneath 指出，该团队特意从一个非常标准的、以 DOM 为中心的方法开始。但是，虽然这样做很好，但这也意味着性能会受到影响，特别是对于更高级的功能。在过去一年左右的时间里，该团队开始研究所谓的 Canvas Kit。这个基于 WebAssembly 的项目采用了与 Android 和 Chrome 相同的 Skia 图形引擎，并使其可用于 web 应用。

“这意味着我们现在可以基本上绕过核心的 HTML——有点像网络平台以文档为中心的部分——真正使用网络平台以应用为中心的部分，而不会留下像文本或密码的自动完成以及所有让网络感觉非常独特的东西，”Sneath 说。

图片来源:谷歌

在桌面上，谷歌宣布 Canonical 将全力以赴开发 Flutter，并将其作为未来所有桌面和移动应用的默认选择。

微软也在扩大对 Flutter 的支持，并与谷歌合作开发 Windows 对 Flutter 的支持。鉴于微软对 Android 的兴趣，这可能并不令人惊讶，事实上，微软今天发布了对 Flutter 引擎的贡献，以帮助支持可折叠的 Android 设备。

谷歌指出，现在亚马逊、微软、Adobe、华为、阿里巴巴、易贝和 Square 等公司总共为 Flutter 和 Dart 提供了超过 15，000 个包。

和往常一样，在这次更新中还有许多其他更小的更新。

展望未来，Sneath 指出，Flutter 团队计划在 Flutter 上花更多时间，将其作为嵌入式设备和其他非传统平台的框架。他还指出，该团队对 Flutter 如何帮助增强环境计算体验感兴趣。

“当我们思考环境计算世界时，环境计算方面背后有这些核心前提，比如:它能被容易地搜索吗？人们能从自己开发的应用程序中赚钱，并以负责任的方式去做吗？我们正在建立对这些服务的支持。更好的分析、更好的广告框架、与 Firebase 和谷歌云的连接，这样人们不仅可以利用 Flutter，还可以利用谷歌提供的更广泛的生态系统服务，”Sneath 解释道。
