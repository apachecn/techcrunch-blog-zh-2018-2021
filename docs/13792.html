Stream 筹集了 3800 万美元，因为其聊天和活动源 API 为 1B 用户的通信提供了动力 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2021/03/04/stream-raises-38m-as-its-chat-and-activity-feed-apis-power-communications-for-1b-users/

如今，我们彼此之间的许多交流都是数字化的，如今，一家实现这一点的公司——通过 API 将聊天体验融入应用程序——宣布在一些非常强劲的增长背景下进行一轮融资。

让开发者通过几行代码将聊天和活动流构建到应用和其他服务中的 Stream 已经筹集了 3800 万美元，这笔资金将用于继续发展现有业务以及开发新功能。

Stream 从活动提要的 API 开始，然后扩展到聊天，今天可以集成到各种平台上构建的应用程序中。目前，它的客户集成了第三方聊天机器人，并将杜比用于 Stream 中的视频和音频，但随着时间的推移，这些都是 Stream 本身希望做得更多的领域。

“端到端加密，聊天机器人:我们希望尽可能多地采用组件，”首席执行官蒂埃里·谢伦巴赫(Thierry Schellenbach)说，他于 2015 年在阿姆斯特丹与初创公司的首席技术官托马索·巴布格利(Tommaso Barbugli)共同创立了这家初创公司(这家初创公司在阿姆斯特丹仍有一个由巴布格利领导的庞大团队，但其总部现在位于科罗拉多州的博尔德，谢伦巴赫最终搬到了那里)。

该公司已经积累了一系列知名客户，包括宜家旗下的 TaskRabbit、NBC Sports、联合利华、Delivery Hero、Gojek、eToro 和斯坦福大学，以及医疗保健、教育、金融、虚拟活动、约会、游戏和社交等领域的许多其他公司。这些应用的流能力总共覆盖了超过 10 亿用户。

这一轮 B 轮融资由 Felicis Ventures 的艾登·森库特(Aydin Senkut)牵头，之前的支持者 GGV 资本和 01 Advisors(该基金由 Twitter 前首席执行官首席运营官、迪克·科斯特罗和亚当·贝恩共同创立)也参与了。

除了他们之外，以前的和新的个人和小投资者也参加了:数据狗公司的首席执行官奥利维尔·波梅尔；GitHub 联合创始人汤姆·普雷斯顿-沃纳(Tom Preston-Werner)；总部位于阿姆斯特丹的骑士资本；Johnny Boufarhat，Hopin 创始人兼 CEO 社交游戏应用 Bunch 的联合创始人兼首席执行官 Selcuk Atli(不久前由 General Catalyst 领投的 2000 万美元的轮融资)。

这份名单是一个显著的指标，表明什么样的初创公司也在悄悄地与 Stream 合作。

该公司没有透露其估值，但表示 c hat 的收入在 2020 年增长了 500%。

事实上，B 轮谈到了一个机会时刻:它是在初创公司筹集了 1500 万美元的 A 轮融资后大约六个月到来的，事实上 Stream 现在并不打算筹集资金。

“我们原本不打算在今年晚些时候筹集资金，但后来艾登找到了我们，让我们很难说不，”谢伦巴赫说。

“最重要的是，他们建立在重要的技术平台上，”森库特在接受采访时补充道，并指出其用户证明了投资的强劲回报。“很少看到一款产品对客户如此重要，而且扩展性很好。这只是一种无上限的能力…我们希望成为故事的一部分。”

这个机会不是激流勇进所追求的。

在基于 API 的通信服务领域，一些更重要的参与者包括消息传递、活动流(你在应用程序中获得的整合更新，这些更新会告诉你何时有人回复了你的帖子，或者有与你相关的新内容，或者你有一条消息，等等)和聊天，包括 SendBird、Agora、PubNub、Twilio 和 Sinch，所有这些都筹集了大量资金，获得了大量客户，或者将自己定位为整合者。

这可能是竞争的表现，但也指向了那里巨大的市场。

事实上，像 Stream 这样的公司现在做得这么好的原因之一是因为他们已经建立了什么以及市场对它的需求。

与 Adyen(阿姆斯特丹的另一个主要科技力量)、Stripe、Rapyd、Mambu 和其他公司在金融科技领域所做的事情相比，Stream 的通信服务可能是最好的。

就像支付之类的东西一样，构建聊天功能的机制可能会很复杂，通常需要将一系列服务和平台编织在一起，这些服务和平台之间不会自然地相互对话。

与此同时，像活动提要或消息功能这样的东西对许多应用程序的工作方式至关重要，即使它们不是产品本身的核心功能。一个很好的例子就是订餐和送货应用程序:它们本质上不是“聊天应用程序”，但它们需要有一个聊天选项，以便当你需要与司机或餐馆沟通时使用。

将这些力量放在一起，我们会看到一系列技术公司的出现，这是很符合逻辑的，它们都做了艰苦的工作，建立了聊天服务的机制，并通过 API 的方式让那些想使用它的人可以访问，API 是当今应用程序中更核心和标准的构建模块之一；越来越多的开发人员渴望获得这些 API，将这些功能嵌入到他们的应用程序中。

Stream 正在做的事情不要与 Zendesk 或 Intercom 等公司在谈论应用聊天时建立的以客户服务为中心的服务混淆。这些功能本身可以是专门的功能，与 CRM 系统、客户服务团队和其他营销分析产品等相关联。相反，Stream 的重点是为消费者提供与其他消费者对话的服务。

一个值得关注的趋势是，像 Stream 这样易于集成的服务是否预示着更多社交应用程序的扩散。

已经有至少一个关键客户——我现在可以说出它的名字了——是一个稳步增长、仍然年轻的社交应用，它已经在 Stream 的 API 上建立了其服务的核心。

只有少数几家公司——以脸书为首，但也包括字节跳动/抖音、腾讯、Twitter、Snap、谷歌(通过 YouTube)和其他一些取决于地区的公司——对社交互动拥有巨大的控制权，更容易、平台无关地访问聊天等核心通信工具可能有助于更多这些具有不同“社交”商业模式的公司找到进入世界的方式。

前 Twitter 首席执行官、01 Advisors 的迪克·科斯特罗在一份声明中表示:“Stream 的技术通过提供一种易于集成和可扩展的消息传递解决方案，解决了产品开发中的一个常见问题除此之外，他们的团队和清晰的愿景使他们与众不同，我们热烈支持他们的使命。"
