Esri 将其旗舰 ArcGIS 平台引入 Kubernetes TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2021/04/06/esri-brings-its-flagship-arcgis-platform-to-kubernetes/

地理信息系统(GIS)、地图和空间分析公司 Esri 今天将举办其(虚拟)开发者峰会。不出所料，它将在活动中发布几项重大声明，从新的设计系统和改进的 JavaScript APIs 到支持在 Kubernetes 上的容器中运行 ArcGIS Enterprise 。

Esri 的产品经理 Trevor Seaton 和 Philip Heede 告诉我，Kubernetes 项目是公司的一项重大任务。传统上，像许多类似的产品一样，ArcGIS 被设计为安装在物理机器、虚拟机或云托管的虚拟机上。虽然软件在哪里运行对最终用户来说并不重要，但应用程序容器化意味着企业可以更容易地根据需要扩展或缩减系统。

基于 Kubernetes 部署的 Esri ArcGIS Enterprise。影像配额: Esri

“我们有很多客户——尤其是一些大客户——提出非常复杂的问题，”西顿解释道。“有时这是不可预测的。他们可能会对季节性事件、商业事件或经济事件做出响应，他们不仅需要了解世界上正在发生的事情，还需要对来自组织外部的许多用户提出的关于他们使用 ArcGIS 部署的系统的问题做出响应。这种不可预测的需求是 Kubernetes 的主要优势之一。”

在 Kubernetes 上部署 Esri ArcGIS Enterprise。影像配额: Esri

该团队本可以选择简单的路线，将现有工具打包，然后收工，但正如 Seaton 指出的那样，Esri 利用这个机会重新设计其工具，并将其分解为微服务。

“这花了我们一段时间，因为我们采用了三四个大型应用程序，它们共同组成了[ArcGIS] Enterprise，”他说。“我们将这些分成了一个更大的微服务集。这使我们能够将特定的服务容器化，并在不增加管理员复杂性的情况下为系统增加许多高可用性和弹性，事实上，我们在这样做的同时降低了复杂性，所有这些都在一个部署脚本中安装。”

虽然 Kubernetes 简化了许多管理体验，但许多使用 ArcGIS 的公司还不熟悉它。正如西顿和希德指出的，该公司没有强迫任何人进入这个平台。它将像以前一样继续支持 Windows 和 Linux。Heede 还强调，像 ArcGIS 这样复杂的完全集成的系统以微服务和多个容器的形式交付，然后客户在自己的基础设施上运行，这种情况仍然很少见，尤其是在这个行业。

影像配额: Esri

除了 Kubernetes 发布之外，Esri 今天还发布了新的 JavaScript APIs，使开发人员能够更轻松地创建应用程序，将 Esri 的服务器端技术和在客户端进行大量分析的可扩展性结合在一起。过去，Esri 会支持像微软的 Silverlight 和 Adobe/Apache Flex 这样的工具来构建丰富的基于 web 的应用程序。Esri 产品经理 Julie Powell 告诉我:“现在，我们真正关注的是单一的 web 开发技术和相关的工具集。

本月晚些时候，Esri 还计划推出其新的设计系统，使开发人员更容易、更快地创建干净、一致的用户界面。该设计系统将于 4 月 22 日推出，但该公司今天已经提供了一个小预告。正如 Powell 所指出的，Esri 面临的挑战是其设计系统必须帮助公司的合作伙伴在他们从 ArcGIS 生态系统中获得的地图和数据的基础上建立自己的风格和品牌。
