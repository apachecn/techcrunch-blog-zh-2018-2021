近年来，开源软件催生了大量有用的软件。我们今天使用的许多伟大的技术都诞生于开源开发:Android、Firefox、VLC media player、MongoDB、Linux、Docker 和 Python，仅举几例，其中许多也发展成为非常成功的盈利公司。

虽然有一些专门的开源投资者，如 Apache Software Foundation 孵化器和 OSS Capital，但大多数开源公司将从传统的风险投资公司筹集资金。

我们的团队从 Speedinvest 这样的传统风险投资公司，OSS 这样的开源公司，甚至从 OpenOcean 这样的混合公司筹集资金，open ocean 是由 MariaDB 和 MySQL 的创始人和高级领导团队创建的。可以理解的是，这些公司有一个重要的但不是唯一的开源焦点。

我们的创新领域是开源 AutoML 服务器，它可以降低模型训练的复杂性，并将机器学习引入数据源。最终，我们觉得机器学习的民主化有可能真正改变现代商业世界。因此，我们成功筹集了 500 万美元的种子资金，以帮助我们将愿景带入当前的市场。

在这里，我们旨在为希望遵循类似路径获得资金的开源创业公司提供见解和建议，并详细说明您的团队在设计商业模式以吸引投资时需要考虑的一些重要风险。

获取开源种子资金的策略

显然，风险资本家发现许多开源软件项目是值得投资的。然而，当成功地将一个创新的想法商业化时，他们需要理解任何内在的风险。对于这些公司来说，寻找能带来利润丰厚的商业机会的低风险投资仍然是一个重要的目标。

根据我们的经验，我们发现这些风险分为三大类:市场风险、执行风险和创始人风险。用简明的方式向潜在投资者解释这三点有助于消除他们的恐惧。最终，低风险、高回报的场景显然会吸引风险资本来源的切实兴趣。

最终，投资公司希望初创公司产生足够的收入，达到超过 10 亿美元的估值。虽然这个数字可能会随着时间的推移而增加，但它仍然是与投资者进行初步融资讨论的一个良好起点。1 亿美元的年收入是达到这一估值水平的良好基准。

开源项目的市场风险

与寻求资金的传统企业相比，开源组织的市场风险往往有所不同。值得注意的是，这些传统创业公司的投资者正在进行更大的信念飞跃。
