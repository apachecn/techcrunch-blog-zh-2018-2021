FintechOS 以低代码方式实现传统银行和保险服务的现代化，获得 6000 万美元

原文：https://web.archive.org/web/https://techcrunch.com/2021/04/20/fintechos-nabs-61-5m-for-a-low-code-approach-to-modernizing-legacy-banking-and-insurance-services/

银行和保险领域的“挑战者”初创公司颠覆了他们的行业，并获得了大量业务，它们构建了比规模更大的现有竞争对手通常提供的工具和服务更为客户友好的工具和服务，这些工具和服务更加个性化，更容易获得，价格通常也更具竞争力。现在，罗马尼亚的一家初创公司正在开发工具，帮助现有公司提供更好的服务，随着业务的增长，该公司宣布将获得一轮重要的融资。

FintechOS 建立了一个低代码平台，旨在帮助大型(老牌)银行和保险公司在其现有基础设施之上和周围建立新的服务和分析，该公司已在 B 轮融资中为€筹集了 5100 万英镑(按今天的利率计算为 6150 万美元，但交易完成时为 6000 万美元)。

FintechOS 的机会是瞄准保险和银行业的一波现有企业，这些企业一直在慢慢地看着 Lemonade(保险行业)和大量挑战者银行(Revolut、N26、Monzo 和许多其他银行)等新参与者涌入并吸引客户，特别是在年轻人口中，而他们一直无法做出回应，主要是因为他们的基础设施太旧太大。正如我们所看到的，让这艘巨轮掉头不是一件小事——这种情况在《疫情生活》的最后一年变得更加明显，并由此导致了向数字互动的巨大转变。

“当我们在 2017 年推出 FintechOS 时，我们已经可以看到，现有的数字化转型解决方案将很难取得切实的成果。相比之下，我们独特的方法迅速激发了金融机构应对数字化和与客户互动的方式的巨变，”FintechOS 联合创始人兼首席执行官特奥多尔·布利达鲁斯(Teodor Blidarus)在一份声明中表示。“过去一年发生的事件只是增加了我们行业发展的压力，因此我们看到对我们强大平台的需求不断增长。我们的最新一轮融资将帮助我们以所需的速度增长，以改善全球金融机构及其客户的成果。”

(不是唯一的。欧洲以外的其他银行在为现有银行带来新工具以帮助他们制造更现代和更具竞争力的产品方面包括 10x 、思想机器、 Temenos 、 Mambu 等等。)

B 轮融资由 Draper Esprit 牵头，Earlybird、Gapminder Ventures、Launchub 和 OTB 风险投资公司(均参与了其 2019 年 12 月的A 轮融资)也参与其中。这家初创公司补充说，目前还有其他支持者没有披露。FintechOS 也没有披露其估值。这家总部位于布加勒斯特的公司到目前为止已经筹集了将近 8000 万美元。

FintechOS 目前在英国和欧洲非常活跃，在那里它以 200%的 CAGR 增长，并表示其服务触及“数百万”人，其一些关键客户包括银行巨头兴业银行和 IdeaBank 以及国际保险经纪公司 Howden。我们的计划是继续在这些市场投资，同时进行国际扩张。

它将增加更多的服务。如今，银行平台旨在帮助银行为消费者和中小企业客户推出更多零售服务，并帮助保险公司开发新的健康、人寿和普通保险产品(保险和金融服务公司多年来的发展有很多协同效应，因此在为这些行业开发工具时，这是一个自然的组合)。

在金融领域，FintechOS 允许银行构建新的数字入职流程、信用卡和贷款产品、储蓄和抵押贷款产品。保险产品包括生成和处理报价、客户入职和管理以及索赔自动化的新方法，这很可能会使 FintechOS 与迄今为止走出其母国的最成功的初创公司 RPA juggernaut UiPath 建立更密切的联系和合作。在所有情况下，它都有助于将来自银行自身系统的数据与更现代的工具结合在一起，并将其与更现代的工具联系起来，以帮助更轻松地处理这些数据。

这是“低代码”，但它通常意味着公司需要与第三方合作来实现所有这些。合作伙伴包括集成商和其他全球服务技术人员，如微软、德勤、凯捷、毕马威等。(初创公司的创始人本身就有咨询背景，因此他们非常了解这些公司在将技术引入大企业的过程中所扮演的角色。)

FintechOS 正在利用几个非常大的趋势，这些趋势可以说是金融和相关保险行业中最大的趋势。

第一个事实是，围绕信贷/贷款、活期存款和储蓄等事物的核心服务不仅构建起来非常复杂，而且实际上已经在很大程度上商品化了——类似于数字支付——因此，将它们打包并转化为可以通过 API 集成的服务，使它们更容易访问，而无需从头构建它们。这让公司转而专注于客户服务，或围绕这些基本服务构建更有趣的工具来定制它们(例如基于人工智能的个性化)。基本功能从围绕它们构建的服务中去中介化可以说是一个更大的趋势，但它在企业中尤其普遍，在后端和前端的创新方面，企业长期以来一直是一个移动缓慢的空间。

第二个趋势是使用无代码和低代码工具，让组织中的更多人在看到某些东西没有尽可能高效地工作时能够坚持下去，并构建工作流本身来改善这一点。这也适用于尝试和测试新产品——这也是金融和保险服务中通常没有做过的事情，但现在可以通过低代码和无代码工具来实现。

“我们的技术不仅帮助金融机构变得以客户为中心，还帮助他们向更多的人和企业提供产品和服务，”FintechOS 的首席财务官兼首席运营官的另一位联合创始人 Sergiu Negut 在另一份声明中说。“在如此多的市场仍未得到充分服务的情况下，针对一个细分市场定制产品的能力提供了提高金融包容性的机会，并坚持了我们的理念，即轻松获得金融服务至关重要。我们很高兴与投资者合作，他们分享了我们对金融科技应该如何改变金融服务业的看法。”

值得注意的是，Draper Esprit 也有支持的思想机器，这是金融科技领域的另一个大玩家，它正在吸收一些帮助新进入者扰乱现有者的知识和模式，并将其打包为现有者的服务。这需要一种不同的方法，不使用低代码，而是使用智能合同，这可能是风投没有将投资视为利益冲突的一个原因。他们也在应对一个巨大的市场，因此至少现在有他们的空间，以及这个空间中的许多其他人，如 10x 、特梅诺斯、曼布、拉皮德和许多其他人。

“当我们见到 Teo 和 Sergiu 时，我们立刻就对他们的愿景深信不疑:一个数据主导的端到端平台，由低代码/无代码基础设施推动，”Draper Esprit 的合伙人 Vinoth Jayakumar 在一份声明中说。“现有的金融服务公司的成本收入比高达 90%，因此我们看到了对基础设施软件的巨大和日益增长的需求，这些软件可以快速、轻松和低成本地实现数字化。德雷珀 Esprit 建立持久的伙伴关系；我们希望与 FintechOS 的团队一起，建立一家持久的金融科技公司，为世界各地的人们带来巨大的金融服务体验。”
