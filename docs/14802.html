运行中的应用在隐私和安全性方面仍然落后 TechCrunch

一些最受欢迎的运行应用在安全性和隐私方面仍然落后。这是安全研究人员的结论，他们相隔五年检查了领先的运行应用程序，发现只有少数应用程序有所改善，而且改善幅度不大。

运行的应用程序在你使用它们的时候会了解你很多。你的健康数据，如你的身高和体重，被用来计算你燃烧了多少卡路里，你的位置数据可以跟踪你挨家挨户的锻炼路线。

但是在坏人手里，这些数据可以识别你住在哪里或者在哪里工作。2018 年，Strava 表示将简化其隐私功能，以允许其用户对其数据有更大的控制权，此前研究人员发现 Strava 应用程序用户无意中分享他们的锻炼数据，并泄露军事基地和秘密政府设施。

现在，英国网络安全公司 Pen Test Partners 的研究人员表示，许多顶级应用程序——Strava、Runkeeper、MapMyRun、Nike Run Club 和 Runtastic——仍然没有使用基本的安全措施来防止黑客入侵或健康和健身数据泄露。

研究人员在测试中发现，只有 Runtastic 在过去五年中设置了更强的密码政策，而其他应用程序仍然允许一些最基本的密码，如“123456”和“password”。恶意黑客通常通过使用已知或容易猜到的密码来锁定用户帐户，从而实现攻击的自动化。更糟糕的是，没有一款应用允许用户设置双因素身份验证，这一功能为防止恶意黑客重复使用窃取的密码设置了额外的障碍。来自谷歌的数据显示，即使是最简单的双因素认证也可以防止大多数自动密码重用攻击。

我们询问了每一个应用程序制造商，为什么他们没有实施双因素认证。没有一家公司发表评论。

研究人员还发现，虽然 Runtastic、Nike Run Club 和 MapMyRun 改善了他们的隐私控制，但 Strava 没有看到“明显的变化”

从他们的报告:“Strava 和 Runkeeper 默认配置为公开共享用户数据。在应用程序中更改这些设置是可能的，但需要一些时间来找到它们并正确设置它们，这可能不是一个普通用户的第一考虑。”

“Nike Run Club、Runtastic 和 MapMyRun 被发现启用了更好的隐私策略设置，这意味着它们不会像其他应用程序那样默认共享用户数据。他们只与朋友或追随者分享你的训练信息，”报告称。
