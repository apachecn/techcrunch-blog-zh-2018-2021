Merge 筹集 450 万美元，帮助 B2B 公司构建面向客户的集成 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2021/05/17/merge-raises-4-5m-to-help-b2b-companies-build-customer-facing-integrations/

帮助其用户构建与第三方工具的面向客户的集成的初创公司今天宣布，它已经获得了由 NEA 领投的 450 万美元的种子轮融资。其他天使投资人包括前 MuleSoft 首席执行官格雷格·肖特(Greg Schott)、Cloudflare 首席执行官马修·普林斯(Matthew Prince)、Expanse 联合创始人蒂姆·朱尼欧(Tim Junio)和马特·克拉宁(Matt Kraning)以及 Jumpstart 首席执行官本·赫尔曼(Ben Herman)。

Merge 于 2020 年推出，其核心重点是为 B2B 公司提供一个统一的 API，以访问目前约 40 个人力资源、薪资、招聘和会计平台的数据，并计划很快扩展到其他领域。但 Merge 的联合创始人沈思丁和吉尔费格强调，这项服务的目的不是取代工作流工具 Workato 或 Zapier，他们是一生的朋友，此前曾在 Expanse 和 Jumpstart 等公司工作。

“我们建造的东西更类似于格子花呢，而不是 MuleSoft 或其他东西，”Feig 说。“我们构建了一个统一的 API，因此我们完全嵌入到客户的产品中，他们与我们构建一个集成，并可以自动向他们的客户提供所有这些集成。除此之外，我们还提供所谓的集成管理，这是一套工具，可以自动检测客户必须参与的问题，自动检测并处理这些问题，而无需再次涉及工程设计。”

当 Merge 的系统检测到集成问题时，可能是因为 API 响应中的数据模式在没有通知的情况下发生了变化(这种情况有一定的规律性)，Merge 的工程师可以在几分钟内修复这个问题，部分原因是团队还构建了一个内部无代码工具来构建和管理这些集成。

图片来源:合并

丁还指出，如今的 B2B 购买者也只是希望他们的工具能够与他们使用的服务集成。“当公司购买一个供应商时，他们希望这个供应商与他们拥有的所有其他供应商进行整合，”她说。“他们不希望购买一个供应商，然后购买一个工作流产品，然后连接这些产品。”

虽然 Merge 目前的重点是几个垂直领域，但计划很快将其扩展到更多领域，可能从 CRM 开始。“Salesforce 拥有相当大的市场份额，所以我们认为这不会是一个有趣的市场，”丁说。“但事实证明，他们的 API 非常复杂，如果我们为他们简化，客户仍然会更愿意与我们集成。”

丁和费格告诉我，这家公司大约在两个月前悄悄成立，其平台上已经有大约 100 家组织，从种子期公司到公开上市的企业都有。该团队将其对安全性和可靠性(及其 SOC II 合规性)的关注归功于能够带来一些更大的公司，尽管其本身是一家种子期公司。

为了将这项服务货币化，Merge 提供了一个免费层(每月多达 10，000 个 API 请求)，并对每个 API 请求收取 0.01 美元的额外使用费。不出所料，该公司还为其大客户提供定制的企业计划。

“与构建和维护无数 API 集成相关的时间和费用是我们从所有行业的投资组合公司中不断听到的一个痛点，”NEA 管理普通合伙人 Scott Sandell 说，他将加入公司董事会。“Merge 正通过其易于使用的统一 API 平台正面解决这一普遍存在的问题。他们的平台具有广泛的适用性，对于任何需要构建、管理和维护多个 API 集成的软件公司来说，都是一次巨大的升级。”
