Netlify snags YC 明矾功能 Peek 增加设计审查能力

原文：https://web.archive.org/web/https://techcrunch.com/2021/05/19/netflify-snags-yc-alum-featurepeek-to-add-design-review-capabilities/amp/

图片来源: Chinga_11 / Getty Images

Netlify ，一家将微服务方法引入网站建设的初创公司，今天宣布它已经收购了 YC 校友 FeaturePeek 。两家公司没有分享购买价格。

有了 FeaturePeek，该公司的设计审核能力得到了重大提升。虽然自 2016 年以来，Netlify 已经在平台中拥有了一个名为“部署预览”的预览功能，但它缺乏一种好的方式供评论者讨论和评论设计。就其本身而言，预览是有用的，但直到今天，在设计上的协作能力仍然是一个缺失的部分。

借助 FeaturePeek，该公司可以扩展部署预览，不仅可以预览设计，还可以让设计流程中的所有利益相关者在设计创建过程中添加他们的意见、编辑和更改，而不必等到最后或在单独的文档或通信渠道中收集意见。

正如 FeaturePeek 的联合创始人 Eric Silverman 在去年获得种子资金时告诉我的那样，他的产品消除了 web 编码人员在最后一分钟得到他们所有评论的挫折感:

“目前，在新工作进入试运行环境之前，没有专门的地方提供反馈，因此我们将在提交或拉取请求时启动临时部署预览，这些完全运行的环境可以与团队共享。最重要的是，我们有自己的覆盖图，你可以在那里记录错误，给截图加注释，录制视频或留下评论。”

Netlify 的首席执行官兼联合创始人马特·比尔曼(Matt Biilmann)表示，当他的公司创建部署预览时，是为了回应客户对该问题提出的解决方案。他们了解到，即使有了自己的预览功能，客户也渴望通信功能。

在经典的构建与购买之争中，该公司开始构建自己的产品，然后遇到了 FeaturePeek 团队，并决定改变方向。“当 FeaturePeek 的创始人 Eric 和 Jason 向我们展示他们的产品时，我们有一个团队正在开发原型。随着演示的进行，我们的下巴越来越接近地面，我们立即知道我们刚刚看到的东西与我们的内部原型和我们在该领域看到的任何其他工具都相距甚远，”比尔曼告诉 TechCrunch。

他补充说，“很快我们就发现，如果两家不同的公司没有从最初的拉动式请求到新功能发布的深度端到端体验，就无法真正实现我们的梦想，所以我们决定联手。”

两家公司的共同努力实际上在今天的新版本 Deploy Previews 中得到了体现，其中包括新的 FeaturePeek 协作/评论功能。

FeaturePeek 成立于 2019 年，经历了 Y Combinator 年夏季批次，融资约 200 万美元。根据 Crunchbase 的数据，Netlify 成立于 2014 年，已筹集超过 9700 万美元。它最近一次融资是在 2020 年 3 月，价值 5300 万美元的 C 轮融资。
