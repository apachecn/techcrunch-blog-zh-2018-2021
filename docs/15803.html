微软的Windows虚拟桌面现在是Azure虚拟桌面TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2021/06/07/microsofts-windows-virtual-desktop-is-now-azure-virtual-desktop/

随着远程工作在疫情期间成为许多公司的默认工作，微软的Windows虚拟桌面等服务引起了大型企业和一批新的小型企业的极大兴趣，这也就不足为奇了。微软的Windows虚拟桌面可以让用户从几乎任何地方访问完全托管的Windows 10桌面体验，这些小型企业突然不得不寻找更好的方法来支持他们的远程员工。这也是微软所看到的，它最初的目标是世界上一些最大的企业的Windows虚拟桌面。因此，随着用户群的变化，微软对该产品的愿景也发生了变化，导致它现在将名称从 Windows虚拟桌面改为Azure虚拟桌面。

“大约一年半前，当我们第一次正式推出Windows虚拟桌面时，世界是一个非常不同的地方，”微软Azure虚拟桌面总经理Kam VedBrat说。“坦率地说，我们看待服务和我们在构建什么，我们在为谁构建，完全不同。当时没有人知道这种全球疫情将会发生，这将导致世界各地的许多组织和数百万人不得不离开办公室，在家工作，以及服务在实现这一目标方面将发挥的作用。”

图片来源:微软

虽然最初的想法是帮助企业将其虚拟桌面环境从数据中心迁移到云，但疫情为 Windows Azure虚拟桌面带来了一系列新的用例。它现在可以托管从虚拟学校实验室到传统远程企业使用案例的任何东西。这些新用户的需求和专业知识也与该服务最初面向的用户有所不同，因此除了今天的名称变更，该公司还推出了一系列新功能，这些功能应该会让新用户更容易开始使用Azure虚拟桌面。

其中包括一个新的快速启动体验，它将很快在公共预览中推出。VedBrat解释说:“我们看到的一个反馈是，由于许多组织都在关注Azure虚拟桌面，以实现混合工作的新场景，他们希望快速启动并运行这些环境，以了解它们如何工作，它们的应用程序如何在其中运行，如何考虑应用程序组和主机池以及其中的一些新概念。”理想情况下，现在只需点击几下鼠标就可以从Azure门户建立一个完整的虚拟桌面环境。

Azure虚拟桌面中的另一项新功能是支持使用微软的统一设备管理服务 Microsoft Endpoint Manager 管理多会话虚拟机(VM)。这标志着Endpoint Manager首次能够处理多会话虚拟机，这是Azure虚拟桌面的最大卖点之一，因为它允许企业在云中运行Windows 10 Enterprise的同一台机器上托管多个用户。

此外，Azure虚拟桌面现在提供了对Azure Active Directory的增强支持，此外还有一个新的按用户访问定价选项(除了在Azure基础架构上运行的成本)，允许用户向外部用户提供应用。例如，微软认为，这将允许软件供应商以SaaS解决方案的形式交付他们的应用。

至于名称的变化，VedBrat认为，虽然Windows显然是体验的核心，但该服务的许多用户也关心底层Azure基础设施，例如存储或网络。“他们着眼于他们正在创建的更广泛的环境，即他们正在云中创建的窗口资产，他们认为这是一个更大的事情，他们将Azure视为其中的一部分。因此，我们觉得在这一点上，为了解决我们的客户正在采取的更广泛的观点，正确的做法是考虑新名称，”他解释说。
