Vivaldi 4.0 发布，内置电子邮件和日历客户端、RSS 阅读器 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2021/06/09/vivaldi-4-0-launches-with-built-in-email-and-calendar-clients-rss-reader/

Vivaldi 一直是基于 Chromium 的浏览器中比较有趣的一个，这在很大程度上是因为它强调在一个以隐私为中心的包中为高级用户构建工具，但也因为它的血统，Opera 直言不讳的前首席执行官 Jon von Tetzchner 是其联合创始人兼首席执行官。今天，Vivaldi 团队推出了其浏览器的 4.0 版本，并推出了一系列新功能，其中包括新的内置邮件、日历和 RSS 客户端的测试版，以及 Vivaldi Translate 的推出，这是一种隐私友好的翻译服务，托管在公司自己的服务器上，由 Lingvanex 提供支持。

维瓦尔第对电子邮件客户来说并不陌生。例如，该公司长期以来一直提供网络邮件服务。但是，在浏览器中构建一个离线电子邮件客户端——以及一个日历客户端——几乎感觉像是回到了浏览器的早期，如 Netscape Navigator 和 Opera，当时这些额外的内置功能几乎是标准的。Von Tetzchner 认为，对于许多浏览器供应商来说，取消这些功能是为了将用户导向特定的方向(包括他们自己的网络邮件客户端)。

“我们选择说，‘好吧，我们不想让商业模式决定我们做什么。我们更关注用户想要什么。我认为(内置电子邮件客户端)有很大的价值。“我们大多数人都使用电子邮件——在不同的程度上，有些人用得很多，有些人用得较少，但基本上每个人都至少有一个电子邮件账户，”他说因此，有一个好的客户，这就是我们的出发点。我的意思是，我们显然在 Opera 做了很多这些事情——其中一些我们没有做——我们正在填补 Opera 过去所做的空白。现在在维瓦尔第，我们正在做这些事情，而且还做得更多。我们从没在歌剧院做过日历。"

图片来源:维瓦尔第

显然，围绕 Vivaldi 邮件和日历的许多决定都是由团队自己的偏好决定的。这意味着，举例来说，邮件客户端尽最大努力去除 Outlook 的常用文件夹结构，以便其过滤系统允许一封邮件出现在多个视图中。由于 Vivaldi 一直是关于定制的，你可以在传统的水平和宽视图之间选择，你可能熟悉其他电子邮件客户端。这里的一个很好的特性是，你可以控制你看到的消息，比如，你可以从邮件列表和自定义文件夹的默认视图中排除邮件。我很喜欢 Vivaldi Mail 区分未看邮件和未读邮件的事实。

维瓦尔第首席执行官乔恩·冯·泰茨赫纳。图片来源:维瓦尔第

正如所料，你几乎可以使用任何支持 IMAP 和 POP 协议的电子邮件提供商，但也有对 Gmail 的内置支持。

新的内置日历也支持大多数标准日历提供商，例如谷歌日历和 iCloud。这里一个有趣的设计变化是，团队决定在日历中显示一个事件的所有可用数据，而不是每个事件只有一两行。Von Tetzchner 告诉我，这是他最喜欢的。

“我认为我们做事情的方式不同了。我们会看看人们是怎么想的，”他说。“但我希望日历能够显示所有内容。通常，对于今天使用的日历，可用于文本的空间大小取决于时隙大小。不需要那样。当时间段是偶数时，看起来更好，但从功能上来说，你实际上可以阅读更多的文本更好。”

Von Tetzchner 指出，他显然希望引导用户远离谷歌和微软，但他认为提供替代品还不够好——它们必须是更好的替代品。

图片来源:维瓦尔第

至于 RSS 阅读器，它仍然是非常基本的，还不提供像导入和导出提要列表这样的功能，例如，这里的想法是帮助用户离开他们各自的回音室，但也避免新闻阅读器专注于新闻建议。这里的总体实现非常好，提要阅读器提供了本地提要阅读器所需的几乎所有特性。当你在网上冲浪时，只要浏览器发现一个 RSS 提要，它也会在地址栏中突出显示，因此订阅新的提要非常简单。你也可以订阅单独的 YouTube feeds(因为即使 YouTube 没有突出显示这一点，每个 YouTube 频道仍然可以作为 feed 使用)。

“有了 feeds，还意味着远离(数据)收集，”他说。“现在的新闻服务，他们看你读了什么，然后建立你的个人资料，借口是你可以得到更多相关的新闻。但依我拙见，你订阅了某些频道，这就足够了。我们试图让你——作为一个用户——控制你在读什么，订阅什么，而不是了解你的习惯或偏好。那些是你的习惯和偏好，不关我们的事。”

所有这一切都归结于维瓦尔第的核心理念，即不以广告为商业模式。“我们没有必要也没有兴趣收集我们用户的数据，”von Tetzchner 告诉我(尽管它正在收集一些基本的聚合数据，如它有多少用户以及他们在世界上的什么地方)。事实上，他认为收集关于用户的详细遥测数据只会推动公司为普通用户制造产品。

这也是新的翻译功能出现的地方，它托管在 Vivaldi 自己的服务器上，所以没有任何数据与任何第三方服务共享。Vivaldi 为此使用了 Lingvanex 的技术，但将其托管在自己的服务器上。结果相当不错，例如，在很大程度上与谷歌翻译相当(两者之间偶尔有细微的差异，谷歌翻译通常会提供更精确的翻译)。

Vivaldi 新的 onboarding 流程允许用户在三种默认布局之间进行选择，这是一个很好的特性，它承认每个人对浏览器都有不同的要求，而且为非高级用户构建 onramp 到 Vivaldi 也很好。对于那些只想要一个基本的和非常像 Chrome 或 Edge 的体验的人来说，有一个“基本”视图，对于那些想要使用浏览器的一些更高级的功能，如面板和状态栏的人来说，有一个“经典”视图，对于那些想要访问所有可用工具的人来说，有一个“完全加载”视图。默认情况下，这最后一个视图也启用了新的 Vivaldi 邮件、提要阅读器和日历功能。

截至目前，维瓦尔第还没有盈利。它从预装书签和搜索引擎合作中获得一些收入。但冯·泰茨赫纳认为，维瓦尔第只需要再增加一点用户群，就能成为一家可持续发展的公司。他似乎对这个想法很满意——以及其每用户收入相对较低的事实。“我们以前做过这种事，也看到了这种效果。建立一家像我们这样的公司需要时间，”他说。“我希望人们喜欢我们正在建造的东西——这是我得到的一种感觉——人们真的喜欢我们正在建造的东西。然后慢慢地，我们会让足够多的用户来支付账单，然后我们再从那里开始。”
