谷歌将允许企业存储他们的谷歌工作空间加密密钥

原文：https://web.archive.org/web/https://techcrunch.com/2021/06/14/google-workspace-encryption-keys/

谷歌将允许企业存储他们的谷歌工作空间加密密钥

尽管谷歌文档仅在去年一年就变得无处不在，但一个经常被无数使用它的工作场所忽视的主要批评是，它不是端到端加密的，允许谷歌或任何提出请求的政府机构访问公司的文件。但谷歌最终通过一轮更新解决了这一关键投诉，该更新将允许客户通过存储自己的加密密钥来保护他们的数据。

该公司的企业产品谷歌工作空间(Google Workspace)包括谷歌文档、幻灯片和表格，它正在增加客户端加密，这样谷歌就无法破译公司的数据。

使用 Google Workspace 的公司目前可以将他们的加密密钥存储在四个合作伙伴中的一个:Flowcrypt、Futurex、Thales 或 Virtru，它们与 Google 的规范兼容。此举主要针对受监管的行业，如金融、医疗保健和国防，这些行业的知识产权和敏感数据受到严格的隐私和合规规定的约束。

(图片:谷歌/供稿)

真正的魔力将在今年晚些时候降临，届时谷歌将公布一个 API 的细节，该 API 将允许企业客户建立自己的内部密钥服务，允许工作场所保留对其加密密钥的直接控制。这意味着，如果政府想要那家公司的数据，他们必须敲开他们的前门，而不是偷偷摸摸地通过向钥匙持有者提供合法要求来实现。

谷歌公布了客户端加密功能如何工作的技术细节，并将在未来几周推出测试版。

近年来，科技公司让企业客户控制自己的加密密钥已成为一种日益增长的趋势。 Slack 和云供应商 Egnyte 逆势而上，允许他们的企业用户存储他们自己的加密密钥，有效地将他们自己排除在监控圈之外。但是谷歌在加密上拖了这么长时间，以至于初创公司正致力于从头开始构建加密的替代方案。

谷歌表示，它还推出了关于如何在 Google Drive 中共享文件的新信任规则，以便在如何共享不同级别的敏感文件方面为管理员提供更多粒度，以及新的数据分类标签，以标记具有“秘密”或“内部”等敏感级别的文件。

该公司表示，它正在通过阻止网络钓鱼和组织内部共享的恶意软件来改善其恶意软件保护工作。目的是帮助减少员工错误地共享恶意文档。
