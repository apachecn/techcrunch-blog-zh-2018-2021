Stripe 不仅仅使用 Stripe 身份进行支付，还为交易提供基于人工智能的身份验证等等 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2021/06/14/stripe-goes-beyond-payments-with-stripe-identity-to-provide-ai-based-id-verification-for-transactions-and-much-more/amp/

图片来源: DKosig / Getty Images

一些创业公司(如这里的、这里的、这里的和这里的)将自己塑造成“身份验证的条纹”，为开发者提供了一种将身份认证集成到平台中的简单方法。如今，Stripe 正在自己填补这一需求:该公司正在推出一款名为 Stripe Identity 的新产品，这是一款自助工具，公司可以使用它来验证用户身份，Stripe 以加密格式管理客户数据，使用计算机视觉和机器学习来“读取”政府 id 并将其与实时自拍进行匹配。

Stripe 表示，这项服务只需 15 秒钟即可完成。

该公司表示，这项服务将于今天开始在 30 个国家进行测试，但与此同时，它已经悄悄地被选定的合作伙伴所使用。它们包括不和谐 (作为其 ID 验证功能的一部分)； Peerspace (在用户和商户登陆时运行 ID 验证)； Shippo (当其识别高风险用户并要求其验证自己时)；和其他匿名客户利用它来防止帐户被接管。

开发者可以到这里请求访问，听起来 Stripe 将在本周晚些时候的开发者大会 Stripe Sessions 上更多地谈论这项服务。

“企业一直要求我们提供一种简单快捷的在线身份验证方法。Stripe Identity 的工程主管罗布·戴利(Rob Daly)说。“现在，任何互联网企业——从五人初创企业到跨国企业——都可以在几分钟内开始安全地验证其用户的身份，而不是几周或几个月。”

如果你看看早期用户的名单，就会发现一些重要的东西:Identity 是 Stripe 如何将其作为强大的服务提供商的影响力扩展到远远超出支付范围的企业的又一个例子。

当然，支付仍然是 Stripe 的核心，身份验证肯定与此有关——首先，它可以帮助防止电子商务欺诈——但正如你所看到的，身份认证还涵盖了许多其他使用案例，例如帮助防止账户被接管，或验证某人的年龄，或遵守其他“了解你的客户”法规，或消灭机器人和巨魔:无需支付。因此，Stripe 确认，您不必成为现有的 Stripe 客户就可以使用 Stripe 身份。

身份验证已经成为实现在线交互过程的重要且广泛使用的组件，无论是金钱还是其他。它与支付有一些共同点，这使它成为构建 API 支持的服务的成熟候选:两者都很复杂，需要集成许多不同的技术和第三方数据提供商。由于构建这样的东西不太可能是需要身份验证的公司的核心竞争力，这就为第三方公司构建这种技术并将其作为服务提供敞开了大门。

Stripe 远不是唯一一家在这一领域工作的公司:其他公司包括 Jumio (今年早些时候筹集了一大笔资金)； Onfido (据传计划 IPO)；Veriff；柏比克斯；通行证等。

以 Stripe 为例，该公司为客户提供了两种使用 Stripe 身份的方式。像 Stripe 的支付产品一样，它可以通过一些代码行集成到结账或其他工作流中。但它也可以作为一种外部服务提供，通过围绕可疑交易或被确定为高风险的其他用户发布的验证链接。

虽然 Stripe 已经进行了大量收购来扩展其技术和服务——例如，上周推出的 Stripe Tax 补充了今年早些时候对 Tax Jar 的收购——但该公司表示，Stripe Identity 是在内部建立的，并且在 10 年的大部分时间里一直被该公司用于验证商户，作为其自身入职和反欺诈检查的一部分。

Stripe 全球风险战略和入职政策主管 Delia Pawelke 在一份声明中表示:“我们从经验中知道，建立一个严格安全的全球身份验证系统需要做多少工作。”“通过条带身份，我们让我们的高级合规性基础架构可供所有用户使用。对于一家在线企业来说，验证某人的身份现在就像接受付款一样简单。”在将其内部工具转变为外部产品的过程中，Stripe 对验证服务进行了一些调整，例如为拍照提供了更多指导。

Stripe 表示，将对“每笔交易”收取费用，但没有向 TechCrunch 透露这些费用将是多少。随着我们了解更多，我们将更新这篇文章。
