边缘计算初创公司 Macrometa 获得由 Pelion Venture Partners 领投的 2000 万美元 A 轮融资 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2021/06/15/edge-computing-startup-macrometa-gets-20m-series-a-led-by/

面向应用开发者的边缘计算云和全球数据网络 Macrometa 今天宣布，它已经筹集了 2000 万美元的 a 轮融资。该轮融资由 Pelion Venture Partners 领投，参与投资的有回归投资者 DNX 风险投资公司(日美企业基金曾领投 Macrometa 的种子轮)、Benhamou Global Ventures (BGV)、Partech Partners、Fusion Fund、Sway Ventures 和 Shasta Ventures。

这家总部位于帕洛阿尔托的初创公司在保加利亚和印度开展业务，计划将 A 轮用于功能开发，获得更多企业客户，并与内容交付网络(CDN)、云和电信提供商整合。它将为其在美国、东欧和印度的工程和产品开发中心招聘员工，并在乌克兰、葡萄牙、希腊、墨西哥和阿根廷增设新的中心。

该公司的最后一轮融资，700 万美元的种子资金，是在八个月前宣布的。其 A 轮融资使 Macrometa 自 2017 年成立以来的总融资额达到 2900 万美元。

作为新一轮的一部分，Macrometa 扩大了董事会，增加了 Pelion 普通合伙人克里斯·库珀为董事，Pelion 高级合伙人扎因·里扎维和 DNX 风险投资公司负责人伊娃·纳哈里为董事会观察员。

Macrometa 的全球数据网络结合了全球分布式 noSQL 数据库和低延迟流数据处理引擎，使 web 和云开发人员能够运行和扩展数据密集型实时云应用程序。该网络允许开发人员在世界各地的 175 个存在点(pop)或边缘区域同时运行应用程序，这取决于哪一个最接近最终用户。Macrometa 声称，在全球范围内，笔记本电脑或手机用户到其边缘云的平均往返时间(RTT)不到 50 毫秒，比 DyanmoDB、MongoDB 或 Firebase 等云平台快 50 到 100 倍。

Macrometa 联合创始人兼首席执行官 Chetan Venkatesh。图片来源: Macrometa

联合创始人兼首席执行官切坦·文卡特什(Chetan Venkatesh)告诉 TechCrunch，自去年种子期以来，该公司已加快了客户获取速度，特别是在大型全球企业和网络规模的公司中。Macrometa 还向开发人员提供其自助服务平台，他们可以免费试用其无服务器数据库、pub/sug、事件处理和有状态计算运行时。

Macrometa 最近成为两家分布式数据公司之一(另一家是 fana)与 Cloudflare 合作，帮助开发人员在其无服务器应用平台 Workers 上开发新应用。Venkatesh 表示，与公共云相比，Macrometa 和 Cloudflare Workers 的结合使数据驱动的 API 和 web 服务的性能提高了 50 到 100 倍，延迟更低。

Venkatesh 说，新冠肺炎疫情大大加快了 Macrometa 的业务，因为其企业和网络规模的客户需要处理远程工作产生的不可预测的数据流量模式。他补充说，疫情还“导致了云的采用和消费的几个长期和永久的转变”，改变了人们购物、消费媒体、内容和娱乐的方式。他说，这“成倍地增加了安全处理应用基础设施需求的需求”。

企业客户如何使用 Macrometa 的一个例子是电子商务提供商，他们利用现有的 CDN 和云后端实施其基础设施，为购物者提供更多数据和基于人工智能的个性化，包括实时推荐、边缘的区域化搜索和本地数据地理围栏，以遵守数据和隐私法规。

Macrometa 的一些 SaaS 客户使用其全球数据网络作为全球数据缓存，以应对使用量的激增，并在其区域数据中心保存数据和 API 结果的区域副本。Venkatesh 补充说，几家大型电信运营商已经使用 Macrometa 的数据流摄取和复杂事件处理平台来取代 Splunk、Tibco 和 Apache Kafka 等传统数据摄取平台。

在一份声明中，Pelion Venture Partners 普通合伙人克里斯·库珀表示，“我们相信计算的下一阶段将专注于边缘，最终将基于云的工作负载更接近最终用户。随着越来越多的工作负载脱离集中式云模式，Macrometa 正在成为事实上的边缘提供商，为全球的开发人员和企业运行数据密集型和计算密集型工作负载。”
