AWS 已经收购了加密信息服务 Wickr

原文：https://web.archive.org/web/https://techcrunch.com/2021/06/25/aws-is-buying-encrypted-messaging-service-wickr/?guccounter=1

亚马逊的云服务巨头亚马逊网络服务公司(Amazon Web Services)正在进军加密信息业务。该公司刚刚宣布收购了安全通信服务Wickr——一款面向政府和军事团体及企业提供服务的消息应用。它声称是唯一符合美国国家安全局设定的安全标准的“协作服务”。

AWS 将继续运营 Wickr，并向 AWS 客户提供服务，“立即生效，”AWS 副总裁兼 CISO Stephen Schmidt 在宣布这一消息的博客中指出。

在简短的公告中没有披露财务条款。根据 PitchBook 的数据，Wickr 已经筹集了不到 6000 万美元的资金(它还提到了不到 3000 万美元的估值，但这似乎是一个非常旧的估计)。与此同时，亚马逊的云和企业部门 AWS 一直是这家电子商务和服务巨头的重要力量。AWS 上个季度的收入为 135 亿美元，同比增长 32%，净收入为 81 亿美元。

亚马逊购买一款旨在为政府机构提供安全服务的消息产品之际，该公司继续卷入围绕 JEDI 合同的纠纷，该合同是一项价值 100 亿美元的交易，旨在向美国提供服务，微软在特朗普政府期间获得了该合同。

目前还不清楚这是亚马逊建立更多基础设施和服务以充实自己产品的努力的一部分，还是仅仅表明它将继续迎合这个市场，不管有没有绝地武士在它的口袋里。

此举还表明，亚马逊可能正计划更大规模地进军通讯领域——有人可能会说这是该公司期待已久的举措。

AWS 部门目前提供通信服务 Chime，使组织能够见面、聊天和拨打商务电话。但它是一个鲜为人知的产品，未能产生竞争对手 services Slack 或微软团队的影响，而且它不像 Wickr 那样专注于端到端加密。

据报道，亚马逊早在 2017 年就在开发一款名为的消息产品，尽管这似乎更适合消费者。该公司还拥有多项社交媒体专利。

快进到 2021 年，有一百万个其他关于消息传递的考虑在 2017 年不会是关键因素，例如加密和其他隐私保护功能。消息传递总体上变得越来越复杂。

亚马逊可能会对以下四个领域感兴趣:(1)将 Wickr 作为一项商业服务提供，继续沿用至今；(2)构建“消息即服务”,供其他公司在其应用中使用，就像其他 AWS 服务一样；(3)在 Wickr 基础设施之上构建消费者消息应用；(4)更多服务连接到 Echo，围绕更大的社交商务/互动游戏扩展更多功能。或者，以上都有。

AWS CISO 公司的 Stephen Schmidt 在评论此次收购时表示:“对这类安全通信的需求正在加速增长。随着向混合工作环境的迁移(部分是由于新冠肺炎疫情),企业和政府机构越来越希望保护他们在许多远程位置的通信。Wickr 的安全通信解决方案帮助企业和政府组织适应其劳动力的这种变化，并且是 AWS 为客户和合作伙伴提供的越来越多的协作和生产力服务的一个受欢迎的补充。”

Wickr 在其网站上的一份通知中说:“自 10 年前成立以来，我们已经成长为服务于世界各地各行各业的组织。我们期待与 AWS 一起，为我们的客户和合作伙伴将我们的解决方案推向更高的水平。”

Wickr 成立于 2011 年，总部位于旧金山，自称是“最安全的”视频会议和协作平台。与其他协作工具不同，Wickr 使用端到端加密，这意味着只有对话两端的人才能解密和读取消息。其他协作工具在消息从用户设备传输到公司服务器时会对消息进行加密，但会以未加密状态存储这些通信。

Wickr 还为用户提供了短暂的消息传递功能，允许用户设置短至几秒钟的自毁定时器。

由于大规模转向在线通信，该公司最近大举进军企业。今年 2 月，Wickr 推出了“全球联盟”(Global Federation)，该功能使企业和政府机构能够使用端到端加密技术与网络外的关键合作伙伴进行安全通信。
