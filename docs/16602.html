Postman 超过 1500 万人的社区中的 API 发布者正在努力为他们的 API 提供更加无缝和集成的开发者体验。从数百次一对一的讨论中提炼出来，我最近在 Postman 上分享了一项关于越来越多地采用带有公共工作区的 API的研究。使用公共工作区的最大原因之一是通过更快的首次调用时间(TTFC)来增强开发人员的入职能力，这是公共 API 需要的最重要的指标。

如果你不投资 TTFC 作为你最重要的 API 指标，你就限制了你的潜在开发者基础的规模。

为了理解一个开发人员的旅程，让我们首先来看看影响他们愿意投入多少时间和精力来学习您的技术并使其工作的因素。

  • 紧迫性:开发人员是否在积极地寻找现有问题的解决方案？还是他们顺便听说了你的技术，产生了轻微的好奇心？
  • 约束条件:开发人员是否试图在截止日期前完成？还是他们有无限的时间和预算去探索可能性？
  • 备选方案:他们的组织是否要求开发人员使用这个解决方案？或者他们是从许多提供商中选择，并考虑其他方法来解决他们的问题？

开发者的 API 之旅

考虑到这一背景，以下阶段描述了开发人员接触新 API 的过程:

第一步:浏览

开发人员浏览您的网站和文档，以了解您的 API 提供了什么。有些人忽略了这一步，宁愿在接下来的步骤中学习你的技术提供了什么。但是判断是在这个非常早期的阶段形成的，可能是在比较你的产品的时候。例如，如果你的文档和入职过程相对来说显得杂乱无章，错误百出，也许这是你的技术的反映。

第二步:注册

注册账户是开发者的第一承诺。这表明他们想用你的 API 做些什么。通常与下一步密切相关的是，注册是生成 API 密匙所必需的。

步骤 3:第一次 API 调用

进行第一次 API 调用是开发人员收到的第一个回报，并且通常是开发人员开始更深入地理解 API 如何适应他们的世界的时候。 Stripe 和 Algolia 在其开发者文档中嵌入了交互式指南，以支持首次 API 调用。 Stripe 和 Twitter 也使用 Postman 公共工作区进行互动入职。由于许多开发人员已经在使用 Postman，在熟悉的领域体验 API 让他们离实现更近了一步。
