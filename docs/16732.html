值得注意的一个 F1 合作伙伴是亚马逊。例如，其公共云项目 AWS 已经为这项运动的屏幕图形提供了动力。当然，有时粉丝们想知道该小组的计算集群是如何得出某些指标的，但 AWS 关于轮胎磨损的记录是有用和及时的。

然而，事实证明，在幕后，亚马逊在 F1 世界比我之前理解的更加活跃。简而言之，我们讨论的技术和 F1 赚钱的故事只是一个更大的难题的一部分。为什么事实证明，AWS 是 F1 2022 新车设计过程的关键。

我敢打赌，你想知道为什么它是如此俯冲。答案是，这款车在设计时考虑了一些非常具体的空气动力学目标。比如减少所谓的“脏空气”，这是一种从 F1 赛车后部飞出的风使跟随它的赛车努力保持在赛道上的现象。

今天的 F1 赛车——我们正处于上一个赛季的中期，使用的是当前一代的 F1 硬件；走吧兰度！ —产生大量污浊空气。这让比赛变得有些尴尬，因为赛道上的赛车不能靠得太近，以免失去至关重要的下压力。你知道，就是让车停在停机坪上而不是墙里的东西。

为了设计一款能够满足 F1 在下一个竞争时代的要求的基础赛车，即减少污浊的空气并允许更近距离的比赛，大量的计算工作必须进入计算流体动力学(CFD)领域。事实证明，AWS 处理了赛车组的计算需求。

这次交流是在 Amazon Chime 上进行的——这是我们第一次使用这个平台——我们与 F1 的数据系统总监罗伯·斯梅德利聊了聊这一切是如何走到一起的。根据前法拉利和威廉姆斯工程师的说法，赛车组织和亚马逊自 2018 年以来一直致力于新车项目。F1 有许多内部大脑来处理自己这边的事情，而亚马逊提供了数千个内核来完成所有复杂的数学运算。

根据史沫特莱的说法，如果他的团队使用单个 F1 车队允许的相同计算能力——一级方程式赛车运动充满了旨在帮助车队保持平等地位或阻止梅赛德斯的规则，这取决于你的观点——每个计算周期需要四天时间来模拟两辆一前一后行驶的新车。

但有了亚马逊提供的 2500 个计算核心，史沫特莱和 F1 的数据研究人员可以在六到八个小时内完成同样的工作。这意味着该小组可以运行更多的模拟和设计更好的汽车。有时候，他们甚至吸收了更多的计算，数据主管告诉交易所，去年有一段时间，他的团队同时对十几个迭代进行模拟。这是由大约 7，500 个内核为数据工作提供动力而实现的。模拟运行花费了 30 个小时。

这就是说，是的，一级方程式中有大量的技术资金帮助车队完成工作并保持财务偿付能力。但是也有一船的技术进入 F1 的真正的内脏和螺栓。作为一名 F1 呆子，看到我的激情与工作相结合，我感到非常高兴。
