Yapily 通过关注基础设施为其开放银行 API 筹集了 5100 万美元

原文：https://web.archive.org/web/https://techcrunch.com/2021/07/21/yapily-raises-51-million-for-its-open-banking-api/

金融科技初创公司 Yapily 已经筹集了由 Sapphire Ventures 领投的 5100 万美元 B 轮融资。该公司一直在为几个欧洲市场开发一个单一的、统一的开放银行 API。开发人员可以利用该编程接口，直接从他们自己的产品中与第三方银行账户进行交互。

创始人兼首席执行官 Stefano Vaccino 告诉我:“我们与该领域大多数公司的核心区别在于我们专注于基础设施。与 Tink 或 TrueLayer 不同，Yapily 在后台运行。你永远不会在任何地方看到 Yapily 的标志，该公司也不提供前端接口。

另一个区别是，Yapily 只专注于 API 集成。由于欧洲最近的监管变化(PSD2)，银行现在必须提供官方 API 来与第三方服务集成。虽然有些仍然不提供 API，但他们中的许多人现在都遵守了规则。

通过专注于官方 API，Yapily 可以提供更快、更可靠的体验。其他致力于统一开放银行 API 的初创公司依赖于官方 API、屏幕抓取和私有 API 的混合。屏幕抓取可能特别慢，私有 API 有时会在一夜之间停止工作。

就覆盖范围而言，Yapily 支持 8 个国家的 1500 多家银行。“我们在英国、法国、西班牙、德国、爱尔兰、奥地利、意大利和荷兰的覆盖率在 90%到 99%之间，”瓦奇诺说。你可以在本页的上看到银行的完整名单。根据 Vaccino 的说法，在欧洲只有 Tink 有类似的覆盖水平。

您可以将 Yapily 的 API 用于几种不同的目的。例如，API 允许您检查银行帐户的余额，检查帐户持有人的姓名，并获取最近两年的交易。

但这家初创公司也注意到，越来越多的客户依赖开放式银行从银行账户发起支付。与卡支付相比，帐户对帐户支付更便宜。信用卡也会过期，导致客户流失。Yapily 的 API 可用于一次性支付、国际支付、批量支付和经常性支付。

随着今天的融资，该公司计划扩大其在整个欧洲的商业存在。除了英国、德国和意大利，Yapily 还将在法国和西班牙招聘新的团队成员。

这家初创公司还将与北欧和波罗的海地区的新市场的银行建立整合，然后超越欧洲。“明年年初，我们将着眼于拉丁美洲，尤其是巴西，”瓦奇诺说。巴西的银行已经有很多开放的银行 API。

目前，Yapily 有大约 100 个客户，如美国运通、QuickBooks、Bux、Vivid Money、Volt 和 Moneyfarm。通过专注于基础设施层，其他金融科技初创公司正在利用 Yapily 在初创公司的 API 上构建应用程序。这是一个有趣的走向市场的策略，看起来效果不错。
