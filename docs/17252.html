OpenAI 升级其自然语言人工智能编码器 Codex，并启动私人测试版

原文：https://web.archive.org/web/https://techcrunch.com/2021/08/10/openai-upgrades-its-natural-language-ai-coder-codex-and-kicks-off-private-beta/

OpenAI 已经对该公司上个月宣布的人工智能编码助手 Codex 进行了一些重大改变。该系统现在接受简单英语的命令，并输出实时的工作代码，让人们无需命名变量就能构建游戏或网络应用。一些幸运的编码者(以及，有人认为，非编码者)将能够在一个免费的私人测试版中试用这个新的 Codex API。

Codex 最好被认为是 OpenAI 的通用语言引擎 GPT-3，但只训练代码，而不是普通的书面材料。这使得它可以做一些事情，比如完整的代码行或整个部分，但当它被宣布时，它真的不是一个非编码人员能够轻松交互的东西。

这个新的 API 改变了这种情况，它解释普通的日常请求，如“让球从屏幕两侧弹开”或“使用公共 API 下载数据并按日期排序”，并以十几种语言之一输出工作代码。

我观看了一个现场演示，OpenAI 的联合创始人 Greg Brockman(首席技术官)和 Wojciech Zaremba (Codex 负责人)从头开始构建了一个简单的游戏，并解释了幕后发生的事情。

布罗克曼解释说:“编程就是要有一个愿景，并把它分成块，然后实际上为这些块编写代码。”Codex 的目的是让编码者在第一部分比第二部分花更多的时间。毕竟，大量的代码是在重复或直接复制其他人以前做过的事情——当然，这可以是创造性的，但是没有人会在做像部署 web 服务器来测试一点代码这样的基本事情时发挥他们的想象力。布罗克曼用一句简单的话做到了这一点——“创建一个这样的网页”或诸如此类的话。

一秒钟后，十几行 JavaScript 代码以完全标准的方式完成了这一任务。

“这是编程最糟糕的部分，”布罗克曼说。“我可能已经写了几十遍这种代码，但我总是忘记它到底是如何工作的。我不知道这些 API，也没必要知道。你可以更轻松地做同样的事情，只需更少的按键或交互。”

因为 Codex 接受了 GitHub 上基本上所有公共代码的训练，除了其他库，它知道所有的标准实践，50 或 100 次有人在他们的代码中包括 web 服务器、键盘控制或对象操作和动画。因为自然语言方面拥有所有 GPT-3 通常的理解，当你说“让它变小并裁剪它”，然后“让它的水平位置由左右箭头键控制”时，你指的是同一个“它”

它还牢记自己的工作，为自己准备了几千字节的编码上下文——因此它知道必须遵守的命名约定、现有的界限和请求，以及用户输入可能隐含的其他信息。

它还知道嵌入在代码集中的一般性。例如，当布罗克曼告诉它“让巨石从天空落下”时，系统并没有问“天空”是什么，即使它还没有在很大程度上空白的画布上定义。它不仅让巨石从屏幕顶部落下，而且下落速度像物体通常会做的那样加快——因为它从其他用法和上下文中对“下落”和“天空”的意思的最佳猜测。

“我们认为它提供了一种与现有软件交互的新方式，”扎伦巴说，他几年前为自己的博士论文制作了一个有限版本，当时他们演示了一个用于 Microsoft Word 的 Codex 插件。当然，字处理程序中的许多任务都有自动化，但是如果您遇到一个奇怪的格式问题，并且想要修复 100 个不同的实例，该怎么办呢？键入“使所有文本的大小和字体相同，并使双倍行距为单倍行距”，它就会这样做，剪掉杂乱的样式，挑选最有可能被认为是“正常”的大小和字体然后输入“将所有标题设为 24 磅粗体”，它会放大显示，以此类推。

这里值得注意的是，这种事情对许多人来说很方便，但对那些由于残疾等原因而缺乏能力做这些事情的人来说至关重要。如果你正在使用语音命令或操纵杆操作文字处理器，能够执行如上所述的复杂任务是非常有用的。一个盲目的编码者，像其他人一样，可以拼凑一个标准的公共测试服务器，但是浏览堆栈溢出、获取最佳代码片段、检查语法、更改相关变量等等的过程几乎肯定会更长。

对于那些在高层流传下来的语法和约定的范围内工作的人来说，通过将模型暴露给文档，Codex 可以很容易地反映这些内容。Codex 还可以将代码从一种语言转换和移植到另一种语言，就像翻译引擎将西班牙语翻译成法语一样。

布罗克曼说，与《GPT 3》一样，他们只是触及了可能的表面，并希望对开发者提出的东西感到惊讶(毕竟，OpenAI 没有预测人工智能地牢)。测试版将是私人的，就像 GPT-3 的测试版一样，但开发者可以申请并描述他们的项目，Codex 团队将对他们进行审查。最终 API 将会是一个付费的公共 API，但是时间和价格仍有待确定。
