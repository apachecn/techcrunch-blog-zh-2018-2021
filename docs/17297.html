今天，OpenAI Codex 的发布，一个新的将自然语言翻译成代码的人工智能系统，标志着计算机软件编写方式转变的开始。

在过去的几年里，关于“无代码”平台的讨论越来越多，但这并不是什么新现象。事实是，自从第一台可编程设备问世以来，计算机科学家们已经在我们如何“编码”计算机软件方面取得了有规律的突破。

在键盘发明之前，第一台计算机是用开关或穿孔卡编程的。编码变成了键入数字或机器语言，直到格蕾丝·赫柏发明了现代编译器和 COBOL 语言，开启了编程语言和平台的几十年创新。像 Fortran、Pascal、C、Java 和 Python 这样的语言不断发展，其中最新的语言(使用更老的语言构建)使程序员能够使用越来越多的人类语言“编码”。

除了语言，我们已经看到了“无代码”平台的发展，包括 20 世纪 80 年代无代码的鼻祖 Microsoft Excel，它使人们能够在可视化界面中编写计算机程序，无论是在学校还是在工作场所。任何时候你在电子表格中写一个公式，或者当你在Code.org或 Scratch 上拖动一段代码时，你都是在给计算机编程或“编码”。【无代码】为代码。每十年，一个突破性的创新使得编写代码变得更加容易，以至于旧的编码方式被新的取代。

这是否意味着编码已死？不要！它不能取代程序员理解代码的需要。这意味着编码变得更加容易，影响更大，因此也更加重要。

这就引出了今天的公告。今天，OpenAl 宣布了 OpenAI Codex，这是一种用自然英语“写代码”的全新方式。一个计算机程序员现在可以用英语来描述他们希望他们的软件做什么，OpenAl 的生成式 Al 模型会自动生成相应的计算机代码，在你选择的编程语言中。这是我们一直想要的——让计算机理解我们希望它们做什么，然后去做，而不需要通过像编程语言这样的复杂中介。

但这不是结束，而是开始。有了人工智能生成的代码，人们可以想象每一个编程工具、每一个编程类的进化，以及新软件的寒武纪大爆发。这是否意味着编码已死？不要！它不能取代程序员理解代码的需要。这意味着编码变得更容易，影响更大，因此也更重要，就像穿孔卡片被键盘取代，或者格蕾丝·赫柏发明了编译器一样。

事实上，今天对软件的需求比以往任何时候都大，而且只会继续增长。随着这项技术的发展，Al 将在生成代码方面发挥更大的作用，这将成倍提高计算机科学家的生产力和影响，并将使越来越多的计算机程序员能够进入这一领域。

已经有一些工具可以让你只通过拖放来编程，或者让用你的声音写代码。这些技术和新工具的改进，如 OpenAI Codex，将使创建软件的能力日益民主化。因此，世界上的代码量以及编码者的数量将会增加。

这也意味着学习如何编程——以一种新的方式——比以往任何时候都更重要。学习编码可以打开机会之门，也有助于解决全球性问题。随着创建软件变得越来越容易和容易，我们应该给每个学校的每个学生基础知识，让他们不仅成为技术的使用者，而且成为创造者。
