Rutter 的电子商务 API 获得了 150 万美元的资金

原文：https://web.archive.org/web/https://techcrunch.com/2021/08/20/rutter-comes-out-of-stealth-with-1-5m-in-funding-for-its-e-commerce-api/

远程优先公司 Rutter 正在开发统一的电子商务 API，使公司能够通过任何平台连接数据。

上周五，该公司宣布从秘密行动中脱颖而出，获得了包括 Haystack、Liquid 2 和 Basis Set Ventures 在内的一批投资者的 150 万美元资金。

创始人郁鑫和彼得·周在学校相遇，并于 2017 年开始在鲁特工作，周称之为“商业格子”，然后参加了 2019 年夏天的 Y Combinator 队列。

他们去年在教育技术领域工作时偶然发现了电子商务 API 的想法。这对夫妇为关心孩子在全球疫情期间如何学习的父母制作了订阅包和学习材料。然后他们的供应商客户在亚马逊上列出他们的店面有问题，所以他们编写脚本来帮助他们，但发现他们必须为每个平台编写单独的脚本。

有了 Rutter，客户只需要一个脚本就可以连接到任何地方。Yu 告诉 TechCrunch，其 API 连接到 Shopify、沃尔玛和亚马逊等电子商务平台，以便技术客户可以建立客户支持和聊天机器人等功能。

Basis Set Ventures 的创始人兼管理合伙人兰·赵雪(LAN Schmidt)通过电子邮件表示，她对 Rutter first 感到“超级兴奋”，因为创始人对产品的热情、毅力和迭代速度。她补充说，这让她想起了另一个团队，他们成功地建立了一个从零到超过 70 亿美元的企业。

“在观察他们(鲁特)几年后，很明显他们建立的东西是强大的:这是在线商务的中枢神经系统，”赵雪补充道。

正如创始人所看到的，电子商务正在发生两次大爆炸:平台方面，无头商务的采用——电子商务网站前端和后端功能的分离，以及新公司的出现，以支持商家。

新的资金将使俞和周能够建立他们的团队，包括雇用更多的工程师。

由于该公司在今年年初正式启动，于没有透露收入指标，但确实表示，拉特的 API 量在过去几个月里翻了两倍和三倍。它还支持与 5000 多家商店有联系的商家。

Rutter 的一些客户正在构建电子商务的一个方面，如退货、保修和结账，但于表示，由于 Shopify 仅占电子商务的 10%，该公司的目标是通过成为“商家找到产品的统一应用商店”，将商家带到市场之外。

“我们认为，在未来，一个商家的电子商务栈将看起来像一个软件公司的 SaaS 栈，”周补充说。“我们希望成为一种粘合剂，为商家将这一堆东西粘在一起。”
