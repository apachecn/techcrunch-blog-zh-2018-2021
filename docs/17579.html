随着收购安全初创公司 CMD TechCrunch，弹性收购热潮仍在继续

原文：https://web.archive.org/web/https://techcrunch.com/2021/08/25/elastic-acquisition-spree-continues-as-it-acquires-security-startup-cmd/

在 Elastic 宣布收购 build.security 的几天后，该公司正在进行另一项证券收购。作为今天下午第二季度收益公告的一部分，Elastic 披露它正在收购加拿大温哥华的安全供应商 CMD。该交易的财务条款尚未公开披露。

CMD 的技术为云基础设施提供运行时安全性，帮助组织更好地了解正在运行的流程。这家初创公司成立于 2016 年，迄今为止已经筹集了 2160 万美元的资金。公司上一轮是 2019 年宣布的 a【1500 万美元 B 轮 ，由 GV 领投。

Elastic 首席执行官兼联合创始人 Shay Banon 告诉 TechCrunch，他的公司将欢迎 CMD 的员工进入他的公司，但没有透露具体有多少人会过来。CMD 首席执行官兼联合创始人 Santosh Krishan 和他的联合创始人 Jake King 都将在 Elastic 担任高管职务。

build.security 和 CMD 都将成为 Elastic 安全组织的一部分。这两项技术将集成到弹性堆栈平台中，该平台提供了对组织运行情况的可见性，以及有助于限制风险的安全洞察。Elastic 近年来一直在稳步增长其安全能力，于 2019 年以 2.34 亿美元收购了end game Security。

Banon 解释说，随着越来越多的组织转向云计算并使用 Kubernetes，他们正在寻找更多的 Linux 内省和保护层。这就是 CMD 技术的用武之地。CMD 的安全服务是用被称为eBPF的开源技术构建的。使用 eBPF，可以挂钩到 Linux 操作系统来实现可见性和安全控制。目前正在为 Windows 工作负载扩展 eBPF。

CMD 并不是唯一一家基于 eBP 的创业公司。 等价 ，2020 年 11 月宣布由安德森·霍洛维茨和谷歌牵头的、2900 万美元 A 轮也活跃在该领域。Linux 基金会最近也宣布成立一个 eBPF 基金会 ，参与方包括脸书、谷歌、微软、网飞和 Isovalent。

从根本上说，Banon 认为 CMD 正在构建的东西和 Elastic 旨在为用户提供的东西之间存在明显的一致性。

“我们在 Elastic 有一句谚语——当你观察时，为什么不保护？”巴农说。“对于 CMD，如果你观察他们所做的一切，你会发现他们也有这种深深的热情和信念，即它始于可观察性。”

Elastic 将 CMD 技术整合到 Elastic Stack 中还需要时间，尽管时间不会太长。巴农指出，收购一家初创公司的好处之一是，它通常比更大、更成熟的供应商更容易整合。

巴农说:“在我们进行的所有这些收购中，我们花时间将它们整合成一个单一的产品线。

巴农解释说:这意味着 Elastic 需要采用其他公司已经开发的技术，并将其整合到它的堆栈中，这有时需要时间。他指出，收购后花了两年时间整合 Endgame 技术。

巴农说:“一般来说，这有助于我们与拥有真正创新技术的小公司联手，这些技术更容易被采用并集成到我们的堆栈中。”。
