辛烷银行$2M 灵活的计费软件

原文：https://web.archive.org/web/https://techcrunch.com/2021/08/31/octane-banks-2m-for-flexible-billing-software/

软件计费初创公司 Octane 周二宣布，以 1000 万美元的估值融资 200 万美元，以推进其现收现付计费软件。

阿卡什·卡诺尔卡和他的联合创始人十年前在卡内基梅隆大学相遇，从那以后，他们走向了不同的方向。在 Khanolkar 的案例中，他经营着一家云咨询公司，看到了 Datadog 和 Snowflake 等公司进入市场和处理亚马逊网络服务的速度。

卡诺尔卡告诉 TechCrunch，他发现所有这些快速增长的公司的共同点是计费软件使用现收现付的商业模式，而不是传统的统一费率计划。

然而，他解释说，监控消费意味着计费变得复杂:公司现在必须跟踪客户每秒钟使用软件的情况，以便每月正确计费。

Khanolkar 说，看到基于消费的计费方式的转变，联合创始人在 2020 年 6 月一起创建了 Octane，这是一个计量计费系统，可以帮助供应商以类似于雪花和 AWS 的方式创建计划，监控使用和收费。

“我们是 API 驱动的，你作为供应商将向我们发送使用数据，在我们这边，我们存储这些数据，然后进行实时汇总，因此在月底，你可以相应地向客户收费，”Khanolkar 说。“我们已经看到工程和产品之间的竞争。工程师在那里创建核心计划，所以我们为产品团队建立了一个无代码体验，以便能够创建新的价格计划，然后进行更改，如添加优惠券。”

在全球云计费市场中，预计到 2025 年将达到 65 亿美元，有一系列 Octane 竞争对手，如 Chargebee 和 Zuora，Khanolkar 表示，他们正在解决订阅管理方面的问题，并在过去几年中取得了成功。现在，一个基于使用和消费的世界即将到来，一系列全新的软件业务，如 Octane，将在那里取得成功。

新一轮融资由 Basis Set Ventures 牵头，包括 Dropbox 联合创始人阿拉什·菲尔多西、Github 首席技术官杰森·沃纳、Fortress 首席技术官阿苏塔·加廖内、Scale AI CRO·切坦·乔德里、Twilio 前高管埃文·库马克、伊斯特班·雷耶斯、Abstraction Capital 和 Script Capital。

“随着以产品为导向的增长和基于使用量的定价模式的兴起，基于使用量的计费是一个关键和基础的基础设施，而这正是我们所缺少的，”Basis Set Ventures 的合伙人常旭通过电子邮件表示。“同时，这是每个部门都关心的事情，因为这是您的收入。我们与之交谈的许多后期公司已经建立了这种关于持续维护成本的内部对话，并希望有一个供应商可以将它外包出去。”

我们对 Octane 团队致力于构建同类最佳且强大的基于使用的计费解决方案印象深刻。他们已经通过与许多工程团队交谈验证了这一机会，因此他们可以解决所有边缘情况，这在像计费这样的关键任务中非常重要。我们确信辛烷将成为科技基础设施不可避免的一部分。"

新资金将主要用于雇佣工程师以及产品、营销和销售人员。Octane 目前有 7 名员工，Khanolkar 预计到今年年底会有 10 名左右。

该公司正在与许多公司合作，主要集中在基础设施和深度计行业。Octane 还看到一些独特的用例出现，如一家建筑公司使用电表来跟踪员工的工作时间，以及一些充电公司出于这些目的使用电表。

“我们没有想到建筑工人会使用它，但理论上，它可以被任何跟踪时间的公司使用——甚至是合法的，”Khanolkar 补充道。

他拒绝谈论公司的收入，但他说现在还有两到三年的时间。

接下来，该公司计划推出新功能，如基于使用情况的价格实验，以帮助客户更好地决定如何为他们的软件定价，这是 Khanolkar 看到的另一个问题。它将建立一些方法，让客户可以根据使用数据尝试不同的计划，以验证哪一个效果最好。

“我们仍然处于基于消费的模式的早期阶段，但我们看到更多的最终用户选择与希望让他们试用软件，然后按需付费的企业合作，”他补充说。
