Grammarly SDK 测试版允许开发人员在任何 web 应用程序中嵌入自动文本编辑技术

原文：https://web.archive.org/web/https://techcrunch.com/2021/09/14/grammarly-sdk-beta-lets-developers-embed-text-editing-into-any-web-app/

Grammarly SDK beta 允许开发者在任何 web 应用中嵌入自动文本编辑

热门自动编辑工具 Grammarly 今天宣布面向开发者发布 Grammarly。该公司正在用文本编辑器 SDK(软件开发工具包)开始这项工作，它使程序员能够将语法文本编辑功能嵌入任何 web 应用程序。

Grammarly 的产品和平台负责人 Rob Brazier 表示，这个 SDK 的测试版让开发人员可以通过几行代码获得 Grammarly 自动化编辑的全部功能。Brazier 告诉我:“实际上，只需几行 HTML 代码，[开发者]就可以将 Grammarly 的帮助添加到他们的应用程序中，他们的所有用户都可以获得原生的 Grammarly 体验，而无需用户安装或注册 Grammarly。”。

在引擎盖下，这些开发人员正在获得高度复杂的自然语言处理(NLP)技术，而不需要任何人工智能理解或经验。相反，开发人员可以利用 Grammarly 已经完成的工作。

虽然目标应用程序的用户不需要成为语法上的客户(事实上这就是我们的想法)，但是如果他们碰巧是，他们可以登录到他们的语法帐户，并访问随之而来的所有功能。“如果他们的用户有语法订阅，这些用户可以将他们的语法帐户链接到开发者的应用程序中。他们可以使用 Grammarly 登录，并[直接]在该应用程序中解锁他们特定订阅的附加功能，”他说。

Brazier 说，因为这是一个起点，该公司希望保持它的基本功能，获得测试版的反馈，然后在未来添加额外的功能。“我们希望从最简单的方式开始，让尽可能多的用户获得这一功能。这就是为什么我们从一个非常简单的产品开始。我认为它会随着时间的推移而发展，变得更加复杂，但它实际上只是几行代码，你就可以开始运行了，”他说。

这是该公司首次涉足开发者工具领域，允许程序员访问语法功能并将其嵌入到他们的应用程序中。这与 Zoom 去年发布 SDK 进军视频服务时所采取的方法没有什么不同(尽管 Zoom 在开发者工具之旅中走得更远)。随着 Grammarly 和 Zoom 等公司越来越受欢迎，似乎下一个合乎逻辑的步骤是展示该平台的优势，在这种情况下是文本编辑，让开发人员利用它。事实上，2007 年推出 Force.com 时，Salesforce 是第一个实施这一想法的 T4 公司。

这种方式也有可能为 Grammarly 提供该产品订阅版本之外的另一个收入来源，尽管 Brazier 表示现在谈论这种形式还为时过早。无论如何，今天的公告只是一个更广泛战略的第一步，该战略将平台的不同部分暴露给开发者，并使他们能够利用 Grammarly 的工程师投入平台的所有工作。感兴趣的开发者可以申请成为测试程序的一部分。
