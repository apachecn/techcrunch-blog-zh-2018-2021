Crypto 的网络化协作将推动 Web 3.0

Harry Alford 是比特币基地云公司(前身为 Bison Trails)的业务开发经理，负责领导主要金融机构、基金、交易所和托管机构的关键协议和合约的销售和业务开发工作。

web 1.0 是静态 Web，Web 2.0 是社交 Web，但 Web 3.0 将是去中心化 Web。它将把我们从一个社区贡献但不拥有或获利的世界，带到一个他们可以通过合作的世界。

通过脱离以大公司利益为中心的传统商业模式，Web3 带来了以社区为中心的规模经济的可能性。这种协作精神及其相关的激励机制正在吸引当今一些最有才华和雄心的开发人员，开启以前不可能的项目。

Web3 可能不是最终答案，但它是当前的迭代，创新在开始时并不总是显而易见的。

正如 Ki Chong Tran 曾经说过的，Web3 是“互联网的下一次重大迭代，它有望从今天主宰网络的中央集权公司手中夺取控制权。”没有单一实体控制的分散式网络使 Web3 支持的协作成为可能。

在闭源商业模式中，用户信任企业来管理资金和执行服务。在开源项目中，用户相信技术可以完成这些任务。在 Web2 中，更大的网络胜出。在 Web3 中，谁一起构建了最大的网络，谁就赢了。

在一个去中心化的世界里，不仅所有人都可以参与，而且激励机制的设计使得参与者越多，每个人就越成功。

向 Linux 学习

Web2 的大部分网站背后的 Linux 改变了互联网的发展模式，并提供了一个协作过程如何驱动技术未来的清晰例子。Linux 不是由现有的科技巨头开发的，而是由一群志愿程序员开发的，他们使用网络协作，即人们在没有中央控制的情况下自由共享信息。

在“The Cathedral&The Bazaar”中，作者 Eric S. Raymond 分享了他对 Linux 内核开发过程的观察以及他管理开源项目的经验。Raymond 描述了一个时代，当时流行的思维模式是开发复杂的操作系统，并由一个小型的、排外的团队——“大教堂”，即公司和金融机构进行仔细协调。

Linux 以完全不同的方式发展。Raymond 解释说，“质量不是由严格的标准或专制来维持的，而是由每周发布并在几天内获得数百名用户反馈的天真简单的策略来维持的，这在某种程度上是对开发者引入的突变的达尔文选择。让几乎所有人都感到惊讶的是，这种方法非常有效。”这个 Linux 开发模型，或者如 Raymond 所说的“集市”模型，假设当暴露在一群没有有效协调的黑客面前时，“错误通常是肤浅的现象”。
