Xata 是用于无服务器应用的数据库服务

原文：https://web.archive.org/web/https://techcrunch.com/2021/09/19/xata-is-a-database-service-for-serverless-apps/

认识一下对托管数据库有新看法的初创公司 Xata 。该公司为您运行您的数据库，并将其转换为 API，以便您可以从您的无服务器应用程序中查询和更新它。Xata 已经筹集了 500 万美元的资金。它的产品还没有准备好上市，但是公司正在分享细节。

Xata 似乎特别适合 Jamstack 网站。Jamstack 一直是大规模开发和部署网站的流行方式。流行的 Jamstack 托管平台包括 Netlify 、 Vercel 和 Cloudflare Pages 。

应用程序部署在全球边缘网络上，大部分逻辑由 API 调用处理。结果是一个网站或应用程序加载速度很快，可以处理大量的流量。

部署 Jamstack 网站非常容易，因为它通常与您的 Git 存储库紧密集成。当您提交代码更改时，无服务器平台会负责部署您的应用程序。与基于 API 的开发工具集成相对容易，而且您不需要自己管理逻辑。

例如，部署一个包含静态内容和 Stripe 结帐模块的网站并不需要太多的努力——Stripe 为您管理支付服务器。如果你想使用一个实时数据库并与之交互，事情会变得复杂一些。传统的数据库软件不依赖于通过互联网的 API 调用来添加一行、搜索多行并找到数据。

Xata 专注于数据库，并希望更容易地将数据库与您的无服务器应用程序集成。您不必关心底层基础设施，因为 Xata 可以为您扩展数据库。您不必更新软件、将数据移动到新服务器等。

您的数据库分布在多个数据中心，以改善响应时间和冗余。它支持许多数据类型，包括图像。之后，与数据库的交互就像任何 RESTful API 一样。

这家初创公司还从流行的无代码初创公司(如 Airtable)汲取了一些灵感。您可以在 web 浏览器中打开数据库，并从那里直接与数据进行交互。例如，您可以过滤当前视图，使用特定标准对数据进行排序，并获得可以在代码中使用的 API 查询。

如果在数据库中存储了大量数据，可以使用自由文本搜索功能搜索数据。您还可以通过创建图表和可视化来利用 Xata 进行分析。

从 web 浏览器与您的数据进行交互的能力是 Xata 的竞争优势。许多公司依赖 Airtable 作为他们的第一个后端来构建新项目的原型。Xata 可以成为这种 air table as-a-back end 数据管理模型的生产就绪版本。

500 万美元的融资由 Index Ventures 牵头。运营商集体，SV 天使，X 因子；firstminute Capital 也参与其中。一些商业天使，如 Elastic 的 Shay Banon 和 Uri Boness，Confluent 的 Neha Narkhede，Vercel 的 Guillermo Rauch，Color Genomics 的 Elad Gil 和 Netlify 的 Christian Bach 和 Mathias Biilmann 也进行了投资。

这家初创公司由莫妮卡·萨布(Monica Sarbu)创立，她曾是 Elastic 的工程总监。所以她可能对扩展数据库略知一二。
