Rize 融资 1140 万美元，扩大其嵌入式金融科技服务

原文：https://web.archive.org/web/https://techcrunch.com/2021/09/28/rize-raises-11-4m-to-scale-its-embedded-fintech-service/

为其他公司提供服务的金融科技即服务公司 Rize 今天早上宣布了 1140 万美元的 A 轮融资。此次融资活动由阿尔法·爱迪生和墨菲斯风险投资牵头。这家初创公司的一些早期投资者也参与了这轮融资。

Rize 是一个竞争空间里有趣的公司；构建供客户使用的金融科技产品的市场拥有众多玩家。Rize 希望通过允许其客户将一个 API 用于多种帐户类型来竞争，将其产品统一到一个开发者挂钩上。

更重要的是，Rize 建立了一个它称之为“合成账户”的东西。对于我们这些最后一次在金融环境中听到“合成”这个词的人来说，这可能听起来令人担忧。不是的。相反，合成账户允许 Rize 为其银行服务的客户提供一种方式，为最终用户创建一个统一的账户，而不局限于单一的托管账户类型。在实践中，合成账户允许 Rize 向最终用户(其客户的客户)提供将不同的保管账户(支票、经纪)组合成单一体验的能力。

根据 Rize 的一份白皮书，通过提供单一的“合成账户核心”，其客户可以为最终用户创造更具凝聚力和可编程的金融体验。随着 Rize 自身扩展其功能集，合成账户系统将允许 Rize 客户向其金融技术产品添加更多服务。

Morpheus Ventures 的霍华德·柯(Howard Ko)在给 TechCrunch 的一份声明中提到了该公司的合成会计工作，称正是其服务的这一特点“吸引”了他的公司加入这家初创公司，因为它“基本上抽象了开发广泛的金融科技产品和服务所需的所有管道，开发人员正在寻求将这些产品和服务构建到一个简化的单一 API 连接中。”

在与高层领导的采访中，Rize 告诉 TechCrunch，它已经在其产品中建立了一个合规管理系统，将合规工作比作电子商务世界的物流——这是一项既复杂又不是人们喜欢承担的任务的关键工作。Rize 希望减轻这一负担，这可能为企业选择其产品而不是竞争对手的银行即服务产品铺平道路。

Rize 已经完成了其产品的核心银行业务——我们今天看到的初创公司是从 B2C 转向 B2B 的结果——接下来是经纪业务的支持。新资本应该有助于公司扩大产品组合。

值得注意的是，Rize 的领导层告诉 TechCrunch，它比预期提前筹集了 A 轮融资。这家初创公司曾预计市场牵引会放缓，但高于预期的需求意味着 Rize 有更大的人力资本需求。因此，外部资金是显而易见的选择。

Rize 以大约 15 人的团队执行了其 B2B 轴心，如今团队成员已经超过 30 人。该公司预计到年底将拥有近 50 名员工。

最终，银行即服务(BaaS)领域将会出现整合。但我们可能看不到那个波峰，直到创业公司可以蚕食的空白空间减少。到那时，像 Rize 这样的初创公司将希望成为等式的收购方。让我们看看 Rize 可以多快地利用这一系列 a 扩大客户和收入规模。
