不管怎样，谁需要 BaaS 的合作伙伴？

原文：https://web.archive.org/web/https://techcrunch.com/2021/09/30/who-needs-a-baas-partner-anyway/

在过去的几年里，一系列初创公司涌现出来，通过提供基于合作银行基础设施之上的技术，使开发者能够通过 API 提升银行账户、支付和卡功能，从而降低推出金融服务的复杂性。

这些银行即服务(BaaS)初创公司承诺向其他公司提供金融科技能力，而不必与合作银行达成交易，与银行核心整合，或雇用测试或推出新金融产品所需的技术或合规人员。

因此，在现有的垂直 SaaS 业务中，推出一款金融科技应用或添加一个银行组件从来没有这么快或这么容易过。为了更好地了解 BaaS 提供商试图解决的问题，我们采访了该领域的几位创始人，包括 Unit 首席执行官 Itai Damti、Bond 首席执行官 Roy Ng 和 Synctera 首席执行官 Peter Hazlehurst 等人。

突破金融服务的复杂性

在金融科技市场的早期，初创公司在构建或推出新的金融服务应用程序时，基本上都是靠自己。将一种新的金融产品推向市场通常意味着找到一家银行合作伙伴并签署一份长期合同，与该银行一起制定并实施合规政策，然后最终开发出支持您希望向最终用户提供的任何金融应用或服务所需的技术。

对于创业公司来说，这意味着大量的时间和金钱的前期投资，只是为了在建立产品市场匹配之前为推出新产品奠定基础。这还意味着大量重复工作，不仅是初创公司需要建立推出金融产品所需的必要基础设施，银行之间也需要重复工作，因为它们签署了协议，并向金融科技合作伙伴提供了访问其银行系统的权限。

Unit 创始人兼首席执行官 Itai Damti 表示:“推出哪怕是简单的东西，甚至只是检查账户，都会涉及大量的工作和痛苦——你需要浏览由 30 至 40 家银行组成的合作伙伴生态系统，这些银行并不总是了解你的业务，然后你需要制定 15 至 20 项合规政策并实施它们。

现在，许多复杂工作可以外包给 BaaS 公司，这些公司已经拥有银行关系、用于将金融服务嵌入其应用程序的 API，以及代表客户运行合规程序的能力。风险投资家已经在排队为他们提供资金，包括过去三个月里由Rize(1140 万美元)、Synctera(3300 万美元)和Unit(5100 万美元)等公司宣布的几轮融资。
