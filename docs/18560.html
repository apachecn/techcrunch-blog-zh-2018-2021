霍华德·派尔是……的创始人

ExperienceFutures.org

这是一家致力于将公平融入数字体验的非营利组织，曾在大都会人寿和 IBM 领导品牌设计计划。

我最近与一家大型科技公司进行了一次对话，他们想知道他们在以人为中心的设计方面的工作是否能防止经验偏见。简短的回答？大概不会。

当我们说体验偏见时，我们不是在谈论我们自己的认知偏见；我们在数字界面层(设计、内容等)讨论它。).事实是，你接触的几乎每一个应用和网站，要么是基于创建团队的认知和能力，要么是为一两个高价值用户设计的。如果用户没有设计惯例的经验，缺乏数字理解，没有技术途径，等等。，我们会说经验对他们有偏见。

解决方案是转变思维模式，让组织创建多个版本的设计或体验，以满足不同用户的需求。

回到我与之交谈的这家科技公司，任何公司在移情设计上的投资都是必不可少的，但是，作为一个已经启动并运行设计功能的人，我们需要解决一些肮脏的秘密。

第一个是 UX 和设计团队经常被一个战略或业务功能指示在非常有限的目标用户上，并且经验偏见从那里开始。如果企业不优先考虑用户，那么设计团队就没有权限或预算来为他们创造体验。因此，即使公司追求以人为中心的设计或采用设计思维，他们也经常只是根据商业利益重复用户档案，而不是根据文化、种族、年龄、收入水平、能力、语言或其他因素来定义多样性。

另一个肮脏的秘密是，以人为中心的设计经常假设人类设计所有的 UX、服务和接口。如果解决体验偏差的方法是根据用户的不同需求创建定制的变化，那么这种手工制作的 UI 模型是行不通的，尤其是当团队缺乏多样性的时候。根据用户需求对各种体验进行优先排序，需要对设计流程进行根本改变，或者利用机器学习和自动化来创建数字体验——这两者都是向体验公平转变的必要条件。

如何诊断和解决经验偏见

解决经验偏见始于理解如何诊断它可能出现在哪里。这些问题有助于理解您的数字体验中的问题所在:

内容和语言:内容对个人有意义吗？

许多应用程序需要特殊的技术理解，使用面向公司或行业的行话，或者假设技术知识。

任何金融服务或保险网站——假设你了解他们的术语、行业和术语。如果代理人或银行家为你翻译的日子一去不复返了，那么数字体验需要为你翻译。

UI 复杂性:基于我的能力，界面有意义吗？

如果我有残疾，我可以使用辅助技术导航吗？我需要学习如何使用用户界面吗？一个用户导航界面的方式可能会因能力或环境的不同而有很大的不同。

例如，为老龄化人口设计将优先考虑更多的文本和不那么微妙的视觉线索。相比之下，年轻人倾向于在颜色编码或预先存在的设计惯例方面做得很好。想想可怕的新冠肺炎疫苗网站，它们让你难以理解如何导航和预约，或者你的每家银行如何以完全不同的方式导航到类似的信息。过去，初创公司的用户界面非常简单，但一个接一个的功能让它们变得复杂，即使对于资深用户来说也是如此——只要看看 Instagram 在过去五年里的变化就知道了。

生态系统的复杂性:你把无缝导航多重体验的责任放在用户身上了吗？

我们的数字生活不是以一个网站或应用为导向的——我们在网上做任何事情都使用一系列工具。几乎每个数字业务或产品团队都渴望将用户锁定在他们的围墙花园中，很少考虑用户可能会遇到的其他工具，无论他们在生活中试图完成什么。

如果我生病了，我可能需要与保险公司、医院、医生和银行打交道。如果我是一名新的大学生，我可能需要在学校的多个系统中工作，还有供应商，住房，银行和其他相关组织。如果用户难以在一个生态系统中整合不同的体验，他们总是要受到责备的。

遗传偏见 : 你是否在使用生成内容的系统、为不同目的构建的设计模式或机器学习来个性化体验？

如果是，你如何确保这些方法为你设计的用户创造了正确的体验？如果我们利用来自其他系统的内容、用户界面和代码，你就会继承那些工具中的偏见。一个例子是现在可用的几十种人工智能内容和副本生成工具——如果这些系统为你的网站生成副本，你就会将它们的偏见引入到你的体验中。

为了现在就开始建立更加包容和公平的体验生态系统，需要新的设计和组织流程。虽然有助于生成更多定制数字体验的人工智能工具将在未来几年的前端设计和内容的新方法中发挥重要作用，但任何组织都可以立即采取五个步骤:

让数字权益成为 DEI 议程的一部分:虽然许多组织都有多元化、公平和包容性的目标，但这些目标很少转化为面向客户的数字产品。在大公司领导过设计，也在数字初创公司工作过，两者的问题是一样的:缺乏对整个组织中不同用户的明确问责。

事实是，无论是大公司还是小公司，各个部门都在竞争影响力和谁更接近客户。数字体验或产品的起点是在业务层面定义和区分不同的用户。如果在最高层存在创建数字和体验公平定义的要求，那么每个部门可以定义它如何服务于这些目标。
没有管理和资金支持，任何设计或产品团队都无法产生影响，首席执行官需要负责确保这一点得到优先考虑。

在你的设计和开发团队中优先考虑多样性 :关于这一点已经有很多文章了，但重要的是要强调，缺乏任何多样性视角的团队将基于他们的特权背景和能力创造经验。

我要补充的是，有经验为不同用户设计的人是很重要的。你的组织如何改变它的招聘过程来改善设计和开发团队？您与谁合作来帮助寻找多样化的人才？你的 DEI 目标仅仅是招聘表格上的复选框吗？在招聘你已经想好的设计师时，这些复选框会被绕过吗？你的机构有明确的和积极的多元化计划吗？他们在包容性设计方面有多精通？

谷歌的一些有价值的举措堪称典范:为了提高人才管道中的代表性，它已经将机器学习课程的资金从以白人为主的机构转移到更具包容性的学校，允许免费使用 TensorFlow 课程，并向 BIPOC 开发人员发送免费门票，以参加谷歌 I/O 等活动。

用重新定义你测试的内容和对象:通常，用户测试(如果发生的话)仅限于最有利可图或最重要的用户群。但是，你的网站如何与老龄人口或从不使用台式电脑的年轻用户合作呢？

经验中公平与平等的一个关键方面是开发和测试各种各样的经验。太多时候，设计团队测试一个设计，并根据用户反馈进行调整(如果他们真的在测试的话)。虽然这可能需要更多的工作，考虑到老年用户、只使用手机的人、来自不同文化背景的人等等的需求，创造不同的设计。允许您将设计与数字权益目标联系起来。

将你的设计目标从一个面向所有用户的设计转变为推出多个版本的体验:数字设计和产品开发的常见做法是根据最重要用户的需求创建任何体验的单一版本。在未来，任何应用程序或网站都不会只有一个版本，而是会有许多针对不同用户的迭代，这与大多数设计组织的资源配置和创作方式背道而驰。

然而，这种转变对于体验公平至关重要。问一些简单的问题:你的网站/产品/应用程序是否有针对老年受众的简单、大字体的变体？在为低收入家庭设计时，纯移动用户能完成你所期望的任务吗，就像那些会转向台式机来完成的人一样？

这不仅仅是让你的网站有一个响应的版本，或者测试各种变化来找到最好的设计。设计团队应该有一个目标，即推出多种专注的体验，这些体验直接联系到不同的优先用户和服务不足的用户。

拥抱自动化，为每个用户群创建内容和副本的变体:即使我们创建设计变体或对广泛的用户进行测试，我也经常看到内容和 UI 副本被认为是事后的想法；尤其是随着组织规模的扩大，内容要么变得更加术语化，要么被过度修饰以至于毫无意义。
如果我们从现有语言中复制(比如营销复制)并将其放入应用程序中，您将如何限制人们对该工具用途或使用方法的理解？如果经验偏差的解决方案是基于个人需求的前端设计的变化，那么我们可以大大加快的一个聪明的方法是了解自动化可以应用在哪里。

我们正处于一个新的人工智能工具悄然爆发的时刻，这些工具将从根本上改变用户界面和内容的创建方式。看看去年上线的复制驱动的人工智能工具的数量——尽管它们主要旨在帮助内容创作者更快地编写广告和博客文章，但不难想象在一个大品牌中定制部署这样一个工具，它可以获取用户数据，并动态地为他们生成 UI 副本和内容。老用户可能会得到更多没有行话的服务或产品的文字描述；Z 世代的用户可能会得到更多的参考文本和更多的图片。

无代码平台展示了一个类似的机会——从 WebFlow 到 Thunkable 的一切都表明了动态生成 UI 的可能性。虽然 Canva 的设计有时可能感觉很普通，但成千上万的企业正在使用它来创建视觉内容，而不是聘请设计师。

许多公司都在使用 Adobe 体验云，但似乎忽略了其中隐藏的体验自动化功能。最终，设计的角色将从手工定制体验转变为动态生成用户界面的策展人——看看过去 20 年来电影中的动画是如何发展的吧。

由机器学习和人工智能驱动的设计变化的未来

上述步骤旨在改变组织使用当前状态技术解决经验偏见的方式。但如果解决体验偏见的未来状态植根于创造设计和内容变化，人工智能工具将开始发挥关键作用。我们已经看到了一波由人工智能驱动的内容工具，如 Jarvis.ai、Copy.ai 和其他工具——然后还有内置于 Figma、Adobe XD 和其他平台的自动化工具。

可以动态生成前端设计和内容的人工智能和机器学习技术在许多方面仍处于萌芽状态，但我会举出一些有趣的例子来说明即将发生的事情。

第一个是谷歌今年早些时候发布的作品，它是为 Android 设备设计的系统，旨在为用户提供高度可定制性，并具有高度的内置可访问性。用户可以定制颜色、类型和布局，让他们拥有高度的控制权——但正在出现的机器学习功能可能会根据用户变量(如位置或一天中的时间)改变设计。

虽然个性化方面最初被定位为给用户更多的定制能力，但通读材料的细节可以发现在设计层与自动化有很多可能的交集。

同样重要的是，要指出组织围绕人们如何体验人工智能的设计原则和交互所做的工作；例如，微软的人类-人工智能体验计划，该计划涵盖了一套核心的交互原则和设计模式，可用于制作人工智能驱动的体验，以及即将推出的预测和设计人类-人工智能交互失败解决方案的剧本。

这些例子是假设的未来的指标交互和设计是由人工智能产生的——但迄今为止，这在现实世界中如何体现的例子非常少。关键是，为了减少偏见，我们需要进化到前端设计的变化和个性化急剧增加的地方，这说明了人工智能和设计的交叉领域正在出现的趋势。

这些技术和新的设计实践将汇聚起来，为组织创造机会，从根本上改变他们为用户设计的方式。如果我们现在不开始考虑体验偏差的问题，随着前端自动化新时代的到来，我们将没有机会解决这个问题。
