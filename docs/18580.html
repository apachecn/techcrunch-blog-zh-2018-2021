投资者分享基础设施作为代码如何接管开发运维

原文：https://web.archive.org/web/https://techcrunch.com/2021/10/04/investors-share-how-infrastructure-as-code-is-taking-over-devops/

代码为 (IaC)的基础设施近年来在开发运维团队中得到了更广泛的采用，但数据中心配置和管理的复杂性继续带来问题和机遇。

我们调查了 IaC 初创公司的顶级投资者，以了解更多信息。

总的来说，他们看到了很大的增长空间，因为所有的手动工作仍然由 DevOps 团队在各种规模的组织中完成。

然而，IaC 本身需要训练有素的工程师在组织内实现和管理，并且缺乏具有 IaC 技能的软件基础设施工程师。这可能有利于试图向客户提供完整解决方案的 IaC 初创公司。

与此同时，一些大公司将继续在内部管理数据中心，从而在内部而不是外部创业公司中构建自动化。

希拉·古拉提，托拉资本公司董事总经理

基础设施作为代码能否成为服务器实施和配置的解决方案，就像云是物理服务器的解决方案一样？您认为 IaC 设置任何云资源的能力将在哪些领域得到最广泛的应用？

现代云世界彻底改变了我们对基础设施的看法。我们生活在一个多云和许多云的世界，这些范式正在重新定义现代云时代。IaC 可用于任何类型的云工作负载或架构，但对于任何构建现代云的人来说，它都是必不可少的。对于现代云架构来说尤其如此，比如无服务器应用、运行 Kubernetes 的容器化应用、AI/ML 等等。现代云架构提供了许多优势，如增加创新、加快上市速度、提高可靠性和降低成本。但是，这增加了管理云基础架构的负担。可用云服务的数量每年都在增长，现代架构由许多松散耦合、相互依赖的服务和 API 组成。结果是，人们必须管理的云资源数量正在以惊人的速度增长。管理这种复杂性的唯一方法是使用 IaC。

今天，我们看到了新一代的 IaC 平台，它们是为满足现代云的需求而全新设计的。例如， Pulumi 及其云工程平台正在帮助基础设施团队和开发人员驯服云复杂性，让他们能够用自己熟悉和喜爱的语言编写代码，并使用软件工程实践来构建、部署和管理云基础设施。与使用领域特定语言的传统 IaC 工具相比，现代 IaC 平台为实践者提供了编程语言的全部能力，这使得管理现代云应用程序的复杂相互依赖关系变得更加容易。它们允许开发人员使用现有的软件开发工具，包括 ide、包管理器和测试框架，这使他们能够更快地构建和重用最佳实践，同时更频繁地进行测试。

你是否认为 IaC 简化流程的能力吸引了竞争行业的初创公司，让他们的产品率先通过生产线？

是的。在我们的投资组合中，接受 IaC 和云工程原则的公司也是最专注于快速产品开发和简化其能力以更快地将新产品推向市场的公司。这需要对工程人才、平台和工程实践进行投资。

重要的是利用新的云创新来加速创新，并利用 IaC 来构建更多产品，以高效地满足客户需求。为了支持这一点，IaC 领域内的各个提供商之间的合作至关重要，以提供协同工作的组件来支持工程团队。

此外，我们已经看到倾向于更现代的开发平台的团队能够更容易地雇用更好的开发人员——这是当今每个科技公司都关心的问题。更向前发展和真正拥抱左移运动的文化是最好的开发人员想要工作的地方！

为什么公司对采用基础设施作为代码犹豫不决？你能概述一下 IaC 提供商如何推销自己来建立吸引力吗？

对于任何对大规模、可靠、高速地交付云应用感兴趣的公司来说，IaC 应该是显而易见的选择。过去的担忧现在已经过时了。然而，由于所需的文化和流程变化、迁移现有应用程序的风险，或者他们的团队可能没有合适的技能，公司可能会犹豫是否采用它。过去，我们也有 IaC 平台，需要使用特定领域的语言。所有这些加在一起会让生产率的上升看起来很陡。
