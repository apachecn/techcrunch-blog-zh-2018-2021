Mindee 的 API 自动解析文档，无需手动数据输入 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2021/10/06/mindees-api-automagically-parses-documents-without-manual-data-entry/

Mindee 提供了一个 API，可以让你将纸质文档中的原始数据转换成结构化数据。这家初创公司正在走出隐形模式，并已获得由 GGV 资本牵头的 1400 万美元的首轮融资。

今天这轮融资的其他投资者包括 Alven、Serena Capital、Bpifrance 的数字风险基金以及几位商业天使，如 Algolia 联合创始人 Nicolas Dessaigne 和 Datadog 联合创始人 Alexis lê-quc。

如果你正在开发一个涉及保存纸质收据、扫描驾照或在系统中输入发票的产品，那么你很可能一直在寻找尽可能自动化这一过程的方法。许多公司依赖谷歌或亚马逊的 OCR 服务(光学字符识别)。

但是将文本转换成原始的数字文本并不是最复杂的步骤。之后，您希望将原始数字文本转换成结构化数据。这要复杂得多，因为公司通常依靠人类来识别正确的信息并将其粘贴到正确的数据字段中。

本质上，您可以通过 Mindee API 处理照片，并将其转换为您产品的相关数据，而无需任何手动数据输入。许多行业都可以使用这种基于 API 的产品，从费用管理到采购、会计、贷款申请等等。

Mindee 的一些客户包括 Qonto、Payfit、Spendesk 和 Lucca。有了新的资金，这家初创公司计划在欧洲获得更多客户，并扩展到美国。

在幕后，该公司利用机器学习对大数据集的文档进行了算法训练。随着时间的推移，随着它处理更多的文档，它应该会变得更好。该公司每个月已经处理了数百万来自客户的页面。

这家初创公司根据交易量向客户收费。一旦你进入免费层，你就要为每处理一页付费。因为它是一个托管的 API，将 Mindee 集成到您的产品中并不需要对您的产品进行大量的修改。您调用一个 API 并接收结构化数据作为回报。

Mindee 的特别之处在于最终用户的体验要好得多。例如，当你试图注册一个需要你的护照信息的服务时，你通常需要输入你的个人信息，拍一张照片，然后等待。它会对某人进行 ping 操作，这样他们就可以确保注册系统和照片上的信息是相同的。

如果 Mindee 设法以近乎实时和近乎完美的准确度捕捉数据，你可以获得更快的体验。这可能会为 Mindee 的客户带来更多的客户。
