Cord 获得 1750 万美元，让更多开发人员加入其 API 进行实时协作 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2021/10/07/cord-gets-17-5m-to-get-more-devs-plugged-into-its-api-for-real-time-collaboration/

Cord 是一家初创公司，提供单一 API，为任何应用程序添加类似 Slack 的协作功能，已获得 1750 万美元的 A 轮融资。

这轮融资由欧洲风险投资指数(European VC Index)牵头，与产品开发基金挂钩，包括聘用工程师、设计师和产品经理。其他参与的投资者包括 NFX 和 Stride，以及天使投资人 Elad Gil，小杰夫·莫里斯，查理·桑赫斯特，盖伊·波德贾尼和马特·罗宾逊。

这家总部位于伦敦的初创公司去年才成立。它还没有透露客户的数量，但今天透露了与“对话数据收集”平台 Typeform 的合作关系——该平台使用 Cord 的 API 为其客户提供在设计在线表格时进行合作的能力。

Cord 表示，它正在运行的一个私人测试版已于 8 月份结束，当时它向公众开放，涉及“数百家”客户，包括其他未披露的 SaaS 制造商、初创公司和规模扩大公司。

这家初创公司告诉 TechCrunch:“我们最好的市场实际上是团队使用的 B2B SaaS，所以我们看到了从 HubSpot 等 CRM 中的用户使用它，Webflow 或 WordPress 等 CMS，Mailchimp 等营销工具，Amplitude 等 BI 工具，Retool 等无代码工具，以及你需要批准、帮助或反馈的任何其他地方。”

“现在，它更多地被来自早期创业公司的团队使用，以扩大规模，我们确实在 Typeform 的企业客户中看到使用，所以一些非常大的数十亿美元的技术公司和消费品牌是 Typeform 的用户 或大型电影工作室，他们在他们的 BI 工具中使用 Cord 。”

Cord 的 API——它标榜为一行 JavaScript——使付费客户能够快速地将一系列协作功能添加到自己的产品中，减轻了开发实时聊天、在线状态、注释、与任务管理器和 Slack 的集成、屏幕录制、音频消息、实时共同浏览和视频聊天等社交附加功能的压力。

显示 Cord 与 Typeform 集成的屏幕截图(图片来源: Cord)

世界上的信息工作者已经发现自己努力保持在所有渠道的顶端，他们需要检查松弛式通知、pings 和其他数字聊天，应该准备好这种社会噪音变得指数级恶化；本质上，它可能会像无处不在的背景辐射一样，扩散到每一个商业软件中。

Index Ventures 的合伙人简·哈默(Jan Hammer)在一份声明中对 Cord 的提价进行了评论，他预计抵制这一趋势将是徒劳的。

“远程工作的增长，加上软件即服务工具的激增，意味着团队在生产力和通信应用程序(如 Slack)之间来回移动会浪费时间。Cord 允许用户留在最能支持他们工作流程的应用中，但仍能获得协作的好处，”他建议道。“我们喜欢 Cord 添加到现有应用程序中——这不是客户必须采用的另一种解决方案，而是他们已经习惯的解决方案。最终，我们相信 SaaS 工具公司将会开箱即用。”

Cord 的卖点是，它的 API 通过让公司增加社交维度变得超级简单，节省了他们“数年”的开发时间——从而抢在更多孤立的同行之前。

“大多数网络产品将在未来几年成为多人游戏体验，”T2 Cord T3 的首席执行官兼联合创始人 Nimrod Priell 在一份声明中表示。“我们已经看到它正在发生: Figma 在 4 年内达到 400 万用户，并一举击败 Adobe、Invision 和 Sketch，成为主导的原型设计应用程序——这在很大程度上要归功于其内置的协作。

“如果 Figma 是‘设计的谷歌文档’，我们就能让每件产品成为各自行业的谷歌文档。”

普里尔是前脸书产品经理。而 Cord 的另一位联合创始人、首席技术官杰克森·加巴德(Jackson Gabbard)是一名早期的脸书工程师，负责开发其平台的一些成型功能以及数万名脸书员工使用的内部工具套件。

在竞争方面，Cord 并不是唯一一家试图向其他公司出售添加实时协作功能的简单途径的初创公司——例如，它指出另一个团队正在构建类似的 API。

还有其他的初创公司，比如 Liveblocks，或者在协作管道的较低层次上，Ably。

虽然谷歌在近十年的大部分时间里一直在为开发者提供一个 API 来为他们自己的应用添加谷歌文档风格的协作。

但 Cord 也辩称，其主要竞争对手是内部开发人员在内部构建协作工具的努力——向其他企业进行 SaaS 式的宣传，让他们不要浪费宝贵的时间和金钱来投资他们的“核心体验和使命”,而是将“高质量沟通体验”的构建外包给另一家第三方提供商，这种体验与 Slack 和 Teams 等核心服务相集成，并由专门的 API 构建团队进行更新。
