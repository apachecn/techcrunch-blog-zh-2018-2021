BaaS 有三个作用:近距离观察一个快速发展的市场

BaaS 市场发展迅速，希望通过利用这些一揽子银行应用编程接口快速提升新金融服务的公司也有了更多选择。

我们最近看了一下谁是银行即服务初创公司的理想客户和合作伙伴…今天我们将深入探讨不同类型的 BaaS 产品以及它们的组合情况。

市场上的一些 BaaS 参与者已经与银行合作伙伴合作，为他们的服务提供 API 访问，其他人提供软件，并正在运行一个市场模型，以匹配社区银行和金融科技公司，至少有一家初创公司已经收购了一家银行，以建立自己的银行 API 客户可以访问。

如果你想开发一个新的金融科技应用，或者想在现有业务中增加银行、借记卡或其他金融服务，了解这些竞争对手如何定位与客户和银行合作伙伴的合作是关键。

统包银行服务

最简单——也是最常见的——BaaS 产品集合了公司推出金融服务可能需要的一切，并通过 API 提供。提供这些类型服务的公司包括 Synapse 、 Unit 和 Bond 等。

通常，这些 BaaS 提供商通过向客户收取平台费和/或分享客户最终产品或服务产生的交换收入或其他费用来赚钱。

对于刚刚起步的金融科技公司或希望在没有建立全新金融科技团队的情况下推出金融服务的垂直 SaaS 玩家来说，与他们的 API 或 API 集挂钩可能是一种低成本的构建、测试和部署方式，而且没有什么麻烦。
