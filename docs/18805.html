谷歌云的 BigQuery Omni 现已全面上市

原文：https://web.archive.org/web/https://techcrunch.com/2021/10/12/google-clouds-bigquery-omni-is-now-generally-available/

谷歌于去年夏天首次发布了基于 Anthos 的云计算数据分析解决方案 BigQuery Omni。今天，在其年度 Cloud Next 活动上，该公司宣布 BigQuery Omni 已全面上市。Omni 与类似服务的不同之处在于，它允许用户使用他们知道的标准 BigQuery 接口来查询位于其他云上的数据，包括微软的 Azure 和 AWS——而无需在这些云之间移动数据副本。

“每个人都知道数据孤岛是不好的，但现实是，现在我们正在建立数据孤岛，只是在不同的云中，”谷歌负责数据库、分析和观察者的副总裁兼总经理 Gerrit Kazmaier 告诉我。“现在有了新一代的数据孤岛，我们认识到世界上大多数公司都生活在一个多云的现实中，通过 BigQuery Omni，我们基本上为他们提供了跨云分析的可能性，我认为这是最简单的，因为你不再需要考虑我将数据移动到哪里，以及管理数据的所有重复和冗余。单一平台和单一处理框架——这是一项工程壮举，该团队基本上采用了 BigQuery，并将它的许多部分带到了其他云中。”

图片来源:谷歌

许多谷歌云客户已经很好地利用了这些功能。Kazmaier 指出，例如，强生公司使用它来将谷歌云中的数据与 AWS 上的 S3 数据相结合，而电子艺界使用它来将广告数据与游戏内购买相结合。

谈到数据仓库，谷歌今天还指出，Dataplex 是一种允许企业管理、监控和治理其跨数据湖和数据仓库的数据的工具，也将在今年晚些时候普遍推出。它目前正在预览中。

就像今天谷歌云的许多其他声明一样，这两项服务都认识到，几乎每个企业现在都在一个多云环境中运营——几乎默认地，这意味着许多有价值的数据分布在各个系统中。使用多个系统来管理这些数据既麻烦又容易出错，但最重要的是，将所有这些数据整合到一个系统中很难(而且通常很昂贵)。谷歌在这方面的做法是将所有自己的服务带到这些不同的云中，然后从 GCP 管理它们。这是 Anthos 用来管理 Kubernetes 集群的方法，不出所料，这种方法现在也允许它在自己的云之外运行一些核心 GCP 服务。
