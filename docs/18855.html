英国初创公司抨击政府降低数据保护的计划 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2021/10/13/so-much-for-levelling-up/?guccounter=1

后英国退出欧盟时代，英国政府希望“改革”国内隐私规则，降低人们数据的保护级别，这已经对该国的科技生态系统产生了更广泛的影响。

上个月，数字、文化、媒体和体育部(DCMS)宣布了一项关于降低隐私标准的咨询——声称“简化”的规则将有利于商业创新。

现在，一个本土的扩展在的一篇批评博客帖子中抨击了这一咨询——警告说，数据保护标准的任何降低“肯定”会损害其欧盟业务，甚至可能削弱其美国业务，因为许多州(如加利福尼亚州)已经通过了与欧洲通用数据保护法规(GDPR)类似的法律。

美国两党议员现在也在敦促通过全面的联邦隐私立法。因此——至少在英国以外——个人数据的流向是加强保护，而不是减少保护。

但在英国内部，部长们正在关注当前包装数据的高标准，并寻找降低这些保护的方法——表面上声称减少隐私权利对企业有益。

解除管制无疑意味着法律不确定性和商业风险的增加——以及潜在的大量商业损失。

在博客文章中， Cronofy ，一家 2014 年成立的英国初创公司，为企业销售日历 API 和日程安排平台，写道，它正在做准备，以防止国内解除管制的炸弹摧毁其业务——称它将在荷兰开设一家新公司，并为客户提供根据荷兰法律与 Cronofy BV 签订合同的能力。

“这将成为我们所有数据处理的新总部，因此我们可以接受荷兰数据监管机构的监督，从而接受欧盟的监督，”首席执行官兼联合创始人亚当·伯德写道。"我们新的总法律顾问是荷兰人，负责监督所有这些."

“英国如何摆脱这种局面？恐怕不太好，”他补充道，暗示重组也意味着 Cronofy 最终将减少对英国技能和工作岗位的投资。

伯德也不是唯一一个抨击英国废除数据保护规则的人。

英国新任命的信息专员约翰·爱德华兹(John Edwards)在上个月与国会议员举行的任命前听证会上为当前的数据保护规则进行了辩护，将英国的 GDPR 描述为“如何不是一个不要做的事情”。

而在本月早些时候，负责 DCMS 事务的前国务大臣埃德·韦泽(现在的韦泽勋爵)警告说，英国必须与 GDPR 保持一致——否则将面临经济和数字业务的“灾难性”后果。

Vaizey 上周告诉 TechCrunch，“当我们还是欧盟成员时，英国在如何制定数据保护立法方面非常有影响力，所以我认为我们应该回避这项立法有点奇怪。”

“你不希望自己处于一个容易受到欧盟攻击的位置，说你的数据保护制度不充分，因此我们不能进行跨境数据交换——那将是灾难性的。因此，不管我们喜欢与否，我们都必须在一定程度上与欧盟保持同步。”

然而，即使是来自 DCMS 的政策噪音似乎也对英国公司造成了损害。

在他的博客文章中，Bird 将 Cronofy 描述为“一家真正的全球性公司”，该公司(目前)总部位于英国，但收入分成 55%在美国，25%在欧盟，9%在英国。这意味着 91%的收入来自出口。

“欧盟 GDPR 立法没有损害我们的美国业务，在许多情况下还是一种优势，”他继续说道。“随着美国公司意识到必须保护人们的信息，从企业成立之初就必须面对数据隐私要求，这使我们处于明显的优势。”

伯德说，在英国退出欧盟“搞定”之前，“相当多”的欧盟客户已经开始担心英国的离开对他们的(敏感日历)数据以及与他的业务的关系可能意味着什么。

“我们将始终尽最大努力保护人们的隐私数据。然而，我们是在英国政府以“强有力的谈判”的名义哗众取宠的背景下做出这些断言的，甚至到了他们投票违反国际法的程度，”他继续说道，他说即使在过渡期结束之前，客户也不相信 Cronofy 能够信守承诺，或者英国政府会费心强制执行合规性，即使它在纸面上保留相同的数据标准。“更重要的是，他们无法让最终用户放心，”伯德补充道。

政府现在关于“简化”英国数据保护标准的噪音是 Cronofy 的“最后一根稻草”。

在咨询文件中，DCMS 谈到要进行“改革，以创建一个雄心勃勃的、有利于增长和创新的数据保护制度”，同时“保持高数据保护标准，而不为负责任的数据使用制造不必要的障碍”——但毫无疑问，该提案的目的是消除层层保护。

例如，部长们正在考虑扩大企业为“创新”目的使用数据的法律许可，无论这意味着什么(提示:任何事情)——并就消除处理特定类型数据的个人同意的需要进行磋商，以及对英国版 GDPR 的其他潜在修正案。

政府也在考虑完全取消一项条款，该条款赋予人们对具有法律/同等影响的纯自动决策进行审查的权利。

(在这方面，专业机构 BCS ，又名英国信息技术特许协会，今天警告不要采取如此激烈的措施——在一篇博客文章中提出，增加现有条款的清晰度将是比保持原样或完全抛弃它更明智的政策。)

伯德写道:“随着政府最近宣布他们希望对英国数据隐私立法进行修改，这些担忧似乎是有充分根据的，”他对英国数据政策的方向发出了警告。

“它希望转向一种‘做并请求许可’的模式，这种模式不是由人类利益驱动的，而是由商业利益驱动的。无论我们对我们的客户说 Cronofy 如何处理数据隐私和控制，相应的强制措施都不会执行。

“我们可以对 ISO 认证、数据管理控制、分段数据托管提出抗议。然而，潜在客户不一定会走那么远，因为我们会根据我们的位置打折。我不怪他们。数据保护既棘手又复杂。为什么还要冒险与欧盟以外的供应商合作呢？”

如果英国的保护级别被降级，迫在眉睫的风险是英国将失去与欧盟的一项关键数据流协议——该协议刚刚实施，因为它是欧盟术语中所谓的“第三国”。

在欧洲拥有客户的英国公司依靠这份欧盟“数据充足性”协议顺利运营，因为它允许个人数据从欧盟自由流动到英国。但如果英国法律被评估为不再等效，欧盟委员会表示将撤销今年夏天签署的安排。

数据流协议已经包括一项日落条款，这意味着 2025 年将自动审查英国标准。

但欧盟立法者警告说，如果有分歧，他们可以随时撤销它。因此，像伯德这样的本土企业家对英国隐私政策的猛烈攻击不可能不被布鲁塞尔方面注意到…

伯德警告说:“这种全国性的自残行为将对未来几十年产生影响。”。“事实证明，恐惧项目(英国退出欧盟的支持者轻蔑地称之为那些希望留在欧盟的人对离开欧盟的反对意见)实际上是事实项目。

“英国政府似乎把它视为一种超越的结果，而不是把它视为一种应该避免的警告。在孤立状态下，克罗诺菲作为附带伤害并不重要。对于英国及其与世界其他地区的关系来说，我们面临的是一个令人担忧的预兆。”

“我期望并希望将 Cronofy 打造成为一家世界一流的英国公司。欧盟成员国身份为我们提供了一个令人羡慕的平台来实现这一目标，并反过来将这一成功投资回英国，”他补充道，强调了他的观点，即英国政府的政策让 Cronofy 别无选择，只能以将欧盟置于核心的方式重组其业务。

我们联系了 DCMS，请他对伯德的博客帖子做出回应。

一位政府发言人给我们发来了这样一份声明:

“我们没有削弱英国的数据保护规则。我们正在就改革我们的数据制度进行磋商，以鼓励创新和改善公共服务。

“任何提案都将建立在英国 GDPR 的基础上，人们将继续享受最强的数据保护标准，企业也将承担起保护个人信息安全的更大责任。”

在改革英国数据保护规则的经济案例上，发言人声称这在 gov.uk 上的预期影响分析报告中有所描述——但也表示该分析仍有待磋商，该官员补充说，政府正在寻求进一步的信息，以有力地量化影响，包括对贸易的影响，因为它建立了一个更详细的案例。

DCMS 还告诉我们，咨询过程旨在引发讨论，强调它尚未引入立法，并表示在收集到各种意见并与感兴趣的各方接触之前，它不会这样做。

如果政府的“改革”最终导致英国的数据充足性状况恶化，那么为了一瞥等待英国初创公司的未来，请参见 EDPB 关于向第三国转移的复杂指导。准备增加你的法律费用预算。
