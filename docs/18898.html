由前 Braintree 和 PayPal 高管创建的 Pagos 筹集了 1000 万美元，用于提供 API 驱动的支付智能技术

原文：https://web.archive.org/web/https://techcrunch.com/2021/10/15/founded-by-former-braintree-and-paypal-execs-pagos-raises-10m-in-seed-funding-to-offer-api-driven-payment-intelligence/

由前 Braintree 和 PayPal 高管创办的支付智能基础设施初创公司 Pagos 已经筹集了 1000 万美元的种子资金。

今年早些时候开始，remote-firstPagos正在建立一个数据“平台”和 API 驱动的微服务，据说可以与任何支付栈集成。最终目标是提高企业现有支付基础设施的性能和“优化”。

这个概念源自创始团队在之前公司的经历。首席执行官Klas bck和首席采购官Albert Drouart在过去的八到九年里，分别在 Braintree/Venmo 和 PayPal 担任高级领导职务(Braintree/Venmo 被 PayPal 收购)。两人还在支付处理公司 Netgiro(后来卖给了 Digital River)共事。该公司的首席技术官丹尼尔·布隆伯格(Daniel Blomberg)在过去 20 年里创办了 7 家创业公司(并卖掉了 5 家)。

(补充说明:贝克和布隆伯格在瑞典乡下的同一个小镇长大，但在过去的二十年里，他们都生活在国外——不同的地方)。

“我们看到，我们的每一个客户都面临着一个挑战，那就是他们没有足够的知识、足够的数据和足够的工具来执行支付处理策略，或者知道如何优化支付处理，”贝克说，他从一开始就负责 Braintree 的国际业务，并在 PayPal 收购这家初创公司后为 PayPal 工作。“这意味着他们的速度慢了很多，做所有需要做的事情和产生他们想要的结果也困难得多。”

贝克补充说，实际上，这可能导致更高的运营成本、收入损失和“不必要的摩擦，使商业战略的执行变得更加困难”。

因此，Pagos 着手建立一个 SaaS 平台，它将其描述为 API 驱动的微服务，以帮助公司优化其支付处理 和 的执行。

“我们希望给他们所需的洞察力和数据来回答‘这是什么意思？我怎样才能做得更好？如何才能执行得更快？”贝克说。我们希望以一种易于使用的方式为他们提供这些工具。”

这足以赢得下划线风投和 point 72 Ventures——他们共同领导了 Pagos 的种子轮——以及天使投资人，包括前 Venmo 总经理和现任 Accel 投资人 Amit Jhawar 谷歌商务总裁比尔·里迪&支付；比利陈(Billy Chen)，菲尼克斯金融合作伙伴副总裁，优步前支付总监；和微软 Dynamics 365 客户洞察总经理 Rich LaBarca。

短期内，Pagos 将提供“即时”支付数据可视化、支付趋势或问题的自动通知以及最新的银行识别号(BIN)详细信息等服务，以管理客户和跟踪成本。展望未来，该公司计划提供网络令牌化和帐户更新服务。

“中型到大型公司在网上销售或通过他们的移动应用程序获得了合理的牵引力。一旦他们开始达到有意义的数字，他们的支付基础设施就会阻碍他们，”贝克告诉 TechCrunch。“我们希望帮助他们扩展，用更少的资源执行更多的任务。”

从第一天起，该公司就一直在全球范围内与客户合作，从 50 人的公司到在线销售数十亿美元产品和服务的其他公司。

贝克指出，该公司计划主要利用新资本扩大其 20 人团队的规模，尤其是工程师。

它的投资者自然是乐观的。

波士顿下划线风投公司的合伙人克里斯·加德纳告诉 TechCrunch，他之所以被 Pagos 吸引，是因为该公司的团队花了几十年时间与目标客户合作，“让他们成为唯一有资格为他们服务的人。”

“但是他们的潜在市场是世界上的每一个电子商务商家——而且有数以百万计的商家，”他通过电子邮件写道。"这是制胜秘诀中的两大要素。"

在他看来，Pagos 产品在服务能力和客户交付模式方面都是独一无二的。随着时间的推移，该团队计划提供十几个单独的微服务来解决特定的支付优化挑战——所有这些都可以通过 API 访问。

“既然没有‘放之四海而皆准’的东西，”他说，“它们可以完全不捆绑销售，可以免费试用，大规模使用时可以单独定价。”

Point72 Ventures 的运营合伙人 Dave Matter 认为，随着商业变得越来越数字化，商家的支付体系变得更加复杂和难以管理。

他在一份书面声明中说:“帕格斯由业内两位最有成就的支付产品专家领导，他们的关系、领域专长和解决这些棘手问题的第一手经验非常宝贵。”。
