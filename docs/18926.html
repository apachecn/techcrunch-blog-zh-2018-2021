Builder.io 旨在让开发者对其数字店面的无代码方法感到满意 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2021/10/18/builder-io-aims-to-make-developers-happy-with-its-no-code-approach-to-digital-storefronts/amp/

图片来源: Builder.io / Builder.io 联合创始人布伦特·洛克斯和史蒂夫·休厄尔

任何电子商务企业都必须不断改变其数字店面，以确保准确的库存得到体现。但是要发布一个新产品或者测试一个主页的位置，通常需要从忙碌的开发者那里寻求指导，这就减慢了这个过程。

Builder.io 认为客户应该能够创造他们想要的商务体验，比如一个拖放系统，他们可以在没有开发者支持的情况下插入他们的技术堆栈。

今天，该公司宣布从 Greylock 获得 1400 万美元的新资金，参与投资的有 Imaginary Ventures 和一批天使投资人，包括 Everlane 创始人兼首席执行官迈克尔·普莱斯曼(Michael Preysman)、Hopin 首席技术官安东尼·肯纳达(Anthony Kennada)和数据狗首席运营官·亚当·布利泽(Adam Blitzer)。

公司联合创始人史蒂夫·休厄尔(Steve Sewell)和布伦特·洛克斯(Brent Locks)在时尚电子商务公司 ShopStyle 合作开发网络应用程序时，就看到了这个问题。首席执行官休厄尔(Sewell)表示，该公司是首批“无头”的公司之一，这意味着将前端应用程序与后端应用程序分离，以便它们可以独立更新，而不会影响其他应用程序。

然而，在更新商店风格页面时，他们遇到了大量非开发人员请求帮助更新和测试页面，这变得势不可挡。

“我们根本没有真正考虑过内容管理，”Sewell 告诉 TechCrunch。“我们对此没有准备。我们有很好的 API 基础设施，但是没有办法管理这些东西。”

通常来说，要想在网页上获得一个新按钮，这个请求可能会交给一名工程师，并在一个月内完成，如果有优先权的话。

他补充说，为了避开时间，营销团队通常会使用其他软件创建自己的页面，然后将其集成到公司的网站上，这会降低网站的速度。

“代码对商家来说是一个瓶颈，现实情况是他们不会构建任何新的东西或测试任何东西，因为这需要太长时间，太昂贵了，”Sewell 补充道。

相反，Builder.io 通过拖放式可视编辑器和无头内容管理系统提供适当的 web 流，以便营销人员、产品经理或设计师可以使用现有组件或创建自己的组件来构建内容，运行 A/B 测试并衡量他们所需的转换。

自 2019 年推出商务工具以来，该公司的客户同比增长了 6 倍，现在与 400 多家客户合作，包括 Everlane、Alo Yoga、Afterpay 和 Vistaprint。它还聘请 Angular 创始人米斯科·赫韦利(Misko Hevery)担任首席技术官。

这种增长导致该公司寻求新的资金来雇佣更多的支持和销售人员。它在 2020 年推出了与 Shopify 的原生集成，随着资金的宣布，还推出了通用连接器，以便商业公司可以在不改变代码的情况下与任何无头商业平台或 API 集成，以及让开发人员构建自己的数据源连接器。

“我们希望达到这样一种境界，即每个人都可以很容易地做出一些东西，并知道如何创造他们可以想象的任何东西，”Sewell 说。

Greylock 的合伙人、建筑公司董事会成员迈克·杜博(Mike Duboe)表示，他在 Stitch Fix 工作时也亲眼目睹了这个问题，当时他是第一个内部增长型员工。

他关注的一个问题是减少对开发者的依赖，允许更多的无头操作。对于每个有人想要旋转的新页面，它会转到提示的末尾。Duboe 补充说，商家通常没有工程师，或者只有几个工程师，任何添加到网站的副产品都会降低速度。

“在它成为词典之前，史蒂夫和布伦特就生活在这种东西里，”他说。“他们也是有趣的补充:史蒂夫有技术领导者的才能，聘请 Angular 的创始人担任首席技术官使该公司成为人才磁铁。我们在那之前就投资了，但他有激情，关心业绩。布伦特是产品和业务方面的领导者，他们都有很深的共鸣。”
