当你开始创业时，有一个伟大的想法是不够的。你必须通过在正确的时间做出正确的决定来很好地执行这个想法。特别是，你必须为你的产品选择正确的技术组合。没有好的技术基础，最后可能会积累很多技术债。

因此，为了帮助创始人了解一个好的技术堆栈应该是什么样的，我们邀请了这方面的两位专家， HashiCorp 的工程执行副总裁 Preeti Somal 和 Pilot 的工程副总裁 Jill Wetzler 到 TechCrunch Disrupt 2021 来讨论从评估供应商到确保你可以依赖开源产品的一切。

确保您的团队能够快速发货

有些开发环境比其他的更熟悉。例如，如果你选择使用一个流行的框架，就更容易找到工程师加入你的团队，现有开发人员的学习曲线也更容易。

你的技术不局限于你的团队使用的语言。选择一个好的 CI/CD(持续集成和持续交付)框架可以帮助你更频繁地发布更新。使用测试套件也是一个好的开发管道的关键元素。

“我看了看我们是如何考虑开发人员的生产力和我们的环境的。有哪些事情可以帮助我们的团队快速行动和快速出货？因为我认为当你谈论一个创业公司时，这就是游戏的名字。韦茨勒说:“这只是归结为如何尽快将代码发布出去。

韦茨勒知道她在这方面谈论的是什么，因为她在之前的工作中经历了与此相反的,当时她在 Twitter 工作。“Twitter 当时正在做出一些决定，我认为这些决定是基于一些人当时的个人偏好。我们开始派生出自己的 git 版本和构建系统。它只是变得一团糟，需要几年的时间来解决。所以你真的要为这些决定付出代价，”她说。

跨不同平台重用代码的能力还可以帮助您更轻松地管理多个项目。如果您负责路线图，并且希望在规划下一季度时有一定的透明度，这一点可能很重要。

“我们在后端生产力方面的投资做得非常好。但是当涉及到前端时，我们确实缺少了许多关键的基础设施，而这些基础设施帮助我们快速构建了前端，”Wetzler 说。她加入 Pilot 后就致力于解决这个问题。
