事实证明，该公司的申请文件有一些曲折之处，这让我们真的值得花时间。所以，我们在挖。

总部位于加州圣马特奥的 Backblaze 的风险投资历史非常有限。那是因为它有不赔钱的记录。事实上，根据 Crunchbase 的数据，它在私人生活中只筹集了为数不多的数百万美元。该公司在其文件中证实了这一点，声明如下:

我们的运营历来高效，外部投资有限。在 2021 年 8 月的一轮私人融资中发行 1000 万美元的可转换票据(我们也称之为未来股权协议(SAFE)的简单协议)之前，自 2007 年成立以来，我们已经筹集了不到 300 万美元的外部股权。

有没有人忘记了，你可以在没有大量外部资金的情况下建立准备上市的软件公司？这里提醒一下。

自然，我们要弄清楚这家公司是如何在没有损失价值九位数的风险资金的情况下，成功发展到目前的规模的。我们还想看看它的毛利润。

为什么？因为通过提供存储，我们的假设是 Backblaze 的毛利润有些黯淡。但是我们可能错了。最后，我们将继续上周末我们谈到的话题，即专业云基础架构提供商以及他们如何与公共云巨头相抗衡。

我们走吧！

Backblaze 如何在不损失上帝所有资金的情况下发展到 IPO 规模

Backblaze 有两个主要产品，都位于其“存储云”之上。或者，正如我所想的，一大桶在线存储容量。该公司提供“B2 云存储”，这是一种包括 API 访问的工具，用于开发人员备份信息，与第三方内容交付网络(cdn)等相结合。它的第二个产品叫做“电脑备份”它能为一台或多台电脑做它在电视上所说的事情。

Backblaze 是为谁打造的？虽然该公司声称其“解决方案是为各行各业各种规模的个人和企业设计的”，但它确实指出“对中端市场组织，或员工人数不到 1000 人的组织，有着特别强的吸引力”。

所有这些都很重要，因为它框定了我们的问题:Backblaze 是如何增长到超过 6000 万美元的年运营率的(基于 H1 2021 的数据；更多关于此事的信息)没有更多的风险投资？
